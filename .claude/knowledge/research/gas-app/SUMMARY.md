# Gas 앱 종합 리서치 — 통합 요약

> 조사 기간: 2026-07-08 · 방법: mobbin-mcp(UI 실측) + WebSearch/WebFetch + rdt-cli/twitter-cli(인증됨) + Workflow 기반 사실검증(모순 7건 교차확인) + 완성도 비평(갭 4건 보강). 총 산출물 11개 파일. 이 문서는 그 전체를 종합한 최종 보고서다. "오늘 해볼까"(친구 투표/카드 UI 기반 아이디어 생성 앱) 팀의 벤치마킹을 위해 작성됨 — 시사점은 어디까지나 참고용 첨언이며 원 리서치 요청(GAS 전체 조사)이 본 목적이다.

---

## TL;DR — 7줄 요약

1. **Gas는 "친구 폴 투표 → 익명 칭찬 알림"이 핵심 루프인 10대 대상 앱**으로, 2022-08 출시 후 3개월 만에 1,000만 유저를 모았다가(광고비 $0) 2023-01 Discord에 인수, 2023-11 종료됐다.
2. **성장의 실체는 창업자 Nikita Bier 본인이 공개한 K-factor > 1.0**(메시지 1건당 신규 유저 획득 1명 초과)이었다 — "익명 메시지를 받은 10대가 발신자가 궁금해 Snapchat에 스크린샷을 공유 → 제3자가 설치"하는 구체적 메커니즘이다(트윗 원문으로 검증 완료).
3. **몰락의 실체는 "Discord가 방치해서"가 아니다** — 다운로드는 인수 발표 8주 전(2022 추수감사절 무렵)에 이미 98% 붕괴했다. 원인은 (a) 인신매매 괴담이라는 외부 신뢰 충격, (b) 창업자 본인이 인정한 "나이 들수록 초대 발송 20%씩 감소"라는 구조적 K-factor 감쇠, (c) "가짜 답장 아니냐"는 의심이 변동보상 루프의 전제(불확실성의 진짜다움)를 깨뜨린 것 — 세 갈래다.
4. **유료화(God Mode, $6.99/주=월 환산 ~$28로 사실검증 완료)와 동일한 "결제해야 발신자 공개" 모델이 카피캣 NGL·Sendit에서 FTC 제재로 이어졌다** — NGL은 $5M 합의금+18세 미만 서비스 영구금지, Sendit은 화해 없이 DOJ 소송 계류 중(2026-03 기준 기각신청 서면공방 완료, 이후 미확인).
5. **한국에도 구조적으로 동일한 카피캣(OMG·HYPE)과 계보 앱(Asked/에스크)이 실재**하며, 에스크는 실제 형사사건(소년재판)까지 갔고, 데이팅앱 아만다는 개인정보보호법 위반으로 과징금 2.24억을 이미 물었다 — 미국 FTC 프레임이 아니라 **한국 개인정보보호법·정보통신망법이 직접 적용되는 리스크**다.
6. **가장 실행 가능성이 높은 단일 발견은 기술 인프라 쪽**: "오늘 해볼까"의 설계 원칙("로그인 요구 시점=카드 발행 순간", Supabase 매직링크)이 Supabase 내장 이메일의 **시간당 2통 프로젝트 전체 한도**와 충돌한다 — 바이럴이 실제로 터지는 순간 로그인 자체가 막히는 자기파괴적 구조. 커스텀 SMTP를 사전에 붙이지 않으면 100% 재현되는 장애다.
7. **심리학적 근거(사회비교이론·다크패턴 taxonomy)로 뒷받침되는 5개 구체적 설계 제안**을 도출했다 — 투표수/순위 기본 비노출, 신원공개 유료화 금지, 당사자 동의 후 결과공개, 다크패턴 체크리스트, 비교대상 동질성 확보.

## 1. Gas 앱이란 무엇인가

Nikita Bier(전작 tbh, 2017)가 만든 10대 대상 익명 칭찬/투표 앱. 학교 인증 → 친구 목록 기반 4지선다 폴("Best person to go camping with" 등) 12개를 연속 응답 → 누군가 나를 선택하면 익명 알림("불꽃") 수신. 12개를 다 풀면 60분 잠금이 걸리고 **"기다리거나 친구를 초대해서 즉시 스킵"** 화면이 뜬다 — 이것이 초대 바이럴의 심장부다. 수익화는 코인(폴 노출 부스트, 폴에 몰래 이름 넣기)과 **God Mode $6.99/주(월 환산 ~$28) 구독**(투표자 힌트) 두 갈래.

## 2. 성장의 실체 — 검증된 K-factor와 그로스 플레이북

Nikita Bier 본인의 트윗을 twitter-cli로 직접 확보·검증했다(원문 대조 완료):

- **"개발 시간의 10%만 폴 기능에, 90%는 friendfinder(친구 매칭 인프라)에 투입했다"** — 콘텐츠가 아니라 "누가 내 친구인가"를 찾는 인프라(전화번호부 샘플링, Redis 친구그래프)가 바이럴의 실체였다.
- **학교당 36시간 내 40-50% 침투, 시간당 3만 명 가입, 미국 10대 3명 중 1명 설치.**
- **K-factor 공개(2023-12 회고, 원문 검증 완료)**: "Gas에서 10대가 익명 메시지를 받으면 첫 반응은 Snapchat에 올려 친구들에게 '누가 보냈는지' 묻는 것이었다. 그래서 메시지 1건당 우리는 1.0명 넘는 유저를 획득했다." — 단, 검증 결과 이 트윗 자체는 "스크린샷"이나 "제3자가 설치"라는 구체적 인과 경로까지 명시하진 않았고(초창기 리서치의 약간의 각색이 있었음), 핵심 수치(>1.0)와 메커니즘 방향은 정확하다.
- **Geofenced exclusivity**: 학교별 Instagram 비공개 계정 → 전교생 팔로우 → 하교시간 동시 공개 → 주 단위 순차 확대. FOMO로 학교 경계를 넘는 입소문을 만듦.
- **메시징 미지원은 안전+운영효율의 이중 설계**: "폴만 지원해 어뷰징 처리량을 99.9% 줄였다"고 본인이 밝힘 — "오늘 해볼까"의 "투표 수신자 로그인 요구 금지" 원칙과 유사한 트레이드오프.

## 3. 몰락의 실체 — "Discord 탓"이 아니라 장르 구조적 붕괴 (신규 발견)

기존 리서치가 놓쳤던 부분을 갭 보강으로 채웠다. **다운로드는 Discord 인수 발표(2023-01-17)보다 8주 앞선 2022-11-24(추수감사절) 전후로 이미 붕괴하기 시작**했고, 인수 시점엔 이미 피크 대비 98% 감소 상태였다.

| 시점 | 사건 |
|---|---|
| 2022-10-05 | 인신매매 괴담 발원(중국발 리뷰 1건→TikTok 확산) → 다운로드 2주 정체, 단일일 3% 삭제 |
| 2022-11-10 | 다운로드 피크(124,558건/일) |
| 2022-11-24 | 추수감사절 전후 다운로드 급락 시작 |
| 2023-01-17 | Discord 인수 발표 (이미 붕괴 8주 경과) |
| 2023-01-29 | 일일 다운로드 약 2,000건(피크 대비 -98%) |
| 2023-11-07 | 서비스 완전 종료 |

원인은 세 갈래:
1. **외부 신뢰 충격**: 인신매매 괴담(허위로 판명됐으나 실질적 이탈 유발).
2. **구조적 K-factor 감쇠**: Bier 본인 발언 — "내가 만든 모든 소셜 앱에서, 나이가 13→18세로 한 살씩 오를 때마다 유저당 초대 발송이 20%씩 줄어든다." "durable한 소셜 제품을 만드는 건 10년에 한 번 나오는 black swan이다." 즉 Gas·tbh류는 애초에 영구 리텐션이 아니라 **짧고 굵은 스파이크**를 최적화하도록 설계됐다. tbh(2017)도 동일 곡선(급성장→Facebook이 "이용률 저조"로 자체 종료)을 그렸다는 것이 "Discord 탓"론의 반증이다.
3. **호기심 소진 + 신뢰 붕괴**: "누가 뽑았는지" 루프는 슬롯머신형 변동보상 구조로, "결과가 진짜"라는 믿음이 전제다. "가짜 답장 아니냐"는 의심(카피캣 NGL은 이 문제로 FTC와 $5M 합의)이 그 전제를 깨는 순간 루프 전체가 무력화된다.
4. **장르 전체의 반복 패턴**: Yik Yak, Sarahah, Yo("leaky bucket"), BeReal 모두 "단일 메커닉+노벨티 의존" 앱이 예외 없이 급성장 후 급쇠퇴를 겪었다.

**"오늘 해볼까" 시사점**: 카드 공개 UX는 Gas·NGL과 동일한 변동보상 구조다. 초대 루프는 "이미 물어볼 지인에게 다 물어본" 시점 이후 구조적으로 감쇠하므로, 코어 루프에 초대 바깥의 재방문 이유(콘텐츠 갱신, 실행 진행 추적 등)가 필요하다. 다운로드/가입 지표만 보면 "이미 죽어가고 있다"는 신호를 놓친다 — D7/D30 리텐션과 "이미 아는 사람 다 초대했는가" 같은 그래프 포화도를 런칭 직후부터 추적해야 조기 경보가 가능하다.

## 4. 유료화와 규제 리스크 — 동일 패턴이 3개국·3개 앱에서 반복

| 앱 | 유료화 구조 | 규제 결과 |
|---|---|---|
| **Gas** | God Mode $6.99/주(~$28/월) — 투표자 힌트 | 법적 제재는 없었음(허위 루머로 평판 타격은 있었음) |
| **NGL** | Peek/Pro $9.99/주 — 가짜 메시지 생성 후 "발신자 공개" 미끼 | **2024-07 FTC+LA지검 $5M 합의**(소비자환급 $4.5M+민사벌금 $50만, 사실검증 완료) + **18세 미만 서비스 영구 금지** |
| **Sendit** | Diamond Membership $9.99/주 — 동일 구조 | **2025-09 FTC가 화해 없이 DOJ에 소송 이첩**(COPPA 위반 11.6만 건 등) — 2026-03 기준 기각신청 서면공방 완료, 그 이후 미확인, 계류 중 |

공통 위법 패턴: (1) 실제 사람이 아닌 가짜/봇 메시지를 실제처럼 위장, (2) "결제하면 발신자를 알 수 있다"고 광고했으나 실제로는 무의미한 힌트만 제공, (3) 자동갱신 미고지, (4) 미성년자 대상 마케팅.

**"오늘 해볼까" 시사점**: "무료 투표+유료 상세리포트" 모델은 구조적으로 이 카테고리의 "무료 폴+유료 리빌"과 유사하다. 결정적 차이를 유지해야 한다 — 유료화 대상이 **"발신자 신원 공개"가 아니라 "아이디어 수요 데이터/실행 플랜"**이어야 동일한 FTC식 리스크를 피할 수 있다.

## 5. 한국 로컬 맥락 — 완전히 빠져 있던 각도를 보강 (신규 발견, 중요도 높음)

원 리서치가 전부 미국/영어권 소스였다는 완성도 비평을 받아 한국어로 재조사했다. **결론: 한국의 리스크는 Gas 자체가 아니라 국산 카피캣과 계보 앱에서 이미 실현되고 있다.**

- **Gas는 한국에 정식 출시된 적이 없고, 국내 보도는 전부 "우호적 성장 스토리" 톤뿐**이다.
- **국산 카피캣 OMG("우리학교 익명 속마음 투표앱")와 HYPE(하입)가 실재**하며, OMG 앱스토어 리뷰에는 이미 자해·성적 대화·특정인 저격·연령 허위기재 등 구체적 위험 신호가 노출돼 있다.
- **Gas의 계보 앱 ASKfm의 한국판 "Asked(에스크)"는 실제로 "사이버 학폭 온상"으로 여러 차례 보도**됐다 — 서울 중학교 사건은 소년재판·기소유예까지 갔다.
- **NGL은 한국 앱마켓에 등록돼 있고 GRAC 기준 "청소년 이용불가" 등급**을 받았다 — 미국 FTC와 무관한 독자적 국내 등급 심사가 이미 작동 중.
- **데이팅앱 아만다는 개인정보보호법 위반(무단 도용 허위계정 276개)으로 2024-09-26 과징금 2.24억+경찰고발** 처분을 받은 실제 국내 집행 선례.
- **대학 익명 게시판 에브리타임에서는 사이버불링으로 인한 실제 사망 사건과 시민단체 기자회견**이 있었다.
- **적용 규제가 미국과 다르다**: FTC/COPPA가 아니라 **개인정보보호법(만 14세 미만 법정대리인 동의)**과 **정보통신망법(명예훼손 최대 7년, 모욕 최대 1년 — 형사처벌)**. 초성 표기만으로도 특정 가능하면 처벌 대상이 될 수 있다(에브리타임 판례 다수).
- 국회에 SNS 청소년보호 법안 5건 계류 중이나 통과 확인 안 됨.

**"오늘 해볼까" 시사점**: (1) 투표/코멘트 내용에 특정인을 저격 가능한 텍스트가 노출되지 않도록 설계해야 한다 — 이것이 로그인 여부보다 실질적인 국내 리스크 축이다. (2) 카드아트에 실존 인물 사진을 쓰지 않는 현재 방향(자체 아트)은 아만다식 리스크를 피하는 데 이미 유효하다. (3) 투표 수신자가 만 14세 미만일 가능성을 배제할 근거가 없다면 법정대리인 동의 흐름을 원천 검토해야 한다.

## 6. 심리학적 근거와 설계 지침 (신규 발견 — 학술 문헌으로 보강)

기존 근거(코넬대 교수·CBC 인터뷰)가 전부 동료심사 없는 언론 코멘트임을 확인한 뒤, 실제 학술 문헌으로 대체·보강했다:

- **소시오미터 이론(Leary)**: 자존감은 사회적 포함/배제를 실시간 감시하는 계기판 — "투표 0표"가 공개적으로 드러나는 UX는 배제 신호로 해석될 수 있다.
- **Nesi & Prinstein(2019, JCCAP, 716명 1년 종단연구)**: "좋아요 같은 정량 지표를 좇는 행동" 자체가 실제 인기와 무관하게 1년 뒤 위험행동 증가를 예측 — 투표수·순위 노출 자체가 위험 경로일 수 있음.
- **Gray et al.(2018, CHI)**: 다크패턴 5분류(Nagging/Obstruction/Sneaking/Interface Interference/Forced Action). **Mathur et al.(2019, CSCW)**: 7카테고리·15유형 실증 taxonomy(Urgency, Social Proof, Scarcity 등).
- **Radesky et al.(2022, JAMA)**: 유아 앱 98.8%가 조작적 디자인 포함.
- **Peeple 선례**: 동의 없는 평가 앱 → 여론 반발로 "당사자 동의 후 게시"로 정책 전환.
- **Instagram 좋아요 수 숨기기 실험**: 정량 지표 가시성 자체를 낮추는 방향으로 업계 최대 플랫폼이 이미 이동.

**근거 기반 설계 제안 5개**(원문 그대로):
1. 투표수/순위를 카드 결과에 기본 노출하지 않는다.
2. "누가 투표했는지 보려면 결제" 구조는 NGL이 연방 제재를 받은 모델과 동일 형태이므로 금지 또는 재설계 — 결제는 실행 플랜/수요 리포트 같은 실질적 가치로 전환.
3. 투표 결과 공개는 당사자 동의 후에만(Peeple 후기 정책 선례).
4. 다크패턴 자가점검: 카운트다운 결제 압박 금지, 활동 메시지(사회적 증거) 신중히, 해지 경로를 가입만큼 쉽게, 필수 기능에 결제 강제 금지.
5. 비교 대상의 동질성 확보 — 전체 랭킹이 아니라 "나와 비슷한 소수 지인" 단위로 비교를 국한.

## 7. 기술 인프라 — 가장 실행 가능성 높은 단일 발견 (신규 발견, 즉시 조치 권고)

기존 인용("HN 게시물 제목", "창업자 트윗 1줄")이 부정확했음을 확인한 뒤, Gas 공동창업자 Dave Schatz의 트윗 스레드 3건 전문을 확보해 실제 장애 원인을 규명하고 이를 **Supabase/Vercel 스택으로 직접 환산**했다.

**Gas의 실제 장애**: (1) 매 요청마다 9만 멤버 Redis SET을 SMEMBERS로 통째로 읽어오면서 값을 쓰지도 않던 코드 → CPU 80% 낭비, (2) 오토스케일 중 신규 연결 TLS 협상 폭주 → 3시간 다운타임 → 복구 중 요청까지 몰려 자기 자신을 DOS.

**"오늘 해볼까" 스택 환산 및 실행 아이템**:
1. **[최중요] 매직링크 로그인 전에 커스텀 SMTP부터 연결한다.** Supabase 내장 이메일은 **프로젝트 전체 기준 시간당 단 2통**으로 제한된다(회원가입·매직링크·재설정 전부 합산). 현재 설계 원칙("카드 발행 순간 로그인 요구", Supabase 매직링크)은 바이럴이 실제로 성공해 카드 발행이 동시다발하는 바로 그 순간 로그인 자체가 막히는 자기파괴적 구조다. 사전에 SendGrid/Resend 등 커스텀 SMTP를 붙이지 않으면 성공 시나리오에서 100% 재현되는 예측 가능한 장애다.
2. 투표/좋아요 집계는 Postgres Changes 실시간 구독이 아니라 폴링+캐시 카운트 또는 Broadcast로 설계한다(구독자 수만큼 인가체크가 선형 증가하는 동일 유형 병목).
3. 투표 데이터는 배열/JSONB 누적이 아니라 카운터 컬럼/집계 뷰로 저장한다.
4. Vercel 서버리스+Supabase는 직결이 아니라 Supavisor 트랜잭션 모드를 쓴다(동시접속 300명 규모에서도 표준조치 후 실패한 실사례 있음).
5. 로그인 없는 투표 제출 엔드포인트에 자체 레이트리밋을 반드시 붙인다(Supabase Auth 기본 레이트리밋은 커스텀 RPC에 적용 안 됨).
6. 런칭 전 현재 플랜의 Realtime 동시연결·메시지/초 한도(Free 티어: 200/100)를 예상 트래픽과 대조한다.

## 8. 사실검증 노트 (투명성을 위한 기록)

리서치 전반을 읽으며 발견한 3개의 실제 모순을 Workflow로 원문 대조해 해소했다:

| 모순 | 정정 결과 |
|---|---|
| God Mode 가격 표기 불일치 ("$6.99 구독"/"$6.99~28"/"주 $1.99~9.99") | **$6.99/주(월 환산 ~$28), 단일가 확정.** "$1.99~9.99" 범위는 뒷받침 근거 없어 오류로 정정. |
| Discord의 Gas 인수가 — 일부 출처 제목에 "$7,000,000" 등장 | **오독으로 확인.** $7M은 인수가가 아니라 Gas가 인수 전 6개월간 iOS에서 번 순수익. 인수가는 여전히 비공개. |
| TBH의 Facebook 인수가 ("$30M 이상" vs "약 1억 달러 추정") | **공식 비공개.** "1억 달러"는 2017년 익명 취재원 기반 추측의 재인용일 뿐이며, SEC 현금흐름표 역산 분석은 오히려 $17M대를 시사 — "$30M+" 쪽이 더 신빙성 있음. |
| K-factor 회고 트윗의 "스크린샷" 세부 묘사 | 트윗 원문 확인 결과 핵심 수치(>1.0)는 정확하나 "스크린샷"·"제3자 설치" 세부는 트윗에 문자 그대로 없는 합리적 각색이었음 — 방향은 정확, 디테일은 경미하게 과장. |
| NGL FTC 합의금 총액 ("$5M" vs "$4.5M") | **모순 아님.** 총 $5M = 소비자환급 $4.5M + LA지검 민사벌금 $50만. 2차 소스가 "$4.5M"만 헤드라인으로 쓴 것일 뿐, FTC 원문(curl로 직접 확인)과 정합. |
