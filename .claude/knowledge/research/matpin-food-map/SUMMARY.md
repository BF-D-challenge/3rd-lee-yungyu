# 맛핀 리서치 통합 요약 (2026-07-12)

> PRD: `docs/prd/b2c-ideas/01-맛핀-prd.md` v1. 수집: WebSearch(6쿼리)·레딧(2쿼리)·유튜브(1쿼리). 트위터·r/foodNYC 직접 검색·다이닝코드/포잉 상세는 **미수집(시간 제한)**. 상세는 COMPETITORS.md / COMMUNITY.md.

## 핵심 발견 5

1. **컨셉은 검증됐지만 독점 아님**: "영상 URL→개인 지도" 카테고리에 GeoTok·Stasht·MunchMap·TokSpot·Rezz 등 5개+ 해외 앱이 이미 경쟁 중. 대부분 무료 + OS 공유 시트 연동(맛핀의 '붙여넣기 우선'보다 마찰 낮음). 차별화는 국내 지도(네이버/카카오 POI)·한국어 추출·여행 동선에서 찾아야 함.
2. **개인 맛집 리스트 수요는 실재**: Beli 평가 58M건(2025 Q2, Yelp 추월)·유료 광고 없이 입소문 성장·80%가 35세 미만. Mapstr 4M 사용자. "메모 앱에서 벗어났다"는 VOC 확인.
3. **회귀 기본값은 기존 지도 앱**: 레딧 VOC에서 Beli 이탈자가 Google Maps로 회귀. 유튜브에서도 신규 앱보다 구글지도/카카오맵 활용법 콘텐츠가 압도적 조회수(346K vs 개인 맛집앱 1~14K). 신규 앱의 대체 편익이 크지 않으면 리텐션이 무너짐.
4. **국내 맛집 앱 수익화는 잔혹사**: 망고플레이트 MAU 200만 찍고도 수익화 실패로 2023 종료. 캐치테이블도 매출 301억(2025)에 영업적자 -151억(2024)~지속. B2C 맛집 발견에 직접 과금은 국내 선례가 없음 — PRD의 "공개 열람 무과금, 정리 노동에 과금" 방향 자체는 옳으나 결제 검증이 더욱 절실.
5. **네이버의 갭은 '입력 단계'뿐**: 네이버지도 저장탭은 업종 분류·리스트 공유·실시간 구독까지 이미 제공. 맛핀이 이길 수 있는 유일한 지점은 "영상→장소 변환 자동화"이며, 저장 이후 관리 경험으로는 못 이김.

## PRD 가정 대조표 (VUVF + Kill 기준)

| PRD 가정 | 판정 | 근거 |
|---|---|---|
| **V1** 멀티플랫폼 저장자는 저장함보다 위치 기반 개인 지도를 원한다 | **부분 지지** | Beli/Mapstr 성장, 동종 앱 5개+ 존재(수요 방증). 단 유저 육성 VOC 인용 미확보, 회귀 기본값은 구글/네이버지도 |
| **V2** 자동정리·대량 추출·여행 동선에 결제한다 | **반박 우세(불명)** | 직접 경쟁 MunchMap 등 무료, 국내 맛집앱 B2C 과금 성공 선례 0(망고 종료·캐치 적자). Mapstr 프리미엄이 유일한 긍정 신호. 1주차 사전 결제 실험이 결정적 |
| **U1** URL 1개 확인 45초 이하 | **불명** | 미검증 — 경쟁 앱(GeoTok 등)이 캡션+음성+OCR 조합으로 자동화 달성 주장 → 기술적으론 도달 가능 신호 |
| **U2** 핀+영상+메뉴로 3분 내 결정 | **불명** | 수집 증거 없음. 1주차 실험 항목 |
| **Vi1** 7일 패스 vs 월 구독 분리 가능 | **불명** | 선례 없음. 캐치테이블도 소비자 과금 아닌 B2B 수수료 모델 |
| **Vi2** 무료 한도로 변동비 20% 이하 통제 | **불명** | 미수집(시간 제한) |
| **F1** 3플랫폼 URL 장소 추출 80% | **부분 지지** | GeoTok·TokSpot 등이 동일 파이프라인 상용화 — 실현 가능성 방증. 단 한국어·국내 POI 정확도는 별도 검증 필요 |
| **F2** 중복·모호 지역 1회 확인 해결 | **불명** | 증거 없음 |
| **최대 위험: 네이버·카카오지도보다 충분히 편한가** | **경고(부분 반박)** | 네이버 저장탭이 관리·공유 이미 커버, VOC상 이탈자들은 지도 앱으로 회귀. 편익은 오직 '입력 자동화 순간'에 존재 — 이 한 순간의 시간 절약이 앱 전환 비용을 이겨야 함. PRD의 45초 기준과 "직접 검색 대비 30% 결정 시간 감소" kill 기준은 타당하며 반드시 실측할 것 |
| **Kill: 공유·복사 0건이면 high-K 폐기** | **지지(설계 타당)** | Beli도 K는 소셜 그래프(친구 랭킹)에서 나옴. 읽기 전용 카드 공유만으로 K≥1은 낙관 금물 — PRD의 보수적 계측 방침 적절 |

## 벤치마킹 시사점 (5)

1. **공유 시트 연동을 앞당길 것**: 해외 동종 앱 전부가 share sheet 진입. 붙여넣기-온리는 저장 순간의 마찰을 키워 U1(45초)보다 앞 단계에서 이탈 유발. MVP 웹앱이라면 PWA share target 검토.
2. **추출 파이프라인은 캡션+음성 전사+화면 OCR 3중 조합**(GeoTok 방식)이 업계 표준 — 캡션 단독 추출로 80% 목표는 어려울 것.
3. **랭킹·게임화 강제는 독**(Beli 이탈 VOC): 저장·결정 플로우는 가볍게, 구조화는 선택으로.
4. **국내 B2C 과금은 '여행 순간' 집중이 유일한 활로**: 반복 구독 선례 없음. PRD의 7일 여행 패스 우선 가설이 월 구독보다 증거와 부합.
5. **콘텐츠 SEO가 저비용 획득 채널**(Stasht 사례): "저장한 릴스 맛집 정리법" 류 하우투 콘텐츠로 유입 → K 루프 실패 시 대체 채널로 유효(PRD 3장의 대체 채널 설계와 일치).
