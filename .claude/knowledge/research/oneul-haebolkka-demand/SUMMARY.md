# 오늘 해볼까 — 수요 검증 리서치 통합 요약

## 종합 판정: 수요 신호 **중~강** — 단, 포지셔닝 조건부

- **문제는 실재(강)**: "뭘 만들지 모르겠다"는 pain이 영·한 3개 언어권에서 교차 확인됨. "Codex 크레딧 남는데 뭘 만들지 모르겠다"(85~135 likes), 아이디어 17개 방치 글(273댓글), 한국어 바이브코딩 회고("딸깍은 쉬운데 '무엇을'이 핵심").
- **수요 확인 pain은 사후형**: "시작 전 몰라서"보다 "검증 없이 만들고 6개월 뒤 침묵" + "정직한 피드백 줄 사람이 없음"(수단 부재) 형태.
- **솔루션 형태에 대한 냉소는 강함(약)**: 유사 아이디어 생성 툴 7개+ 도배, 최상위 댓글 일관 "툴 말고 유저 5명과 대화하라", "AI 검증은 확증편향 기계". 솔루션 피칭 글은 점수 0~2, 문제 공감 글만 흥함.

## 설계 반영 (개선안 v2와의 정합)

1. **"아이디어 생성기" 포지셔닝 금지** — 레드오션 + 불신 지대. "내 것(씨앗)에서 시작해 오늘 바로 만들게 하는 실행 촉발 + 수요 확인"으로 말할 것.
2. 커뮤니티가 처방하는 대안("유저 5명과 대화하라")이 곧 투표 카드 기능 — **마케팅 카피를 냉소 진영의 언어로 쓸 수 있음** ("툴 믿지 말고, 12명한테 물어보세요").
3. v2의 씨앗 기반 전환·수요 리포트 과금은 시장 실증 패턴과 일치. 랜덤 생성 단독이었다면 IdeasAI의 전철.
4. 배포 채널: 디스콰이엇은 경쟁자가 아닌 획득 채널. 솔루션 피칭 글은 죽으므로 **문제 공감형 콘텐츠**로 진입.
