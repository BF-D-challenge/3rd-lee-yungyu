# 결론 — 결과를 “영상으로 말할 수 있는 아이디어”로 바꾼다

## 한 문장 결론

`오늘 해볼까`는 자동으로 영상을 만들어주는 기능부터 만들 것이 아니라, 네 장을 뽑은 결과를 **첫 6초 훅 + 검증된 증거 + 한 끗 변화 + 오늘 만들 장면 + 사람 질문**으로 바꾸는 `영상 준비 브리프`를 먼저 제공해야 한다.

## 현재와 권장안

| 현재 | 권장 |
|---|---|
| 완성 아이디어 제목과 설명 블록 | 10초 안에 읽히는 UVP와 전후 차이 |
| “어떻게 생각해?” | “너도 이 문제를 겪나요?” |
| 네 카드의 내부 조립 과정을 길게 설명 | 첫 화면은 문제·결과·증거, 카드는 접어서 보조 |
| 친구에게 응원 요청 | 실제 문제 보유자에게 판단 요청 |
| 공유 링크 한 개 | 링크 + 20~30초 영상 대본 + 9:16 화면 순서 |
| 공유 후 전체 제작 문구 잠금 해제 | 사람 반응으로 수요 영수증을 쌓고 다음 영상 소재 생성 |

## 권장 출력 구조

1. **이름 — 결과를 말하는 한 줄**
2. **문제 질문** — 숫자·시간·반복 행동 중 하나
3. **전과 후** — 지금 얼마나 귀찮은지 → 무엇 하나로 끝나는지
4. **검증된 원본** — 이미 작동하거나 돈이 된 근거 한 줄
5. **한 끗 변화** — 한국의 누구·어떤 순간을 위해 바꿨는지
6. **오늘 만들 장면** — 입력 1개 → 처리 1회 → 결과 1개
7. **사람 질문** — 문제 경험 / 사용 의향 / 한 줄 이유

## 맛핀 25초 예시

**0~3초**  
“저장한 맛집 릴스 200개, 지금 당장 하나 찾을 수 있나요?”

**3~7초**  
“저장은 3초인데 다시 찾는 건 20분입니다.”

**7~13초**  
영상 링크를 붙여넣자 상호·메뉴·위치가 추출되고 지도 핀 하나가 생기는 화면을 보여준다.

**13~18초**  
“해외에서는 맛집 리스트와 영상→장소 변환 앱이 이미 작동합니다. 저는 네이버 지도에 맞는 이 한 화면만 오늘 만듭니다.”

**18~25초**  
“당신도 저장한 맛집 영상을 다시 못 찾나요? 예·아니오만 눌러주세요.”

## 가장 작은 제품 실험

자동 영상 생성은 아직 만들지 않는다. 먼저 결과 화면에 아래 두 산출물만 추가해 48시간 검증한다.

1. `20초 영상 대본 보기`
2. `이 문제를 겪는 친구에게 보내기`

10명의 빌더가 각자 실제 타겟 2명에게 보내 총 20명을 관찰한다.

- 공유 완료율
- 링크 오픈율
- “나도 겪는다” 선택률
- 한 줄 이유 작성률
- 응답자 → 자기 아이디어 만들기 전환률

## 구현 우선순위

1. 결과 첫 블록을 이름·변화 한 줄·문제 질문·전후 대조로 재구성한다.
2. 현재 공유문을 위 4개 요소가 들어간 20초 대본으로 바꾼다.
3. 수신자 첫 행동을 `응원 보내기`에서 `너도 이 문제를 겪나요?`로 바꾼다.
4. 답변 뒤에 한 줄 의견과 응원을 받는다.
5. 실제 반응을 다음 영상의 증거 카드로 보여준다.
6. 이 루프가 반응을 만들 때만 자동 자막·9:16 카드·영상 렌더링을 붙인다.

이 순서는 영상 제작 자동화보다 먼저 **좋은 아이디어가 사람 반응을 얻는지**를 검증한다.
