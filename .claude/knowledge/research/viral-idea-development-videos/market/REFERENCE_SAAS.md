## 가장 참조할 만한 3개

### 1. Eden — 판정 방식

Eden은 큰 계정의 큰 조회수를 그대로 인기라고 부르지 않는다. 게시물이 그 크리에이터의 중앙값보다 얼마나 더 잘됐는지 `outlier multiplier`로 계산하고, 주제·플랫폼·팔로워 규모·최소 배수를 함께 필터링한다.

우리에게 가져올 규칙은 다음과 같다.

```text
outlier 배수 = 게시물 조회수 / 해당 계정 최근 게시물 조회수 중앙값
```

- 3배 미만: 참고만
- 3~5배: 약한 후보
- 5~10배: 카드 재료 후보
- 10배 이상: 사람이 원본 메커니즘을 직접 해석
- 같은 문제·결과가 서로 다른 계정 3곳 이상에서 5배를 넘을 때만 강한 신호

Eden의 의미 검색과 팔로워 범위 필터도 중요하다. 유명 계정의 팬덤 효과와 아이디어 자체의 힘을 분리할 수 있다.

### 2. Shortimize — 모음 화면

Shortimize는 `AI`, `Productivity`, `Maps`, `Weight loss`, `Mental health`, `Astrology`처럼 `오늘 해볼까`의 카드 도메인과 바로 맞는 카테고리별로 “바이럴 영상 20개”를 보여준다. 카테고리는 매일 갱신되며, 검색도 자막·오디오 같은 **Hook**과 화면에 실제로 등장하는 **Context**를 나눠 제공한다.

우리 화면에 가져올 구조는 다음과 같다.

```text
카테고리 → 오늘 튄 영상 20개 → 문제 훅 → 작동 장면 → 제품 원본 후보
```

영상 전체를 무작정 저장하지 않고, `문제 한 문장 / 입력 / 처리 / 결과 / 반응 이유` 다섯 칸만 추출해야 기존 IdeaLabScenario와 섞이지 않는다.

### 3. 1of10 — 무료 실험

1of10은 무료 플랜에서도 outlier 검색이 가능하다. 영상 하나가 우연히 튄 것보다 같은 주제의 5배 이상 outlier가 여러 채널에 반복되는 것을 신호로 본다.

첫 실험은 결제 없이 다음처럼 할 수 있다.

1. `app I built`, `problem so I built`, `indie app launch`, `vibe coding` 검색
2. 최근 90일·5배 이상만 남김
3. 같은 문제와 결과를 말하는 영상을 묶음
4. 묶음이 3개 이상인 문제만 원본 후보로 넘김

## 자동 승격 금지

바이럴 영상은 `사람들이 끝까지 보고 공유했다`는 증거이지 `그 제품에 돈을 낸다`는 증거가 아니다. 따라서 바이럴 신호는 새 원본을 자동 통과시키지 않고 다음 순서로만 사용한다.

```text
바이럴 outlier
→ 문제·입력·처리·결과 추출
→ 해외 작동 제품 또는 결제 근거 연결
→ 한국의 돈 낼 사람·필요 순간 확인
→ 3×3×3 카드 자연스러움 감사
→ 통과한 원본만 앱 승격
```

즉, 바이럴 데이터는 **후보를 빠르게 찾는 레이더**이고, 기존 품질 감사는 **앱에 넣는 문지기**다.
