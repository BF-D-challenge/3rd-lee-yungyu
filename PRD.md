## 정의

**무엇을 만들지 모르는 초보 1인 빌더가, 좋아하는 것(🌱씨앗)에서 출발해 오늘 만들 아이디어 1개를 뽑고, 지인 투표로 수요까지 확인하는 도구.**

- 문제: AI 코딩 시대의 첫 병목은 개발이 아니라 ① 바로 실행할 만큼 구체적인 아이디어 ② 만들기 전 수요 확신.
- 포지셔닝 금칙어: **"아이디어 생성기 · AI 추천 · 랜덤"** — 순수 생성의 지불의사는 0(리서치 실증). 서사는 항상 **"내 것(🌱) → 오늘 바로 → 지인에게 물어보기"**. 냉소 진영의 처방("툴 말고 유저와 대화하라")이 곧 투표 카드다.

## 타겟

| 페르소나 | 후크 | 주 결제 |
|---|---|---|
| P1 지민(막막형, 러닝) | 씨앗을 넣는 순간 결과가 전부 내 얘기 | 실행 플랜 990 |
| P2 현우(개발자, 확신 부족) | 투표 카드 = 수요 조사 도구 | 수요 리포트 |
| P3 소연(취향형, 인스타) | 씨앗 고정 + 스토리에 올릴 만큼 예쁜 카드 | 오늘 패스 1,900 |
| P4 태호(도메인 15년, 시간 빈곤) | 업계 단톡방 수요 투표 | 수요 리포트(9,900 실험) |

## 확정 원칙 (변경 시 사용자 승인)

1. **입력은 전부 객관식 칩**, 주관식은 맨 끝 탈출구(`✏️ 직접 쓸래요`).
2. **슬롯은 빈 슬롯 모델**(2026-07-06 확정, 정적 데모 v7 충실): 필수 4칸(🌱씨앗·😖불편·📦형태·🎬장면) + 옵션 💭마음(전체 뽑기 시 ~35%만 등장). 빈 시작 → 덱에서 한 장씩 채움 / 채워진 카드 탭=그 축만 교체 / ✕=제거 / 🎲전체 다시 뽑기(유일한 캡 소모). **curAxis 게이트**: 빈 필수 칸이 남은 동안 그 칸이 조준(펄스)되고 덱 전체가 그 칸으로 향함 — 빈 칸부터 순서대로 채워진다. **씨앗 고정 릴 대신 취향 질문 시트**(첫 아이디어는 무방해 완성 — 두 번째 전체 뽑기부터 미설정 시 가끔(40%) 등장, 제출/스킵 후 재질문 없음. 좋아하는/잘아는→대분류) — 이후 덱·뽑기가 취향 80%/다양성 20% 가중. **불편·형태는 씨앗(또는 취향) allowlist 앵커 준수**(러닝에서 물류 불편이 주류면 실패), 골든 조합 일치 시 ✨타이틀. **레이아웃/인터랙션은 정적 데모 원본과 동일**(2026-07-07 확정): 경계 있는 스테이지 박스 + `rotateX(8deg)` 틸트 테이블 + 스테이지 내장 부채꼴 덱 + 떠오르는 결과 오버레이 + 🔒고정·한 번에 뽑기·카드 아트. **스킨만 다크·Aurora·글래스로 유지**(원본 라이트 색 미채택).
3. **투표·공유는 무료**(바이럴 K), **결제는 상세·추가에서**: ① 스핀 캡→실행 패스 1,900 ② 확정 직후→플랜 990 ③ 발행 선예약+대시보드→수요 리포트 1,900~9,900. **4주차까지 전부 가짜 문**(클릭·이메일만 측정), 구독 없음.
4. **로그인은 카드 발행(S3) 단 한 곳**(구글 원탭). **투표 수신자에겐 로그인·설치 요구 0자.**
5. 디자인 = **미니멀 센터 컬럼**(2026-07-06 확정: 전 화면 중앙 정렬 단일 컬럼 max 560px·모바일 우선·화면당 primary CTA 1개·pill 버튼·짧은 카피 — moonlight/tarotoo 문법, 히어로는 simplicity f11 프레임: 검은 배경+중앙 9:16 카드+pill 하나. 단 인터랙션은 정적 데모처럼 화려하게 유지) × **다크(#0a0a0b) × glassmorphism 2.0**(앰비언트 오브 + `.glass/.glass-strong/.glass-gold`) × Moonlight 카드 인터랙션 × **toss simplicity급 인트로**(원 컨티뉴어스 샷). **주요 색상 = Aurora Mesh**(2026-07-06 확정): 좌하단 그린·시안 → 우상단 로열블루·네이비(파랑 ~75% 지배) 그라데이션을 범용 `.aurora` 클래스로 — 주요 CTA·히어로/공유 카드·앰비언트에 적용, 토큰은 `--aurora-*` CSS 변수(테마 교체 가능). 골드는 씨앗🌱·선택 상태의 보조 액센트로 유지. 애니메이션 opacity/transform만, `prefers-reduced-motion` 존중.
