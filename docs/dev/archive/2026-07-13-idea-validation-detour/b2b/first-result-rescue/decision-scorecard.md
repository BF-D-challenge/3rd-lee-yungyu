# 첫결과 구조대 — Pass / Kill 판정표

판정일: `[YYYY-MM-DD]`  
표본: 적합 강사 5명. 타겟 밖 인터뷰는 분모에서 제외하고 대체 모집한다.

## 사전 고정 정의

- 문제 인정: 최근 3개월 실제 코호트에서 D+7 첫 결과물 미제출자를 찾거나 놓친 사례를 날짜·흐름과 함께 설명함.
- 수기 결과 사용: 목록을 받고 24시간 안에 최소 1명에게 실제 확인 행동을 함.
- 선결제: 표시 가격 9,900원을 실제 결제함. 결제 링크 동의·무료 신청·구두 약속은 제외함.
- 외부 결과물 문제: LMS 진도율만으로는 찾을 수 없는 사진·문서·링크 등 강의 밖 산출물 제출이 핵심임.

## 최종 판정

### PASS

위 5개 기준을 **모두** 충족한다. 다음 단계는 API 없는 3화면 MVP이며 가격을 낮추지 않는다.

### HOLD

문제 인정·사용 기준은 통과했지만 선결제가 1명이다. 같은 타겟 5명을 한 번만 추가 모집해 동일 가격으로 재검증한다. 문구 수정은 가능하지만 기능 추가·할인은 금지한다.

### KILL

아래 하나라도 해당하면 종료한다.

- 선결제 0명
- 최근 실제 문제 인정이 2명 이하
- LMS·폼의 기존 미응답 목록으로 충분하다는 응답이 3명 이상
- 결과물 내용이나 수강생 개인정보를 수집해야만 가치가 생김
- 수기 목록을 받고도 실제 확인 행동을 한 운영자가 2명 이하
- 통과를 위해 가격 할인·조교 대행·AI 채점 등 다른 제품으로 범위를 넓혀야 함

## Lost Reason 집계

`no_problem / existing_tool / no_budget / no_trust / privacy / timing / other` 중 첫 이유 하나만 집계한다. 가장 많은 거절 이유를 제품 방향을 바꿀 근거로 쓰되, 선결제 기준 자체는 바꾸지 않는다.
