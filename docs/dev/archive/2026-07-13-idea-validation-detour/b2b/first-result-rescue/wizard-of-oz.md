# 첫결과 구조대 — Wizard-of-Oz 수기 운영

## 제공 범위

- 입력 1개: 결제자 임시 ID 명단 CSV
- 보조 상태: 운영자가 정한 첫 결과물 제출 여부
- 결과 1개: D+7 미제출자 ID 목록과 개인 확인 메시지
- 하지 않는 일: 결과물 채점, 수강생 연락 대행, 환불 예측, LMS·결제 API 연동

## 개인정보 최소화

- `S001` 같은 임시 ID만 받는다. 이름·이메일·전화번호·결제 금액은 받지 않는다.
- 결과물 원본 대신 `submitted_at` 유무만 받는다.
- 운영자에게 ID 매핑표를 보내지 말라고 안내한다.
- 원본과 결과 파일은 실험별 폴더에 분리하고, 결과 전달 14일 뒤 삭제한다.
- 삭제 요청이 오면 즉시 삭제하고 로그에는 `deleted` 상태만 남긴다.

## 운영 절차

1. **범위 확인**: 코호트 시작일, D+7 날짜, 첫 결과물의 한 문장 정의, 예상 인원, 결과 전달일을 적는다.
2. **유료 예약 확인**: 가격 9,900원, 수기 제공, 환불 조건, 삭제일을 다시 보여준다. 결제 링크는 `[placeholder]` 상태이며 이 문서만으로 결제를 받지 않는다.
3. **입력 수령**: `student_id,paid_at,submitted_at` 세 열만 있는 CSV를 받는다.
4. **형식 검사**: 중복 ID, 빈 `paid_at`, 날짜 형식을 확인한다. 오류가 있으면 추측하지 않고 운영자에게 돌려보낸다.
5. **목록 생성**: `paid_at + 7일 <= 기준시각`이고 `submitted_at`이 빈 ID만 추린다.
6. **수동 검수**: 두 번 세어 총 결제자 수·제출자 수·미제출자 수가 맞는지 확인한다.
7. **결과 전달**: 미제출자 ID, D+7 경과일, 복사 가능한 확인 메시지만 제공한다.
8. **사용 확인**: 24시간 뒤 운영자가 실제로 확인 메시지를 보냈는지, 목록 오류가 있었는지 묻는다.
9. **종료·삭제**: 전달 14일 뒤 입력·결과 파일을 삭제하고 삭제 완료만 기록한다.

## 참가자 메시지 템플릿

### 파일럿 시작

> `[코호트명]` 수기 파일럿 범위를 확인합니다. 가격은 1회 9,900원이며, 자동화 제품이 아니라 운영자가 수기로 D+7 미제출자 목록을 만듭니다. `student_id, paid_at, submitted_at`만 받고 이름·연락처·결과물 원본은 받지 않습니다. `[전달일]`까지 목록을 제공하지 못하면 전액 환불하며, 파일은 `[삭제일]`에 삭제합니다. 동의하시면 `[예약/결제 링크 placeholder]`로 진행해주세요.

### 입력 요청

> 첨부 템플릿의 세 열만 보내주세요: `student_id, paid_at, submitted_at`. ID는 `S001`처럼 바꾸고 실제 이름과 연결한 표는 보내지 마세요. 제출하지 않은 사람의 `submitted_at`은 비워주세요.

### 입력 오류

> 추측해서 처리하지 않기 위해 확인이 필요합니다. `[ID]`의 `[필드]`가 비어 있거나 형식이 다릅니다. 해당 셀만 수정해 다시 보내주세요. 다른 개인정보는 추가하지 말아주세요.

### 결과 전달

> `[기준일]` 기준 결제자 `[N]명` 중 첫 결과물 D+7 미제출자는 `[N]명`입니다. 첨부 목록에는 임시 ID와 경과일만 있습니다. 아래 확인 문장을 복사해 운영 중인 채널에서 직접 보내주세요. 결과물의 품질이나 환불 가능성은 판정하지 않았습니다.

### 24시간 후 사용 확인

> 어제 받은 미제출자 목록으로 실제 확인 메시지를 보내셨나요? `전원/일부/보내지 않음` 중 하나와, 보내지 않았다면 가장 큰 이유 하나만 알려주세요. 제품 칭찬보다 실제 행동을 기록하려는 질문입니다.

### 삭제 완료

> 약속한 `[삭제일]`에 입력 CSV와 결과 파일을 삭제했습니다. 결제·판정 장부에는 익명 실험 ID와 상태만 남습니다.
