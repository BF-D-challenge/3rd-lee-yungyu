# 펫시터 인계팩 — Pass / Kill 판정표

판정일: `[YYYY-MM-DD]`  
표본: 적합 독립 펫시터 5명. 앱 소속 보고만 하고 직접 예약이 없는 인터뷰이는 분모에서 제외하고 대체 모집한다.

## 사전 고정 정의

- 반복 문제: 최근 30일 실제 돌봄에서 요청 재확인·보고 작성·누락·재문의 중 하나가 발생했고 구체적인 마지막 건을 설명함.
- 주간 반복: 앱 밖 직접 예약 보고를 주 1회 이상 보냄.
- 수기 결과 사용: 제작 링크를 실제 보호자에게 기존 채널로 발송함.
- 선결제: 월 9,900원을 실제 결제함. 무료 사용·결제 링크 동의·구두 약속은 제외함.
- 무가입: 보호자가 새 계정이나 앱 설치 없이 링크를 확인함.

## 최종 판정

### PASS

위 6개 기준을 **모두** 충족한다. 다음 단계는 요청표·기록·결과 링크의 3화면 MVP이며 예약·결제 중개는 추가하지 않는다.

### HOLD

문제·발송 기준은 통과했지만 선결제가 1명이다. 같은 타겟 5명을 한 번만 추가 모집해 동일 월 가격으로 재검증한다. 할인이나 방문 돌봄 중개 기능은 추가하지 않는다.

### KILL

아래 하나라도 해당하면 종료한다.

- 선결제 0명
- 주 1회 이상 반복하는 펫시터가 2명 이하
- 기존 카톡 템플릿·중개 앱 돌봄일지로 충분하다는 응답이 3명 이상
- 수기 링크를 실제 보호자에게 보낸 펫시터가 2명 이하
- 보호자가 별도 앱 가입을 요구받음
- 주소·출입 비밀번호·의료 판단을 수집해야만 핵심 가치가 생김
- 통과를 위해 예약 중개·결제·GPS 추적 등 더 큰 제품으로 범위를 넓혀야 함

## Lost Reason 집계

`no_problem / existing_tool / no_budget / no_trust / privacy / client_preference / timing / other` 중 첫 이유 하나만 센다. 가장 많은 거절 이유는 후속 판단에 사용하되 선결제 기준은 바꾸지 않는다.
