# 펫시터 인계팩 — Wizard-of-Oz 수기 운영

## 제공 범위

- 입력 1개: 돌봄 요청표 1개
- 결과 1개: 요청별 완료·미완료, 실행 메모, 사진이 있는 보호자 확인 링크 1개
- 하지 않는 일: 예약·결제 중개, 위치 추적, 건강·투약 판단, 보호자 연락 대행, 출입 정보 저장

## 개인정보 최소화

- 펫시터는 `J001`, 반려동물은 `PET01`, 보호자는 `C001` 같은 임시 ID만 쓴다.
- 주소·공동현관/도어락 번호·열쇠 위치·전화번호·결제 정보는 절대 받지 않는다.
- 투약은 `요청됨/완료/미완료`만 표시하고 약명·용량을 이 도구에서 해석하거나 추천하지 않는다. 실제 지시는 보호자와 기존 채널에서 확인한다.
- 사진은 위치정보를 제거한 최소 장수만 쓰고, 공개 공유나 학습 데이터에 사용하지 않는다.
- 링크는 추측하기 어려운 임시 URL과 만료일을 쓰며, 실험 종료 또는 요청 후 14일 안에 삭제한다.

## 운영 절차

1. **적합성 확인**: 앱 밖 직접 예약, 주 1회 이상 보고, 보호자에게 공유할 예정인 돌봄 한 건이 있는지 확인한다.
2. **범위·유료 예약 확인**: 월 9,900원, 수기 운영, 환불 조건, 금지 데이터, 삭제일을 다시 보여준다. 결제 링크는 `[placeholder]` 상태다.
3. **요청표 생성**: 임시 ID, 돌봄일, `식사/산책/배변/놀이/기타` 요청과 보호자 원문 메모만 받는다. 주소와 출입 정보 칸은 만들지 않는다.
4. **사전 확인**: 모호하거나 건강 판단이 필요한 요청은 링크에서 처리하지 않고 펫시터가 보호자에게 기존 채널로 재확인하게 한다.
5. **실행 기록**: 펫시터가 각 요청에 `완료/미완료/해당 없음`, 시각, 짧은 사실 메모, 선택 사진을 남긴다.
6. **수기 검수**: 요청 수와 결과 수를 맞추고 빈 상태를 표시한다. 누락을 완료로 추정하지 않는다.
7. **링크 제작**: 첫 화면에 전체 완료 수, 그 아래 요청별 원문과 결과를 나란히 배치한다. 건강 상태 평가는 넣지 않는다.
8. **펫시터 승인**: 보호자에게 보내기 전 펫시터가 사실·사진·개인정보를 최종 확인한다.
9. **실제 발송·사용 확인**: 펫시터가 기존 카톡으로 직접 발송하고, 24시간 뒤 보호자 확인·재문의 여부를 기록한다.
10. **종료·삭제**: 만료일에 요청표·사진·링크를 삭제하고 익명 행동 지표만 남긴다.

## 참가자 메시지 템플릿

### 파일럿 시작

> 첫 달 수기 파일럿은 월 9,900원입니다. 자동화 제품이 아니라 운영자가 요청별 결과 링크를 수기로 제작하며, 시작하지 않거나 약속한 링크를 제공하지 못하면 전액 환불합니다. 주소·출입 비밀번호·연락처·결제 정보는 받지 않고, 파일은 `[삭제일]`에 삭제합니다. 동의하시면 `[예약/결제 링크 placeholder]`로 진행해주세요.

### 요청표 작성 안내

> `[요청표 링크 placeholder]`에 임시 ID와 식사·산책·배변·놀이 등 돌봄 요청만 적어주세요. 주소·출입 번호·전화번호는 입력하지 마세요. 투약이나 건강 관련 지시는 이 링크로 새로 정하지 말고 보호자와 기존 채널에서 직접 확인해주세요.

### 모호한 요청 확인

> `[요청 번호]`는 완료 조건을 추측하기 어렵습니다. 이 링크에서는 판단하지 않고 `확인 필요`로 두었습니다. 돌봄 전에 보호자에게 기존 채널로 직접 확인해주세요.

### 결과 링크 검수 요청

> 보호자에게 보내기 전 `[결과 링크 placeholder]`를 확인해주세요. 요청 `[N]개` 중 완료 `[N]개`, 미완료 `[N]개`, 확인 필요 `[N]개`입니다. 사진의 위치정보·주소·출입 정보가 보이지 않는지도 최종 확인해주세요. 승인 후 기존 카톡으로 직접 보내주세요.

### 24시간 후 사용 확인

> 어제 링크를 실제 보호자에게 보냈나요? `보냄/보내지 않음`과 보호자의 `확인/재문의/응답 없음` 중 하나를 알려주세요. 칭찬보다 실제 사용 여부를 기록하려는 질문입니다.

### 삭제 완료

> 약속한 `[삭제일]`에 요청표·사진·결과 링크를 삭제했습니다. 결제·판정 장부에는 익명 실험 ID와 상태만 남습니다.
