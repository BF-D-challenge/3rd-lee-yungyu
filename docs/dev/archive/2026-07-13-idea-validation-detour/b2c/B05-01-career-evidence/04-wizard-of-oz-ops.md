# 경력성과증거팩 — Wizard-of-Oz 수기 운영

## 운영 범위

- 참가자 10명, 4주
- 참가자당 이력서 문장 최대 5개
- 첫 카드 SLA 24시간, 수정 1회
- 자동 문장 생성, 합격 예측, 사실 인증, 채용 추천은 하지 않는다.

## 안전한 증거 규칙

### 절대 받지 않는 자료

- 회사 내부 원본 문서, 사내 링크, 미공개 소스코드
- 고객 이름·연락처·계정 ID가 담긴 자료
- 계약서, 급여명세서, 미공개 매출 원장
- 접근권한이 참가자 개인에게 없는 자료

### 허용하는 대체 증거

| 등급 | 입력 | 카드 표시 |
|---|---|---|
| A | 공개 URL·공개 저장소·공개 출시 자료 | 공개 확인 가능 |
| B | 회사·고객·계정·절대액을 가린 본인 소유 캡처 | 비식별 자료, 외부 검증 안 됨 |
| C | 파일 없는 회고: 시점·비교 기준·역할·측정 방법 | 본인 진술, 외부 검증 안 됨 |

운영자는 사실을 인증하지 않는다. 숫자가 의심스럽거나 출처를 안전하게 설명할 수 없으면 수치를 빼고 행동·범위 중심으로 바꾼다.

## 단계별 절차

### 0. 참가 코드 발급

이름 대신 `CE-001` 형식의 코드를 발급한다. 결제 기록과 작업 자료의 참가 코드는 같게 쓰되, 연락처는 결제 도구 안에만 두고 운영 CSV에 복사하지 않는다.

### 1. 안전 안내와 입력 요청

```text
[CE-___] 첫 카드에 넣을 이력서 문장 1개를 보내주세요.

회사 내부 원본, 고객정보, 사내 링크는 보내지 마세요. 근거는 ① 공개 링크 ② 식별정보를 가린 캡처 ③ 파일 없는 본인 회고 중 하나로만 받습니다. 애매하면 파일 대신 “언제·무엇과 비교했고·내가 무엇을 했는지”만 적어주세요.
```

### 2. 입력 사전 검사

1. 파일마다 회사명, 고객명, 이메일, 전화번호, 사내 URL, 계정 ID, 절대 매출액을 확인한다.
2. 하나라도 남아 있으면 열람·복사하지 않고 참가자에게 되돌려 삭제 또는 대체 증거를 요청한다.
3. 받은 자료가 참가자 본인 소유인지 확인 문구를 받는다.

```text
이 자료는 민감 정보가 남아 있어 사용하지 않겠습니다. 원본을 다시 보내지 말고, 표시된 항목을 가린 캡처 또는 파일 없는 회고로 바꿔주세요. 현재 전달본은 삭제 처리합니다.
```

### 3. 증거 카드 수기 제작

고정 구조만 채운다.

```text
[이력서 문장]
[내 역할: 동사로 시작, 최대 2개]
[측정 기준: 비교 대상·기간·단위]
[근거 1~3: A/B/C 등급 + 안전한 위치]
[30초 설명: 상황 → 행동 → 결과 → 한계]
[확인 필요: 과장·누락 가능성이 있는 한 항목]
```

새로운 성과나 숫자를 추론하지 않는다. 참가자가 제공하지 않은 역할을 추가하지 않는다.

### 4. 전달과 수정

```text
[CE-___] 첫 증거 카드가 준비됐습니다: [개인 링크 placeholder]

면접에서 30초 안에 설명 가능한지 읽어보세요. 사실과 다른 부분은 직접 고쳐주세요. 회사 자료를 추가로 보내지 않고도 한 번 수정할 수 있습니다.
```

### 5. 실제 사용 확인

카드 전달 후 72시간, 실제 지원 또는 면접 다음 날에 각각 묻는다.

```text
카드를 받은 뒤 실제로 한 행동 하나만 알려주세요.
1) 지원 전에 열어봄 2) 면접 전에 열어봄 3) 면접 답변에 사용 4) 아직 사용 안 함
사용했다면 근거를 찾는 데 걸린 시간만 숫자로 답해주세요.
```

### 6. 재사용·결제 확인

```text
다음 지원에서도 새 성과 문장을 카드로 만들겠나요? 계속 쓰려면 4주 패스 4,900원입니다. [결제 링크 placeholder]
결제하지 않는다면 가장 큰 이유 하나만 알려주세요: 문제 없음 / 기존 도구 충분 / 가격 / 기밀 우려 / 결과 불신 / 지금 시기 아님 / 기타.
```

### 7. 삭제

- 거절된 민감 파일: 발견 즉시 삭제하고 시간만 기록
- 허용된 작업 입력: 카드 전달 7일 뒤 삭제
- 결과 카드: 파일럿 종료 7일 뒤 삭제, 참가자가 보관할 링크를 먼저 제공
- 삭제 요청: 24시간 안에 처리
- 원본 내용은 CSV에 적지 않고 삭제 시각과 위반 여부만 기록

## 환불 처리 메시지

```text
환불 요청을 접수했습니다. 첫 카드 전달 전이면 4,900원 전액, 전달 후이면 제공하지 않은 카드 수에 해당하는 금액을 환불합니다. 예약금은 첫 자료 제출 전 취소 시 전액 환불됩니다. 처리 예정: [날짜 placeholder].
```
