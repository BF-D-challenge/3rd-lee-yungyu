# 경력성과증거팩 — Pass·Kill 판정표

## 집계 규칙

- 분모는 `3개월 안에 실제 지원 예정`이며 최근 6개월 이내 지원 행동이 있는 적격 참가자 10명이다.
- 말로 한 결제 의향은 결제로 세지 않는다.
- 1,000원 예약금은 보조 신호일 뿐, 4,900원 정가 결제에 포함하지 않는다.
- 운영자·지인 할인, 테스트용 0원 결제는 제외한다.
- 경력 사실의 진위가 아니라 `안전한 근거 연결`에 대한 행동만 평가한다.

## 판정 지표

| 게이트 | Pass | Hold | Kill |
|---|---:|---:|---:|
| 최근 실제 고통 | 10명 중 5명 이상이 최근 지원에서 근거 탐색·문장 약화 경험 | 3~4명 | 0~2명 |
| 입력 행동 | 5명 이상이 기밀 없이 안전한 입력 제출 | 3~4명 | 0~2명 또는 기밀 없이는 불가능 |
| 결과 사용 | 3명 이상이 실제 지원·면접 준비에 카드 사용 | 1~2명 | 0명 |
| 정가 결제 | **3명 이상이 4주 4,900원 실제 결제** | 1~2명 | 0명 |
| 차별성 | 결제자 과반이 문장 생성이 아닌 누적 근거 연결을 이유로 선택 | 혼재 | 과반이 ChatGPT·노션으로 충분하다고 답함 |
| 재사용 | 2명 이상이 두 번째 문장을 만들거나 다음 지원 때 다시 사용 | 1명 | 0명 |
| 안전성 | 기밀 자료 없이 전원 운영 가능, 중대 사고 0건 | 반복적인 재안내 필요 | 원본 회사 자료가 핵심 입력이거나 삭제 사고 1건 이상 |

## 최종 결정

### Experiment Pass

다음을 모두 만족한다.

1. 최근 실제 고통 5/10 이상
2. 실제 정가 결제 3/10 이상
3. 결과 실제 사용 3/10 이상
4. 두 번째 사용 행동 2명 이상
5. 중대 개인정보·기밀 사고 0건

### Hold 후 한 번만 수정

- 정가 결제 1~2명이고 거절 이유 1위가 가격 또는 시기일 때만 메시지·세그먼트를 한 번 수정한다.
- 예약금은 3명 이상이지만 정가 결제가 0명이면 가격 차이가 아니라 약한 약속으로 보고 Pass하지 않는다.
- 수정 실험은 같은 분모 10명을 새로 모집하며 기존 응답을 합산하지 않는다.

### Kill

다음 중 하나면 종료한다.

- 적격 10명에게 제안했는데 정가 결제 0명
- 5명 미만이 안전한 근거를 제출
- 과반이 문장 생성·기존 폴더·노션으로 충분하다고 답함
- 회사 기밀 원본 없이는 핵심 결과를 만들 수 없음
- 자료 반출·허위 경력 인증 기대가 반복되거나 중대 안전 사고 발생
