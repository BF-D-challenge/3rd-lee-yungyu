# 부모님공문 — Pass·Kill 판정표

## 집계 규칙

- 분모는 `아이 둘 이상 또는 공지 채널 3개 이상`이며 지난 4주에 실제 공지를 처리한 적격 10가구다.
- “4주 연속 사용”은 매주 공지 제출·카드 열람·완료 체크를 모두 한 가구만 센다.
- 말로 한 결제 의향과 1,000원 예약금은 정가 결제로 세지 않는다.
- 가족 2인 사용은 서로 다른 두 사람이 링크를 열고 최소 한 번씩 담당 또는 완료를 바꾼 경우만 센다.
- 개인정보 원본을 오래 보관해야만 만들 수 있는 결과는 실패로 본다.

## 판정 지표

| 게이트 | Pass | Hold | Kill |
|---|---:|---:|---:|
| 최근 실제 고통 | 10가구 중 5가구 이상이 최근 4주 내 누락·재탐색·가족 재확인 경험 | 3~4가구 | 0~2가구 |
| 횡단 문제 | 5가구 이상이 아이·기관·채널을 3개 이상 오감 | 3~4가구 | 과반이 학교 앱 하나로 충분 |
| 4주 연속 사용 | **3가구 이상** | 1~2가구 | 0가구 |
| 가족 공동 사용 | 3가구 이상에서 두 번째 가족이 실제 체크 | 1~2가구 | 0가구 |
| 정가 결제 | **2가구 이상이 4주 4,900원 실제 결제** | 1가구 | 0가구 |
| 자발 재사용 | Week 3에 운영자 재촉 없이 3가구 이상 제출 | 1~2가구 | 0가구 |
| 개인정보 안전 | 원본 100%가 결과 전달 후 24시간 내 삭제, 중대 사고 0건 | 재가림 요청이 많으나 SLA 충족 | 삭제 SLA 위반 또는 중대 노출 1건 이상 |

## 최종 결정

### Experiment Pass

다음을 모두 만족한다.

1. 최근 실제 문제 5/10가구 이상
2. 4주 연속 사용 3/10가구 이상
3. 실제 정가 결제 2/10가구 이상
4. 두 번째 가족 구성원 실제 사용 3가구 이상
5. 과반이 학교 앱 하나가 아니라 여러 아이·채널 통합을 핵심 가치로 선택
6. 원본 삭제 SLA 100%, 중대 개인정보 사고 0건

### Hold 후 한 번만 수정

- 연속 사용 1~2가구 또는 정가 결제 1가구이고, 거절 이유 1위가 운영 요일·가격처럼 좁은 변수일 때만 한 번 수정한다.
- 예약금만 3가구 이상이고 정가 결제가 0이면 약한 약속으로 보고 Pass하지 않는다.
- 수정 실험은 새 가구 10곳으로 실행하며 기존 숫자를 합산하지 않는다.

### Kill

다음 중 하나면 종료한다.

- 적격 10가구에게 제안했는데 정가 결제 0가구
- 4주 연속 사용 0가구
- 과반이 학교·학원 앱 하나 또는 기존 가족 캘린더로 충분하다고 답함
- 가족의 두 번째 사용자가 한 명도 실제 체크하지 않음
- 원본 스크린샷을 장기 보관하거나 학교 계정 로그인이 핵심 가치에 필요
- 삭제 SLA 위반 또는 중대 개인정보 사고 발생
