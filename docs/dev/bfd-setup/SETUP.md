# ⚙️ 세팅 안내 (처음 한 번만)

이 레포는 **세팅 도구 모음**이에요. 이 폴더를 클로드(Claude Code)로 연 다음, 이렇게 말하세요:

> ```
> 세팅해줘
> ```

그러면 클로드가 **본인 베프디 작업 레포**에 챌린지에 필요한 기능과 보안 설정을 넣어줘요.

- **🔐 보안 설정** — 클로드가 실수로 비밀 값(토큰·비밀번호)에 접근하거나 위험한 명령(강제 푸시·전체 삭제 등)을 못 하도록 차단룰을 걸어요. **(가장 중요, 모두 필수)**
- **safe-push** — 안전하게 깃에 올려주는 기능. 설치 후 **"올려줘"** 한마디면 올리기 전에 "이건 올릴까요? 무시할까요?"를 물어봐 줍니다.
- **prd-maker** — 4주 챌린지 PRD를 같이 깎아주는 기능. 새 프로젝트 시작할 때 **"PRD 짜줘"** 라고 하세요.
- **자동 배포 파일** — `.github/workflows/deploy.yml`·`.gitignore`·`.env.example` 을 작업 레포에 넣어, **"올려줘"만 하면 Vercel에 자동 배포**되게 해줘요.

> 💡 스킬과 보안 설정은 **컴퓨터 전역이 아니라 본인 작업 레포 안(`.claude/`)에만** 들어가요. 그래서 개인 컴퓨터 설정을 건드리지 않고, 레포와 함께 다녀서 재클론해도 그대로예요.

## 🤖 (클로드용) 설치 지침

사용자가 이 레포 폴더에서 **"세팅해줘"** / **"환경 세팅해줘"** / **"설치해줘"** 라고 하면, 아래를 순서대로 수행하세요.

> ⚠️ **핵심 원칙**
> - 대상은 **비개발자**입니다. 전문용어 대신 쉬운 말로 안내하세요.
> - 모든 설치는 **컴퓨터 전역(`~/.claude`)이 아니라, 사용자의 베프디 작업 레포 안(`<작업레포>/.claude/`)** 에만 합니다. 전역 설정은 절대 건드리지 마세요.
> - **1.5단계 보안 설정은 절대 건너뛰지 마세요.** 이게 이 세팅의 가장 중요한 목적입니다.
> - 사용자 확인 없이 기존 설정 파일을 덮어쓰지 마세요.

### 0. 작업 레포 위치부터 물어보기 (제일 먼저)

세팅은 **사용자의 베프디 작업 레포**를 대상으로 합니다. 지금 열려 있는 이 폴더(BF-D-setting)는 도구일 뿐이에요.

먼저 이렇게 물어보세요:

> "세팅을 넣을 **본인 베프디 작업 레포**가 어디예요? 둘 중 하나로 알려주세요:
> ① 이미 컴퓨터에 받아뒀으면 **그 폴더 경로**
> ② 아직이면 **깃허브 주소** (제가 받아드릴게요)"

- ②를 주면 사용자가 원하는 위치에 `git clone <주소>` 로 받아준 뒤, 그 폴더를 작업 레포로 씁니다.
- 확인된 작업 레포의 절대경로를 아래 지침에서 `<작업레포>` 로 씁니다. (아래 bash 예시의 `WORK` 변수에 넣으세요.)
- ⚠️ 이 BF-D-setting 폴더 자신을 작업 레포로 착각하지 마세요. 둘은 **다른 폴더**입니다.

### 1. 스킬을 작업 레포에 설치

이 레포 안의 `skills/` 아래 스킬들을 **작업 레포의 `.claude/skills/`** 로 복사합니다.
**아래 명령을 그대로 쓰세요.** (재설치해도 안전 — 스킬 설명은 최신으로 갱신하고, 개인 무시 목록은 보존합니다.)

```bash
WORK="/여기에/작업레포/절대경로"   # 0단계에서 확인한 작업 레포 경로

# safe-push (깃 안전하게 올리기)
SAFE="$WORK/.claude/skills/safe-push"
mkdir -p "$SAFE/config"
cp skills/safe-push/SKILL.md "$SAFE/SKILL.md"
# 개인 무시 목록은 이미 있으면 절대 건드리지 않고, 없을 때만 새로 만듦
[ -f "$SAFE/config/ignore-list.json" ] || cp skills/safe-push/config/ignore-list.json "$SAFE/config/ignore-list.json"

# prd-maker (4주 챌린지 PRD 같이 깎기)
PRD="$WORK/.claude/skills/prd-maker"
mkdir -p "$PRD"
cp skills/prd-maker/SKILL.md "$PRD/SKILL.md"

echo "스킬 복사 완료:"
ls "$SAFE" "$PRD"
```

- ⚠️ `cp -R skills/safe-push "$SAFE"` 같은 폴더 통째 복사는 **쓰지 마세요.** 이미 폴더가 있으면 `safe-push/safe-push/`처럼 중첩돼 스킬이 깨집니다. 위처럼 **파일 단위로** 복사하세요.
- 경로에 특정 사람 이름이 든 절대경로를 하드코딩하지 말고, 0단계에서 확인한 `WORK` 변수를 쓰세요.

### 1.5. 보안 설정 적용 (가장 중요 ⚠️ — 절대 건너뛰지 말 것)

작업 레포의 **`<작업레포>/.claude/settings.json`** 에 보안 차단룰을 넣습니다. 클로드가 실수로 민감 파일(.env·키)에 접근하거나 위험한 명령을 실행하지 못하게 막는, **이 세팅의 핵심**입니다.

- 적용 전 **"보안 차단룰을 넣을게요, 진행할까요?"** 한 번만 확인하세요.
- **파일이 없으면** `templates/claude-settings.json` 을 `<작업레포>/.claude/settings.json` 으로 그대로 복사하면 됩니다.
- **파일이 이미 있으면** bash로 덮어쓰지 말고, 파일을 **Read** 한 뒤 `permissions.deny` 배열에 **빠진 패턴만 추가**해서 안전하게 **Edit/Write** 하세요. (기존 설정 전부 보존, 중복 추가 금지.)
- 이 `.claude/settings.json` 은 **작업 레포에서만** 적용되고, 깃허브에 함께 올라가도 안전합니다(비밀 값이 아니라 차단룰 목록이라서).

넣어야 할 `permissions.deny` 패턴 (= `templates/claude-settings.json` 내용):
```
# 민감 파일 접근 차단
Read(.env)
Read(.env.*)
Read(*.pem)
Read(*.key)
Read(*.cert)
Read(credentials.json)
Read(service-account*.json)

# 환경 변수 노출 차단
Bash(env)
Bash(printenv)
Bash(*SECRET*)
Bash(*PASSWORD*)
Bash(*TOKEN*)
Bash(*CREDENTIAL*)

# 파괴적 명령 차단
Bash(rm -rf /*)
Bash(git push --force*)
Bash(git reset --hard*)
```

### 2. 자동 배포 파일을 작업 레포에 넣기

`templates/` 의 파일들을 작업 레포에 복사합니다. 이게 있어야 **"올려줘" → Vercel 자동 배포**가 됩니다.
(이미 있으면 덮어쓰기 전에 한 번 물어보세요. 사용자가 이미 만든 게 있을 수 있어요.)

```bash
# 자동 배포 워크플로우
mkdir -p "$WORK/.github/workflows"
cp templates/deploy.yml "$WORK/.github/workflows/deploy.yml"

# .gitignore (앞에 점을 붙여 복사)
[ -f "$WORK/.gitignore" ] || cp templates/gitignore "$WORK/.gitignore"

# .env.example 견본
[ -f "$WORK/.env.example" ] || cp templates/env.example "$WORK/.env.example"

echo "배포 파일 복사 완료:"
ls "$WORK/.github/workflows" "$WORK/.gitignore" "$WORK/.env.example" 2>/dev/null
```

> 💡 `.gitignore` 가 이미 있으면 덮어쓰지 말고, 빠진 항목(`.env`, `node_modules/` 등)만 있는지 확인하고 없으면 추가하세요.
> 💡 Vercel 실제 배포를 켜려면 시크릿 3개(`VERCEL_TOKEN`·`VERCEL_ORG_ID`·`VERCEL_PROJECT_ID`)를 **작업 레포**의 GitHub Settings → Secrets 에 등록해야 합니다. (README 참고) 안 해도 세팅은 완료되고, 배포만 조용히 건너뜁니다.

### 3. 깃 기본 준비 확인 (쉬운 말로 안내)

작업 레포 폴더(`$WORK`)에서 아래를 점검하고, 안 되어 있으면 도와주세요.
- 깃이 깔려 있는지 (`git --version`)
- 깃허브에 로그인돼 있는지 (`gh auth status`) — 안 되어 있으면 `! gh auth login` 을 직접 실행하도록 안내
- 깃 사용자 이름·이메일이 등록돼 있는지 (`git config --global user.name`, `user.email`) — 없으면 물어보고 등록
- **한글 파일 이름 깨짐 방지**: `git config --global core.quotepath false` 를 적용하세요. (이 설정이 없으면 `메모.txt` 같은 한글 파일명이 `"\352\260..."` 처럼 깨져 나와서, safe-push가 위험·개인 파일을 이름으로 알아보기 어려워집니다. 사실상 필수입니다.)
