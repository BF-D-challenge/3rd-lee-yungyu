## 원 프로그램 구조 (리버스 엔지니어링)

| 회차 | 원 프로그램 세션 | 담당 멘토 | 이 플레이북에서의 미션 |
|---|---|---|---|
| 1회차 (1주) | 프로덕트 세션 & 바이브 코딩 시작 | 윤민정 (프로덕트) | **팔리는 설계**를 먼저 그리고, 빌딩 환경을 준비한다 |
| 2회차 (2주) | 맞춤형 빌딩 세션, MVP 핵심 기능 빌딩 | 김시현 (구현·배포·결제) `[추정]` | **핵심 퍼널만** 동작하는 MVP를 만든다 |
| 3회차 (3주) | 완성 → 배포 → 판매 → 마케팅 | 셋 공동 `[추정]` | 세상에 내놓고 **첫 결제·첫 공유**를 만든다 |
| 4회차 (4주) | 콘텐츠 마케팅 전략 + 데모데이 | 네오 (마케팅) `[추정]` | 유입 채널을 실험하고 **숫자로 발표**한다 |

**왜 이 순서인가**: 원 프로그램은 "코딩부터"가 아니라 **"프로덕트 설계부터"** 시작합니다. 잘된 제품은 설계 시점부터 수익화·성장을 치밀하게 계획한 제품이었고(28:10), 무슨 기능이 필요하고 뭘 없애도 되는지는 업으로 해도 어렵기 때문(32:18). **1주차에 결제 위치와 공유 루프를 그려놓지 않으면 2주차에 뭘 뺄지 판단할 수 없어요.** 이 순서만 지켜도 절반은 따라간 겁니다.

## 1주차 — 팔리는 설계 그리기

**🎯 이번 주가 끝나면**: PRD 1장 + "돈 받는 곳"과 "공유 루프"가 표시된 퍼널 그림 1장이 있다.

### ✅ 체크리스트 (위에서부터 순서대로)
- [ ] ⚡ 위의 "오늘 당장" 3줄 완료
- [ ] 종이에 퍼널 그리기: `진입 → 가치 맛보기 → 💰결제 → 결과 → 📤공유 → 친구 진입`. 6칸을 내 제품으로 채운다
- [ ] 💰 위치 검사: 결제가 퍼널 **안**에 있나? "나중에 수익화"라고 쓰여 있으면 다시 (근거는 아래)
- [ ] 📤 문구 검사: 공유 카드에 뭐라고 뜨나? "나 이거 했다"(이기적)면 "너한테 ○○가 도착했어"(이타적)로 고친다
- [ ] 🔁 루프 검사: 공유받고 들어온 친구가 **다시 공유자가 되는 버튼**이 결과 화면에 있나?

### 💬 막히면 복붙
```
/minjung-coach 내 퍼널 그림이야: [6칸 내용 붙여넣기]. 결제 위치랑 공유 루프 뜯어줘
```

### 🧠 왜 이렇게 하나 (근거 — 안 읽어도 따라할 수 있음)
- **결제를 퍼널 안에**: 990원 사주는 대운 풀이가 결제 전엔 뚝 끊긴다(52:41). 혼자 만든 서비스는 재사용률이 낮으니, 계속 쓸 확률이 낮다면 **한 번 쓸 때 돈을 받아야** 돈 벌 가능성이 높다(57:13, 58:40~59:16). 토스의 "밸류 퍼스트"를 1인 창업자는 뒤집어야 한다(56:38).
- **이타적 공유**: "나 100원 받게 눌러줘"(X) → "친구가 주식을 선물했어요"(O). 받는 사람 입장에서 선물처럼 보여야 트래픽 창출력이 몇 배 바뀐다(48:57).
- **루프 연결**: 트리꾸미기는 메시지 쓴 직후 "내 트리 만들기"를 띄워 수신자를 발신자로 바꿨다(42:08). 많은 초대 제품이 여기가 끊겨 있다(50:44). 바이럴 = 광고비 0 = 런웨이(48:20).
- **재사용 꿈이 있다면**: 부정하지 말고 순서 조정 — 앱인토스 같은 고빈도 플랫폼에 올라타기(59:56~1:01:20, 대본이 여기서 끊김. 게임·콘텐츠형이 잘 된다).

## 2주차 — 핵심 퍼널만 만들기 `[추정]`

**🎯 이번 주가 끝나면**: 내 컴퓨터에서 `진입 → 가치 → 결제 잠금 → 결과 → 공유 카드`가 끝까지 클릭된다.

### ✅ 체크리스트
- [ ] 1주차 퍼널 그림에서 **6칸 밖 기능을 전부 지운다** (마이페이지·설정·히스토리·로그인… 대부분 안 필요)
- [ ] 클로드에게 아래 "빌딩 시작" 프롬프트 복붙 → 한 칸씩 만든다 (한번에 전부 시키지 말 것)
- [ ] 기능 하나 완성될 때마다 `올려줘` (= 커밋. 망하면 되돌릴 지점이 생긴다)
- [ ] 결제는 **잠금 화면까지만** — "990원 내면 열려요" 화면이 뜨면 이번 주는 합격 (실결제는 3주차)
- [ ] 공유 카드 이미지가 생성되고, 카드에 이타적 문구가 박혀 있다

### 💬 막히면 복붙
```
빌딩 시작: [퍼널 6칸 붙여넣기]. 이 순서대로 하나씩 만들자. 한 단계 끝날 때마다 나한테 보여주고 다음으로 가줘.
```
```
에러 났어: [에러 메시지 그대로 붙여넣기]
```
```
같은 버그 수정을 3번 넘게 시도했어. 지금까지 시도 다 버리고, 원인 후보 3개를 먼저 생각한 다음 깨끗한 상태에서 다시 고쳐줘.
```

### 🧠 왜 이렇게 하나
- **뺄 것부터 정하는 이유**: 무슨 기능이 필요한지 판단은 업으로 해도 어렵다(32:18). 처음부터 재사용 서비스를 내려는 욕심을 내지 마라(51:18).
- **한 칸씩 + 매번 커밋**: YC Tom Blomfield — "새 기능은 깨끗한 Git 상태에서 시작하고, AI가 산으로 가면 주저 없이 리셋하라. 실패한 수정을 겹겹이 쌓지 말고, 찾은 해법만 들고 깨끗한 코드에서 다시 적용하라" ([YC: vibe coding 팁](https://www.youtube.com/watch?v=BJjsfNO5JTo), 대본 밖 보강).
- **막히면 혼자 끙끙대지 않기**: 에러 대처가 비개발자의 제일 큰 벽이다(32:57, 34:48). 에러 메시지를 그대로 붙여넣는 게 가장 빠르다.

## 3주차 — 내놓고, 첫 결제 만들기 `[추정]`

**🎯 이번 주가 끝나면**: 누구나 들어올 수 있는 라이브 URL + 첫 결제 1건(또는 첫 공유 유입 1건)이 있다.

### ✅ 체크리스트

- [ ] 결제 연결 — 어려우면 **수동이라도**: 토스아이디 송금 링크·계좌이체 안내면 충분. 프라이싱은 990원처럼 심리 장벽 낮게(52:41)
- [ ] 내 폰 카톡으로 공유 카드를 **실제로 나에게** 보내보고, 받는 화면 스크린샷 확인 (990원 사주의 카톡 공유처럼, 39:25)

- [ ] 반응 기록: 몇 명이 열었나 / 몇 명이 결제했나 / 몇 명이 재공유했나

### 🧠 왜 이렇게 하나

## 4주차 — 채널 실험 + 숫자로 발표 `[추정]`

**🎯 이번 주가 끝나면**: 채널별 유입 숫자 기록 + 5분 발표 자료 1벌이 있다.

### ✅ 체크리스트
- [ ] 내 제품이 **스스로 만드는 콘텐츠**를 찾는다 — 사주 결과·MBTI 리포트처럼 결과물 자체가 공유거리(38:48, 55:20)
- [ ] 채널 2개 × 콘텐츠 1개씩 실험 (예: 인스타 릴스 1개, 커뮤니티 글 1개) — 어느 쪽 유입이 많은지만 기록
- [ ] 공유 문구 A/B: 이기적 버전 vs 이타적 버전, 클릭 수 비교(48:57)
- [ ] 아래 템플릿에 숫자 채우면 발표 자료 끝

### 📣 데모데이 발표 템플릿 (5분 — 빈칸만 채우세요)
1. **문제 & 타겟** (30초): "_____의 _____한 순간을 노렸습니다"
2. **라이브 데모** (2분): 진입 → 가치 → 결제 → 공유, 핵심 퍼널만
3. **설계 의도** (1분): "결제는 _____에 뒀고, 공유받은 사람은 _____로 다시 공유자가 됩니다" — 잘된 제품과 사라진 제품의 차이가 이 설계다(28:10)
4. **숫자** (1분): 방문 ___명 → 결제 ___건 (전환 __%), 공유율 __%. 작아도 숫자로 — "165만 명·50억"처럼 숫자가 이야기다(38:05)
5. **다음 4주** (30초): "_____를 빼고 _____를 키웁니다"

### 💬 막히면 복붙
```
데모데이 발표 자료 만들자. 내 숫자야: [방문/결제/공유 수치]. 위 5단 구조로 정리해줘.
```
