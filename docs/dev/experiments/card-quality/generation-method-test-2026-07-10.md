# 카드 생성 방식 소규모 재실험 (2026-07-10)

## 배경

이전 Claude 세션은 H0/H1/H2/H3 비교로 방향을 잡았다.

- H0: 현행 기계식 템플릿
- H1: 실제 앵커 이름과 매출 수치를 노출
- H2: 같은 아이디어지만 근거를 문체로만 암시
- H3: 타로 리딩 톤

그때의 결론은 "Sonnet + 린 프롬프트 + 실제 앵커 + 실명/숫자/상황을 앞에 둔다"였다. 다만 전량 생성은 끝까지 실행되지 않았고, 현재 `public/data/golden.json` 6,610장은 다시 기계식 흔적이 강하다.

이번 재실험의 목적은 화면을 더 만지기 전에, 남은 전량 재작성에 쓸 생성 방식을 다시 작게 확정하는 것이다.

## 평가 기준

각 후보를 5점 만점으로 본다.

- 니치 구체성: "나 이거 아는데" 싶은 좁은 사람/상황이 보이는가
- 오늘 실행감: 반나절 MVP로 실제 만들 수 있는가
- 의외성: 흔한 체크리스트/기록앱 재탕을 피했는가
- 자연스러움: 한국어 생활 문장으로 읽히는가
- 정합성: seed, pain, format이 서로 충돌하지 않는가

## 테스트 표본

| id | 조합 | 이유 |
|---|---|---|
| T1 | 러닝 + 지속 어려움 + 연속 기록 | 쉬운 소비자 니치 |
| T2 | 러닝 + 성과 안 보임 + 투표 카드 | 쉬운 조합이지만 형식 해석이 갈림 |
| T3 | 골프 + 스윙 봐달란 말 못 함 + 공유 링크 | 부정합을 피해야 하는 어려운 조합 |
| T4 | 카페 운영 + 반복 수동 + 대시보드 | 실무형 조합 |
| T5 | 간호 인수인계 + 반복 수동 + 공유 링크 | 전문 맥락 조합 |
| T6 | 보험 보장분석 + 비교 피로 + 계산기 | 민감한 전문 판단을 피해야 하는 조합 |

## 후보 방식

### A. 기계식 템플릿 유지

현재 데이터의 대표 문장:

> 점심 직후, 러닝 33개 뒤지다 지속 어려움에 식은땀, 연속기록표로 바로 정리돼 안심

판정:

- 문제: 숫자, 시간, 감정, 해결이 서로 붙어 있을 뿐 실제 사람이 떠오르지 않는다.
- 문제: "골프 79개 뒤지다", "간호 인수인계 25개 뒤지다"처럼 맥락이 깨진다.
- 점수: 1.8 / 5

결론: 전량 재작성 대상. 기준선으로만 둔다.

### B. 앵커 선행형

생성 순서:

1. 조합을 먼저 "사람의 실제 장면"으로 번역한다.
2. 비슷한 해외 제품 앵커에서 검증된 메커니즘만 빌린다.
3. oneliner 앞부분에 사람/장면을 놓고, 뒤에 도구 형태를 붙인다.
4. evidence에는 앵커와 금액을 짧게 노출한다.

샘플:

- 러닝 지속 어려움: "혼자 뛰다 3일째 끊긴 초보 러너가 쉬어도 실패로 보이지 않는 7일 러닝 달력을 만든다"
- 골프 스윙 민망함: "동호회 단톡엔 못 올린 스윙 영상을 익명 링크로 보내 낯선 골퍼에게 한마디만 받는다"
- 카페 운영 반복 수동: "마감 때마다 손으로 적는 원두 재고와 폐기 수량을 하루 매출 옆에 붙여보는 카페 마감판"
- 간호 인수인계 반복 수동: "매번 새로 쓰는 인수인계 문장을 환자별 체크 링크로 넘겨 누락만 먼저 보이게 한다"

판정:

- 장점: 사람, 상황, 형식이 바로 보인다.
- 장점: 근거를 숨기지 않아서 "그럴듯함"이 생긴다.
- 리스크: 앵커가 엉뚱하면 오히려 신뢰가 깨진다. 앵커는 카테고리보다 메커니즘 유사성을 우선한다.
- 점수: 4.4 / 5

결론: 최종 기본 방식.

### C. 운세/리딩 선행형

생성 순서:

1. 카드 앞면부터 운세처럼 쓴다.
2. 고통을 은유적으로 말한다.
3. 뒷면에서 해결 도구를 밝힌다.

샘플:

- "오늘의 카드는 끊긴 리듬을 탓하지 말라고 말합니다. 당신에게 필요한 것은 의지가 아니라 다시 켤 수 있는 작은 표시입니다."

판정:

- 장점: 카드 컨셉에는 잘 맞는다.
- 문제: 앞면만 보면 어떤 앱을 만들지 늦게 보인다.
- 문제: 6,610개 전량에 쓰면 문장이 쉽게 비슷해진다.
- 점수: 3.5 / 5

결론: `frontStory`와 상세 뒷면 톤에는 사용하되, 핵심 oneliner 생성 방식으로 쓰지 않는다.

### D. MVP 선행형

생성 순서:

1. MVP 4개를 먼저 정한다.
2. 그 MVP를 한 줄 설명으로 압축한다.
3. 근거를 뒤에 붙인다.

샘플:

- "오늘 했음, 쉼, 메모, 공유 링크가 있는 러닝 기록판"

판정:

- 장점: 만들기는 쉽다.
- 문제: 아이디어가 기능 목록처럼 보이고 사용자가 사라진다.
- 문제: 의외성이 낮다.
- 점수: 3.1 / 5

결론: `buildPrompt`와 `mvp` 작성 순서에는 유용하지만, 카드 카피의 시작점으로 쓰지 않는다.

## 확정 방식

최종 방식은 **B. 앵커 선행형 + C의 리딩 톤 일부 + D의 MVP 검산**이다.

생성 순서는 다음으로 고정한다.

1. 조합을 실제 사람의 하루 장면으로 바꾼다.
2. pain을 "불편한 감정"이 아니라 "반복되는 행동 실패"로 쓴다.
3. format을 기능명이 아니라 사용 흐름으로 번역한다.
4. 앵커는 카테고리보다 메커니즘 유사성을 우선한다.
5. oneliner는 사람/상황 → 해결 흐름 → 도구 형태 순서로 쓴다.
6. evidence는 짧게 노출한다. 숨기지 않는다.
7. frontStory는 운세 톤을 쓰되, 최소 2단계는 실제 고통 장면이어야 한다.
8. MVP 4개는 전부 클릭/입력/공유 가능한 화면 요소로 쓴다.
9. 같은 seed 배치 안에서는 카드마다 차별 축을 먼저 정한다.
10. 금지: "N개 뒤지다", "식은땀", "{seed} 하다가 {pain}에 막히는 사람", 범용 기록앱, 범용 체크리스트, 의미 없는 대시보드.

## 차별성 게이트

카드가 많아질수록 `사람/상황 → 문제 → 도구 → 안심` 문장 구조가 반복되어 서로 닮아 보인다. 그래서 배치 작성 전에 각 카드의 **차별 축**을 고정한다.

카드마다 아래 5개 중 최소 3개가 달라야 한다.

- 심리 원리: 예) Abstinence Violation Effect, Evaluation Apprehension, Choice Overload, Social Accountability, Self-Efficacy
- 사회적 맥락: 혼자 회복, 친구에게 자랑, 낯선 사람에게 익명 질문, 소규모 모임 조율, 전문가에게 전달
- 결과물 형태: 회복 기록판, 반응 카드, 매칭 목록, 진단 계산기, 공유 브리프, 요약 대시보드
- 감정 해소: 안도, 용기, 명확함, 인정, 소속감, 재시작감
- 앵커 메커니즘: 기록 유지, 피드백 수집, 후보 비교, 추천 필터링, 진행률 시각화

배치 단위 하드룰:

- 20장 안에서 같은 심리 원리는 최대 4장까지만 쓴다.
- 같은 oneliner 구조는 최대 3장까지만 허용한다.
- 같은 이름/인물/시간대는 반복하지 않는다.
- 같은 seed라도 "기록", "공유", "선택", "회복", "매칭" 중 서로 다른 아이디어 종으로 나눈다.
- 샘플 3장을 뽑았을 때 서로 바꿔 끼워도 말이 되면 실패다.

## 한 장 작성 체크리스트

- 이 카드의 사용자가 5초 안에 떠오르는가?
- 왜 오늘 만들어야 하는지 행동이 보이는가?
- 같은 format 카드 100장과 구분되는 소재가 있는가?
- 같은 seed의 앞뒤 카드와 심리 원리·사회적 맥락·결과물 형태가 다른가?
- 앵커 이름과 금액이 이 아이디어를 도와주는가?
- oneliner를 소리 내 읽었을 때 사람이 쓴 문장 같은가?
