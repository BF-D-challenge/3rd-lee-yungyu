# 모바일 최종 카드 품질 루프 - 2026-07-10

## 목표

구조 검사를 통과한 카드 중에서 사용자의 취향에 맞고, A/B가 서로 다르며, 근거가 제안 카드의 메커니즘과 맞는 두 장을 노출한다. 대량 기계 재생성 대신 작은 가설을 같은 데이터로 비교하고 승자만 런타임에 반영한다.

## 기준선

- 원본 카드: 73장
- 구조 validator: 73/73 통과
- visible-smell: 0건
- 기존 human-fit 72점: 73/73 통과
- 실제 결함: 선택한 3-8개 취향 중 앞 2개만 사용, `로컬` 카드 0장으로 전체 풀 폴백, `B2B`·`공유 반응` 각 1장, 근거 앵커 의미 불일치

구조와 문장 점수 통과는 사용자에게 설득력 있는 카드라는 보장이 아니었다.

## 가설 실험

`scripts/rollout/experiment-mobile-card-pairing.mjs`에서 같은 난수 시드로 20,000개 A/B 페어를 비교했다.

| 가설 | 변경 | 결과 |
|---|---|---|
| H0 | 첫 두 취향의 단일 카테고리에서 무작위 선택 | 기준선 |
| H1 | 선택한 모든 취향에 동일한 추첨 기회 | 3번째 이후 취향 노출은 생겼지만 `로컬` 오매칭 유지 |
| H2 | 다중 카테고리·포맷·문구 신호로 적합 풀 구성 | 취향 적합 100%, 그러나 같은 포맷 증가와 노출 카드 축소 |
| H3 | H2 + 두 번째 카드 차별성 재정렬 | 차별성 90.3%, 노출 카드 53장으로 축소 |
| H4 | 적합도 가중 추첨 + 차별성 재정렬 + 완화 풀 | 품질과 전체 카드 노출을 함께 유지 |
| H5 | 씨앗·카테고리·포맷·심리 충돌만 감점하고 최대 3후보 비교 | **최종 승자. H4보다 단순하고 충돌률이 낮음** |

### H0 대비 H4

| 지표 | H0 | H4 |
|---|---:|---:|
| 라우팅 프로필 적합률 | 95.7% | 100% |
| 3번째 이후 취향 노출률 | 0% | 53.0% |
| `로컬` 오매칭률 | 72.8% | 0% |
| 같은 포맷 페어 | 15.5% | 7.4% |
| 차별성 통과율 | 84.1% | 91.2% |
| 노출 카드 | 73 | 73 |

H4는 카테고리, 포맷, 제목·원라이너 신호, 직접 수요 근거를 취향 적합 점수로 사용했다. 후속 감사에서 사회 맥락·결과 형태·정서적 해소가 거의 전부 고유 자유문장이라 문자열 비교가 실제 의미 차이를 증명하지 못한다는 점을 발견했다.

H5는 적합도 가중 추첨은 유지하고 두 번째 카드 후보를 최대 3장만 본다. 같은 씨앗 8점, 카테고리 6점, 포맷 4점, 심리 3점을 감점해 가장 충돌이 적은 후보를 고른다.

### H4 대비 H5 - 승인 67장

| 지표 | H4 | H5 |
|---|---:|---:|
| 라우팅 프로필 적합률 | 100% | 100% |
| 같은 포맷 | 8.0% | 4.1% |
| 같은 카테고리 | 1.2% | 0.4% |
| 차별성 통과율 | 90.8% | 95.4% |
| 특정 카드 최대 노출 비중 | 7.2% | 5.2% |
| 노출 카드 | 67 | 67 |

## 의미 앵커 가설

73장 전수 수동 검토에서 기존 validator가 놓친 의미 불일치 12장을 찾았다.

- 6장: 제목·원라이너·MVP·근거 앵커를 함께 수정
- 6장: 적합한 근거를 확인할 때까지 `src/data/card-quality-gates.json`으로 런타임 격리
- 검사 오탐: 정상 기술 용어 `CORS` 전체 차단을 `CORS 에러 못 읽음` 문구만 차단하도록 축소

수정한 앵커는 로컬 TrustMRR 원문에서 제품 설명과 30일 매출을 다시 확인했다. 예: `3AK Track & Field`, `CORSPROXY`, `takeout-tools`, `Ticketdesk AI`.

## 적용 결과

- 구조 정상: 73장
- 런타임 승인: 67장
- 의미 재검토 격리: 6장
- 승인 카드 human-fit 72점: 67/67
- 최종 페어 실험: 라우팅 프로필 적합 100%, `로컬` 오매칭 0%, 같은 포맷 4.1%, 차별성 통과 95.4%, 승인 67장 전부 노출
- `two_cards_completed`에 선정 버전, 카드 키, 취향 ID, 적합 점수, 페어 차별 점수를 기록
- 친구 선택 후에는 원 제작자의 전체 취향을 넘기지 않고 선택 카드 취향과 가까운 보조 취향만 승계

## 다음 검증

실사용에서는 `selection_variant=fit-contrast-v2`의 `two_cards_completed -> round_shared`와 `/vs`의 `praise_sent` 전환을 본다. 다음 패스는 공유율 또는 칭찬 전환이 낮은 카드 키를 우선 검토하며, 구조 통과만으로 격리를 해제하지 않는다.

현재 적합 점수는 카테고리·포맷·명시 문구를 이용한 라우팅 지표다. `돈 받을 수 있음`, `하루 MVP`, `AI 도구`, `로컬` 같은 강한 약속의 실질 적합성을 뜻하지 않는다. 다음 데이터 패스에서 카드별 `monetizable`, `oneDay`, `aiNative`, `local` 태그를 사람이 검수하고 해당 칩의 하드 게이트로 사용한다.
