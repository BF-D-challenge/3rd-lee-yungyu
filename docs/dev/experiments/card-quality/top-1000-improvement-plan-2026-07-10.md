## 목표

2026-07-10 05:00 KST까지 상위 1,000개 카드를 모두 다시 쓴다. 단, 3시간 안에 한 사람이 1,000장을 장인처럼 오래 붙잡는 방식은 불가능하므로, **품질 규칙을 먼저 잠그고 4개 재작성 레인이 병렬로 전량 1차 재작성한 뒤 고위험군을 2차 리뷰**한다.

1. 상위 1,000개 카드 후보는 유지한다.
2. 1,000개 카드의 핵심 문장과 카드 구조를 전량 재작성한다.
3. 고위험/고노출 200~250장은 전량 재작성 이후 2차 정밀 리뷰한다.
4. 새 필드와 검증 기준을 확정해 이후 배치가 같은 품질 기준을 따르게 한다.
5. 앱 화면과 공유 화면에서 새 카드 구조가 깨지지 않는지 최소 검증한다.

최종 완료 기준은 **1,000개 전량이 기존 기계식 조합 문장에서 벗어나고, 05:00 KST에 검증 가능한 재작성본과 실패/리뷰 목록이 남는 것**이다.

## 핵심 판단

지금 문제는 카드 개수가 아니라 카드의 **니즈 출처와 차별성**이다.

기존 방식은 `seed × pain × format`을 너무 많이 펼쳤고, 그 결과 카드들이 다음처럼 비슷해졌다.

- 비슷한 문장 구조
- 비슷한 타깃 표현
- 비슷한 evidence 문장
- 비슷한 MVP 네 가지
- 같은 seed 안에서 기능만 조금 바뀐 느낌

따라서 3시간 작전에서는 먼저 **니즈 원자와 차별 축 규칙**을 고정한 뒤, 1,000개를 4개 배치로 나눠 전량 재작성한다. 이후 노출 가능성이 높거나 품질 냄새가 강한 200~250개를 2차로 다시 본다.

- UI 대개편

## 품질 스키마

전량 재작성 카드에는 다음 필드를 붙인다. 값이 약한 카드는 릴리즈에서 숨기지 않고 `review` 또는 `fail`로 남겨 다시 고친다.

```json
{
  "needSource": "direct | adjacent | inferred | external",
  "psychologyPrinciple": "Abstinence Violation Effect",
  "whyItMatters": "하루 실패를 전체 실패로 해석하면 복귀보다 포기가 쉬워진다",
  "differentiationAxis": {
    "psychology": "실패 위반 효과",
    "socialContext": "혼자 회복",
    "outputShape": "회복 기록판",
    "emotionalResolution": "재시작감",
    "anchorMechanism": "기록 유지"
  }
}
```

검증 기준:

- `needSource`가 없으면 실패
- `psychologyPrinciple`이 있으나 `whyItMatters`가 비어 있으면 실패
- `differentiationAxis`의 5개 하위 값 중 2개 이상이 같은 seed 안에서 반복되면 리뷰 큐
- 화면용 문장에 전문 용어가 직접 노출되면 리뷰 큐
- `buildPrompt`가 반나절 구현 단위로 읽히지 않으면 실패

## 전량 재작성 배치 기준

1,000장은 다음 기준으로 4개 배치에 나눠 동시에 쓴다.

1. 같은 seed가 한 레인에 몰리지 않게 분산한다.
2. `template-scene`, `generic-evidence`, `weak-anchor`, `samey-mvp` 냄새가 섞이게 나눈다.
3. 각 레인은 250장 안에서 title/oneliner/frontStory/mvp/buildPrompt를 모두 다시 쓴다.
4. 같은 seed 안의 카드가 비슷해 보이면 `differentiationAxis`를 먼저 바꾸고 문장을 다시 쓴다.
5. evidence는 아이디어를 꾸미는 문장이 아니라 "왜 이 니즈가 있을 수 있는지"를 설명해야 한다.

2차 정밀 리뷰 200~250장은 다음 우선순위로 뽑는다.

1. 랜덤 뽑기나 공유 화면에 노출될 가능성이 높은 카드
2. 같은 seed 안에서 title/oneliner 구조가 반복되는 카드
3. target이 너무 넓거나 누구의 문제인지 흐린 카드
4. evidence와 MVP가 따로 노는 카드
5. 오늘 반나절 안에 만들기 어려운 MVP를 제안하는 카드
6. 심리 원리가 없으면 왜 중요한지 설명되지 않는 카드

모든 카드는 "미완성"으로 방치하지 않는다. 각 카드는 다음 중 하나로 분류한다.

- `rewritten-pass`: 재작성 후 검증 통과
- `rewritten-review`: 재작성은 됐지만 사람 리뷰 필요
- `rewritten-fail`: 재작성됐으나 validator 또는 QA에서 실패
- `second-pass`: 2차 정밀 리뷰 대상
- `rule-pass`: 현재 문장이 최소 기준을 통과하고 규칙만 적용해도 되는 카드
- `needs-evidence`: 근거가 약해 리서치 또는 수동 판단이 필요한 카드
- `drop-candidate`: 상위 1,000개에는 남기되 릴리즈 전 교체 후보
