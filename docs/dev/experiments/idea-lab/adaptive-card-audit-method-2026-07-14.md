# Candidate 카드 적응형 감사 방법 — 2026-07-14

## 목적

`Candidate` 2,576개를 빠르게 좋은 카드로 만들기 위해, 모든 후보를 처음부터 27조합으로 똑같이 펼치지 않는다. 먼저 27개의 모든 1:1 관계를 확인할 수 있는 최소 9조합을 검사하고, 위험 신호가 있는 후보만 18조합 또는 27조합으로 확대한다.

기존 `idea-source-final-ledger.jsonl`의 8,406개 원본 판정과 기존 27조합 감사 문서는 폐기하지 않는다. 기존 문서는 원본 근거와 기준 보정 자료로 사용하고, 이 문서의 적응형 방식은 2026-07-14 이후 후보 카드 배치에 적용한다.

## 1. 사전 단계: 원본 클러스터

원본 행은 삭제하거나 최종 `Merge`로 덮어쓰지 않는다. 다음을 별도 파생 파일에 기록한다.

- 같은 이름·URL·입력→처리→결과 지문: 안전한 중복 그룹
- 의미가 가까운 입력·처리·결과: `provisional_cluster`
- 기존 앱 52개와 가까운 원본: `Merge-to-Existing` 검토 그룹
- 대표 원본과 그룹 구성원 원본의 모든 키

대표 원본은 원문 근거, `needs_enrichment=false`, 구체적인 입력·처리·결과, 작은 MVP, 시장 신호 순서로 고른다. 유사도 숫자 하나만으로 자동 탈락·자동 Merge하지 않는다.

## 2. 최소 조합: Latin 9

세 축은 결제자 P1~P3, 순간 M1~M3, 한 끗 T1~T3이다.

```text
P1-M1-T1   P1-M2-T2   P1-M3-T3
P2-M1-T2   P2-M2-T3   P2-M3-T1
P3-M1-T3   P3-M2-T1   P3-M3-T2
```

이 9개는 다음 관계를 각각 한 번씩 포함한다.

- 결제자×순간 9가지
- 결제자×한 끗 9가지
- 순간×한 끗 9가지

따라서 두 축 사이의 어색함을 찾는 최소 검사 수다. 9개보다 적으면 결제자×순간의 일부를 보지 못한다.

## 3. 확대 규칙

### Fast Pass — 9조합

다음 조건을 모두 만족하면 9조합 결과로 통과시킬 수 있다.

- Latin 9개 하드게이트 통과
- 신입·중급·숙련 UX 3/3
- 원본 충실도 경고 없음
- 기존 앱 중복 경고 없음
- 상대방·기관·상시 수집 의존 없음
- 입력 1개→처리 1회→결과 1개가 명확함
- 문구 기계 복제 경고 없음

### Review — 18조합

UX 판정 불일치, 입력·권한 경계, 높은 포트폴리오 유사도, 한 끗별 결과물 차이가 있으면 두 번째 Latin 회전 9개를 추가한다.

```text
첫 회전: T = (P + M) mod 3
두 번째 회전: T = (P + M + 1) mod 3
```

### Full Audit — 27조합

수정이 발생한 후보, 고위험 도메인, 상대방·기관 의존 후보, 원본 근거 부족 후보, 18개에서도 이견이 남는 후보는 세 번째 회전까지 포함해 27개를 모두 본다.

## 4. 놓침률 감시

Fast Pass 후보의 10%를 무작위로 선택해 나머지 18개도 확인한다.

- 추가 실패율 0~2%: 9조합 유지
- 추가 실패율 2~5%: 경계 유형은 18조합
- 추가 실패율 5% 초과: 해당 유형은 처음부터 18/27조합

이 검사는 품질을 줄이는 것이 아니라, 9조합이 27조합을 얼마나 잘 예측하는지 계속 측정하는 장치다.

## 5. 승격 규칙

감사 보고서를 먼저 문서화한다. `sample-data.ts`에는 다음만 추가한다.

- `Fast Pass` 또는 `Review/Full Audit` 후 최종 Pass
- 정확한 리서치 키·URL 연결
- 기존 런타임 ID·원본 이름 충돌 없음
- 타입·조합 수·Idea Lab E2E 통과

`Candidate` 원장 판정만으로 앱에 자동 승격하지 않는다.
