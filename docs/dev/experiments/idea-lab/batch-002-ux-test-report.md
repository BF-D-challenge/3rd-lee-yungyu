# 아이디어 배치 002 — UX 테스트·한국 적합성 보고서

검사일: 2026-07-12

## 테스트 조건

- 입력: `batch-002-ascii-cards.md`의 동일 규격 아이디어 카드 10개
- 공통 과업: `10초 안에 이해 → 한국에서 실제 사용 장면 발견 → 결제 이유 판단`
- 페르소나: UX Tester 고정 경험 3층(P1 신입, P2 중급, P3 숙련)
- 독립성: 각 페르소나가 각 카드를 독립 셀로 초기화해 평가
- UX 통과: 세 과업이 모두 Yes인 페르소나가 2/3 이상
- 최종 통과: UX 통과 뒤에도 한국 행동 근거, 빈도/손실, 현지 대체재, 결제 결과, 1인 MVP 게이트를 모두 통과
- 한계: 아래 결과는 합성 사용자 평가에 의한 가설이며 실제 한국 사용자 인터뷰·행동 데이터가 아니다.

## 페르소나별 여정 요약

### P1 · 신입

금액이나 즉시 결과가 먼저 보이는 카드에서는 빠르게 진입했다. 다만 파일 업로드, 외부 전송, 일괄 캘린더 등록처럼 되돌리기 어려운 행동 앞에서 순서·개인정보·미리보기 부재를 크게 불안해했다. `학원공지한장`만 결제 반복성이 약하다고 판단했다.

### P2 · 중급

계산 근거와 결과 정확도를 가장 엄격하게 확인했다. 급여 계산 기준, OCR 오인식, 명세서에서 사용 여부를 추정하는 문구, 자세 분석의 안전성을 문제 삼았다. `러닝각도`는 분석 신뢰가 부족해 결제하지 않겠다고 판단했다.

### P3 · 숙련

반복 처리와 기본 도구 대비 이득을 집중해서 봤다. 변경 근무표 갱신이 없는 `근무표찍기`, 여러 장 일괄 처리가 없는 `학원공지한장`, 기본 카메라로 대체 가능한 `언박싱증거`의 결제 가치를 인정하지 않았다.

## 공통 문제와 최소 수정

1. **확인 없는 불가역 행동**: 캘린더 등록·집주인 전송·가족 공유 전에 미리보기, 수정, 되돌리기를 둔다.
2. **AI 추정의 과장**: `미사용`, `안 쓰는`, `누락 확정`, `자세 처방`처럼 입력만으로 확정할 수 없는 표현을 `확인 필요`, `예상`, `실험`으로 바꾼다.
3. **민감정보 신뢰**: 카드·통신 명세서는 기기 내 처리, 자동 마스킹, 원본 미저장을 CTA 바로 옆에서 약속한다.
4. **반복 가치 부재**: 월 1회 변환 도구는 구독이 아니라 단건·평생 구매로 판다. 구독하려면 변경분 탐지와 갱신까지 소유해야 한다.

## 카드별 결정

### C01 급여차이 — 통과

- 유지할 약속: 예상 급여와 실제 명세서의 항목별 차이
- 필수 수정: 계산 전 시급·근무·휴게·수당 기준 확인
- MVP: 한 주 근무 입력 + 명세서 총액 → 예상액·차이·계산 근거

### C02 근무표찍기 — 통과

- 유지할 약속: 단체 근무표에서 내 줄만 캘린더로
- 필수 수정: 전체 미리보기, 중복 표시, 되돌리기; 후속 버전에 변경분 갱신
- 가격 가설: 월 구독보다 3,900원 평생 구매

### C03 숨은구독 — 통과

- 유지할 약속: 앱스토어 밖 반복 결제까지 한 명세서에서 발견
- 필수 수정: `미사용`을 단정하지 않고 사용 여부 확인; 기기 내 분석·원본 미저장
- MVP: PDF/CSV 1개 → 반복 상호 목록 → 사용자 확인 → 해지 경로

### C04 하자증거 — 통과

- 유지할 약속: 사진·요청·답변을 날짜순 증거팩으로
- 필수 수정: 요청문 전송 전 수신자·문구·첨부 미리보기
- MVP: 하자 1건, 사진·시간·요청문·공유 PDF

### C05 학원공지한장 — 탈락

- 이유: 사용 장면은 명확하지만 한 장 변환이 기존 캘린더 입력보다 충분히 큰 결제 이득이 아님
- 재검토 조건: 여러 아이·여러 공지를 일괄 처리하고 변경 공지를 기존 일정에 갱신하는 행동 데이터

### C06 내말속도 — 통과

- 유지할 약속: 20초 읽기 뒤 내 속도로 움직이는 촬영
- 필수 수정: 촬영 중 느리게·정지·빠르게 수동 복구, 가능하면 음성 기반 실시간 적응
- MVP: 대본 + 20초 보정 + 카메라 위 스크롤

### C07 러닝각도 — 조건부 통과

- 유지할 약속: 문제 하나와 다음 1km 실험 하나
- 필수 수정: 전신 촬영 가이드, 분석 가능 여부·신뢰도 표시, 의료 진단·통증 조언 금지
- MVP: 10초 측면 영상 → 착지 위치 한 지표 → 다음 촬영 전후 비교

### C08 이사반납 — 통과

- 유지할 약속: 빈집·계량기·열쇠·공과금 인계를 한 흐름으로
- 필수 수정: 필수 항목 완료 전 전송 잠금, 완료·누락·미정산 최종 요약
- MVP: 퇴거 체크 8개 + 정산 링크 + 집주인용 PDF

### C02-09 통신비구멍 — 조건부 통과

- 유지할 약속: 부모님 명세서에서 해지 확인할 유료 항목과 실제 절감 후보 찾기
- 필수 수정: 사용 여부를 단정하지 않고 약정 영향까지 묻는 114 질문 생성
- MVP: 명세서 1장 → 항목 분류 → 확인 질문·전화 연결

### C10 언박싱증거 — 탈락

- 이유: 촬영 순서는 유용하지만 기본 카메라와 메모로 대체 가능하고, 보상 접수 성공을 제품이 소유하지 못함
- 재검토 조건: 주문처·택배사별 보상 접수 양식을 실제 제출까지 연결

## 결론

배치 002는 **8개 통과, 2개 탈락**이다. 다음 제작 순서는 즉시 회수 금액과 반복성이 모두 높은 `급여차이 → 숨은구독 → 근무표찍기`가 우선이며, 고액 손실 방어형 `하자증거 → 이사반납`이 그다음이다. 합성 UX 평가는 가설이므로 실제 제작 전 각 상위 아이디어에서 한국 타겟 5명에게 입력 자료를 받아 수동 결과물을 판매하는 컨시어지 테스트가 필요하다.
