## 페르소나 관찰

### P1 · 신입

구체적인 손실액·처리 건수가 보이면 빠르게 이해했다. `SRT`, `CSV`, `안전영역` 같은 준비 용어와 계약·의료·금융정보의 외부 전송에서 멈췄다. 미리보기와 일상어가 필수다.

### P2 · 중급

금액·날짜·발송이 걸린 흐름에서는 원문, 합계, 대상 확인을 요구했다. 자동 추출보다 수정·되돌리기·저장 실패 복구가 결제에 더 중요했다.

### P3 · 숙련

한 건씩 촬영·등록·확인하는 제품을 거의 결제하지 않았다. 일괄 입력과 예외만 검토하는 흐름, 기본 도구가 소유하지 못하는 돈 회수·증거·발송 결과가 있는 아이디어만 인정했다.

## 최종 통과 12개

1. `오답리턴` — 여러 문제 일괄 촬영과 오답 자동 분리가 전제
2. `말버릇컷` — 60초 전후 말버릇 횟수 비교
3. `미수금톡` — 약속일 이후 단계별 미수금 메시지
4. `범위밖` — 계약 범위와 새 요청을 원문 근거로 비교
5. `라이더진짜시급` — 여러 플랫폼 정산 캡처 일괄 입력
6. `중고인수증` — 앱 설치 없는 양측 확인 상태표
7. `계약날짜` — 법률 판정 없이 계약서 원문과 날짜 확인
8. `자막핏` — 한국어 자막의 플랫폼 안전영역 검수
9. `발표배분` — 장표별 리허설 시간과 초과 장표
10. `퇴근도장` — 근로자 개인 소유 출퇴근 기록
11. `회비대조` — 회원 명단과 입금 CSV의 예외 매칭
12. `공구입금` — 주문폼과 입금 내역 대조 후 발송표

## 탈락·보류 이유

- `환불왔나`: 카드 연결 없이 환불 도착을 자동 확인하지 못해 사용자가 일을 다시 해야 함
- `예약금마감`: 캘린더·예약 플랫폼 알림보다 결제 우위가 약함
- `박스어디`: 한국 App Store에 QR 이사박스 동일 제품이 존재
- `부모님공문`: UX는 좋지만 국내 행동·결제 근거가 부족해 보류
- `진료메모`: 국내 환자용 진료 녹음·요약·공유 제품이 이미 존재하고 의료정보 부담이 큼
- `영상한컷`: 무료 프레임 추출 앱이 다수 존재
- `30칸`: UX 0/3. 이전 설치형 테스트 승인과 무관하게 최종 실패
- `짐검사`: 인천공항·항공사 공식 무료 정보와 기존 무료 앱이 이미 해결

## 결론

배치 003은 **12개 통과, 7개 탈락, 1개 보류**다. 제작 우선순위는 돈 회수와 반복성이 함께 있는 `미수금톡 → 범위밖 → 공구입금 → 라이더진짜시급`, 고액 분쟁 방어형 `중고인수증 → 계약날짜`, 반복 훈련형 `말버릇컷 → 자막핏 → 발표배분` 순이다.

실사용 확인 질문:

1. 미수금·추가 요청을 최근 한 달에 몇 번 직접 추적했으며, 실제 손실 금액은 얼마였나?
2. 라이더·총무·공구 셀러가 앱에 넣을 수 있는 원본 파일은 실제로 CSV인가, 캡처인가?
3. 계약서·출퇴근 기록처럼 민감한 자료를 올리기 위해 필요한 최소 신뢰 약속은 무엇인가?
