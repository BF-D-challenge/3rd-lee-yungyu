# EXH-001 후보 9개 27조합 감사 — 2026-07-14

## 현재 상태

- 앱은 아직 **43개 / 1,161조합**이다.
- 이 문서를 보여주기 전에 `sample-data.ts`에는 후보 9개를 넣지 않았다.
- 후보 9개는 각각 결제자 3×순간 3×한 끗 3, 총 **243조합**을 작성했다.
- 9칸 교차 예비 감사 **81/81**, 27조합 전수 감사 **243/243**이 하드게이트를 통과했다.
- 신입·중급·숙련 카드 이해도는 후보 9개 모두 **3/3 Pass**였다.
- 따라서 9개 모두 앱 승격 가능 판정이다. 승격하면 **52개 / 1,404조합**이 되어 50개 체크포인트를 넘는다.

## 왜 27개가 모두 자연스러운가

모든 결제자는 같은 원본 처리 흐름을 실제 업무에서 반복한다. 모든 순간은 결과가 필요한 바로 전·직후이고, 모든 한 끗은 원본의 입력→처리→결과를 유지한다.

- AIRIX: 브랜드·대행사·컨설턴트 모두 보고·경쟁사 제보·출시 점검에서 답변 비교표를 쓸 수 있다.
- 명함: 영업·채용·프리랜서 모두 행사 직후·후속 전·CRM 반영 전에 연락처·표 결과를 쓸 수 있다.
- 브랜드 카드: 운영자·디자이너·대행사 모두 외주 전·불일치 발견 후·리브랜드 공유 때 사이트 근거를 쓸 수 있다.
- 이메일: 영업·채용·프리랜서 모두 무응답·약속일 경과·주간 정리 때 후속 기한 근거를 쓸 수 있다.
- 호텔: 여행자·출장 코디네이터·여행 기획자 모두 결제 전·취소 조건 변경·공유 전에 최종가 비교를 쓸 수 있다.
- 다음 세트: 개인 운동자·PT 코치·취미 리프터 모두 세트 사이·실패 후·다음 주 시작 전에 같은 숫자 계산을 쓸 수 있다.
- 인터벌 영상: 운동자·원격 코치·체육관 운영자 모두 시작 전·기록 직후·영상 전송 전에 시간과 영상을 맞출 수 있다.
- Apps Script 폼: 운영자·강사·셀러 모두 공동 사용 전·Forms 한계 발견·시트 오류 후에 입력 화면을 만들 수 있다.
- 숏폼 사전검사: 크리에이터·대행사·쇼핑몰 마케터 모두 업로드 전·경고 후·납품 전에 같은 규칙 근거를 쓸 수 있다.

## 임베딩 교차검사

임베딩은 자동 합격·탈락기가 아니라 원본 이탈과 기존 카드 충돌을 사람이 볼 보조 신호로만 썼다.

### 원본 충실도

- local fastembed: **243/243 통과**, 최저 `0.947230`, 중앙값 `0.978780`, 최고 `0.990157`
- OpenAI `text-embedding-3-small`: **243/243 통과**, 최저 `0.895461`, 중앙값 `0.929235`, 최고 `0.948288`
- 두 모델 모두 0.8 미만 없음

### 기존 43개와 충돌 후보 수동 확인

로컬 모델의 임계값 `0.819693` 이상인 새 후보 관련 원본 쌍 8개를 직접 비교했다.

| 쌍 | 최대 유사도 | 왜 같은 원본이 아닌가 |
|---|---:|---|
| Captime ↔ track-form-coach | 0.871432 | 타이머로 영상을 구간화 vs 영상에서 관절·자세를 분석 |
| RightInbox ↔ safe-message-reply | 0.859257 | 답장이 없는 보낸 메일의 기한 추적 vs 들어온 메시지의 안전한 즉시 답장 |
| FitnessAI ↔ meal-habit-loop | 0.842184 | 직전 세트 숫자 계산 vs 식사 사진·습관의 다음 행동 |
| 명함→연락처 ↔ 수수료 계산 | 0.826911 | 명함 OCR·연락처 저장 vs 실적·수수료 규칙 계산 |
| Captime ↔ interactive-video-checkpoint | 0.826831 | 운동 중 타이머·카메라 기록 vs 완성 영상에 학습 문제 삽입 |
| 명함→연락처 ↔ RightInbox | 0.826311 | 새 사람의 연락처 생성 vs 이미 보낸 이메일의 무응답 추적 |
| Captime ↔ FitnessAI | 0.825044 | 영상 구간 기록 vs 다음 중량·횟수 계산 |
| RightInbox ↔ connected-knowledge | 0.821155 | 메일 기한·무응답 상태 vs 여러 문서의 합의·충돌 근거 연결 |

첫 검사에서 Captime의 대표 프레임 비교와 RightInbox의 답장 초안 큐가 기존 원본과 문구상 가까웠다. Captime은 `라운드별 시간 변화`, RightInbox는 `초안 없는 무응답 기한 큐`로 고쳐 원본 경계를 더 분명히 했다.

## UX 반영

- `vCard` → `휴대폰 연락처 파일(vCard)`
- `기다릴 날짜` → `회신을 기다릴 마지막 날짜`
- 운동 난이도 → `1~10`, 사용 가능한 원판 단위 추가
- 카메라 → `이 기기의 카메라 권한`, 기기 저장 명시
- 시트 입력 → URL·머리글 양자택일 대신 `열 머리글 복사`로 첫 MVP 고정
- 숏폼 규칙 → 내장 규칙·갱신일·오탐 제외·재검사 명시
