생성일: 2026-07-12  
목적: 일반 대중용 포트폴리오에서는 유입 장벽 때문에 통과하지 못하지만, 깊은 도메인 지식·기존 고객·제휴 채널이 있는 사람에게는 강해지는 아이디어를 보존한다.

## 보관 원칙

다음 조건을 모두 만족하면 폐기하지 않고 이 보관함에 넣는다.

1. 문제와 결과는 이해하기 쉽고 실제 효용이 있다.
2. 1인 빌더가 핵심 MVP를 만들 수 있다.
3. 일반 빌더는 필요한 순간의 사용자를 만나기 어렵다.
4. 특정 업계 종사자·사업자·크리에이터가 이미 가진 고객 또는 채널을 사용하면 유입 문제가 사라진다.

이 후보들은 일반 포트폴리오의 `New Pass` 및 100개 목표에 포함하지 않는다. 사용자가 자신의 업종·경험·고객 채널을 입력해 **커스텀 아이디어를 만들 때만** 다시 꺼낸다.

## 커스텀 아이디어 생성 시 사용법

사용자에게 먼저 세 가지를 확인한다.

1. **어떤 업계의 일을 깊이 아는가?**
2. **이미 만날 수 있는 고객은 누구인가?**
3. **예약·구매·접수·방문 중 어떤 순간을 통제하는가?**

답과 일치하는 후보가 있으면 원래 기능을 그대로 추천하지 않고, 사용자가 실제로 통제하는 순간에 맞춰 다시 조립한다.

예시:

> `렌터카 업계 경험 + 예약 고객 문자 발송 가능`  
> → 렌터카 인수반납 대조를 일반 앱이 아니라 **예약 직후 자동으로 열리는 6장 촬영 링크**로 변환한다.

다시 꺼낸 후보는 `보유 채널의 실재성 → UX 2/3 → 제품성 12점 이상 → MVP 4주` 순서로 재검사한다. 사용자가 실제 접근권을 갖고 있지 않으면 일반 포트폴리오로 복귀시키지 않는다.

## 향후 저장 형식

새 후보는 아래 정보를 반드시 남긴다.

- 원 아이디어 ID와 제목
- 일반 포트폴리오에서 걸러진 이유
- 적합한 도메인 전문가
- 필요한 기존 고객·채널·제휴
- 전문가가 통제하는 정확한 사용 순간
- 재사용 가능한 입력 → 처리 → 결과 구조
