앱 데이터 반영: 없음

> 후속 v2.1 전체 감사: 이 문서의 New Pass 70개에 `사용 순간 접근성` 게이트를 적용한 결과는 [사용 순간 접근성 전체 감사 — 70개](./idea-moment-access-audit-70.md)에 있다. **일반 검증 후보 62 / Custom Reserve 8**이며, 아래 52개·70개 수치는 게이트 적용 전 후보 수다.

## 결론

- 최근 배치 006~009의 400개를 전수 재분류했다.
- 기존 UX의 세 번째 값 `결제 이유`를 제거하고 `이해/한국 사용 장면`과 카드의 첫 입력·결과 명료성으로 다시 계산했다.
- 수정 UX는 **374개 통과 / 26개 실패**다. 결제 추측 하나 때문에 탈락했던 283개가 제품성 심사로 복귀했다.
- 최근 400개 최종 재분류는 **New Pass 52 / Merge 155 / Hold 164 / Fail 29**다.
- 배치 001~005에서 과도한 경쟁·빈도·B2C 제한으로 내려간 후보 중 **18개를 New Pass로 복구**했다.
- 이번 재심사에서 새로 사용자 검증 대상으로 올릴 후보는 **총 70개**다: B2C 44개, 1인 사업자 B2B 26개.
- `Merge 155`는 나쁜 아이디어가 아니다. 이미 통과한 그룹의 카드 재료로 흡수하므로 새 원본 그룹 수에 중복 계산하지 않는다.
- Pass는 실제 결제 검증 완료가 아니라 대표 카드 사용자 검증 자격이다. 앱의 32개 그룹·1,728조합은 아직 바꾸지 않는다.
