# 원본 8,406개 전수 확인 — 다중 실험 프로그램

작성일: 2026-07-13  
상태: L1 Draft, 앱 데이터 반영 없음  
목표: 8,406개 모든 원본에 최소 하나의 의미 판정과 추적 가능한 이유를 남긴 뒤, 고품질 후보만 9조합·27조합 감사로 올린다.

## 1. `전부 확인`의 완료 정의

원본 8,406개를 모두 27조합으로 만드는 것은 품질 심사가 아니라 낭비다. 대신 모든 원본이 아래 L4까지 도달해야 전수 확인 완료로 센다.

| 수준 | 의미 | 적용 범위 | 현재 |
|---|---|---:|---|
| L0 | 원본 수집·고유 키 확인 | 8,406 | 완료 |
| L1 | 텍스트·키워드·시장·임베딩 기계 심사 | 8,406 | 완료 |
| L2 | 입력→처리→결과→순간 메커니즘 정규화 | 8,406 | 대기 |
| L3 | 결과 소유·Moment·기본 기능·범용 AI·상대방 의존 게이트 | 8,406 | 대기 |
| L4 | `shortlist / merge / reserve / reject / enrich` 중 하나와 이유 확정 | 8,406 | 대기 |
| L5 | 결제자 3×순간 3×한 끗 3 중 9칸 교차 예비 감사 | L4 shortlist만 | 조건부 |
| L6 | 27조합 전수 감사·UX 2/3·앱 승격 | L5 통과만 | 조건부 |

`enrich`는 탈락이 아니다. 설명이 부족해 의미 판단을 할 수 없다는 상태이며, 보강 뒤 다시 L2로 들어온다.

## 3. 실험 목록

### X01. 소스 완전성·설명 보강 실험

- 질문: 지금 가진 텍스트만으로 핵심 메커니즘을 판정할 수 있는가?
- 전체 적용: 8,406개
- 현재 결과: 4,165개 기계 심사 가능, 4,241개 보강 필요
- A안: 설명 길이 기준만 사용
- B안: 이름·카테고리·검색어·순위 조합으로 설명 없는 행을 구제
- 성공 기준: 수동 100개에서 `판정 가능/불가능` 일치율 90% 이상
- 실패 시: 설명 부족 행을 Fail로 만들지 않고 URL 보강 큐로 유지

### X02. 아이디어 문법 키워드 실험

- 질문: 현실 입력·처리·결과·순간 단어가 함께 나타나는가?
- 전체 적용: 8,406개
- 1차 결과: 현재 원본 회수율 상위 1,000개 26.3%, 단독 필터 실패
- 남은 용도: 의미 요소 누락 태그와 검색 설명
- 성공 기준: 탈락 판정에는 사용하지 않으며, L2 정규화 뒤 필드 누락 탐지 90% 이상

### X03. 24개 문제 렌즈 검색 실험

- 질문: 돈 차이·마감 증거·버전 변화·인계 상태 등 어떤 문제 렌즈에 걸리는가?
- 전체 적용: 8,406개
- 1차 결과: 렌즈 합집합 890개, 현재 원본 9/38 회수
- 성공 기준: 신규 상세 검토 후보가 무작위보다 3배 이상 많고, 한 렌즈가 후보의 20%를 넘지 않음
- 실패 시: 렌즈를 탈락기가 아닌 다양성 표본 추출기로만 사용

### X04. 시장 신호와 작은 MVP의 직교 실험

- 질문: 시장 신호가 강한가와 1~3화면 MVP가 가능한가는 별개인가?
- 전체 적용: 8,406개
- 1차 결과: 시장 신호 회수율 73.7%지만 대형 플랫폼 오염 확인
- A축: 매출·순위·리뷰·등장 횟수
- B축: 입력 1개·처리 1회·결과 1개 가능성
- 성공 기준: `시장 강함/MVP 작음` 사분면의 수동 상세 검토 정밀도 50% 이상
- 실패 시: 시장 점수는 순서 보조로만 사용

### X05. 메커니즘 정규화 A/B 실험

- 파일럿: 소스별 20개, 총 60개
- A안: 원본 설명을 자유롭게 한 문장으로 요약
- B안: 아래 7필드를 강제로 작성

```text
검증된 원본 메커니즘
현실적인 입력
처리 한 번
즉시 결과
사용자가 찾는 순간
결과의 실제 소유자
1~3화면 MVP
```

- 성공 기준: 수동 정답 30개에서 필드 완전성 90%, 과장·추론 오류 10% 이하
- 승자 처리: 이 기준을 통과한 형식만 8,406개 L2 전수 심사에 사용

### X06. 결과 소유자 삼각형 실험

- 질문: 입력·상태·실행 결과를 사용자, 제품, 플랫폼 중 누가 소유하는가?
- 전체 적용: L2 완료 8,406개
- A안: `플랫폼 키워드` 탐지
- B안: `누가 원본 상태를 보고 실제 실행할 권한이 있는가` 반사실 질문
- 기존 결과: 제목 키워드 탐지는 위험 후보의 28.2%만 탐지
- 성공 기준: 기존 플랫폼·상대방 의존 판례 39개에서 재현율 80%, 오탐 20% 이하
- 승자 예상: B안. 단어가 아니라 권한 관계를 판정해야 함

### X07. Moment 반사실 실험

- 질문 1: 문제를 발견한 뒤 시작해도 완전한 결과가 나오는가?
- 질문 2: 사전 기록이 필요하다면 사용자가 그 전에 제품을 찾는 검색·업무 순간이 있는가?
- 전체 적용: L2 완료 8,406개
- 판정: Moment 2/1/0
- 성공 기준: 기존 70개 Moment 판정과 90% 이상 일치
- Moment 0: 일반 포트폴리오 Fail, 도메인 채널이 있으면 Reserve
- Moment 1: Hold/보강, 자동 New 금지

### X08. 기본 기능·범용 AI 대체 결투 실험

- 질문: 같은 입력을 OS·플랫폼 기본 기능 또는 범용 AI에 넣었을 때 5분 안에 같은 결과가 나오는가?
- 전체 적용: L2 완료 8,406개
- 비교 결과에 반드시 쓸 것: `원본 상태`, `실행 권한`, `누적 상태`, `다음 행동`, `배포 채널`
- 성공 기준: 기존 commodity-ai·platform-owner 판례 재현율 80% 이상
- 통과 조건: 전용 제품만의 누적 상태·실행·검증 포맷·배포 중 하나가 있음

### X09. 이전 대화 판례 쌍대 비교 실험

- 질문: 절대 점수보다 가장 가까운 Pass/Fail/Merge와의 차이를 설명하는 것이 정확한가?
- 파일럿 60개에 두 방식 적용
- A안: 16점 절대 점수
- B안: 가까운 Pass 2개·Fail 2개·Merge 2개를 붙여 `왜 다른가` 비교
- 성공 기준: 수동 L4 판정 정확도 80%, Merge 재현율 80% 이상
- 1차 원본 설명 임베딩 실패 이유: 판례와 원본의 추상화 수준이 달랐음. X05 정규화 뒤에만 재시험

### X10. 포트폴리오 중복·이상치 구조 실험

- 전체 적용: L2 완료 8,406개
- A안: 원문 임베딩 클러스터
- B안: `타겟·순간·입력·결과` 구조 거리 + 임베딩
- 판정: New / Merge / Outlier
- 성공 기준: 기존 Merge 11개 중 9개 이상 Merge 회수, 현재 38개 서로 다른 그룹의 잘못된 병합 2개 이하
- Outlier: 낮은 유사도라는 이유로 버리지 않고 별도 10% 구조적 신규성 감사

### X11. 독립 검사관 불일치 실험

- 전체 원본은 최소 한 판정자가 L3까지 검사
- 경계 후보는 세 검사관이 독립 판정: `가치 검사관 / 권한 검사관 / MVP 검사관`
- 경계 조건: 판정 확률 0.35~0.65, 검사 간 불일치, 새 클러스터 이상치
- 성공 기준: 세 검사관 중 2/3 합의 + 하드게이트 검사관의 치명적 반례 없음
- 불일치 후보는 자동 탈락 금지, 사람 검토 큐로 이동

### X12. 탈락 후보 그림자 감사

- 각 100개 배치에서 Reject 예정 후보의 10%를 점수와 무관하게 재검토
- 소스별 최소 3개 보장
- 성공 기준: 그림자 표본의 거짓 탈락률 5% 미만
- 5% 이상이면 해당 배치 이후 진행 중단, 원인 규칙 수정 후 이전 배치 재실행

### X13. 9칸 교차 조합 예비 감사

- 적용: L4 shortlist만
- 결제자 3·순간 3·한 끗 3이 각각 세 번 등장하는 9개 라틴 스퀘어 조합
- 성공 기준: 9/9 자연스러움, 입력 접근·결과 소유·1~3화면 MVP 하드 실패 0
- 실패 시: 카드 풀 수정 후 한 번 재시험. 다시 실패하면 Reject/Merge

### X14. 27조합 전수 감사와 앱 승격

- 적용: X13 통과 후보만
- 27/27 자연스러움·MVP 가능
- UX 2/3 이상
- 해외 원본 충실도와 포트폴리오 중복 수동 확인
- 문서 감사 결과를 먼저 공개한 뒤 `sample-data.ts` 승격


## 5. 전수 진행 대시보드 기준

매 배치가 끝날 때 아래 숫자를 기록한다.

```text
배치 ID
전체 / L2 완료 / L3 완료 / L4 완료
shortlist / merge / reserve / reject / enrich
그림자 감사 수 / 거짓 탈락 수
TrustMRR / App Store / Chrome 분포
새 클러스터 수 / Merge 비율
검사관 불일치 수
```

전수 완료 조건:

- L4 확정 합계가 정확히 8,406
- 고유 키 8,406, 누락·중복 0
- `enrich`는 보강 뒤 재심사되어 최종 상태 0
- 누적 그림자 거짓 탈락률 5% 미만
- 모든 Reject에 최소 하나의 구체적인 `rejection_code`와 한 문장 이유 존재
