# 원본 8,406개 빠른 선별 — 소규모 실험 1차 결과

검사일: 2026-07-13  
상태: L1 Draft, 앱 데이터 반영 없음  
실험 원본: TrustMRR 1,863 + App Store 4,512 + Chrome Web Store 2,031 = 8,406  
정량 결과: `docs/research/idea-source-screening-experiments.json`  
임베딩 점수: `docs/research/idea-source-embedding-scores.jsonl`

## 결론

키워드 하나나 원본 설명 임베딩 하나로 8,406개를 자동 탈락시키면 현재 수준의 아이디어를 대량으로 놓친다.

- 아이디어 문법 키워드의 현재 원본 38개 회수율은 상위 1,000개에서 **26.3%**였다.
- 24개 문제 렌즈 BM25 검색은 상위 1,000개에서 **23.7%**였다.
- 제목만 사용한 위험 키워드 탐지는 판정 가능한 39개 중 **11개, 28.2%**만 맞혔다.
- 기존 38개 원본과의 leave-one-out 다국어 임베딩 근접도는 상위 1,000개에서 **15.8%**였다.
- 4 Pass 대 42 Fail 판례의 임베딩 마진은 상위 1,000개에서 **5.3%**였다.
- 시장 신호는 상위 1,000개에서 **73.7%**를 회수했지만, 상위 표본에는 대형 플랫폼·범용 앱이 과도하게 많았다.
- 시장 신호·렌즈·문법·임베딩을 섞은 혼합안은 상위 1,000개에서 **78.9%**로 가장 높았지만, 수동 표본 정밀도는 여전히 낮았다.

따라서 키워드는 `Fail` 판정기가 아니라 **설명 가능한 후보 탐색·태깅 장치**로만 사용한다. 다음 실험은 원본 설명을 먼저 동일한 `입력 → 처리 → 결과 → 순간 → 결과 소유자` 구조로 정규화한 뒤 비교해야 한다.

## 1. 평가 방법

### 카나리아 회수율

현재 앱과 원본 장부가 정확히 연결된 38개를 점수 계산에서는 숨기고, 각 방식이 상위 후보에 몇 개를 다시 올리는지 측정했다.

주의할 한계가 있다.

- TrustMRR: 36개
- App Store: 2개
- Chrome Web Store: 0개

현재 38개만 최적화하면 TrustMRR 편향을 학습한다. 이 지표는 기존 수준의 누락 경보로만 사용하며 App Store·Chrome 품질 정답으로 사용하지 않는다.

### 위험 키워드 탐지

최종 판정 62개 중 `platform-owner / commodity-ai / counterparty / domain`처럼 키워드로 탐지 가능하다고 가정한 39개를 제목만 보고 분류했다. 판정 이유는 입력하지 않아 정답 누설을 막았다.

### 소표본 직접 감사

각 순위의 1·11·21·31·41·51·61·71·81·91위, 총 10개를 고정 표본으로 읽었다. `깊은 검토 가치 있음`은 최종 Pass가 아니라 하드게이트 상세 검토로 보낼 가치가 있다는 뜻이다.

## 4. 왜 실패했는가

### 원본 설명과 최종 아이디어의 추상화 수준이 다르다

원본은 `AI voice platform`, `fitness app`, `customer feedback SaaS`처럼 넓게 적혀 있다. 현재 카드의 품질은 이 원본에서 한국 타겟·필요한 순간·작은 결과를 다시 깎아 만든 뒤 생긴다. 넓은 원본 설명을 완성된 한국 카드 판례와 바로 임베딩하면 유사도가 품질을 나타내지 않는다.

### 긴 설명이 키워드 점수를 독점한다

문법 키워드 상위 300개 중 App Store가 292개였다. 긴 앱 설명에는 `file / result / before / track` 같은 단어가 우연히 많이 포함된다. 소스 층화로 쏠림은 줄였지만 좋은 원본 회수율은 크게 오르지 않았다.

### 시장 신호는 품질이 아니라 존재감이다

시장 신호 상위에는 ChatGPT, Instagram, Cash App, 은행 앱, 대형 피트니스 앱이 올라왔다. 수요는 강하지만 1인 빌더가 1~3화면으로 가져올 좁은 메커니즘이 자동으로 보이지 않는다.

### 위험은 제목 단어가 아니라 권한 관계다

`항공권 이름 수정`, `차지백 제출팩`, `여행 취소수수료`는 제목에 `backup`이나 `restore`가 없지만 실제로는 항공사·결제사·예약 플랫폼이 결과를 소유한다. 위험 키워드 28.2%는 이 관계를 단어만으로 잡을 수 없음을 보여준다.

## 6. 다음 소규모 실험 제안

다음 회차에서는 전체 8,406개를 더 점수화하지 않는다. 소스별 20개씩 총 60개만 다음 비율로 뽑는다.

- 혼합 v2 상위: 소스별 8개
- 문제 렌즈 상위이면서 혼합 v2에 없는 이상치: 소스별 6개
- 완전 무작위 감시 표본: 소스별 6개

60개 각각을 동일한 중간 표현으로 바꾼다.

```text
원본 메커니즘
현실적인 입력
처리 한 번
제품이 즉시 소유하는 결과
사용자가 찾는 순간
결과의 실제 소유자
1~3화면 MVP
가장 가까운 Pass / Fail / Merge 판례
```

그다음 두 실험을 비교한다.

1. 정규화된 구조의 키워드·규칙 심사
2. 이전 대화 판례를 붙인 쌍대 비교 심사

성공 기준은 다음과 같다.

- 고정 30개 수동 표본에서 깊은 검토 가치 정밀도 50% 이상
- 플랫폼 주인·범용 AI·상대방 의존 위험 탐지 80% 이상
- TrustMRR뿐 아니라 App Store·Chrome에서 각각 상세 검토 후보 3개 이상
- 기존 38과 완전히 같은 결과는 `New`가 아니라 `Merge`로 분리

이 기준을 통과하기 전에는 `sample-data.ts`에 새 원본을 넣지 않는다.
