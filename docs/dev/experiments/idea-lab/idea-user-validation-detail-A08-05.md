> 2026-07-12 결정: `계약 종료 + 보증금 미반환 + 분할 반환 합의`가 겹치는 시장이 너무 좁다. 별도 New 후보가 아니라 `이사반납` 또는 금전 약속 대조 카드 재료로만 보관하며, 사용자 검증 큐에 다시 올리지 않는다.

🏠 **보증금 잔금표** — 집주인의 “금요일에 500만 원”을 실제 입금까지

**“이번 주 금요일에 500만 원 먼저 보낼게요.” 그 약속, 지금 카톡 어디에 묻혀 있나요?**  
보증금 3,000만 원 중 1,000만 원만 들어오고, 나머지는 날짜와 금액이 계속 바뀝니다. 카톡에는 약속이 있고 은행에는 입금이 있지만, **얼마가 언제 들어오기로 했고 지금 얼마가 남았는지** 한눈에 보이지 않습니다.

보증금 잔금표는 집주인의 약속 메시지와 실제 입금 캡처를 넣으면 `약속일·약속액·실입금·남은 금액`을 한 장으로 맞춥니다. 법률 판단이나 회수 보장은 하지 않고, **약속과 실제의 차이만** 남깁니다.

🎯 **타겟**

계약 종료 뒤 보증금을 한 번에 돌려받지 못해 집주인과 분할 반환 날짜·금액을 카톡으로 조율하는 세입자.

⚔️ **기존에 잘되는 앱·방법 vs 차별점**

- 카카오톡 — 집주인의 약속 원문은 남지만 날짜·금액·남은 잔금을 계속 다시 계산해야 함
- 은행 앱 — 실제 입금은 정확하지만 집주인이 언제 얼마를 보내기로 했는지는 모름
- 메모·스프레드시트 — 직접 정리할 수 있지만 약속이 바뀔 때마다 다시 옮겨 적어야 함
- 보증기관·법률 서비스 — 정식 보증·분쟁 절차를 다루지만 일상적인 분할 반환 약속표는 아님

→ 보증금 잔금표의 유일 주장: **“보증금 분할 반환 약속과 실제 입금을 한 장에서 맞추는 세입자 전용 잔금표.”**

🗺️ **전체 플로우**

보증금 총액·계약 종료일 입력 → 집주인 약속 메시지 캡처 → 날짜·금액 추출 후 사용자가 확인 → 실제 입금 캡처 추가 → 남은 금액과 지켜지지 않은 약속 표시 → 사실만 담은 확인 메시지 복사

💳 **결제와 MVP**

- 반환 사건 1건 3,900원, 자동 갱신 없음
- 첫 약속 1건은 무료, 두 번째 약속부터 전체 잔금표 잠금 해제
- 3화면 MVP: 사건 만들기 → 캡처·금액 확인 → 잔금표·메시지
- 은행 자동 연동 없음. 캡처 또는 금액 직접 입력만 사용
- 민감한 계약·계좌 정보는 저장하지 않거나 처리 직후 삭제하는 것을 기본값으로 함

반환 약속이 또 바뀌어도 카톡을 다시 뒤지지 마세요. **약속과 입금의 차이 한 장부터 만드세요.**

## 포트폴리오 중복 경계

- `미수금톡`: 프리랜서가 청구 후 미입금을 독촉하는 반복 B2B
- `이사반납`: 퇴거 사진·열쇠·정산을 인계하는 이사 사건
- `보증금 잔금표`: 집주인의 분할 반환 약속과 실입금을 대조하는 고액 B2C 사건

메커니즘은 인접하지만 타겟·순간·입력·결과가 모두 같지는 않다. 사용자가 차이를 충분히 느끼지 못하면 새 그룹이 아니라 Merge한다.
