# 사용자 상세 검증 — B09-01 첫결과 구조대

검증일: 2026-07-12  
현재 상태: 사용자 공통 12문항 평가 대기  
앱 데이터 반영: 없음

📈 **첫결과 구조대** — 결제한 수강생 중 7일째 결과물 0개인 사람만

**“수강생 30명 중, 강의를 결제하고도 첫 결과물 하나 없이 7일을 넘긴 사람은 몇 명인가요?”**  
진도율 70%도 이력서 한 줄, 식단 사진 한 장, 완성한 디자인 한 장이 없으면 수강생에게는 **결과 0개**입니다. 그 사람이 환불이나 “돈만 받고 방치됐다”는 후기를 남긴 뒤에야 찾으면 늦습니다.

첫결과 구조대는 결제자 명단과 `첫 결과물` 제출 링크를 연결해, **결제 후 7일 안에 결과물을 하나도 내지 못한 사람만** 운영자에게 보여줍니다. 전체 수강생 대시보드가 아니라 오늘 먼저 연락해야 할 사람의 목록입니다.

🎯 **타겟**

카카오톡·노션·Zoom·Google Forms를 섞어 매달 10~100명의 유료 강의·챌린지·코칭을 혼자 운영하는 강사.

⚔️ **기존에 잘되는 앱 vs 차별점**

- Teachable — 강의·영상·퀴즈 진도 리포트는 강력하지만 학습자가 만든 실제 결과물은 별도다.
- Circle — 수업 완료율·활동·워크플로를 추적하지만 Circle 밖에서 운영되는 카톡·노션 코호트에는 바로 쓰기 어렵다.
- Kajabi — 영상 시청과 평가 진행을 보여주지만 `첫 이력서`, `첫 식단 사진`, `첫 디자인`처럼 강의별 실제 성과를 공통으로 잡지는 않는다.
- Google Forms·Sheets — 결과물은 받을 수 있지만 결제자 명단·D+7 마감·미제출자를 운영자가 매번 맞춰야 한다.

→ 첫결과 구조대의 유일 주장: **“진도율이 아니라, 돈을 내고도 7일 안에 첫 결과물을 못 만든 수강생만 찾아주는 유일한 코호트 구조대.”**

🗺️ **전체 플로우**

결제자 명단 붙여넣기 → 이번 코호트의 `첫 결과물` 한 가지와 D+7 설정 → 수강생에게 제출 링크 공유 → 사진·파일·링크 중 하나 제출 → D+7에 결과물 0개인 사람만 표시 → 개인화된 확인 메시지 복사 → 제출하면 구조 완료

💳 **결제와 MVP**

- 첫 코호트 20명까지 무료
- 다음 코호트부터 1회 9,900원, 자동 갱신 없음
- 3화면 MVP: 명단·결과 정의 → 수강생 제출 링크 → 미제출자 구조 목록
- LMS API·자동 결제 연동·AI 채점 없음. 명단 붙여넣기와 제출 여부만 사용
- 범용 AI는 문장을 만들 수 있지만 수강생별 제출 상태와 D+7 경과를 계속 소유하지 못한다.

수강생이 환불 버튼을 누른 뒤 찾지 마세요. **첫 결과가 없는 7일째, 먼저 한 사람을 구조하세요.**

## 고정 공통 12문항

평가 문항은 [아이디어 사용자 공통 평가표](./idea-user-evaluation-12.md)와 동일하며, 이 후보를 위해 수정하지 않는다.

1. 10초 안에 누구의 어떤 문제인지 이해되는가?
2. 한국에서 실제로 발생하는 구체적인 사용 순간인가?
3. 해결하지 않으면 잃는 돈·시간·불안·기회가 충분히 큰가?
4. 사용 완료 뒤 받는 결과가 하나의 실체로 명확한가?
5. 문제를 발견한 뒤에도 필요한 입력을 현실적으로 구할 수 있는가?
6. 제휴 없이 첫 사용자 20명을 만날 구체적인 경로가 있는가?
7. 기존 앱·플랫폼 기본 기능·범용 AI보다 선택할 이유가 분명한가?
8. 이 제품만의 유일 주장을 한 문장으로 설명할 수 있는가?
9. 누가 언제 왜 직접 돈을 내는지 납득되는가?
10. 1인 빌더가 4주 안에 입력 1개·결과 1개의 MVP를 만들 수 있는가?
11. 개인정보·법률·의료·금전 판정 위험을 피하면서 핵심 가치를 제공할 수 있는가?
12. 기존 포트폴리오에 합치지 않고 새 아이디어로 둘 만큼 다른가?
