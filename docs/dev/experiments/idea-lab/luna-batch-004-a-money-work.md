## 최종 통과 후보 상세

### M04 보증수리 만료함

- 타깃: 휴대폰·노트북·가전·운동기기를 여러 개 보유하고, 구매처가 여러 곳인 1인 가구와 프로슈머.
- 기존 앱 대비 차이: 보험·쇼핑 앱의 구매내역을 모으려 하지 않고, 사용자가 영수증 사진과 구매일만 넣는다. 제품 상태가 `보증 중 → 30일 전 → 만료`로 누적되고, 만료 전에 무상수리 접수용 증거 PDF를 내보낸다.
- 전체 플로우: 영수증/주문 화면 사진 추가 → 제품명·구매일·보증기간 확인 → 만료일 카드 생성 → 30/7/1일 알림 → 수리 접수번호·결과 기록 → 증거 묶음 내보내기.
- 가장 작은 MVP: 제품 1개 등록, 영수증 이미지 첨부, 만료일 수동 입력, 알림, CSV/PDF export의 3화면.
- 결제 이유: 한 번의 무상수리·교환만 성사돼도 월 구독료보다 이득이다. 광고 없는 1년 보관/내보내기 패스로 과금한다.

### F08 세금계산서 입금표

- 타깃: 월 5건 이상 세금계산서를 발행하는 프리랜서·소규모 제작자.
- 기존 앱 대비 차이: 홈택스 발행 기능이나 회계장부를 복제하지 않고, 발행한 건과 실제 입금의 연결·미입금 큐·독촉 이력만 소유한다.
- 전체 플로우: 세금계산서 번호/금액/입금 예정일 입력 → 입금일 체크 → 미입금 자동 분류 → 1·7·14일 후속 알림 → 고객에게 보낼 상태 링크 또는 CSV export.
- 가장 작은 MVP: 거래 1건 입력, 상태 4개(예정/부분/완료/지연), 알림, CSV export.
- 결제 이유: 한 건의 지연을 줄이는 것만으로도 월 9,900원 이상 가치가 있다. 거래 건수 제한을 둔 월 구독으로 과금한다.

### F09 플랫폼 정산 차이

- 타깃: 배달·중고거래·재능 플랫폼을 두 곳 이상 병행하는 1인 사업자.
- 기존 앱 대비 차이: 플랫폼 주문을 자동 수집하지 않는다. 사용자가 내려받은 정산 CSV나 캡처를 넣으면 `주문 합계-수수료-환불-실입금`의 차이를 월별로 고정해, 플랫폼 고객센터에 보낼 증거를 만든다.
- 전체 플로우: 정산 파일/캡처 추가 → 기간과 기대 입금액 입력 → 실입금 입력 → 차액 규칙 표시 → 이의제기 메모/증거 PDF → 월별 차액 내보내기.
- 가장 작은 MVP: CSV 1종과 수동 입력, 차액 계산, 상태/메모, PDF export.
- 결제 이유: 누락 1건을 찾으면 즉시 손실 회수가 가능하다. 플랫폼별 템플릿 추가와 월별 보관을 유료화한다.

### F11 협찬 사용권 만료

- 타깃: 브랜드 협찬을 월 2건 이상 받는 소형 크리에이터·사진가.
- 기존 앱 대비 차이: 게시물 예약 앱이 아니라 계약의 `게시일·사용기간·매체·승인자`를 상태로 강제한다. 광고주가 확인한 범위와 종료일을 함께 export한다.
- 전체 플로우: 계약 캡처/게시 URL 입력 → 사용기간·매체·단가 입력 → 초안/승인/게시/종료 상태 → 종료 14/3일 알림 → 광고주 확인 링크 → 정산·사용권 기록 export.
- 가장 작은 MVP: 협찬 1건 등록, 날짜 알림, 승인 버튼, 링크/PDF export.
- 결제 이유: 사용권 연장이나 무단 재사용을 한 건만 막아도 가치가 크다. 협찬 건수 기반 월 구독 또는 연간 보관 패스로 과금한다.

### F12 외주 최종본 잠금

- 타깃: 디자인·영상·개발 외주를 월 3건 이상 납품하는 1인 사업자.
- 기존 앱 대비 차이: 드라이브의 파일 관리가 아니라 `최종본 확인`이라는 양측 확인 상태를 소유한다. 확인 전에는 납품 완료/잔금 청구 상태로 닫히지 않는다.
- 전체 플로우: 파일 링크와 버전 입력 → 고객에게 확인 링크 발송 → 승인/수정요청 선택 → 수정 횟수 기록 → 승인 시 납품 완료와 잔금 알림 → 로그 export.
- 가장 작은 MVP: 링크 1개, 버전 3개, 승인/수정요청 버튼, 시간·확인 로그.
- 결제 이유: 최종본 분쟁과 잔금 지연을 한 건 줄이는 결과에 돈을 낸다. 프로젝트 수 제한형 구독으로 과금한다.

### F13 알바 급여명세 보관함

- 타깃: 여러 매장에서 일하거나 시급·근무일이 자주 바뀌는 알바 노동자.
- 기존 앱 대비 차이: 근무시간을 대신 판정하지 않는다. 급여명세서 사진과 실제 입금액을 월별로 묶어 차액과 질문 목록을 만든다.
- 전체 플로우: 명세서 사진 추가 → 입금액/근무기간 입력 → 명세서상 지급액과 대조 → 차액·확인 필요 상태 → 고용주에게 보낼 질문 → 월별 export.
- 가장 작은 MVP: 명세서 1장, 입금액 입력, 차액 계산, 증거 export.
- 결제 이유: 차액이 확인될 때 회수 가능한 금액이 월 이용료보다 크다. 3개월 보관 패스로 과금한다.

### F14 외주 재계약 알림

### H15 관리비 급증표

- 타깃: 오피스텔·원룸에서 매달 관리비 고지서를 직접 확인하는 세입자.
- 기존 앱 대비 차이: 관리비를 납부하는 앱이 아니라 두 달 고지서의 항목·금액·증거를 비교해 `문의할 한 항목`과 보낼 문서를 만든다.
- 전체 플로우: 고지서 사진 2장 업로드 → 항목별 금액 입력/확인 → 전월 대비 5만원 이상 증가 표시 → 사진과 계산을 포함한 문의팩 생성 → 답변일 리마인드 → 처리 결과 기록.
- 가장 작은 MVP: 고지서 2장, 항목 5개 수동 입력, 차액 계산, PDF export, 답변 알림.
- 결제 이유: 잘못 청구된 공용전기·수선비 한 건을 확인하면 즉시 회수 가능하다. 월 4건 고지서 보관 패스로 과금한다.

### H18 수리비 선지급 장부

### H20 집주인 연락기록

- 타깃: 수리·공제·납부 약속을 카카오톡으로 주고받는 세입자.

- 가장 작은 MVP: 사건 카드, 캡처 첨부, 상태 4개, 확인 링크, PDF export.
- 결제 이유: 분쟁이 생겼을 때 즉시 제출 가능한 정리본을 갖는 고액 손실 방지 패스에 결제한다. 사건 단위 과금이 적합하다.
