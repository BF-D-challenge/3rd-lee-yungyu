## 조사·판정 메모

해외 수요 앵커는 `docs/research/trustmrr-acquire/ideas.jsonl`의 공개 매출 행을 사용했다. 국내 하드게이트는 통계청 온라인쇼핑동향, 한국소비자원 1372 소비자상담·피해구제, 국토교통부 자동차·대중교통 통계, 문화체육관광부 국민여행조사, 농림축산식품부 반려동물 관련 조사에서 확인해야 하는 행동을 기준으로 잡았다. 다만 이번 실행에서는 Firecrawl CLI가 `api.firecrawl.dev` DNS 오류로 원문을 내려받지 못했다. 따라서 국내 수치·URL을 날조하지 않고, 공공기관·플랫폼·OS가 이미 무료로 주는 결과는 보수적으로 실패시켰다. `해외·국내 근거`의 국내 부분은 검증해야 할 앵커이며, 숫자 근거가 아니다.

P 칸은 순서대로 `10초 이해 / 한국 실제 사용 / 결제 이유`다. `Y/Y/Y`인 P가 2명 이상이고 UX 합의가 2/3 이상이어야 다음 게이트로 갔다. `MVP`는 로그인·결제·관리자 없이 직접 입력 1~2종으로 구현 가능한 첫 세로 조각을 뜻한다.

## 최종 통과 후보 상세

### LC01 차키교대

- 타겟: 차 한 대를 부부·성인 자녀가 번갈아 쓰며 주차·주유도 나눠 내는 2~4인 가구.
- 기존 앱 대비 차이: 캘린더는 예약만, 메신저는 최신 상태만 남긴다. 차키 인수 시각, 현재 주차 위치, 연료 영수증, 다음 사용자 확인을 한 건으로 닫고 미확인 상태를 표시한다.
- 전체 플로우: 차 사용 예약 → 키를 받은 사람이 사진/주행거리/연료를 확인 → 반납 시 주차 위치와 비용 첨부 → 다음 사용자가 확인 → 월말 이용·비용 요약.
- 가장 작은 MVP: 가족 이름·사용 시작/종료·사진 2장·비용 숫자를 받는 한 화면과 `인수 전/사용 중/반납 확인` 상태.
- 결제 이유: 가족 단위 월 3,900~5,900원. “누가 차를 가져갔고 주차비를 누가 냈나”를 매번 묻는 비용과 분쟁을 줄인다.

### LC02 차량기한

- 타겟: 중고차를 직접 관리하지만 정비소 영수증과 자동차 검사일을 흩어 보관하는 1~2대 보유자.
- 기존 앱 대비 차이: 차량 관리 앱의 차계부 전체가 아니라, 영수증 원문과 주행거리 전후를 함께 보존하고 `다음 마감 + 놓치면 생기는 비용`만 보여준다.
- 전체 플로우: 영수증/검사증 사진 추가 → 차종·날짜·주행거리 확인 → 다음 점검·검사 마감 생성 → 방문 전 체크 → 완료 영수증과 전후 기록 비교.
- 가장 작은 MVP: 사진 1장, 날짜/주행거리 직접 입력, 마감 카드, 완료 전후 사진.
- 결제 이유: 차량 1대 기준 연 9,900원 또는 점검팩 2,900원. 검사·보증·소모품을 놓쳐 생기는 5만원 이상 비용을 막는 결과에 돈을 낸다.

### LC07 부모님교대

- 타겟: 부모님과 30분 이상 떨어져 살며 형제 2명 이상이 비의료적 생활 일을 나누는 자녀.
- 기존 앱 대비 차이: 가족 캘린더가 일정만 표시하는 것과 달리, 부탁 → 담당자 수락 → 완료 사진/영수증 → 다른 가족 확인의 인수인계 상태를 강제한다.
- 전체 플로우: 이번 주 할 일 등록 → 담당자 지정·마감 → 현장에서 사진/메모 첨부 → 완료 확인 → 미완료 자동 재배정 후보 표시 → 주간 인수인계 요약.
- 가장 작은 MVP: 공유 링크 한 개, 할 일 카드, 담당자 확인, 사진 1장, 미완료 마감 표시.
- 결제 이유: 가족방 월 4,900원. 방문·장보기·공과금 납부 누락으로 다시 이동하거나 연체하는 비용보다 싸다.

### LC08 돌봄교대장

- 타겟: 부모님 또는 장애 가족의 식사·외출·세탁·집안 정리를 가족이 교대하는 가구. 의료 기록은 입력하지 않는다.
- 기존 앱 대비 차이: 메모장이 아니라 교대 시각에 “마지막으로 한 일 / 다음에 할 일 / 확인한 사람”을 잠그고, 누락된 항목만 다음 담당자에게 넘긴다.
- 전체 플로우: 오늘 교대 생성 → 체크 항목 완료 → 사진·짧은 메모 첨부 → 다음 담당자 확인 → 미완료 항목 인계 → 주간 누락 리포트.
- 가장 작은 MVP: 항목 5개 템플릿, 완료 시각, 사진, 다음 담당자 확인, 공유 링크.
- 결제 이유: 가족 월 5,900~9,900원. 여러 가족이 같은 내용을 전화로 재확인하는 시간과 돌봄 누락 리스크를 줄인다.

### LC11 반려돌봄팩

- 타겟: 월 1회 이상 펫시터·호텔·지인에게 반려동물을 맡기는 보호자와 돌봄 제공자.
- 기존 앱 대비 차이: 사진을 많이 보내는 것이 아니라 맡기기 전 상태 → 맡긴 동안의 정해진 체크 → 돌려받은 뒤 상태 확인을 하나의 양쪽 확인 기록으로 만든다.
- 전체 플로우: 맡기기 전 먹이·산책·주의사항 확인 → 제공자 체크와 사진 → 보호자 확인/질문 → 귀가 시 상태 사진 → 전후 비교·기록 export.
- 가장 작은 MVP: 반려동물 프로필 1개, 체크 항목 5개, 사진 타임라인, 양쪽 확인 링크.
- 결제 이유: 건당 2,900원 또는 월 4,900원. 맡긴 기간의 누락·오해·상태 분쟁을 줄이는 증거팩이 직접 결과다.

### LC14 가전보증팩

- 타겟: 20만원 이상 가전을 여러 대 보유하고 이사·AS 때 구매 증빙을 자주 찾는 자취·신혼·다인가구.
- 기존 앱 대비 차이: 영수증 OCR만 하는 것이 아니라 모델명·시리얼·구매일·보증 종료·AS 접수·수리 전후 사진을 한 타임라인으로 잠근다.
- 전체 플로우: 영수증과 제품 라벨 촬영 → 항목 확인 → 보증 마감 생성 → 고장 전후 사진 → AS 접수번호·기사 방문 기록 → 수리 결과와 비용 비교.
- 가장 작은 MVP: 가전 3대, 사진 3종, 보증 종료 카드, AS 전후 기록.
- 결제 이유: 10대까지 월 3,900원 또는 이사/AS 증거팩 4,900원. 구매 증빙 누락으로 유상 처리되는 손실을 막는다.

### LC17 공동구매검수

- 타겟: 회사·동호회·가족 단위로 월 1회 이상 식품·생활용품을 묶어 주문하는 총무.
- 기존 앱 대비 차이: 주문 추적이 아니라 수령 순간에 참여자별 수량·파손·사진·환불 책임을 분리하고, 모두 확인된 행만 정산 완료로 닫는다.
- 전체 플로우: 참여자·품목·수량 등록 → 배송 수령 → 박스·품목별 사진 → 누락/파손 담당자 확인 → 환불 마감 → 참여자별 수령·비용 상태 종료.
- 가장 작은 MVP: CSV 또는 직접 입력 20행, 수령 사진, 누락 상태, 담당자 확인, 정산표.
- 결제 이유: 총무 월 4,900원 또는 건당 1,900원. 한 번의 누락·파손·환불 분쟁이 구독료보다 크다.

## 실패 후보 한 문장 사유

- LC03: 비용 기록은 반복되지만 카드앱·스프레드시트가 무료다.
- LC04: 지연 증거의 핵심 정보가 교통사업자에 있고 손실·결제가 작다.
- LC05: 예약·환불 상태를 각 예약 플랫폼이 이미 제공한다.
- LC06: 여행 전 일회성 체크리스트를 별도 앱으로 살 이유가 약하다.
- LC09: 가족 캘린더·메신저와의 차이가 결제 수준까지 크지 않다.
- LC10: 장보기 누락은 작고 장보기앱·메신저로 충분하다.
- LC12: 반복성은 높지만 무료 공유 캘린더와 펫앱이 강하다.
- LC13: 지속 입력이 번거롭고 쇼핑몰 정기배송이 더 편하다.
- LC15: 계절성 문제라 연간 결제가 약하고 관리사무소 안내와 겹친다.
- LC16: 배치 002의 하자증거와 제품 약속이 중복된다.
- LC18: 배치 003의 환불왔나와 핵심 상태가 중복된다.
- LC19: 만료 정보 조회에 머물러 필수 규칙 3을 넘지 못한다.
- LC20: 플랫폼 약관·중개가 핵심이라 제품이 결과를 소유하지 못한다.

## 집계·중복 검사

- 정확한 후보 수: **20개**
- 최종 통과: **7개** — LC01, LC02, LC07, LC08, LC11, LC14, LC17
- 최종 실패: **13개**
- 중복 검사: 배치 001~003의 `급여차이·근무표찍기·숨은구독·하자증거·학원공지한장·내말속도·러닝각도·이사반납·통신비구멍·언박싱증거·오답리턴·중고인수증·계약날짜·환불왔나·예약금마감·박스어디·부모님공문·진료메모·미수금톡`과 동일 제목·이름 바꾸기 후보를 제외했다. LC16·LC18은 의미상 인접해 각각 기존 후보와 중복으로 **실패**시켰고, 나머지 18개는 기존 32개 그룹·배치 후보의 핵심 결과를 재사용하지 않는다.
- 조사 상태: Firecrawl 원문 저장은 DNS 오류로 미완료. 국내 수치가 필요한 공개 전 단계에서는 위 7개도 통계청·한국소비자원·국토교통부·문체부·농식품부 원문으로 재검증해야 한다.
