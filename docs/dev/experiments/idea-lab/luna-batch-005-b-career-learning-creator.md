## 2. Pass 상세 브리프

### B05-01 경력성과증거팩

- Hook: 면접에서 “성과가 뭐였나요?”를 들은 뒤 파일과 수치를 찾지 않도록 이력서 문장 하나를 증거 묶음으로 잠근다.
- 구체 타겟: 최근 3개월 안에 이직 지원을 3건 이상 하며 프로젝트 파일은 있지만 성과 수치를 정리하지 못한 1~5년차 직무 전환자.
- 기존 앱과 차이: Teal·Huntr식 지원 목록이 아니라, 지원서의 주장 문장과 실제 파일/수치의 관계와 확인 상태를 소유한다. 공고를 찾아주거나 답변을 생성하지 않는다.
- 전체 플로우: 이력서 한 문장 입력 → 파일 1~3개와 수치 첨부 → 문장-증거 연결 확인 → 면접용 3개 카드와 원본 링크 내보내기.
- 4주보다 작은 MVP: 텍스트 한 문장, 파일 3개, 수동 증거 연결, 확인 체크, PDF 한 장 내보내기. 로그인·크롤링·회사 OAuth는 제외한다.
- 결제 시점·가격 가설: 첫 면접 일정이 잡힌 순간 1회 2,900원, 이직 기간 월 4,900원. 사용자가 “답변을 잘 썼다”가 아니라 잃을 수 있는 면접 기회를 줄이는 결과에 지불한다.

### B05-06 실기준비물장부

- Hook: 시험장 앞에서 줄자·장갑·재료 하나가 빠진 일을 막고, 종목별 가방을 회차마다 다시 맞춘다.
- 구체 타겟: 조리·미용·제과·용접 등 국가자격 실기를 준비하며 재료와 공구를 직접 챙기는 수험생.
- 기존 앱과 차이: Q-Net의 접수·시험 안내가 아니라 개인이 실제로 산 수량, 쓴 수량, 빌린 물품, 다음 시험에 남은 준비물을 소유한다.
- 전체 플로우: 종목 준비물 목록 붙여넣기 → 보유/구매/대여 수량 입력 → 연습 후 소진 체크 → 시험 전 부족 품목 한 개와 구매 메모 표시.
- 4주보다 작은 MVP: 종목별 목록 템플릿 5개, 수량 증감, 사진 첨부, 시험일 카운트다운, 부족 품목 화면. 쇼핑몰 연동은 제외한다.
- 결제 시점·가격 가설: 실기 접수 직후 시험 회차당 1,900원 또는 준비 기간 월 3,900원. 재료 재구매와 시험 응시 기회 손실을 줄이는 순간 결제한다.

### B05-11 학습과제증명

- Hook: 강의를 들었다는 말 대신 과제 전후와 피드백을 직무 역량의 증거 한 건으로 꺼낸다.
- 구체 타겟: 부트캠프·직무 강의에서 과제를 5건 이상 수행했지만 포트폴리오에 무엇을 넣을지 못 고르는 취업 준비생.
- 기존 앱과 차이: Thinkific·인프런의 수강 진도가 아니라 여러 학습원에 흩어진 과제, 피드백, 개선 전후와 역량 태그를 소유한다.
- 전체 플로우: 과제 원본·수정본·피드백 업로드 → 역량 한 개와 연결 → 개선 전후 확인 → 포트폴리오용 사례 카드 내보내기.
- 4주보다 작은 MVP: 파일 3개와 피드백 텍스트, 역량 태그 6개, 전후 비교, 카드 PDF. 학습 콘텐츠 판매와 AI 평가자는 제외한다.
- 결제 시점·가격 가설: 포트폴리오 제출 전 1회 3,900원 또는 취업 준비 월 4,900원. 수강료가 아니라 제출 가능한 증거 결과에 지불한다.

### B05-14 강의과제피드백큐

- Hook: 혼자 강의하는 사람이 카톡·메일·폼에 흩어진 과제 중 “아직 안 읽은 것”을 놓치지 않는다.
- 구체 타겟: 월 20명 이상에게 실습 과제를 받고 48시간 안에 피드백해야 하는 1인 온라인 강사·코치.
- 기존 앱과 차이: Thinkific의 플랫폼 안 과제 기능이 아니라, 클래스101·인프런·구글폼 등 여러 채널에서 받은 제출물을 한 큐로 모아 검토·피드백·재제출 상태를 소유한다.
- 전체 플로우: 제출 PDF/링크 1건 추가 → 학생·회차·마감 확인 → 피드백 작성 → 완료/재제출 상태 → 다음 3건 큐 표시.

- 결제 시점·가격 가설: 제출물 20건을 넘기는 강사에게 월 7,900원. 답변 생성이 아니라 누락 방지와 SLA 준수라는 운영 결과에 지불한다.

### B05-19 협찬납품청구팩

- Hook: 협찬 게시물을 올렸는데 브랜드가 요구한 산출물과 URL을 다시 찾느라 청구가 늦어지는 일을 끝낸다.
- 구체 타겟: 월 2건 이상 브랜드 협찬을 진행하는 팔로워 1만~10만 명대 마이크로 크리에이터.
- 기존 앱과 차이: `협찬 사용권 만료`가 권리 기간을 소유한다면, 이 제품은 캠페인별 게시 증거·산출물·승인·청구 상태를 소유한다. Frame.io식 창작물 승인 전체를 만들지 않는다.

- 4주보다 작은 MVP: 캠페인 한 개, 산출물 5개, URL·이미지 업로드, 수동 승인 체크, PDF/ZIP 내보내기. SNS API·자동 노출 수집은 제외한다.
- 결제 시점·가격 가설: 게시 완료 후 청구팩을 내보낼 때 건당 2,900원 또는 월 5,900원. 지급 지연·재요청을 줄이는 직접적인 현금화 순간에 지불한다.

- 기존 앱과 차이: 포트폴리오 빌더가 아니라 성과 수치의 출처와 역할 확인 상태를 소유한다. 단, 문서 템플릿과 코칭 서비스가 강한 대체재다.
- 전체 플로우: 작품 링크·프로젝트 메모 입력 → 문제/역할/결과 필드 작성 → 수치 출처 첨부 → 사례 카드 export.

### B05-08 훈련비환급팩

## 4. Fail 사유

- B05-02: UX 합의가 1/3이라 즉시 실패하며 채용 사이트 제출 이력·메일 보관으로 대체 가능하다.
- B05-03: 공고-경력 빈칸 분석은 Jobscan/범용 AI로 대체되고 `hidden-job-finder`의 채용공고 입력과 인접하다.
- B05-04: 면접 안내 상태는 기존 `스크린샷약속`과 캘린더·메일의 조합이 이미 소유한다.
- B05-07: 응시자격·서류 제출의 핵심 결과를 Q-Net 공식 절차가 직접 해결하며 법적 판정 위험이 있다.
- B05-09: UX 합의가 1/3이고 자격별 인정시간은 공식 기관 확인이 필요해 독립 제품으로 안전하게 소유하기 어렵다.
- B05-10: 내일배움카드 출석·장려금 상태는 고용24와 훈련기관이 관리한다.
- B05-13: 질문 큐는 `댓글약속장부`·`발표질문로그`와 미해결 질문 상태가 겹치고 강의 플랫폼 게시판이 대체한다.
- B05-15: 원고-장표 연결은 `강의자료변경표`·`파일버전찾기`와 문서 비교 결과가 겹친다.
- B05-16: 수료증과 교육 제출 PDF는 각 플랫폼·기관 발급 기능과 PDF 병합으로 충분하다.
- B05-17: UX 합의가 1/3이고 기존 사용권 관리와 인접하며 후기 동의의 법무 판단 위험이 있다.
- B05-18: 촬영 소품 회수는 체크리스트·사진 앨범으로 해결되며 반복 결제 근거가 약하다.
- B05-20: 콘텐츠 출처 카드는 `lookup-brief`의 출처 정리와 입력·근거 결과가 실질적으로 겹친다.
- B05-21: 뉴스레터 링크 검수는 브라우저/메일 테스트와 `다운로드검수`의 제출 전 검수에 가깝다.

## 6. 집계 및 중복 감사

- 최종 집계: **Pass 5 / Hold 3 / Fail 17 = 25**.
- 표 행 검증: `B05-01`부터 `B05-25`까지 연속 ID 25행이며, 헤더·상세 브리프·사유 행은 후보 행 수에 포함하지 않았다.
- 기존 32개 그룹 대비 제목 중복: **0**. 기존 `lookup-brief`, `hidden-job-finder`, `interview-copilot`, `screenshot약속`, `강의자료변경표`, `댓글약속장부`, `발표질문로그`, `협찬 사용권 만료` 등과 제목을 다르게 썼다.
- 실제 입력 중복 감사: 공고·이력서→조건 빈칸, 카톡 캡처→마감 상태, PDF 두 개→변경 문장, 후기/권리기간 등 인접 루프는 Fail 또는 Hold로 내렸다. Pass는 파일·과제·게시 증거를 각각 경력 성과, 준비물 재고, 역량 증거, 피드백 큐, 청구팩이라는 다른 소유 상태로 제한했다.
- 소유 상태 중복 감사: 단순 목록·일정·진도·요약·생성 결과는 기존 앱 또는 플랫폼 소유로 보아 탈락시켰고, Pass는 개인이 여러 기관/채널의 원본 증거를 한 번 확인 가능한 상태로 묶는 경우만 남겼다.
- 즉시 결과 중복 감사: `오늘 할 일`, `요약`, `추천`, `문장 생성`을 사용하지 않았다. Pass의 결과는 각각 면접 증거 3개, 부족 준비물 1개, 개선 사례 카드, 피드백 큐 3개, 청구 증거팩이다.
