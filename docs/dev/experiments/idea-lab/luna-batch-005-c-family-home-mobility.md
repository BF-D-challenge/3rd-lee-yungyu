## 1. 방법

후보는 Luna가 독립 발굴했고, P1·P2·P3는 서로 분리된 Luna 컨텍스트에서 같은 ASCII 카드 형식으로 평가했다. 최종 판정은 루트 하드게이트 감사 결론을 적용했다. Firecrawl은 사용하지 않았다. 합성 UX 평가는 실제 사용자를 대상으로 한 검증이 아니라 선별 가설이다.

`P1/P2/P3`는 각각 `이해/한국/결제`이며, `Y/Y/Y`인 페르소나 수를 UX 합의로 집계했다.

## 3. Pass/Hold 상세 브리프

### C04 집소모품마감 — Hold

- **Hook:** “지난번 필터를 언제 갈았지?”를 라벨 사진 한 장과 주기 입력으로 이번 달 교체 큐로 바꾼다.
- **구체 타겟:** 정수기·공기청정기·화재경보기 등 교체주기가 다른 소모품 10개 이상을 직접 관리하는 3~4인 자가 가구의 집안 관리 담당자.
- **기존 앱과 차이:** HomeBinder 같은 해외 원본의 자산·주기 기록 메커니즘을 좁혀, 목록 관리가 아니라 이번 달 미처리 교체 큐와 완료 상태를 소유한다. 다만 캘린더·기기 알림이라는 한국 무료 대안이 있다.
- **전체 플로우:** 제품 라벨 사진 촬영 → 품목명·마지막 교체일·주기 확인/수정 → 담당자 지정 → 이번 달 큐 생성 → 교체 사진 또는 완료 체크 → 다음 주기 시작.
- **4주보다 작은 MVP:** 로그인·결제 없이 품목 5개, 사진 1장, 주기·담당자 입력, 월간 큐와 완료 토글만 구현한다.
- **결제 시점·가격 가설:** 10개 품목을 등록하고 첫 반복 큐를 만든 뒤, 월 2,900원 또는 연 29,000원을 제시한다. 실제 유료 반복과 초기 입력 유지가 확인되기 전에는 보류한다.
- **다음 실제 검증:** 소모품 10개 이상을 직접 관리하는 가구 5팀에게 6주간 품목을 등록하게 하고, 월간 큐 재방문과 결제 의향을 확인한다.

### C10 정비견적차이 — Pass

- **Hook:** 정비소의 약속과 실제 청구가 달라졌는지 견적서와 정비명세서를 나란히 비교해 추가금·누락 작업을 표시한다.
- **구체 타겟:** 중고차 또는 보증기간이 지난 차량을 보유하고, 정비 때마다 항목·금액을 확인해야 하는 운전자.
- **기존 앱과 차이:** CARFAX Car Care가 차량 관리 기록을 앵커로 삼는다면, C10은 정비 예약·진단·법률판정을 하지 않고 실제 문서 쌍의 항목과 금액 차이만 소유한다. 자동차관리법 제58조 및 자동차365를 한국의 문서·규정 앵커로 삼되, 링크는 시장·행동 근거이지 서비스의 정확성을 보증하지 않는다.
- **전체 플로우:** 견적서 촬영 → 정비명세서 촬영 → OCR 항목명·수량·금액 보정 → 약속/실제 매칭 → 차액·추가·누락 표시 → 비교 PDF 저장/공유.
- **4주보다 작은 MVP:** 로그인·결제·차량 데이터 연동 없이 두 이미지 업로드, 수동 OCR 보정, 항목 매칭과 차액 표 한 화면만 만든다.
- **결제 시점·가격 가설:** 비교 결과에서 차액을 확인한 직후 정비 1건당 2,900원을 제시한다. 법률 상담이나 과다청구 판정이 아니라 문서 비교 결과에만 과금한다.
- **다음 실제 검증:** 최근 정비 문서 쌍을 가진 운전자 5명에게 실제 문서를 받아 매칭 오류를 수동 기록하고, 차액 확인 직후 2명 이상이 건당 결제하는지 테스트한다.

### C25 가족안전확인 — Hold

- **Hook:** 부모님에게 온 의심 문자 한 건을 가족이 확인했는지, 연락했는지, 신고했는지 끝난 상태로 닫는다.
- **구체 타겟:** 고령 부모의 문자·전화를 함께 확인하는 성인 자녀·형제 가족.
- **기존 앱과 차이:** Truecaller 같은 해외 원본은 발신자 식별 메커니즘의 앵커이고, C25는 진위판정을 하지 않는다. 대신 캡처된 사건의 가족 검토·연락·차단·신고 상태를 소유한다. KISA 스미싱 안내와 118은 행동·신고 대안 앵커라서, 유료 서비스가 이를 대체한다고 과장하지 않는다.
- **전체 플로우:** 의심 문자 캡처 업로드 → 발신번호·수신자 입력 → 가족 검토 요청 → 가족 확인/전화 여부 체크 → KISA 118 또는 신고 링크 열기 → 차단·신고·확인 완료 상태 저장.
- **4주보다 작은 MVP:** 가족 3명, 사건 1건, 캡처 1장, 승인 링크, 상태 5개와 신고 링크만 구현한다. 자동 진위판정·전화 차단·금융기관 연동은 제외한다.
- **결제 시점·가격 가설:** 가족이 두 번째 사건을 확인하고 상태를 닫은 뒤 가족방 월 3,900원을 제시한다. 카카오톡 무료 공유와 KISA 안내 대비 추가 결제 이유, 가족 응답률을 먼저 검증한다.
- **다음 실제 검증:** 고령 부모를 돕는 가족 5팀에게 2주간 실제 또는 모의 의심 문자 3건을 처리하게 하고, 가족 응답률·카카오톡 대비 시간 절감·유료 전환을 측정한다.

## 4. Fail 22개 이유

- **C01:** Splitwise와 무료 정산 대안이 있고 가족별 납부 확인을 실제로 입력·응답할지 미검증이다.
- **C02:** 연 1회 사용에 보험사 알림·금융 플랫폼 대체가 강해 결제 가치가 약하다.
- **C03:** 정부24가 핵심 민원 결과를 제공하고 가족 인계는 메시지 공유로 대체 가능하다.
- **C05:** K-apt와 스프레드시트 대비 월별 수동 고지서 입력의 결제 결과가 약하다.
- **C06:** 센서 앱·무료 기록과 경쟁하며 습도 수동 입력을 지속할 이유가 약하다.
- **C07:** 가족 창고 갱신 빈도와 유인이 낮고 사진·메모·무료 재고도구로 대체된다.
- **C08:** 민감한 출입정보를 별도 저장하는 보안 리스크가 핵심 가치보다 크다.
- **C05-09:** 단지 앱·관리사무소가 승인 상태를 직접 처리해 별도 앱이 결과를 소유하지 못한다.
- **C11:** 수동 귀속 부담이 있고 EV 앱·간편 정산 대안보다 효율 천장이 낮다.
- **C12:** 하이패스 서비스와 스프레드시트가 명세서 분류·정산을 대체한다.
- **C13:** 기본 카메라와 보험사 앱으로 사고 직후 기록이 가능하고 사고 빈도가 낮다.
- **C14:** 안전 설치 정확도와 책임을 앱 체크만으로 신뢰하기 어렵다.
- **C15:** 가족 자전거 공유 빈도가 낮고 채팅·사진으로 반납 확인이 가능하다.
- **C16:** Tricount·Splitwise 같은 무료 정산 도구가 반려동물 비용에도 적용된다.
- **C17:** 기존 사료잔량과 이름 혼선이 있고 쇼핑몰 정기배송이 더 직접적인 대안이다.
- **C18:** 수면과 돌봄 교대의 제목·결과가 모호하고 무료 수면앱 및 기존 돌봄교대장과 겹친다.
- **C19:** 참석 확인은 캘린더·메신저로 충분해 반복 결제 효익이 약하다.
- **C20:** 가족이 식재료 상태를 계속 입력해야 해 반복 클릭 비용이 크다.
- **C21:** 분기별 사용 빈도가 낮고 국민재난안전포털의 무료 체크리스트가 있다.
- **C22:** 기존 `계약날짜 — 전월세 계약서에서 놓치면 비싼 날짜만`과 동일한 입력·결과다.
- **C23:** 기존 `하자증거`·`관리실작업`과 내부 결과가 인접하고 카카오톡·캘린더·업체 앱으로 대체된다.
- **C24:** C04와 내부 중복이며 렌탈사가 배송·교체 핵심 상태를 이미 제공한다.

## 6. 요약 및 기존 후보 중복 검사

- 최종 집계: **Pass 1 / Hold 2 / Fail 22**.
- 최종 표는 **25행**을 확인했다.
- C22는 기존 `계약날짜`와 동일한 입력·결과라 탈락했다.
- C23은 기존 `하자증거`·`관리실작업`과 상태 결과가 중복된다.
- C24는 C04 및 기존 `가전보증팩`과 주기·소모품 상태가 내부 중복이다.
- C17은 기존 `사료잔량`과 이름 혼선 및 잔량→구매 결과가 겹친다.
- C18은 기존 `돌봄교대장`과 이름·돌봄 인계 맥락이 혼선된다.
