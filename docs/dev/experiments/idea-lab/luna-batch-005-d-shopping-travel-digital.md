## 2. Pass 상세 브리프

### D01 반품 보낼 날

- Hook: 반품 접수는 했는데 “언제까지 실제로 보내야 하지?”를 주문마다 놓치는 사람에게 오늘 보낼 상자와 증거를 보여준다.
- 구체 타겟: 월 2회 이상 온라인 주문을 반품하는 맞벌이 직장인.
- 기존 앱과 차이: 쇼핑몰 배송/환불 화면을 복제하지 않고, 접수 화면과 송장을 입력받아 발송 마감과 완료 증거의 상태만 소유한다.
- 전체 플로우: 접수 캡처 업로드 → 마감일·택배 방식 입력 → 발송 사진/송장 입력 → 미완료 반품 한 장과 증거 PDF.
- 4주보다 작은 MVP: 사진 2장과 날짜 2개를 받는 2화면 웹앱, 로컬 저장, PDF 내보내기.
- 결제 시점·가격 가설: 반품 증거 PDF를 내보낼 때 월 2,900원 또는 10건 1,900원.

### D02 배송 누락 묶음

### D09 숙소 체크인 약속

- Hook: 현장에서 호스트 메시지가 바뀌었을 때 최신 체크인 약속을 찾는다.
- 구체 타겟: 연 2회 이상 직접 숙소를 예약하는 해외 자유여행자.
- 기존 앱과 차이: 숙소 예약·메시지 기능을 복제하지 않고 약속 버전과 마지막 확인자만 소유한다.
- 전체 플로우: 예약/메시지 캡처 → 체크인 항목 4개 → 양측 확인 → 최신 카드. MVP는 수동 입력과 공유 링크다.
- 4주보다 작은 MVP: 숙소 1건, 확인 항목 4개, 변경 타임라인.
- 결제 시점·가격 가설: 여행당 1,900원 가설. 카카오톡·예약 앱보다 낫다는 7일 재사용을 먼저 확인한다.

### D22 앱 권한 점검표

- Hook: 새 앱을 설치할 때 사진·위치·마이크 중 불필요하게 열린 권한 하나를 찾는다.
- 구체 타겟: 업무·생활 앱을 매달 3개 이상 설치하는 개인정보 민감 사용자.
- 기존 앱과 차이: 기기 전체 권한을 자동 수집하지 않고 사용자가 캡처를 넣어 점검 이유와 다음 확인 상태를 남긴다.
- 전체 플로우: 권한 화면 캡처 → 앱별 사용 이유 입력 → 검토 상태 → 끌 권한 목록. MVP는 수동 체크 1화면이다.
- 4주보다 작은 MVP: 앱 10개, 권한 6종, 캡처 첨부, 로컬 저장.
- 결제 시점·가격 가설: 보류. 월 1회 반복성이 약해 주기적 재사용과 5명 결제 테스트 전에는 결제 가설을 확정하지 않는다.

## 4. Fail 사유

- D04 가격 내림 청구함: 가격비교·가격보호가 강한 플랫폼 기능과 직접 겹치고, 판매자별 청구 가능성이 불명확하다.
- D14 동네 예약 대기표: 캘린더·문자·예약 플랫폼으로 충분하며 독립 앱에 반복 결제할 결과가 약하다.
- D20 디지털 유산 봉투: 일회성 정리 성격이 강하고 가족·법무·상속 서비스가 더 강한 대안이다.
- D21 와이파이 방문증: 공유기 게스트 네트워크와 비밀번호 변경으로 핵심 결과가 이미 해결된다.
