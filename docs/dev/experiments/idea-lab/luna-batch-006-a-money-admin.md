## 집계·중복 감사

- 후보 표 데이터 행: **정확히 25개**, ID `A06-01`~`A06-25` 연속.
- 최종 판정 합계: **Pass 0 + Hold 0 + Fail 25 = 25**.
- UX 합의: **0/25**가 2/3 이상. 전 후보가 0/3이므로 즉시 Fail이며, 통과 수를 맞추지 않았다.
- 기존 32개 앱 그룹과의 감사: 보험 청구·환급·서류 추적은 기존 기관 결과를 대체할 수 없어 Fail. `하자증거`, `환불왔나`, `급여차이`, `통신비구멍`, `중고인수증`, `정비견적차이`와 인접한 후보도 결과 소유권이 겹치면 Fail 처리.
- 배치 001~004 중복: `하자증거`, `환불왔나`, `급여차이`, `미수금톡`, `수리비 선지급 장부`, `보증수리 만료함`, `플랫폼 정산 차이`와 입력·소유 상태·즉시 결과가 겹치는 후보는 위 표와 Fail 한 줄에 명시했다.
