## 판정 방법

로컬 TrustMRR·기존 앱 32개 그룹·배치 001~005·B05를 먼저 대조했다. 표의 해외 근거는 매출·사용량이 확인된 원본 흐름의 앵커이고, 한국 근거는 한국에서 같은 행동이 일어나는지와 기본 대안이 무엇인지 확인한 앵커다. 합성 UX는 실제 사용자 검증이 아니라 선별 가설이다.

## 2. Hold 상세 브리프

### B06-01 면접과제 제출봉투

- Hook: 채용 과제를 보냈는데 요구사항 하나를 빠뜨렸는지, 회사가 받았는지 다시 찾지 않게 한다.
- 구체 타겟: 한 달에 채용 과제를 2건 이상 제출하는 1~5년차 직무 전환자.
- 기존 앱과 차이: 지원 공고·면접 일정을 관리하는 도구가 아니라, PDF·노션·메일 등 외부 채널로 제출한 과제의 요구사항과 제출 증거를 한 사건으로 소유한다. 답변 문장 생성은 하지 않는다.
- 전체 플로우: 과제 안내 한 건 입력 → 요구사항 5개와 제출 URL/메일 첨부 → 제출시각·수신확인 체크 → 누락 1개와 제출 증명 PDF.
- 가장 작은 MVP: 요구사항 수동 입력, 파일 3개, 제출 URL 1개, 확인 상태 3개, PDF 내보내기. 채용 사이트 로그인·메일 OAuth·자동 추적은 제외한다.
- 결제 시점·가격: 제출 직전 건당 2,900원 가설. 현재 국내 구직자가 지원 추적·문서 템플릿보다 이 증거에 별도 결제하는지 미확인이므로 Hold.

### B06-03 퇴사인수증

- Hook: 퇴사 후 “그 업무를 인수받지 못했다”는 말이 나오기 전에 전달과 수락을 같은 화면에 남긴다.
- 구체 타겟: 한 달 안에 퇴사하거나 팀 이동하며 운영 업무 10개 이상을 넘기는 1~5년차 사무·운영 직군.
- 기존 앱과 차이: 문서 공유나 할 일 목록이 아니라 항목별 수신자 확인, 수락, 재질문을 소유한다. 회사의 인사·전자결재 기록을 대체하지 않는다.
- 전체 플로우: 인수 목록 10개 입력 → 파일 링크·담당자 연결 → 수신자용 확인 링크 → 미수락 항목과 확인 로그 PDF.
- 가장 작은 MVP: 목록 10개, 링크 3개, 확인 링크 1개, 상태 3개, 로그 PDF. 조직 계정·사내 메신저 연동·법적 책임 판정은 제외한다.
- 결제 시점·가격: 퇴사 통보 직후 건당 3,900원 가설. 사내 메일·문서·인수인계 양식 대비 결제 우위와 외부 수신자 확인율을 확인하기 전 Hold.
