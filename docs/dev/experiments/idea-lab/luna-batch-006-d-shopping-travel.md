검사일: 2026-07-12

## 2. Pass/Hold 상세 브리프

### D06-05 해외 주문 취소기록 — Hold

- Hook: 해외 주문을 취소한 뒤 판매자 승인과 카드 환급이 따로 움직여 “끝난 건지” 모를 때 한 주문의 두 상태를 함께 확인한다.
- 구체 타겟: 분기 2회 이상 해외직구를 하고 취소·환급을 직접 처리하는 전자기기 구매자.
- 기존 앱과 차이: 판매자 환불 판정을 대신하지 않고, 사용자가 넣은 취소 화면·답변·명세의 버전과 마지막 확인일만 소유한다.
- 전체 플로우: 취소 화면 입력 → 판매자 답변과 환급 예정일 기록 → 카드 환급 확인 → 미완료 주문의 제출 묶음.
- 4주보다 작은 MVP: 주문 1건, 증거 5개, 상태 4개, 수동 날짜, PDF 내보내기. 결제 연동과 자동 환율은 제외한다.
- 결제 시점·가격 가설: 환급 확인이 늦어진 사건의 증거 PDF를 내보낼 때 건당 1,900원. 실제 해외직구 취소자 5명 중 2명 선결제가 재진입 조건이다.

### D06-12 호텔 객실 상태팩 — Hold

- Hook: 체크인 직후 찍은 객실 상태와 프런트의 조치 약속을 체크아웃 전에 다시 꺼낸다.
- 구체 타겟: 해외 숙소를 연 3회 이상 직접 예약하는 가족 여행의 예약 담당자.
- 기존 앱과 차이: 숙소 검색·예약·보상을 만들지 않고, 입실 시각이 있는 사진과 프런트 확인 상태를 한 사건으로 보관한다.
- 전체 플로우: 객실 사진 3장 입력 → 하자 유형과 체크인 시각 기록 → 프런트 답변 추가 → 조치 완료 확인 → 체크아웃 전 증거 PDF.
- 4주보다 작은 MVP: 숙소 1건, 하자 6종, 사진 8장, 수동 확인 버튼, 공유 가능한 PDF. 숙소 API와 보상 판정은 제외한다.
- 결제 시점·가격 가설: 체크아웃 전 증거 묶음 내보내기 시 숙소당 2,900원. 숙박업소의 실제 확인 링크 완료율과 사건당 결제를 먼저 검증한다.

### D06-19 체크인 시간 변경 확인 — Hold

- Hook: 늦은 도착으로 열쇠 약속이 바뀌었을 때 호스트와 게스트가 마지막 시간을 같은 카드에서 확인한다.
- 구체 타겟: 연 2회 이상 셀프 체크인 숙소를 쓰는 해외 자유여행자.
- 기존 앱과 차이: 예약·채팅을 복제하지 않고, 메시지에서 사용자가 옮긴 약속 버전과 양측 확인 상태만 소유한다.
- 전체 플로우: 예약서와 메시지 입력 → 체크인 항목 4개 기록 → 호스트·게스트 확인 → 변경 시 새 버전 → 현장용 최신 카드.
- 4주보다 작은 MVP: 숙소 1건, 항목 4개, 확인 링크 1개, 변경 타임라인. 예약 API·자동 알림·결제 대행은 제외한다.
- 결제 시점·가격 가설: 여행당 1,900원. 카카오톡·예약 앱보다 빠른 현장 확인을 5개 여행에서 검증하고 2명 결제해야 Pass로 올린다.

## 3. Fail 한 줄 이유

- D06-01: 주문 플랫폼이 배송지 변경의 원문과 출고 상태를 소유하며 별도 결제 결과가 없다.
- D06-02: 배치 005 D02 배송 누락 묶음과 입력·소유 상태·즉시 결과가 동일하다.
- D06-03: 통관·납부 결과는 관세청·배송사가 소유하고 연 1회성에 가깝다.
- D06-04: 택배사 배송조회와 기사 연락이 핵심 결과를 직접 제공한다.
- D06-06: 배치 005 D14 예약금마감·D09 숙소 약속과 중복되고 반환 판단은 예약사가 소유한다.
- D06-07: 항공사가 예약 원문을 소유하며 민감한 여권 정보의 별도 보관 위험이 크다.
- D06-08: 항공사 좌석·결제 화면이 같은 결과를 직접 제공하고 사용이 일회성이다.
- D06-09: 예약 플랫폼·메신저가 투어 예약과 변경 연락을 소유하며 D05 D07과 인접하다.
- D06-10: 항공·교통 예약사가 원문을 소유하고 배치 005의 여행 인계 결과와 중복된다.
- D06-11: 보험사가 청구 서류·심사 상태를 소유하고 배치 005 A03과 중복된다.
- D06-13: 배치 005 D16 렌탈 반납·C13 사고 증거와 차량 전후 증거 결과가 인접하다.
- D06-14: 배치 005 D08 해외 결제 보관함과 입력·상태·정산표 결과가 동일하다.
- D06-15: 배치 005 D19 파일 전달 확인과 최신 파일·상대 확인 결과가 사실상 동일하다.
- D06-16: 면세점 주문·수령 시스템이 품목과 수령 상태를 직접 소유한다.
- D06-17: 배치 005 D23 해외 eSIM 개통봉투와 설치·활성화 상태가 동일하다.
- D06-18: 배치 004 LC05 여행변경팩과 예약 취소·남은 비용 결과가 중복된다.
- D06-20: 배치 004 LC04 환승증빙과 항공 지연 증거팩 결과가 인접하고 항공사가 운항 원문을 소유한다.
- D06-21: 카드사·결제망이 청구 환율과 최종 금액을 소유해 금융 판정 위험이 있다.
- D06-22: 예약 업체가 양도 가능 여부와 승인 상태를 소유하고 여행 일회성이다.

- D06-24: D06-18과 예약 변경·추가금 상태가 중복되고 여행사·숙소가 원문을 소유한다.
