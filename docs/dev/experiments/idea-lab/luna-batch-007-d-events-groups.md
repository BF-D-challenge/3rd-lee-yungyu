## 2. Pass/Hold 상세 브리프

### D07-02 행사 역할확인 — Hold

- **Hook:** 행사 전날 “누가 무엇을 맡았지?”를 담당자 완료 사진과 상대 확인이 끝난 한 건으로 보여준다.
- **구체 타겟:** 월 1회 이상 10~40명 규모의 사내 소모임·학교 모임·동호회 행사를 직접 운영하는 총무.
- **기존 앱과 차이:** 일정·할 일 목록을 만드는 것이 아니라 역할별 담당자 수락 → 현장 증거 → 운영자의 확인이라는 닫힌 상태를 소유한다. 단체 채팅과 공유 문서는 최신 미완료 역할을 강제하지 않는다.
- **전체 플로우:** 행사 1건 생성 → 역할 8개 입력 → 담당자 초대·수락 → 사진/메모로 완료 → 운영자 확인 → 미완료 한 건 재지정.
- **가장 작은 MVP:** 로그인 없는 행사 1건, 역할 8개, 초대 링크 1개, 사진 1장, `담당/완료/확인` 3상태와 미완료 화면.
- **결제 시점·가격 가설:** 행사 초대 링크를 3명 이상이 열고 두 역할이 확인된 직후 건당 2,900원 또는 총무 월 4,900원을 제시한다. 실제 유료 전환과 반복 행사가 아직 검증되지 않아 Hold다.

### D07-19 봉사 교대인계 — Hold

- **Hook:** 행사 현장에서 다음 교대자가 받아야 할 준비·수거·미완료 한 건을 사진과 확인자로 넘긴다.
- **구체 타겟:** 월 2회 이상 플리마켓·지역축제·동호회 행사를 운영하며 3교대 이상 자원봉사자를 조율하는 소규모 운영자.
- **기존 앱과 차이:** 참가 신청이나 봉사시간 인증을 대체하지 않고, 구역별 현장 상태와 다음 담당자의 수락을 소유한다. 1365는 활동·시간의 공식 기록이고, 이 후보는 현장 인계만 다룬다.
- **전체 플로우:** 구역 5개와 교대시간 입력 → 담당자 수락 → 현장 체크·사진 → 다음 담당자 확인 → 미완료 항목 재인계 → 행사 종료 요약.
- **가장 작은 MVP:** 행사 1건, 구역 5개, 교대 2회, 사진 1장, 다음 담당자 확인 링크, 미완료 목록. 실시간 위치·푸시·봉사시간 연동은 제외한다.
- **결제 시점·가격 가설:** 두 번째 교대에서 인계가 완료된 뒤 행사당 3,900원. 운영자 5팀의 실제 재사용과 현장 인계 실패 감소가 확인되기 전까지는 Hold다.

### D07-20 행사 잔여물품 인계 — Hold

- **Hook:** 행사 뒤 남은 현수막·공구·비품을 누가 어디에 보관했는지 다음 행사 담당자가 바로 확인한다.
- **구체 타겟:** 월 1회 이상 반복 행사를 운영하며 공용 비품 20개 이상을 보관하는 동호회·소규모 비영리 운영자.
- **기존 앱과 차이:** 일반 재고 앱처럼 수량만 세지 않고 행사 종료 시점의 잔여 사진 → 보관자 확인 → 다음 담당자 인수 상태를 한 사건으로 잠근다. 다만 Sortly·공유 문서와의 결제 우위가 아직 약하다.
- **전체 플로우:** 행사 종료 사진 업로드 → 품목·수량·보관 장소 확인 → 보관자 확인 → 다음 행사 담당자 인수 → 누락·파손 항목 표시.
- **가장 작은 MVP:** 품목 20개, 사진 2장, 보관자·인수자 링크, `남음/보관/인수` 상태, 행사별 PDF 목록. 바코드·창고 관리·자동 재고는 제외한다.
- **결제 시점·가격 가설:** 다음 행사 담당자가 인수 완료한 직후 행사당 2,900원 또는 월 4,900원. 5개 운영팀의 2회 재사용과 선결제가 확인되기 전까지 Hold다.
