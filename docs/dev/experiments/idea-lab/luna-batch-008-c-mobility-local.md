## 조사·판정 방법

로컬 규칙·루브릭·검증 로그·005 통합본·앱 샘플 데이터·TrustMRR `ideas.jsonl`과 기존 `batch-*`, `luna-batch-*` 보고서를 먼저 끝까지 읽었다. Firecrawl과 인터넷 조회는 사용하지 않았다. TrustMRR는 해외 수요 앵커로만 사용했고, 한국 근거는 기존 로컬 보고서에 이미 기록된 공식 기관·국내 앱 URL을 출처 앵커로 재사용했다. 시장 규모·사용자 수·매출을 새로 추정하지 않았다.

각 카드는 `실제 입력 → 제품이 소유하는 상태 → 즉시 결과`가 한 번에 드러나는지 검사했다. P1 신입, P2 중급, P3 숙련은 서로 독립된 합성 워크스루로 `10초 이해/한국 실제 사용/결제 이유`를 Y/N 평가했다. 합성 UX는 실제 사용자 검증이 아니라 선별 가설이다. UX 2/3 미만은 Hold로 남기지 않고 즉시 Fail 처리했다. `월 1회 반복 또는 5만원 이상 손실`, `강한 한국 앱·공공기관·OS·무료 대안`, `누적 상태·반복 큐·증거·양측 확인·검증 데이터 소유`, `4주 MVP`를 하드 게이트로 적용했다.
