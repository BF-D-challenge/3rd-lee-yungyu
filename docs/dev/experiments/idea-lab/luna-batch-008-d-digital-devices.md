# Luna 배치 008 D — 기기·계정·파일·사진·개인정보의 반복 상태

검사일: 2026-07-12  
범위: 개인이 직접 보유한 휴대폰·노트북·저장장치·사진·계정의 백업, 교대, 수리, 폐기 전후 상태. OS·클라우드·제조사·계정 제공자의 공식 결과를 복제하지 않고, 사용자가 여러 기기와 사람 사이에서 직접 확인해야 닫히는 상태만 후보로 남겼다. 의료·법률·보안 인증 판정은 제품 소유 범위에서 제외했다.

## 조사·판정 기준

- 공통 규칙의 입력과 하드 게이트, 트랙 맵, 이전 `batch-*`·`luna-batch-*` 보고서를 대조했다. 특히 배치 005 D의 계정 복구·사진 공유·파일 전달·디지털 유산·앱 권한·eSIM·보증서·구독 해지 후보는 동일 입력→상태→결과로 중복 감사했다.
- 로컬 TrustMRR의 백업·파일·개인정보·자산관리 메커니즘 신호와 Apple, Google, Microsoft, KISA, 개인정보보호위원회 등 국내외 기본 대안을 발산 앵커로 사용했다. 해외 매출 신호는 한국 수요의 증거로 해석하지 않았다.
- P1 신입 / P2 중급 / P3 숙련은 각각 `10초 이해/한국 실제 사용/결제 이유`다. UX 합의는 세 항목이 모두 Y인 페르소나 수이며 2/3 미만은 즉시 Fail이다.
- 하드 게이트 순서: 반복·손실 / 국내 강한 대안보다 나음 / 제품이 소유할 결제 결과 / 4주 MVP. 은행 OAuth, 상시 백그라운드 수집, 비밀번호·복구코드 원문 수집은 제외했다.

## 2. Pass/Hold 상세 브리프

### D08-02 백업 복원 시험 — Hold

- **Hook:** “백업 완료” 표시만 믿지 않고, 새 기기에서 실제로 열리는 파일 한 개를 확인해 복원 실패를 미리 발견한다.
- **구체 타겟:** 업무 파일과 가족 사진을 두 대 이상의 기기에 보관하며 분기 1회 이상 기기를 교체·초기화하는 프리랜서·가족 대표.
- **기존 앱과 차이:** 백업 저장소를 대체하지 않는다. 사용자가 지정한 시험 파일, 복원 기기, 실제 열림 확인을 한 사건으로 묶어 `백업됨/복원됨/열림` 상태를 소유한다.
- **전체 플로우:** 백업 화면 캡처와 시험 파일 1개 입력 → 복원할 기기 지정 → 파일 복원·열림 확인 → 실패 파일과 다음 행동 출력.
- **가장 작은 MVP:** 사건 1건, 파일 5개, 수동 상태 3개, 확인 사진 1장, PDF 내보내기. 클라우드 OAuth·자동 복원·파일 내용 분석은 제외한다.
- **결제 시점·가격 가설:** 복원 시험 결과 PDF를 내보낼 때 건당 2,900원 또는 연 4,900원. 실제 기기 교체자 5명 중 2명 선결제 전까지 Hold다.

### D08-05 가족 태블릿 교대카드 — Hold

- **Hook:** 가족이 태블릿을 바꿔 쓸 때 충전기·필수 앱·다운로드 상태를 다음 사용자에게 확인받아 “어디까지 준비됐지?”를 없앤다.
- **구체 타겟:** 아이·부모·보호자가 한 대 이상의 태블릿을 주 3회 이상 교대 사용하는 가정의 관리 담당자.
- **기존 앱과 차이:** 가족 계정·자녀보호 설정을 대체하지 않고, 기기별 물리 구성품과 다음 사용자의 인수 확인을 소유한다. D08-11은 구성품 단독이라 중복으로 Fail 처리했다.
- **전체 플로우:** 기기와 구성품 사진 입력 → 필수 앱·다운로드 목록 5개 작성 → 다음 사용자 확인 → 미준비 항목 재확인.
- **가장 작은 MVP:** 기기 2대, 항목 8개, 초대 링크 1개, `준비/인수/미확인` 상태. 가족 계정 연동·위치 추적·푸시는 제외한다.
- **결제 시점·가격 가설:** 두 번째 교대 인수 확인 뒤 월 3,900원. 5가정의 2주 반복 사용과 유료 전환을 확인하기 전 Hold다.

### D08-08 수리 후 기능 대조 — Hold

- **Hook:** 수리 맡기기 전과 돌아온 뒤의 증상·부품·데이터 상태를 비교해 다시 문의할 차이 한 건을 바로 찾는다.
- **구체 타겟:** 휴대폰·노트북 수리를 연 1회 이상 맡기며 수리 전후 사진과 접수증을 보관하는 개인·소형 사업자.
- **기존 앱과 차이:** 수리 진단이나 책임 판정을 하지 않는다. 사용자가 찍은 전후 사진과 체크 항목의 변화를 저장하고, 수리점에 재확인할 사실만 출력한다. 배치 004 `사진전후대조`와 인접해 결제 우위 검증이 필요하다.
- **전체 플로우:** 수리 전 사진·접수증 입력 → 증상·부품 항목 체크 → 수리 후 사진 입력 → 전후 차이와 미확인 항목 출력.
- **가장 작은 MVP:** 사건 1건, 항목 10개, 사진 6장, 전/후/확인 상태, 공유 가능한 PDF. 수리점 API·자동 진단·법률 문구는 제외한다.
- **결제 시점·가격 가설:** 수리 후 비교 PDF를 내보낼 때 건당 2,900원. 오프플랫폼 수리 이용자 5명 중 2명 결제 전 Hold다.

### D08-19 외장 백업 교대장 — Hold

- **Hook:** 외장 저장장치에 백업은 했지만 같은 장소에 같이 둔 상태를 피하도록, 최근 백업 장치와 보관 위치를 교대로 확인한다.
- **구체 타겟:** 고객 파일·가족 사진 등 5만원 이상의 손실이 예상되는 자료를 월 1회 이상 외장장치에 백업하는 1인 사업자.
- **기존 앱과 차이:** 백업 파일을 분석하지 않고 장치별 최근 백업·분리 보관·다음 담당 확인 상태를 소유한다. 클라우드 백업 서비스의 저장 결과는 대체하지 않는다.
- **전체 플로우:** 장치 사진·백업 로그·보관 위치 입력 → 최근 백업 확인 → 다른 장소 보관 확인 → 다음 교대 알림용 카드 출력.
- **가장 작은 MVP:** 장치 2개, 백업 주기 1개, 보관자 확인 링크, `백업/분리/확인` 상태. 자동 감시·위치 추적은 제외한다.
- **결제 시점·가격 가설:** 두 번째 교대 확인 뒤 월 4,900원. 5개 소규모 사업자의 4주 재사용과 결제를 확인하기 전 Hold다.

### D08-22 휴대폰 폐기 전 삭제봉투 — Hold

- **Hook:** 오래된 휴대폰을 넘기기 전 계정 해제·초기화·폐기 인계 증거가 모두 있는지 한 장으로 닫는다.
- **구체 타겟:** 연 1회 이상 휴대폰·태블릿을 중고 판매나 재활용으로 넘기며 가족 기기를 함께 관리하는 사람.
- **기존 앱과 차이:** 초기화 명령을 실행하지 않고, 사용자가 직접 캡처한 계정 해제·초기화·인계 접수 상태를 사건 증거로 묶는다. D08-06은 중고폰 거래에 한정된 양측 증거라 구분되지만, 실제 결제 우위는 미검증이다.
- **전체 플로우:** 기기 정보 입력 → 계정 해제·초기화 화면 추가 → 폐기·인계 접수증 추가 → 누락 단계와 증거 PDF 출력.

- **결제 시점·가격 가설:** 폐기 증거 PDF 출력 시 건당 1,900원. 실제 폐기·재활용 이용자 5명 중 2명 결제 전 Hold다.

## 3. Fail 한 줄 이유

- **D08-01:** Apple·Google 이전 도구가 상태와 결과를 직접 소유하고 일회성이다.
- **D08-03:** 사진 백업 상태는 Google Photos·iCloud가 직접 제공한다.
- **D08-04:** 저장소 동기화·복원과 B05 `파일버전찾기` 대안이 강하다.
- **D08-06:** OS 초기화가 핵심 결과를 소유하고 중고 거래 1회성이다.
- **D08-07:** OS 보안 설정·수리점 안내와 배치 005 D17 계정 보호가 중복된다.
- **D08-09:** Find My·Google Find My Device가 위치·잠금 결과를 직접 소유한다.
- **D08-10:** 인증·복구 상태는 각 계정 제공자가 소유하고 D08-23·배치 005 D17과 중복된다.
- **D08-11:** D08-05와 물품 인계 상태가 중복되고 공유 문서로 충분하다.
- **D08-12:** OS·앱스토어·라이선스 제공자가 설치 결과를 소유하며 교체 시점이 드물다.
- **D08-13:** 브라우저 동기화가 북마크 이전·열림 결과를 직접 제공한다.
- **D08-14:** 연락처 동기화가 결과를 소유하고 개인정보 입력 부담이 크다.
- **D08-15:** 메신저가 백업·복원 상태를 소유하며 민감정보 위험이 있다.
- **D08-16:** 배치 005 D18·D19의 공유 링크·확인 상태와 입력·결과가 중복된다.
- **D08-17:** 사진 저장·편집 앱이 원본 상태를 소유하고 D18 사진 공유와 인접하다.
- **D08-18:** 클라우드가 파일 이동·복원 결과를 소유하고 일반 파일 관리로 대체된다.
- **D08-20:** 배치 005 D24 보증서 보관과 인접하며 사진·메모로 충분하다.
- **D08-21:** OS 알림 설정이 결과를 직접 소유하고 배치 005 D22 권한 점검과 중복된다.
- **D08-23:** 배치 005 D17 계정 복구 봉투와 완전 중복이며 비밀정보 위험이 있다.
- **D08-24:** D08-03·D08-17과 사진 백업·원본 상태가 겹친다.
- **D08-25:** D08-10·D08-14와 번호·인증·연락처 확인 결과가 중복되고 통신사·OS가 소유한다.

## 5. 집계 및 중복 검사

- 후보 표 행 수: **정확히 25개** (`D08-01`~`D08-25`). 헤더와 상세 브리프는 후보 행에 포함하지 않았다.
- 최종 판정: **Pass 0 / Hold 5 / Fail 20 = 25**.
- UX 합의 2/3 미만을 Hold로 남긴 후보: **0개**. 1/3 후보는 즉시 Fail했고, 2/3 이상 후보만 국내 대안·결제·제품 소유 상태를 추가 심사했다.
- Hold는 D08-02, D08-05, D08-08, D08-19, D08-22의 5개다. `D08-22`까지 포함해 **Hold 5개**이며, 표의 최종 열과 상세 브리프가 일치한다.
- 기존 앱 32개 그룹·배치 001~005 중복: D08-03·10·16·21·23·24는 배치 005 D17·D18·D19·D22·D24 및 기존 파일·사진·계정 후보와 겹쳐 Fail 처리했다. D08-08은 배치 004 `사진전후대조`와 인접해 Hold로 보수 판정했다.
- 동시 배치 중복: 확인 가능한 006·007 보고서의 금융·자격·가족·쇼핑·행사·계약 상태와 D08의 기기·파일·백업·폐기 상태는 입력·소유 상태·즉시 결과가 다르다. 이름만 바꾼 D08-11, D08-24 등은 표에서 Fail 처리했다.
- MVP 감사: 모든 Hold는 수동 입력 1~2개 → 상태 처리 1회 → 누락·차이·증거 1개로 축소 가능하며, OAuth·대규모 크롤링·상시 백그라운드 수집을 요구하지 않는다.
- Firecrawl 사용: **0회**.
