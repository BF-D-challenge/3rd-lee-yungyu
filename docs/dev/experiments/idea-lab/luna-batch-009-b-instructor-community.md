## 2. Hold 상세 브리프

### B09-01 코호트 첫 7일

- Hook: 결제 후 환영 메일을 읽었는지가 아니라, 새 수강생이 첫 결과를 실제로 만들었는지만 운영자가 본다.
- 구체 타겟: 매달 10~50명 규모의 코호트·온라인 코칭을 혼자 운영하며 가입 후 첫 주 이탈을 수동으로 확인하는 강사.
- 기존 앱과 차이: 결제·콘텐츠를 대체하지 않고, 카톡·메일·폼에 흩어진 `가입→첫 행동→완료` 확인과 다음 연락 상태를 한 사건으로 소유한다. Circle은 온보딩 공간과 이벤트를 제공하므로 독립 결제 우위가 아직 약하다.
- 전체 플로우: 결제자와 환영 링크 수동 입력 → 첫 행동 링크 또는 확인 사진 1개 등록 → 운영자 확인 → 멈춘 멤버와 보낼 연락 1건 출력.
- 가장 작은 MVP: 코호트 1개, 멤버 20명, 상태 3개, 확인 링크 1개, CSV 내보내기. 결제·LMS API·상시 추적·자동 메시지는 제외한다.
- 결제 시점·가격 가설: 첫 코호트에서 미완료 멤버 목록을 내보낼 때 월 7,900원. 실제 강사 5명 중 2명 선결제와 첫 주 완료율 우위 확인 전 Hold다.

### B09-02 코칭 기준선 카드

- Hook: “좋아졌어요” 대신 첫 상담의 숫자 하나와 다음 세션의 같은 숫자를 맞춰 변화 여부를 바로 본다.
- 구체 타겟: 월 4회 이상 1:1 세션을 판매하고 고객별 목표 수치를 직접 기록하는 커리어·언어·퍼포먼스 코치.
- 기존 앱과 차이: 예약·결제를 하지 않고 세션 사이의 기준선, 측정일, 고객 확인을 누적한다. 의료 진단·재무 수익 판정은 하지 않는다. 코칭 CRM과 개인 노트가 강한 대안이다.
- 전체 플로우: 첫 세션의 기준 수치와 목표 입력 → 측정일에 고객 수치 입력 → 기준선 대비 변화와 다음 확인 수치 1개 출력.
- 가장 작은 MVP: 고객 1명, 지표 3개, 세션 4회, 수동 입력, 변화 카드 PDF. 외부 캘린더·민감정보 분석·AI 판정은 제외한다.
- 결제 시점·가격 가설: 다음 세션용 변화 카드를 내보낼 때 고객당 월 4,900원. 코치 5명 중 3명이 기존 노트보다 빠르다고 평가하고 1명 선결제하기 전 Hold다.

### B09-03 주간 실행보고

- Hook: 커뮤니티 글 수가 아니라 이번 주 실제로 해낸 결과와 증거가 비어 있는 멤버만 운영자가 본다.
- 구체 타겟: 매주 실천 보고를 받는 20~100명 규모의 유료 습관·커리어·창작 커뮤니티 운영자.
- 기존 앱과 차이: 게시·채팅을 대체하지 않고, 주차별 한 줄 결과·증거·운영자 확인·미제출 상태를 누적한다. Circle의 게시·다이제스트 기능과 겹칠 수 있다.
- 전체 플로우: 주간 질문 1개 발행 → 멤버가 결과 한 줄과 링크/사진 제출 → 운영자 확인 → 미제출·보류 멤버 목록과 주간 요약 출력.
- 가장 작은 MVP: 커뮤니티 1개, 멤버 30명, 주차 4회, 링크/사진 1개, 상태 3개. 소셜 API·자동 독촉·콘텐츠 생성은 제외한다.
- 결제 시점·가격 가설: 첫 주간 요약을 내보낼 때 월 9,900원. 운영자 5명 중 3명이 4주 재사용하고 2명이 결제하기 전 Hold다.

## 5. 상태 집계 및 중복 검사

- 후보 표 행 수: **정확히 25개**, ID는 `B09-01`부터 `B09-25`까지 연속이다.
- 최종 판정: **Pass 0 / Hold 3 / Fail 22 = 25**.
- UX 합의: **3/3 3개, 2/3 7개, 1/3 15개**. `Y/Y/Y` 페르소나 2명 미만인 1/3 후보는 모두 Fail이며 Hold가 없다.
- Hold는 B09-01, B09-02, B09-03뿐이다. 모두 국내 대안·결제 결과를 추가 검증해야 하므로 Pass로 올리지 않았다.
- 금지 이름 검사: `강의과제피드백큐`, `학습과제증명`, `강의수정요청함`, `강의후기동의`, `강의판매정산표`, `강의쿠폰회수`, `학습질문큐`, `오류재현노트`, `댓글약속장부`, `발표질문로그`는 새 후보 제목으로 사용하지 않았고, 중복 대상 설명에만 기록했다.
- 기존 앱 32개 그룹·배치 001~005: 자료 허브, 질문·피드백, 전후 증거, 약속 확인, 파일·행사 인계 계열을 B09-04~06, B09-11~15, B09-17~19, B09-22, B09-24~25에서 Fail 처리했다.
- 배치 006~008 및 현재 저장소에 보이는 타 트랙: B06의 채용·과제·수료, B07의 반복 연습·그룹, B08의 크리에이터·파일·행사 운영과 입력·상태·즉시 결과를 대조했다. 009 A/C/D 파일은 현재 저장소에 존재하지 않아 동시 결과로 가장할 수 없으며, 확인 가능한 보고서 부재를 시장 부재로 해석하지 않았다.
- MVP 감사: Hold 3개는 모두 수동 입력 1개→상태 처리 1회→즉시 목록/카드 1개로 축소되며 OAuth·대규모 크롤링·양면 마켓플레이스·의료 진단·법률 판정·상시 백그라운드 수집이 없다.
- Firecrawl 사용: **0회**.
