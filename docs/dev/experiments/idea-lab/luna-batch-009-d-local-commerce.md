# Luna 배치 009 D 트랙 — 반품·보증·수리·렌탈·지역 상거래

검사일: 2026-07-12  
범위: 온라인 플랫폼의 주문·환불 결과를 복제하지 않고, 한국의 매장·수선점·렌탈점·지역 판매자와 거래할 때 사용자가 직접 받은 증거와 양측 확인을 닫는 보조 흐름.

## 조사·판정 기준

- 공통 규칙의 입력·하드 게이트·UX 합성 규칙과 001~008의 `luna-batch-*` 보고서를 모두 대조했다. D06의 배송·예약 사건, D07의 행사·공동비용·인계, D08의 기기·파일·계정 상태는 제목이 아니라 `입력 → 소유 상태 → 즉시 결과`로 비교했다.
- 로컬 TrustMRR의 증거·수리·렌탈·자산관리 흐름을 발산 앵커로 삼고, 한국소비자원·국가기술표준원·소상공인시장진흥공단·국세청의 공식 경로를 국내 행동·강한 대안 확인에 사용했다. 해외 매출 신호를 한국 수요의 증명으로 해석하지 않았다.
- P1 신입 / P2 중급 / P3 숙련의 값은 각각 `10초 이해/한국 실제 사용/결제 이유`다. 세 값이 모두 Y인 페르소나가 2명 미만이면 즉시 Fail이다. UX 2/3 이상이어도 국내 사업자·플랫폼이 핵심 결과를 소유하거나 결제·MVP가 약하면 Fail 또는 Hold로 낮췄다.
- Firecrawl은 사용하지 않았다. 은행 OAuth, 대규모 크롤링, 양면 마켓플레이스, 법률·품질 판정은 제품 범위에서 제외했다.

- 구체 타겟: 안경을 연 1회 이상 맞추고 재제작·수령 확인을 직접 처리하는 성인.
- 기존 앱과 차이: 도수 판정이나 렌즈 품질 판정을 하지 않고, 주문서·매장 답변·수령 사진의 버전과 확인 시각만 소유한다.
- 가장 작은 MVP: 주문 1건, 항목 4개, 사진 3장, 매장 확인 링크 1개, `주문/완성/수령` 상태와 공유 PDF. 안경점·결제 연동은 제외한다.
- 결제 시점·가격 가설: 재제작 또는 수령 확인 PDF를 내보낼 때 건당 1,900원. 안경점 이용자 5명 중 2명 선결제가 확인되기 전 Hold다.

### D09-08 자전거 정비 인수카드 — Hold

- Hook: 정비소가 완료했다고 한 항목과 고객이 시험한 항목을 서로 확인해 미완료 한 건을 찾는다.
- 구체 타겟: 자전거를 월 1회 이상 타고 정비소 이용 후 직접 인수하는 생활 자전거 사용자.

- 가장 작은 MVP: 정비 1건, 항목 8개, 사진 4장, 고객 확인 링크, `접수/완료/인수` 상태. 매장 POS·위치 추적은 제외한다.

### D09-22 지역 예약 수령 확인 — Hold

- Hook: 지역 판매자가 상품을 준비했고 누가 언제 받았는지 양쪽 확인이 끝난 예약만 보여준다.
- 구체 타겟: 월 2회 이상 공방·농가·동네 매장에서 예약 상품을 찾아가는 총무·가족 대표.
- 기존 앱과 차이: 주문·결제·재고를 만들지 않고, 판매자가 보낸 준비 증거와 수령자의 확인 시각만 소유한다.
- 가장 작은 MVP: 예약 1건, 사진 2장, 판매자 확인 링크 1개, 수령자 확인 1개, `예약/준비/수령` 상태. 주문 API·마켓플레이스·자동 알림은 제외한다.
- 결제 시점·가격 가설: 수령 확인 PDF 또는 공유 링크를 만들 때 건당 1,900원. 5개 지역 판매자와 10건 픽업 중 2건 유료 전환 전 Hold다.
