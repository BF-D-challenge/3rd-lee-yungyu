## 수량

- 배치 006, 007, 008, 009를 순서대로 실행한다.
- 각 배치는 A/B/C/D 네 트랙이며 각 트랙은 **정확히 25개**다.
- 전체 목표는 **16개 파일·400개 후보**다.
- 한 트랙 안, 같은 배치 안, 배치 간, 기존 앱 32개 그룹과 배치 001~005 전체에서 입력·소유 상태·즉시 결과가 겹치면 Fail이다.

## 발굴·웹 조사

- Firecrawl은 사용하지 않는다.
- 로컬 TrustMRR·App Store 자료로 발산한다.
- Pass/Hold 가능 후보만 일반 웹 검색, Jina Reader, 공식 기관, 제품 공식 페이지, 앱스토어 중 가능한 수단으로 확인한다.
- 보고서마다 최소 3개의 직접 URL을 기록한다. 수집 실패를 시장 부재로 해석하지 않는다.
- 해외 원본은 80~99% 또는 완전 동일해도 되지만 브랜드·로고·고유 카피·에셋은 복제하지 않는다.

## 제품 하드 게이트

1. 한국 타겟이 월 1회 이상 겪거나 예상 손실이 5만원 이상이다.
2. 국내 플랫폼·공공기관·OS·강한 무료 앱이 핵심 결과를 소유하면 Fail이다.
3. ChatGPT·Claude 한 번 질문으로 끝나면 Fail이다. 제품이 시간 누적, 반복 큐, 증거, 양측 확인, 검증 데이터 중 하나를 직접 소유한다.
4. 1인 바이브 코딩 4주 MVP다. 큰 범위는 실제 입력 1개→처리 1회→즉시 결과 1개로 줄인다.
5. 은행 OAuth, 대규모 크롤링, 양면 마켓플레이스, 의료 진단, 법률·세무 판정, 상시 백그라운드 수집이 필요하면 Fail이다.
6. 제목은 타겟의 손실·마찰과 즉시 결과가 10초 안에 보이는 한국어 UVP다. `AI 생성기·추천`, 추상 카테고리명은 금지한다.

## UX 합성 평가

모든 후보는 같은 ASCII 카드 규격으로 본다고 가정한다.

- P1 신입: 제목과 첫 결과의 10초 이해·진입
- P2 중급: 오류·확인·되돌리기·반복 효율
- P3 숙련: 기본 도구 대비 우위·결제 결과·효율 천장

각 페르소나는 `10초 이해/한국 실제 사용/결제 이유`를 Y/N으로 남긴다. 세 항목 모두 Y인 페르소나가 2명 미만이면 즉시 Fail이다. UX 2/3 이상도 국내 대안·결제·MVP 중 하나가 실패하면 Fail이다. 합성 UX는 실제 사용자 검증이 아니라 선별 가설이다.

## 보고서 형식

각 출력 파일은 다음을 포함한다.

1. 정확히 25행 후보 표
   - ID, 제목/UVP, 입력→소유 상태→결과, 해외·한국 근거
   - P1/P2/P3, UX 합의, 하드 게이트, 최종 Pass/Hold/Fail
2. Pass/Hold만 상세 브리프
   - Hook, 구체 타겟, 기존 앱 차이, 전체 플로우, 가장 작은 MVP, 결제 시점·가격
3. Fail 한 줄 이유
4. 출처 URL, Pass/Hold/Fail 수, 기존·동시 배치 중복 검사

## 파일·판정 규칙

- 담당 파일만 생성한다. 앱 데이터와 기존 보고서를 수정하지 않는다.
- UX 2/3 미만을 Hold로 남기지 않는다.
- 25개 수를 맞추려고 통과 수를 맞추지 않는다.
- Pass가 0개여도 정상이다.
