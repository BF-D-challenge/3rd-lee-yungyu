# TASK 1 생존 카드 — 합성 UX 입력

검사 방법: 각 카드를 10초만 읽고 아래 세 질문에 답한다.

1. 누구의 어떤 문제인가?
2. 첫 입력은 무엇인가?
3. 끝나면 어떤 결과를 받는가?

코드나 구현은 보지 않는다. 아래 문구만 평가한다.

---

## 1. 국내 채널 3종 복사본

```text
누구: 스마트스토어와 SNS를 함께 운영하는 소형 브랜드 담당자
언제: 신상품 원고가 완성된 직후
입력: 600자 이하 원고 + 사진 1장
결과: 네이버 블로그 도입부 / 인스타그램 캡션 / 카카오채널 공지
행동: 각 문구를 복사하고 완료한 채널을 체크한다
```

## 2. 문서 3개의 합의·충돌 근거표

```text
누구: 매주 고객 인터뷰를 정리하는 제품 리서처
언제: 서로 반대되는 고객 발언을 발견한 오후
입력: PDF·TXT 최대 3개 + 확인할 질문 1개
결과: 합의 / 충돌 / 확인 불가 주장과 각 파일·페이지·원문
행동: 근거 위치를 열어 결정에 쓸 문장을 확인한다
```

## 3. 오답 한 장에서 찾는 약한 개념 3개

```text
누구: 퇴근 후 자격시험을 준비하는 직장인
언제: 모의문제를 틀린 직후
입력: 수업 PDF 1개 + 풀이가 보이는 오답 사진 1장
결과: 놓친 개념 최대 3개 / 학생 풀이 근거 / 수업 자료 페이지
행동: 같은 개념의 2분 구술 질문으로 다시 확인한다
```

## 4. 10초 달리기 관절선 영상

```text
누구: 회원의 자세를 영상으로 설명하는 러닝클럽 코치
언제: 옆에서 찍은 달리기 영상을 받은 직후
입력: 달리기 10초 영상 1개
결과: 어깨·골반·무릎·발목 선 + 가장 크게 변한 프레임 1장
행동: 다음 세트에서 다시 볼 자세 한 가지를 전달한다
주의: 부상·통증·완치 판단은 하지 않는다
```

## 5. 전단 한 장으로 만든 모바일 안내 페이지

```text
누구: 이번 주말 팝업을 여는 1인 판매자
언제: 행사 전단 이미지가 완성된 직후
입력: 전단·메뉴 사진 1장
결과: 상호 / 시간 / 가격 / 주소가 있는 공개 모바일 링크
행동: 지도 열기 / 전화하기 / 신청하기 버튼을 확인해 공유한다
```

## 6. 업무 메시지의 사실·질문·다음 행동 3칸

```text
누구: 여러 고객과 일정을 조율하는 프리랜서
언제: 수정 요청과 마감 변경이 한 메시지에 온 순간
입력: 업무 메시지 캡처 1장
결과: 확정된 사실 / 확인할 질문 / 내 다음 행동 + 원문 근거
행동: 이름·연락처·프로필을 가린 뒤 답할 항목 누락을 확인한다
```

## 7. 공급사 엑셀을 스마트스토어 등록표로

```text
누구: 네이버 스마트스토어에 신상품을 매주 올리는 소형 브랜드 운영자
언제: 공급사 신상품 엑셀이 도착한 금요일 오후
입력: 공급사 XLSX·CSV 1개
결과: 스마트스토어 필수 열 매핑 / 오류 행 / 업로드 초안 XLSX
행동: 상품 20개씩 제목·가격·옵션·이미지를 검수한다
```

## 8. 댓글 파일의 질문·스팸·정상 비판 검수표

```text
누구: 여러 소셜 채널을 관리하는 D2C 브랜드 커뮤니티 매니저
언제: 영상 하나에 댓글 500개가 붙은 밤
입력: 댓글 CSV 또는 한 줄 한 댓글 TXT 1개
결과: 답할 질문 / 반복 스팸 / 유해 표현 / 정상 비판 + 분류 이유
행동: 위험 댓글 20개를 원래 플랫폼에서 직접 확인한다
```

## 9. 유튜브 한 편의 이중 자막 학습표

```text
누구: 교재 대신 유튜브로 영어를 공부하는 직장인 초급 학습자
언제: 자막이 빨라 같은 장면을 세 번 돌려본 순간
입력: 자막이 공개된 유튜브 URL 1개
결과: 원문 / 한국어 뜻 / 재생 시점 + 오늘 쓸 표현 3개
행동: 표현 하나의 30초 구간을 듣고 따라 말한다
```

## 10. 카드 거래 파일에서 찾은 반복 결제

```text
누구: 영상·업무·AI 구독을 여러 개 결제하는 1인 직장인
언제: 이번 달 카드값이 예상보다 커진 직후
입력: 최근 3개월 카드 CSV·XLSX 1개
결과: 반복 결제 / 오른 금액 / 예상 다음 날짜와 다음 달 합계
행동: 사용자가 각 결제를 직접 확인한다. 계정 연결·자동 해지는 없다
```

## 11. 7일 안에 되찾는 비활성 탭 보관함

```text
누구: 자료 조사 때 탭을 수십 개 여는 기획자·리서처
언제: 열린 탭이 50개를 넘어 제목도 보이지 않는 순간
입력: 브라우저 탭 권한 1개
결과: 7일 이상 안 본 탭의 제목·도메인·마지막 사용 시각
행동: 제외할 탭을 고른 뒤 로컬 보관하고 7일 안에 복원한다
```

## 12. 손그림 한 장을 편집 가능한 흐름도로

```text
누구: 회의실 화이트보드 흐름도를 문서로 옮기는 제품 기획자
언제: 회의가 끝나 화이트보드를 지우기 직전
입력: 정면에서 찍은 손그림 사진 1장
결과: 상자·화살표·짧은 글자를 편집할 수 있는 SVG
행동: 원본과 나란히 보고 애매한 글자·연결만 고쳐 내려받는다
```
