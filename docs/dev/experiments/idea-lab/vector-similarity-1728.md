# 아이디어 1,728조합 해외 원본 충실도 점검

검사일: 2026-07-12

## 결론

새 정책에 맞춰 비교 대상을 바꿨다. 생성 아이디어끼리 닮았는지는 탈락 기준이 아니다. 각 조합이 **자기 해외 원본의 제품 메커니즘을 얼마나 유지하는지**만 임베딩으로 검사한다.

- OpenAI 주 모델: 1,728/1,728 통과
- 유사도 범위: 0.855~0.955
- 다국어 로컬 모델: 기존 1,620/1,620 통과. 새 설치형 그룹 54개는 OpenAI 주 모델로 우선 검증
- 기존 로컬 유사도 범위: 0.827~0.994
- 해외 원본 또는 다른 조합과 사실상 동일한 아이디어: 허용
- 1인 바이브 코딩 MVP 하드 게이트 실패: 아이디어를 핵심 세로 조각으로 쪼갠 뒤 재검사

## 정책

1. 해외에서 검증된 제품 메커니즘을 그대로 가져와도 된다.
2. 목표 코사인 유사도는 0.80~0.99다.
3. 0.99 초과는 자동 탈락시키지 않는다. 사실상 같은 메커니즘도 허용한다.
4. 브랜드명·로고·고유 카피·디자인 에셋 복제는 제품 아이디어 복제와 별개이며 허용하지 않는다.
5. 다른 생성 아이디어끼리 닮은 것은 포트폴리오 중복 정보일 뿐 품질 실패가 아니다.
6. 임베딩 통과와 MVP 제작 가능성은 별도 하드 게이트다.

## 측정 대상

- 해외 원본: 32개
- 원본별 타겟 3 × 순간 3 × 끗 조합 6
  - 한 끗 3개
  - 두 끗 조합 3개
- 전체 조합: 1,728개

원본 충실도 임베딩에는 마케팅 문구가 아니라 다음 제품 메커니즘만 넣는다.

- 원본 제품 핵심
- 원본의 상세 작동 방식
- 보존할 3단계 흐름
- MVP에서 적용할 한 끗 제목

타겟과 순간은 UVP를 구체화하지만 원본 제품 메커니즘의 변화량에는 포함하지 않는다.

## 결과

| 모델 | 최소 | 중앙값 | 상위 90% | 최대 | 0.80 이상 |
|---|---:|---:|---:|---:|---:|
| OpenAI `text-embedding-3-small` | 0.855 | 0.934 | 0.948 | 0.955 | 1,728/1,728 |
| 다국어 MiniLM 교차검증 | 0.827 | 0.960 | 0.982 | 0.994 | 기존 1,620/1,620 |

로컬 모델에서 27개가 0.99를 넘었다. 해외 원본과 사실상 동일한 아이디어를 허용하므로 이 조합들도 통과다.

가장 낮은 조합도 OpenAI 0.855, 로컬 0.827이었다.

- 원본: `1Lookup`
- 조합: `연락 전 30초 상대 브리핑 + 현재 탭 상대 확인 버튼`
- 판단: 원본의 정보 입력·여러 출처 확인·근거 정리 흐름이 유지되므로 통과

## 1인 바이브 코딩 MVP 하드 게이트

임베딩을 통과해도 다음 항목 중 하나라도 실패하면 그대로 공개하지 않는다.

1. 1명이 반나절~2일 안에 배포할 수 있는가
2. `입력 1개 → 처리 1회 → 결과 1개`로 설명되는가
3. 화면이 1~3개인가
4. 로그인·결제·관리자·팀 권한 없이 핵심 가치가 작동하는가
5. 자체 모델 학습·대규모 크롤링·여러 OAuth·상시 백그라운드 작업이 없는가
6. 브라우저 기본 기능·관리형 API 1~2개 또는 설치형 테스트 앱의 단일 기기 권한으로 실제 데이터를 처리할 수 있는가

### 범위가 크면 쪼개는 순서

1. 원본에서 사용자가 처음 가치를 느끼는 결과 1개만 남긴다.
2. 자동 수집·외부 연동을 URL·텍스트·파일 직접 입력으로 바꾼다. 단, 설치형 테스트 앱의 단일 기기 권한으로 실제 입력을 받을 수 있으면 허용한다.
3. 상시 모니터링을 사용자가 누르는 `지금 확인` 1회 실행으로 바꾼다.
4. 네이티브 권한이 핵심 가치가 아니면 모바일 웹/PWA로 바꾼다. 사진·카메라처럼 실제 입력 자체가 핵심이면 설치형 테스트 앱으로 유지한다.
5. 영상·음성은 30초 또는 파일 1개만 처리한다.
6. 그래도 반나절~2일 안에 실제 작동이 불가능하면 핵심 결과를 한 단계 더 줄인다.

예시:

| 큰 범위 | 핵심 MVP 조각 |
|---|---|
| 여러 SNS에 자동 발행 | 글 붙여넣기 → 채널별 미리보기 3개 |
| 지역 일감 상시 감시·알림 | 요청 글 붙여넣기 → 내 조건 적합 판정 1개 |
| 링크드인 자동 영업 | 프로필 URL/텍스트 붙여넣기 → 첫 연락 문장 1개 |
| 모든 회사 채용 페이지 크롤링 | 회사 채용 URL 1개 → 새 공고 여부 확인 |
| 실시간 영상 자세 분석 | 30초 영상 1개 → 다음 동작 교정 1개 |

## 재현

```bash
node scripts/research/export-idea-lab-combinations.mjs > /tmp/idea-lab-combinations-1728.jsonl

python scripts/research/check-idea-vector-similarity.py \
  --input /tmp/idea-lab-combinations-1728.jsonl \
  --mode source-fidelity \
  --provider openai \
  --batch-size 100 \
  --top 30
```

생성 아이디어끼리의 참고용 중복 분포가 필요할 때만 `--mode portfolio`를 사용한다. 그 결과는 탈락 판정에 쓰지 않는다.
