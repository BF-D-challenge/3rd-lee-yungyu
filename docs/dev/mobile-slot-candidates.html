열릴 때 저장된 선택값을 먼저 읽고 상단 고정 selected row에 복원한다. 선택값 복원이 끝나기 전 빈 상태를 잠깐 보여주지 않는다.

검색 입력은 120ms 디바운스한다. 결과 변경 시 목록 컨테이너 높이는 유지하고 기존 결과 opacity 1→0, 새 결과 0→1을 각각 90ms로 교차시킨다.
검색 중에도 selected row와 적용 CTA는 고정한다. 모바일 키보드가 열리면 결과 영역만 줄이고 CTA는 safe-area 위에 sticky로 유지한다.
추가·해제 모션은 첫 선택 화면과 같은 260ms spring을 재사용한다. 제거된 칩은 scale 1→.92와 opacity 1→0 후 목록으로 되돌아간다.
최대 8개 규칙, 카운터 고정 폭, 오류 live region을 첫 화면과 공유한다.

닫기 또는 바깥 탭은 이번 세션의 미적용 변경을 버리고 이전 저장값으로 복구한다.
prefers-reduced-motion에서는 검색 결과와 시트에 100ms opacity만 적용한다.

구현 프롬프트 · 카드 A·B 직접 열기열기 
완성 아이디어 카드 두 장을 직접 여는 모바일 게임판을 구현한다. 4축 조합 슬롯은 만들지 않는다.

상태 모델: idleA → flyingA → revealingA → idleB → flyingB → revealingB → complete. 각 상태에서 허용되는 입력을 명시적으로 제한한다.
카드 A와 B 표면 자체를 유일한 기본 뽑기 버튼으로 만든다. 초기에는 A만 활성화하고 1.9s ease-in-out 조준 링을 반복한다. A가 안착한 뒤 280ms 후 B를 활성화한다.
부채꼴 덱은 모바일 카드 84×122px, 반경 640px, 슬리버 간격 12px를 기준으로 한다. 최초 한 번만 -12deg→0deg 진입 스윕을 3.2s cubic ease로 실행하고 이후 1.1deg/s로 rAF 드리프트한다.
활성 카드를 탭하면 즉시 전체 드로 입력을 잠근다. 중앙 덱은 비행 시작점을 보여주는 장식이며 클릭·포커스 대상이 아니다. 덱 아펙스 원본을 ghost 처리하고 body clone을 만든다.
클론은 시작점과 목표 중앙 사이에 quadratic bezier를 사용한다. 제어점 y는 min(startY,targetY)-120px, duration 420ms, cubic ease-in-out. 이동 중 회전은 현재 각도→0, scale은 1→목표 폭 비율로 보간한다.
도착 즉시 clone을 제거하고 목표 카드에 내용 데이터를 주입한다. 안착 팝은 scale .9→1.02→1, 300ms cubic-bezier(.34,1.3,.5,1).
도착 즉시 앞면 플립을 시작한다. rotateY 0→180deg, 520ms cubic-bezier(.4,0,.15,1). 동시에 카드가 y 0→-10→-3px로 500ms 리프트되고 오라는 opacity 0→.8→.3, 650ms ease-out으로 남는다.
드래그는 MVP에서 제외한다. 후속 강화 시에만 활성 카드에서 시작하는 보조 입력으로 추가하며 탭과 같은 상태 머신을 재사용한다.
두 번째 카드가 도착한 뒤 650ms 동안 결과를 읽는 여백을 주고 'BF.D에게 골라달라고 하기'와 보조 액션 '다른 두 장'을 fade-up 320ms로 노출한다.
완료 전에는 지속 CTA를 두지 않는다. 카드 표면만 기본 액션이며, 각 카드의 터치 타깃은 최소 48×48px다. 모바일 카드 내부에는 별도 스크롤을 만들지 않는다.
prefers-reduced-motion에서는 덱 드리프트·아크·리프트·블룸을 제거하고 데이터 주입 후 100ms cross-fade로 앞면을 보여준다. 입력 잠금과 상태 순서는 동일하게 유지한다.

구현 프롬프트 · BF.D 한 장 선택열기 
BF.D 공유 링크의 로그인 및 A/B 긍정 선택 화면을 구현한다.

상태 모델: loading → authRequired → authenticating → ready → choosing → chosen. OAuth 복귀 URL에는 원래 slug를 보존한다.
진입 시 발신자 맥락은 400ms fade-up, 카드 A/B는 50ms 뒤, 로그인 시트는 100ms 뒤 나타난다. 카드 두 장의 높이와 폭은 처음부터 확정해 시트 제거 시 레이아웃이 이동하지 않게 한다.
authRequired에서는 카드 내용을 읽을 수 있지만 opacity .68, inert, aria-disabled true로 둔다. 로그인 시트는 role=dialog, aria-modal=true, aria-labelledby를 갖고 포커스를 Google CTA 안에 가둔다. 선택 영역을 가리지 않게 하단에 배치한다.
Google 로그인 탭은 로그인 화면과 같은 press, 지연 로더, 중복 방지 규칙을 재사용한다.
인증 복귀 시 시트를 opacity 1→0과 translateY 0→12px, 220ms로 닫고 선택 영역의 inert와 aria-disabled를 해제한다. 카드 A와 B를 50ms 차이로 opacity .68→1 전환하고 첫 카드에 포커스를 보낸다.
카드 두 장은 role=radiogroup 안의 role=radio다. 선택은 aria-checked로 노출하고 좌우 방향키로 포커스와 선택을 함께 이동한다.
카드 탭 즉시 choosing으로 전환하고 두 카드를 모두 비활성화한다. 서버 응답을 기다리지 않고 낙관적으로 선택 상태를 표시한다.
선택 카드는 translateY -2px, scale 1.02, 강조 링과 체크를 200ms로 적용한다. 미선택 카드는 scale .98과 opacity .72를 같은 200ms에 적용해 제목과 설명을 계속 읽을 수 있게 한다. 레이아웃 크기는 변하지 않는다.
선택 뱃지의 하트는 scale 1→1.18→1, 300ms ease로 팝한다. 투표 저장은 fire-and-forget하되 실패 시 로컬 큐에 남기고 화면 흐름은 막지 않는다.
로컬 큐에 남긴 경우 성공 체크 대신 role=status에 '응원을 기기에 저장했어요. 연결되면 자동으로 보낼게요.'를 표시한다. 재시도 액션이 있는 스낵바는 자동으로 닫지 않는다.
520ms 뒤 카드 영역은 유지한 채 하단 안내만 칭찬 선택 상태로 cross-fade한다. 더블탭, 멀티탭, 재방문 중복 응원을 방지한다.
부정 라벨과 설치 유도 문구는 넣지 않는다. 두 선택 모두 응원이라는 의미를 유지한다.
prefers-reduced-motion에서는 scale과 이동을 제거하고 선택 링 및 opacity를 100ms로 전환한다.

상태 모델: praiseEntering → praiseReady → praiseSelected → submitting → handoff. 카드 선택 결과는 화면 상단에 계속 유지한다.
칭찬 영역은 카드 선택 후 520ms 시점에 opacity 0→1, translateY 10px→0, 320ms ease-out으로 나타난다. 프리셋 세 개는 40ms 간격으로 전개한다.
프리셋은 '나도 필요해', '완성하면 알려줘', '너라면 만들어'처럼 전부 긍정형으로 유지한다. 한 개만 선택 가능하다.
칩 탭 시 선택 칩 배경을 transform scaleX 0→1, transform-origin left, 360ms cubic-bezier(.65,0,.35,1)로 채운다. 텍스트와 이모지는 scale 1→1.18→1, 300ms로 팝한다. 다른 칩은 opacity .35로 낮춘다.
칭찬을 선택하기 전에는 CTA를 disabled 상태로 둔다. 선택 순간 CTA를 240ms 색 전환으로 활성화하고 문구는 '응원하고 내 카드 만들기'로 고정한다.
CTA 탭 즉시 모든 입력을 잠그고 scale .98 press를 90ms 적용한다. 응원 데이터와 선택 카드 seed를 병렬 저장한다. 서버 저장을 확인했을 때만 성공 체크를 220ms 보여준다.
다음 URL에는 seed와 via=vote를 포함한다. 게임판 복귀 시 선택한 취향 배너를 2.6s 노출하고 카드 A 조준 상태를 즉시 준비한다.
복귀 게임판은 새 페이지 전체를 밀어 넣지 않는다. opacity cross-fade 180ms 후 배너가 y -8→0으로 240ms 내려오고 덱의 기존 3.2s 첫 진입 스윕은 최초 방문일 때만 실행한다.
네트워크 실패가 나도 사용자의 칭찬과 handoff를 막지 않는다. 로컬 큐에 보존하고 다음 세션에 재전송하되 성공으로 가장하지 않는다. role=status에 '응원을 기기에 저장했어요. 연결되면 자동으로 보낼게요.'를 표시한다. 재시도 액션이 있는 스낵바는 사용자가 처리할 때까지 유지한다.
