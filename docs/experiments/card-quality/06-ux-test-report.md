# 통합 카드 UX 테스트 리포트

> 시안: 오늘 해볼까 통합 결과 카드(단일) · 방법: ux-tester 3페르소나(신입·중급·숙련) 휴리봇 워크스루 + 맥락 없는 독립 리뷰어 2명 → opus 합의 집계.
> 과업: T1 3초 파악 · T2 필요성·반나절 실현성 판단 · T3 다음 행동 찾기.

## 1. 여정

**🟢 김서연 (신입) — "그림은 예쁜데, 내 문제인지 3초엔 안 잡혀요"**
T1 부분성공(load 3) → T2 성공 → T3 성공. 감정은 우상향(1→2→2). 가장 아픈 막힘은 T1: 제목이 솔루션 이름('타이머 앱')이라 "누구의, 왜"를 알려면 Zone 0~1을 2단계로 읽어야 해 3초를 넘김. 아트가 먼저 눈에 들어오는데 핵심 정보는 텍스트에만 있어 시선이 헛돎. 체험: "누군가의 생활 문제로 반나절 프로젝트를 주는 거구나 — 근데 첫 3초엔 그림만 봤다."

**🟡 박지민 (중급) — "위계는 선명한데, 믿음이 안 서서 손이 멈춰요"**
T1 성공(load 1, 위계 극찬) → T2 부분성공(load 3) → T3 부분성공(load 4). 감정은 우하향(2→0→-1), 유일하게 음수까지 떨어진 층. 가장 아픈 막힘은 T3: 버튼 누른 뒤 복사 성공 피드백이 없고, 발행 칩의 로그인 타이밍·역할(투표/반응?)이 불명시라 행동을 미룸. 체험: "CTA는 명확한데 복사·발행 후 다음 스텝이 흐려서 손가락이 버튼 위에서 멈췄다."

**🔵 이도현 (숙련) — "3초 파악은 되는데, '내가 왜' 동기가 약해 넘기고 싶어져요"**
T1 성공(00:06, 초고속) → T2 부분성공(load 3) → T3 부분성공. 감정 평탄(1→0→0). 가장 아픈 막힘은 T2: Captime 월$673는 시장성 증명일 뿐 "내가 해야 할 이유"가 아니고, 유튜브 홈트러가 내 네트워크와 멀 수 있어 동기가 약함. T3에선 프롬프트 미리보기가 선택인지 필수인지 몰라 심리적 마찰. 체험: "제목만으로 뭔지는 알겠는데, '왜 나'가 약해 버튼까지 마찰이 있다."

## 2. 정량·공통

**과업별 완전성공**
- T1(3초 파악): **2/3** (신입만 부분 — 제목이 문제 아닌 솔루션 이름)
- T2(필요성·반나절 실현성): **1/3** (신입만 성공, 중급·숙련은 MVP 스코프·동기에서 막힘)
- T3(다음 행동): **1/3** (신입만 성공, 중급·숙련은 버튼 피드백·발행 흐름에서 망설임)

**공통 막힌 지점 (합의)**
- **MVP 칩 우선순위/오늘 스코프 불명** — 3/3 (+독립 직관 HIGH). 신입 "4개 다 만드나?", 중급 "필수/선택 구분 없음", 숙련 "오늘 vs Week2 구분 없음"
- **'붙여넣어 만들기' 버튼 불명확 + 클릭 피드백 없음** — 3/3 (+독립 직관 HIGH). 어디로 복사/이동하는지, 눌러도 반응이 있는지 모름
- **T3 발행/프롬프트 미리보기 흐름 불명** — 3/3. 미리보기 선택/필수, 발행 로그인 타이밍이 안 보임
- **oneliner 문제(분홍)/이점(초록) 위계 혼재** — 2/3 (신입·숙련, +독립 2명 HIGH). 중급만 "선명하다"

**평균 감정**: valence **+0.78** / 3 (신입 +1.67, 중급·숙련 각 +0.33) · **load 2.33** / 5. 즉 첫인상은 우호적이나 실행·발행 단계에서 확신이 깎이는 곡선. 특히 중급이 T3에서 -1까지 하락.

## 3. 종합 — 위계는 질서정연한가?

**판정: 세로 골격(훅→근거→실행→발행)은 인지되나, '무게 배분'이 역전돼 있어 절반만 질서정연하다.** 두 곳에서 무너진다.

1. **훅 안에서 1차 정보가 분산** — oneliner에 문제/이점이 한 문장·두 색으로 섞여 3초 패턴 인식을 방해 (숙련은 "두 줄이라 5~6초", 신입은 "회색 문제 문구가 2차로 읽힘").
2. **실행 섹션이 위계에서 밀림** — 정작 핵심 질문("뭘 만드나")의 답인 '오늘 만들 것'이 작은 글씨로 축소되고, MVP 칩은 우선순위 없이 동일 비중이라 "만들 만한가" 판단이 안 섬. 근거·실행이 뒤로 물리며 T2·T3 성공률이 1/3로 주저앉음.

**독립리뷰 교차**
- ⚠️ **양쪽 발견(최우선)**: ① 실행 섹션 위계 축소·MVP 스코프 불명(정보구조 HIGH ↔ 3/3 시나리오) ② CTA 버튼 추상성(직관 HIGH ↔ 3/3) ③ oneliner 문제/이점 색 혼재(시각위계 HIGH ↔ 신입·숙련) ④ 근거 2줄 텍스트의 신호 약화(시각위계 MED ↔ 신입 지표 모호)
- 💡 **독립만**: 소유권 모호("내 아이디어인가 예시인가", Zone0 라벨이 '오늘의 카드'인데 입력한 적 없음 — 신원성 HIGH) · 근거 구간이 '보기→이해→실행' 흐름에 갑자기 끼어드는 단절(Flow) · "지인에게 3초" 톤이 CTA인지 메타 카피인지 모호
- 📋 **시나리오만**: 발행 칩 로그인 타이밍/역할(투표vs반응) 불명 · 프롬프트 미리보기 선택/필수 · 신입의 커뮤니티 용어('땅끄부부 20일차') 인지부담

## 4. 결론 + 액션

첫인상 호감(+0.78)은 확보됐다. 붕괴점은 **"만들 만한가(T2)"와 "바로 뭘 하나(T3)"** — 여기서 3층 모두 1/3로 주저앉는다. 위계 골격을 다시 세우는 게 아니라, **강조(무게)를 실행·근거 쪽으로 재배분**하는 게 처방이다.

- 🔴 **P0 (3/3 합의)**
  1. **MVP 칩에 우선순위·오늘 스코프 부여** — [필수 3: 타이머·연속일·저장] 금색 강조 / [선택 2: 캘린더·공유] 옅게. '주간 캘린더'·'공유 링크'는 로컬저장 전제와 모순이라 스코프 라벨로 해소.
  2. **'붙여넣어 만들기' 버튼 = 목적지 명시 + 클릭 피드백** — "→ ChatGPT용 프롬프트 복사"로 문구 구체화, 클릭 즉시 "✓ 클립보드에 담겼어요" 상태 전환.
  3. **'오늘 만들 것' 섹션 위계 승격** — 축소된 3~4줄을 카드 중앙 1차 정보로 끌어올림.

- 🟠 **P1 (2/3 · HIGH)**
  4. **oneliner 한 줄 완성형으로 통일** — 문제/이점 색 혼재 제거, 문제→해결은 근거(Zone2)에서 명시하거나 '→' 구조 문구로.
  5. **발행 플로우 명확화** — 칩 위 인포칩 "로그인 후 친구에게 투표 요청 (선택)", 프롬프트 미리보기는 선택임을 표시.

- 🟡 **P2 (1/3·LOW 또는 독립만)**
  6. 근거 지표 단위 명시("$673/mo revenue · 니즈 검증됨"), 헤더 아트에 카테고리 실루엣 오버레이, "내 카드 vs 예시" 소유권 신호(독립리뷰 발견).

**실유저 확인 질문**
1. 카드를 처음 본 3초 안에, 이게 **"당신을 위해 추천된 것"**으로 느껴지나요, 아니면 예시 템플릿 같나요? (소유권 — 독립만 발견이라 실사용자 검증 필요)
2. MVP 칩 4개를 봤을 때 **"오늘 반나절에 뭘 만들면 되는지"** 바로 답이 나오나요? 몇 개를 만든다고 생각했나요?
3. '붙여넣어 만들기'를 누르기 직전, **무슨 일이 일어날지** 예상되나요? 눌러본 뒤 예상과 같았나요?

## 부록 · 독립 리뷰 (맥락 없음)

**[정보구조 · 시각위계]** 첫인상: 제목은 명확하지만, 사용자의 핵심 질문 "뭘 만들라는 건지, 만들 만한지"의 답이 페이지 중단부에 약하게 배치돼 있음. 맨 아래 버튼까지 가기 전에 실행 내용을 제대로 파악하기 어려움.

- (HIGH·정보구조) 핵심 정보의 위계 비약 — '오늘 만들 것' 섹션이 3~4줄 작은 글씨로 축소됨
- (HIGH·시각위계) 상황과 해결의 혼제 — oneliner 내 분홍(문제)과 초록(기회)이 한 문장에 섞여 시선 분산
- (MED·시각위계) 근거 섹션의 시각적 신호 약화 — 회색 2줄 텍스트(누구/증명)가 2차 정보인지 참고사항인지 불명확
- (MED·네비게이션) 섹션 경계의 약한 구분 — 근거 영역과 '오늘 만들 것' 사이 구분선 하나로는 정보 전환 신호 부족
- (MED·정보구조) 라벨 위계 모호 — 'Zone0' 라벨(작은 대문자)이 제목 위에만 있고, '오늘 만들 것' 라벨(보라 소문자)이 같은 시각적 강도로 대우되지 않음
- (MED·네비게이션) 인지 흐름 단절 — 제목(1차)→oneliner(1.5차)→근거(약함)→실행내용(작음)→버튼으로, 실행 가능성의 핵심이 '다음 뭘 할지'보다 뒤로 물림

**[직관]** 첫인상: 운동 기록 앱 아이디어 카드처럼 보이는데, 이게 나한테 추천된 건지 예시 템플릿인지 불명. "붙여넣어 만들기"라는 버튼이 있어서 뭔가 만드는 건 알겠는데, 정확히 뭘 어떻게 만드는 건지 3초에는 안 보임.

- (HIGH·신원성(Identity) — 사용자가 소유권을 느껴야 함) 이게 '내 아이디어인가' vs '추천/예시 아이디어인가' 구분 불명 — Zone0 라벨이 "오늘의 카드"인데, 내가 입력한 적 없는데 왜 나타나는가?
- (HIGH·역량(Action) — 사용자가 명확한 다음 스텝을 모름) "오늘 만들 것" MVP 칩 4개 중 어느 것이 '오늘의 구현 범위'인가? 모두 만드나, 1개만 택하나, 아니면 제시된 순서 그대로 만드나?
- (HIGH·명확성(Clarity) — 주요 CTA가 추상적) "붙여넣어 만들기" 버튼이 정확히 뭘 하는가 불명 — 코드 복사? 프롬프트 생성? 튜토리얼 링크? 버튼 문구만으로는 추론 불가
- (MED·맥락(Context) — 인물 관계가 모호함) "유튜브 홈트 3주차, 지쳐가는 맨몸운동러" — 누가 이 사람인가? 서비스가 추론한 '내 페르소나'인가, 아니면 참고용 벤치마크 유저인가?
- (MED·시각계층(Visual Hierarchy) — 색 의도가 암묵적) 분홍(통증) vs 초록(이점) 색 구분이 설계 의도인지 명확하지 않음 — 텍스트만 봐서는 색이 왜 필요한지 이해 못함
- (MED·음성(Tone) — 캐주얼 표현이 의도를 흐림) "마음에 들면 카드 발행하고 지인에게 3초" — '3초'가 뭔가? 공유가 빠르다는 메타 메시지인가, 아니면 '지금 당장 누르라'는 CTA인가?
- (LOW·흐름(Flow) — 정보 배치가 목표-중심이 아님) 전체 수직 구조가 '보기→이해→동의→실행' 흐름인데, 중간에 "2차 근거" 구간이 갑자기 끼어드니 '왜 이 정보가 필요한가' 불명
