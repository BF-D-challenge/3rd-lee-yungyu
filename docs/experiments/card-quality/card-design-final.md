# 결과 카드 설계 — 확정본 (2026-07-08)

> 참조 구현: [`card-design-final.html`](./card-design-final.html) (홈트 C3 예시로 렌더). 카피 규칙은 [`../../../prds/오늘-해볼까-카드품질-실험계획.md`](../../../prds/오늘-해볼까-카드품질-실험계획.md) + [`DESIGN.md`](./DESIGN.md), 카피 v6 규칙은 `scripts/experiments/card-quality/`의 생성 워크플로우.

사용자가 카드 카피(v6)와 단일 통합 카드 레이아웃을 모두 확정.

## 1. 카피 규칙 (v6, 롤아웃 프롬프트)

생성 모델 = **Sonnet**. 근거 = `research/trustmrr-acquire/ideas.jsonl` 실매출 앵커 90/10. 결정론 검증기로 이진 결함 0.

- **생생한 니치를 oneliner 맨 앞에** (실명·숫자·구체 시나리오). 평범/일반화 금지, "의외의 긴장"(왜 하필 이 사람이·이 순간에).
- **한국 브랜드 치환**: 낯선 외래어는 풀이 말고 한국 대중 브랜드로(스트라바→나이키런클럽, 지도→네이버·카카오맵, 배달→배민, 레시피→만개의레시피). 해외 앵커명은 evidence 줄에만.
- **도구 가시화**: oneliner에 뭘 만드는 도구인지(타이머·트래커·투표 등)가 보여야.
- **페르소나 정합**: 상황이 정체성 규정(포트폴리오=취준생/신입).
- **뉴로마케팅 통증→이점 대조**: 통증(감정어)→이점(감정어)을 카드에 구조적으로. 실매출=증명 트리거.
- 필드: title(≤14, 명료·은유금지) · oneliner(≤60) · target · mvp[4] · evidence(실명+수치) · todayAction("오늘 반나절"~) · buildPrompt(복붙 목업 프롬프트).

## 2. 단일 통합 카드 레이아웃 (확정)

**정보 위계 = 사용자 여정 4묶음** (라벨 + 좌측 색 바):
1. 🔴 **누구의·어떤 순간** (상황·통증) — 타깃 + 통증 순간 (자기중심)
2. 🟢 **그래서 이 카드는** (해결·이점) — 이점 (대조)
3. 🟡 **믿을 근거** (증명) — 실매출 수요검증
4. 🟣 **오늘 만들 것** (실행) — MVP 필수/다음 + 오늘 한 줄

상단 = 🌱 씨앗 출처 + 카드번호(소유권 신호) + 아트. 제목은 여정 위.

**액션 위계 = 한 면 Primary 1개 + 고스트 세컨더리 1개 (앞·뒤 하단 같은 위치·크기 고정):**
- 메인 = **🛠 오늘 만들기 시작** (앞·뒤 공통, 꽉 찬 보라). 클릭=프롬프트 복사.
- 세컨더리 = 프라이머리와 **같은 크기 고스트 버튼**. 앞면 "더 자세히 보기↻"(뒤집기) / 뒷면 "📮 발행해서 먼저 물어보기".
- 하단 액션존을 flex spacer로 **바닥 핀 고정** → 뒤집어도 버튼이 제자리.
- 뒷면 "← 접기"는 상단 backnav로(하단 대칭 유지). 반응칩 🔥🙌👀은 버튼 아니라 "지인이 받는 것" 설명 문구.

**뒷면(더 자세히) = 이 카드를 쓸 엔드유저의 하루 타임라인** (통증→제품→해소 색점 아크) + 프롬프트 미리보기.
