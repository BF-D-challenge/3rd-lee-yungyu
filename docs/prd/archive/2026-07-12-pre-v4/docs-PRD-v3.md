# 오늘 해볼까 — PRD (한 장)

> BF.D 4주 챌린지 · 이윤규 · v3 재통합 2026-07-08. **최종 PRD(기준 명세)는 [prds/오늘-해볼까-prd.md](prd/오늘-해볼까-prd.md)**, 실행 순서는 [TASK.md](TASK.md), 개별 개선안·과거 버전·리스크 대장(R1~R26)은 전부 [prds/archive/](prd/archive/)로 정리했다.

## 정의

**무엇을 만들지 모르는 초보 1인 빌더가, 좋아하는 것(🌱씨앗)에서 출발해 오늘 만들 아이디어 1개를 뽑고, 지인 응원으로 수요까지 확인하는 도구.**

- 문제: AI 코딩 시대의 첫 병목은 개발이 아니라 ① 바로 실행할 만큼 구체적인 아이디어 ② 만들기 전 수요 확신.
- 포지셔닝 금칙어: **"아이디어 생성기 · AI 추천 · 랜덤"** — 순수 생성의 지불의사는 0(리서치 실증). 서사는 항상 **"내 것(🌱) → 오늘 바로 → 지인에게 물어보기"**. 냉소 진영의 처방("툴 말고 유저와 대화하라")이 곧 응원 카드다.

## 타겟

| 페르소나 | 후크 | 결제 접점 |
|---|---|---|
| P1 지민(막막형, 러닝) | 씨앗을 넣는 순간 결과가 전부 내 얘기 | 실행 브리프는 무료(공유하면 열림) — 캡 소진 시 오늘 패스 1,900 |
| P2 현우(개발자, 확신 부족) | 응원 카드 = 수요 조사 도구 | 수요 리포트 |
| P3 소연(취향형, 인스타) | 씨앗 고정 + 스토리에 올릴 만큼 예쁜 카드 | 오늘 패스 1,900 |
| P4 태호(도메인 15년, 시간 빈곤) | 업계 단톡방 수요 투표 | 수요 리포트(9,900 실험) |

> ~~실행 플랜 990원~~은 폐지됐다 — 실행 브리프(5섹션)는 무료 완제품, 대신 **공유해야 "오늘 만들기 프롬프트"가 열리는 공유 게이트**로 결제 대신 바이럴을 강제한다.

## 확정 원칙 (변경 시 사용자 승인) — [구현됨] / [목표: 구현 대기] / [미정] 표기

1. **[구현됨] 입력은 전부 객관식 칩**, 주관식은 맨 끝 탈출구(`✏️ 직접 쓸래요`).
2. **[구현됨] 빈 슬롯 채우기 + 🔒고정/리롤**: 필수 4칸(🌱씨앗·😖불편·📦형태·🎬장면) + 옵션 💭마음(~35%). 빈 시작 → 덱에서 한 장씩 채움 / 탭=그 축만 교체 / ✕=제거 / 🔒고정한 칸은 유지한 채 🎲전체 다시 뽑기(유일한 캡 소모). **씨앗 고정 릴 대신 취향 질문 시트**(첫 아이디어는 무방해 완성, 두 번째 전체 뽑기부터 미설정 시 40% 등장, 제출/스킵 후 재질문 없음) — 이후 덱·뽑기가 취향 80%/다양성 20% 가중. 불편·형태는 씨앗 allowlist 앵커 준수, 골든 조합 일치 시 ✨타이틀.
3. **[구현됨] 카드 콘텐츠 = trustmrr 실매출 앵커 90/10 생성**(H0~H3 블라인드 실험 승자 규칙 v6/v7). 확정 화면은 flip 카드(앞=설명+고통 타임라인, 뒤=해결+근거+MVP), **공유해야 만들기 프롬프트가 열리는 공유 게이트**.
4. **[구현됨] 응원은 전부 긍정형**(부정 선택지 없음, Gas 앱 벤치마크) — 4칩(🔥나도 이거 필요해/🙌완성하면 알려줘/👀지켜볼게/💪너라면 만들어) + A/B 응원 대결(`/vs/[slug]`)도 동일 원칙.
5. **[구현됨] 응원·공유는 무료**(바이럴 K), **결제는 상세·추가에서**: ① 스핀 캡→오늘 패스 1,900 ② 발행+대시보드 5표 이상→수요 리포트 1,900~9,900. 실행 플랜 990은 폐지. **전부 가짜 문**(클릭·이메일만 측정), 구독 없음.
6. **[구현됨, 플래그로 제어] 로그인은 카드 발행(S3) 단 한 곳**(구글 원탭 — 실 Supabase OAuth 코드 준비, env 플래그로 on/off). **응원 수신자에겐 로그인·설치 요구 0자.**
7. **[목표: 구현 대기] 디자인 = "다크 무대 위의 moonlight 게임판"**(2026-07-07~08 디자인 리셋 확정 — `prds/오늘-해볼까-prd.md` 원칙 7 / `prds/오늘-해볼까-디자인-리셋.md` 상세): 랜딩 폐기 → `/`가 곧 게임판(인트로 내장, 매 방문 재생), 세리프 Playfair 전면 + f11 블루 단일 액센트, 타로/오로라 카드아트 폐기 예정. **아직 코드는 착수 전**(현재는 다크+glass+aurora+타로 상태 유지 중).

## 여정 (현재 구현 기준)

```
[L] 랜딩(히어로+4섹션) — ⚠ 디자인 리셋 목표 상태에서는 폐기 예정
 → [S0] 씨앗 온보딩(/start, 여정 밖 독립 페이지)
 → [S1] 빈 슬롯 🎰 n/5(/slot) — 부채꼴 덱 채움 · 탭=교체 · ✕=제거 · 🔒고정 · 🎲전체 다시 뽑기(캡 소모)
       중간 취향 시트(1회) / 캡 소진→페이월①
 → [확정 분기] flip 카드(앞 고통 타임라인/뒤 해결+근거) → 공유해야 프롬프트 언락
 → [S3] 발행(/publish) ★로그인 (9:16 카드 + 카톡/링크/스토리)
 → [S5] 수신자 응원(/c/[slug], 무로그인, 4칩 긍정 전용) → "너라면?" 프리필 → S1 = K 루프
 → [S5-VS] A/B 응원 대결(/vs/[slug]) — 최근 카드 2장, 역시 무로그인·긍정 전용
 → [S6] 대시보드(/dashboard) — 무료: N명 도착+첫 반응 1개 / 5표 이상→리포트 결제 유도(가짜 문)
```

**응원(투표) 데이터는 이미 Supabase에 연결돼 있다**(`src/lib/backend/votes.ts`, 개인 계정 SQL Editor로만 스키마 적용 — MCP 금지). 로그인·결제만 아직 가짜.

## 남은 미착수 항목 (우선순위 순)

1. per-slug 서버 OG 메타 — `/c/[slug]`·`/vs/[slug]`가 전부 클라이언트 컴포넌트라 카톡 링크 프리뷰가 빈 화면
2. 수요 리포트 4층(판정·유형분해·리드목록·아웃리치) — 현재 결제해도 `●` placeholder뿐
3. 금칙어 잔존 2건 — `demand-board.tsx`의 "판정: ●●●" · "검증된 고유 투표자"
4. 디자인 리셋 P1~P5(moonlight 게임판 롤아웃)

## 위험 가정 (검증 순서)

1. 공유가 실제로 일어나는가 — Supabase 연동 완료로 **지금 바로 실측 가능**(더 이상 3주차를 기다릴 필요 없음)
2. 씨앗-conditioned 조합의 체감 품질(카드 콘텐츠 v6/v7 롤아웃으로 개선 시도)
3. 결제 클릭 지불의사(특히 리포트 — 단 본문이 아직 비어 있어 클릭해도 실망 리스크)
4. 수신자→발신자 K 루프 폐쇄
