## 1. 문제 정의

### 겉으로 드러난 문제

초보 1인 개발자는 AI 코딩 툴 덕분에 **앱을 만들 수는 있게 됐다.** 그런데 정작 **무엇을 만들지를 몰라서 시작조차 못 한다.**

### 진짜 문제 (5 Whys로 파고들어 찾음)

AI 코딩 시대에 개발자의 첫 번째 벽은 '개발 실력'이 아니다. 두 가지다.

1. **지금 당장 만들 수 있을 만큼 구체적인 아이디어**를 얻는 것
2. 만들기 전에 **"이거 수요가 있을까?"라는 확신**을 얻는 것

1번만 풀면 우리 타겟 4명 중 1명(무엇을 만들지 막막한 사람)만 만족한다. 이미 아이디어가 있는 사람들(개발자·특정 분야 전문가)은 2번에서 멈춰 있다.

### 지금 사람들은 어떻게 버티고 있나 (현재 방식)

- ChatGPT/Claude에 "만들 아이디어 추천해줘"라고 물음 → 답이 너무 뻔함 → 계속 다시 묻는 무한 루프
- 노션에 아이디어 30개를 쌓아두고 하나도 못 고름 ("이거 수요 있나?"에서 멈춤)
- v0·Lovable·Claude Code를 열지만 **맨 첫 프롬프트를 못 정함**
- 커뮤니티 조언은 "툴 쓰지 말고 유저 5명이랑 얘기해봐" — 맞는 말이지만, 그걸 실제로 하게 만드는 장치가 없음

### 왜 지금인가 (Why now)

AI 코딩 툴로 **제작 비용이 급락**하면서, 병목이 "만들기"에서 "무엇을 만들지 + 수요 확신"으로 옮겨갔다.

단, 리서치로 확인한 함정이 있다.[^demand] "AI 아이디어 생성기"라는 형태 자체에 사람들은 냉소적이고, **순수 생성에 돈 낼 의사는 0에 가깝다**(대표 실패 사례: IdeasAI). 유일하게 돈이 되는 건 **1회성 소액 리포트**(예: DimeADozen $9)뿐이었다.

→ 그래서 **포지셔닝 금칙어**를 정했다. **"아이디어 생성기 · AI 추천 · 랜덤"이라고 절대 말하지 않는다.** 우리가 파는 이야기는 항상 이것이다:
**"내 것(🌱씨앗)에서 출발해 → 오늘 바로 만들고 → 지인에게 물어보는 도구."**
냉소파가 말하는 처방("유저와 대화하라")을 곧 이 제품의 **투표/응원 카드** 기능으로 만든 것이다.

## 2. 타겟 & 그들이 원하는 것

### 타겟 사용자 (페르소나 4명 — 상세 프로필은 archive[^improve])

'페르소나'는 대표 사용자를 한 명의 가상 인물로 그려본 것이다. **P1이 핵심 타겟**이고, 나머지는 확장 대상이다.

| 페르소나 | 상황 | 무엇에 끌리는가 (후크) | 결제 접점 |
|---|---|---|---|
| **P1 지민** (29·주니어 마케터) — **핵심 타겟** | "사이드 프로젝트 할 거야"만 6개월째. 좋아하는 건 있음(러닝) | 씨앗을 넣는 순간 결과가 전부 내 얘기가 됨 | 실행 브리프 자체는 **무료**(공유하면 열림) — 뽑기 캡 소진 시 오늘 패스 1,900원 |
| P2 현우 (33·백엔드 5년차) | 노션에 아이디어 30개, 못 고름 | 응원 카드 = 수요 조사 도구 | 발행 후 5표 이상 모이면 수요 리포트 |
| P3 소연 (26·프리랜서 디자이너) | 빵집 투어 계정 운영, 랜덤을 싫어함 | 씨앗 고정 + SNS에 올릴 만큼 예쁜 카드 | 오늘 패스 1,900원 |
| P4 태호 (41·물류 15년차) | 업계 비효율을 매일 봄, 시간 없음 | 업계 단톡방에 수요 투표 돌리기 | 수요 리포트 (9,900원 상방 실험) |

> **변경 이력**: P1의 결제 접점이던 "실행 플랜 990원"은 **폐지됐다**[^kill990]. 실행 브리프(5섹션)는 이제 완전 무료 완제품이고, 대신 **공유해야 "오늘 만들기 프롬프트"가 열리는 공유 게이트**로 바뀌어 결제가 아니라 **바이럴 K를 강제하는 장치**로 재배치됐다.

### 사용자가 원하는 일 (Job Story)

'Job Story'는 **"어떤 상황일 때 → 무엇을 하고 싶고 → 그래서 뭘 얻고 싶은가"**를 한 문장으로 쓴 것이다.

- **상황:** AI 코딩 툴로 뭔가 만들고 싶은데, 무엇을 만들지 막막할 때
- **하고 싶은 것:** 내가 좋아하는 것/잘 아는 것(🌱씨앗)에서 출발한 조합을 슬롯머신처럼 돌려 **오늘 만들 아이디어 1개를 확정**하고, 그걸 카드로 만들어 **지인에게 수요 응원을 받는 것**
- **얻고 싶은 것:** "내 것"이라는 애착과 "누가 이걸 원한다"는 확신을 갖고, **오늘 바로 만들기 시작**하는 것

### 왜 이 제품이어야 하나 (경쟁 대비)

- **누구를 위한 건가:** 무엇을 만들지 모르거나 확신이 없는 초보 1인 개발자
- **기존 대안들이 못 주는 것:** ChatGPT 추천(내 취향과 무관·일반적), 노션 목록(검증이 없음), 커뮤니티 피드백(글이 묻혀 반응 없음) — **어느 것도 "내 것에서 출발 + 수요 확인"을 한 흐름으로 주지 못한다.**
- **우리가 주는 것:** 내 취향이 반영된 카드 덱에서 빈칸을 채워 아이디어를 좁히고 → 확정한 카드를 지인 응원으로 검증. 결과는 **오늘 만들 아이디어 1개 + 지인 N명의 수요 신호 + 시작 프롬프트.**

## 3. 성공을 어떻게 잴 것인가 (임팩트 가설)

### 가장 중요한 단 하나의 지표 (North Star Metric)

- **성장 지표 = 카드 공유율** = (카드를 공유한 횟수) ÷ (아이디어를 확정한 횟수)
  → 아이디어를 정한 사람이 실제로 지인에게 물어보러 가는가?
- **수익 지표 = 세션당 '가짜 문' 클릭 가치**
  → 결제는 대부분 여전히 '가짜 문'(아래 [용어](#용어-한눈에) 참고)이므로, 클릭·이메일 남긴 정도로 "돈 낼 의사"를 측정한다.

> **[구현됨] 앞당겨진 부분**: 응원(투표) 데이터는 원래 "3주차 실측 후에만" Supabase에 연결할 계획이었지만, **이미 실 데이터베이스로 연결돼 있다**[^votesdb](`src/lib/backend/votes.ts` → 개인 계정 프로젝트, localStorage는 오프라인 폴백). 덕분에 성장 NSM(공유율)을 콘솔 로그 근사치가 아니라 **실제 도착·응원 수**로 지금 바로 확인할 수 있다.

### 앞단에서 미리 보는 지표 (선행 지표)

1. 씨앗 입력 완료율 · 씨앗 종류별 아이디어 확정률 (= 씨앗이 얼마나 좋은가)
2. 발행한 카드 중 응원 5표 이상 받은 비율 (= 수요 검증이 실제로 돌아가는가)
3. **바이럴 K** = 공유 수 × (응원한 사람이 다시 아이디어를 뽑아보는 전환율)
4. 로그인 마찰 = (로그인 성공) ÷ (로그인 요청) — 로그인 요구가 사람을 얼마나 튕겨내는가

> 참고: 아직 GA4(구글 애널리틱스)를 안 붙인 단계에서는, 이벤트 기록 코드(`src/lib/track.ts`)가 콘솔 로그와 브라우저 저장소(localStorage)에 대신 쌓아둔다.

### 숫자가 아닌 성공의 모습

> "내가 좋아하는 걸로 오늘 만들 게 정해졌고, 친구들이 진짜 쓸지도 알게 됐어요."

## 4. 솔루션 (MVP)

'MVP'는 검증에 꼭 필요한 최소 기능만 담은 첫 버전이다.

### 한 문장 정리

**오늘 해볼까**는, 무엇을 만들지 모르는 초보 1인 개발자가 **좋아하는 것(🌱씨앗)에서 출발해 오늘 만들 아이디어 1개를 뽑고, 지인 응원으로 수요까지 확인하는 도구다.**

### 화면에 등장하는 카드 축

아이디어는 여러 '축'의 카드를 조합해서 만든다.

| 축 | 뜻 | 필수/옵션 |
|---|---|---|
| 🌱 씨앗 | 내가 좋아하거나 잘 아는 것 (예: 러닝) | 필수 |
| 😖 불편 | 해결하려는 불편·문제 | 필수 |
| 📦 형태 | 만들 결과물의 모양 (앱·봇·뉴스레터 등) | 필수 |
| 🎬 장면 | 그게 쓰이는 상황·장면 | 필수 |
| 💭 마음 | 건드리려는 감정·심리 | 옵션 (전체 뽑기 때 ~35%만 등장) |

### 절대 바꾸지 않는 확정 원칙 (변경 시 만든 사람 승인 필요)

1. **[구현됨] 모든 입력은 객관식(칩 선택)이 기본**이다. 직접 타이핑하는 주관식은 맨 끝의 탈출구(`✏️ 직접 쓸래요`)로만 둔다.

2. **[구현됨] 아이디어 만들기는 '빈 슬롯 채우기 + 고정/리롤'** 방식이다.
   - 필수 4칸(🌱씨앗·😖불편·📦형태·🎬장면) + 옵션 1칸(💭마음)으로, **빈 상태에서 시작**해 부채꼴 덱에서 한 장씩 뽑아 채운다.
   - 채워진 카드를 탭하면 그 축만 교체, ✕는 제거, 🎲는 **전체 다시 뽑기**(이것만 뽑기 횟수를 소모).
   - **🔒 고정(Prioritize)**: 마음에 든 칸을 잠그면 🎲 전체 다시 뽑기에서 잠긴 칸은 그대로 두고 나머지만 리롤한다.
   - '씨앗을 처음부터 고정'하는 대신 **취향 질문 시트**를 중간에 한 번 띄운다(첫 아이디어는 방해 없이 완성, 두 번째 전체 뽑기부터 취향이 없으면 40% 등장, 제출/스킵 후 재질문 없음). 취향이 들어가면 이후 덱·뽑기가 **취향 80% / 다양성 20%**로 가중된다.
   - **불편·형태 카드는 씨앗과 어울리는 것만** 나온다(허용 목록 앵커). 잘 맞는 조합(골든 조합)이 완성되면 ✨특별 타이틀.

3. **[구현됨] 카드 콘텐츠는 "실제 근거처럼 그럴듯하게"** — 카드 품질 실험(H0~H3, 20조합×4안 블라인드 채점)의 승자 규칙(v6/v7)이 전량 적용됐다[^cardcopy].
   - 생생한 니치를 한 줄 맨 앞에(실명·숫자·구체 시나리오), 해외 서비스명은 한국 대중 브랜드로 치환(예: 스트라바→나이키런클럽).
   - `research/trustmrr-acquire/ideas.jsonl`(실 매출 1,863개 제품)에서 유사 제품을 앵커로 매칭해 "검증된 메커니즘 90% + 이 씨앗·니치로 10% 비틀기"로 생성, 카드에 근거 한 줄(`evidence`) 노출.
   - 확정 화면은 **flip 카드**(타로 비율 300:485): 앞면 = 설명 + "고통 타임라인", 뒷면 = 해결 + 근거 + MVP. **공유해야 "오늘 만들기 프롬프트"가 열리는 공유 게이트**로 K 루프를 강제한다.

4. **[구현됨] 응원은 전부 긍정형** — 부정 선택지 자체가 없다(Gas 앱 벤치마크[^gas]). 카드 발행 후 응원은 4칩(🔥나도 이거 필요해 / 🙌완성하면 알려줘 / 👀지켜볼게 / 💪너라면 만들어), 응원 직후 "너라면?" 씨앗 프리필로 재뽑기 루프를 닫는다. **A/B 응원 대결(`/vs/[slug]`)**도 같은 원칙 — 두 카드 중 하나를 고르되 둘 다 응원(강제 양자택일이 아니라 강제 긍정).

5. **[구현됨] 돈 버는 지점은 뒤에 둔다.** 응원·공유는 **무료**(그래야 입소문이 퍼진다). 결제는:
   - ① 뽑기 횟수 소진 → 오늘 패스 1,900원
   - ② 카드 발행 시 대시보드에서 응원 5표 이상 모이면 → 수요 리포트 1,900~9,900원
   - ~~실행 플랜 990원~~은 폐지됐다(원칙 2번 참고).
   - **이 결제는 전부 '가짜 문'**이다. 실제 결제 없이 클릭·이메일만 측정한다. 구독은 없다.

6. **[구현됨, 플래그로 제어] 로그인은 딱 한 곳, 카드를 발행하는 순간(S3)에만** 요구한다 (구글 원탭). **응원하는 사람에게는 로그인·설치를 한 글자도 요구하지 않는다.** 실제 Supabase 구글 OAuth 코드가 준비돼 있고, 환경변수(`NEXT_PUBLIC_GOOGLE_LOGIN`)와 Supabase 프로바이더 설정이 갖춰지면 실 로그인이 켜지며, 아니면 데모용 가짜 통과로 자동 폴백한다.

7. **[목표 — 구현 대기] 디자인 톤은 "다크 무대 위의 moonlight 게임판"으로 이동한다**[^designreset]. 2026-07-07~08 디자인 리셋 인터뷰에서 전 분기 확정된 목표 상태:
   - 랜딩 페이지(Hero/HowItWorks/FAQ) **폐기** → `/`가 곧 게임판, 인트로가 게임판 안에 내장된 원 컨티뉴어스 샷(매 방문 재생, 3~3.5초, 즉시 스킵 가능)
   - 타이포는 세리프 **Playfair** 전면 통일, 액센트는 **f11 블루** 단일(퍼플 퇴역)
   - 카드 표면은 검정 중앙 + 블루 림 글로우 3레이어, 덱 뒷면은 moonlight식 인그레이빙(로제트) 패턴
   - **타로/오로라 카드아트 런타임 경로(`src/lib/card-art.ts` 3티어 리졸버·`TarotCard`)는 폐기 대상으로 확정됐고 현재 코드에서는 제거됐다.** `public/cards/aurora/**` 자산은 비교·회귀 확인용으로만 보존한다.
   - 나머지 세부 분기(D1~D17)와 구현 순서(P1~P5)는 원문서[^designreset] 참고.

### 사용자가 밟는 여정 (현재 구현 기준)

```
[L]  랜딩 (히어로 + 미니멀 4섹션) — ⚠ 디자인 리셋 목표 상태에서는 폐기 예정(원칙 7)
 → [S0] 씨앗 온보딩 (/start, 여정 밖 독립 페이지)
 → [S1] 빈 슬롯 채우기 🎰 (/slot, n/5칸) — 부채꼴 덱 · 탭=교체 · ✕=제거 · 🔒고정 · 🎲=전체 다시 뽑기(캡 소모)
        중간 취향 시트 1회 · 캡 소진 시 오늘 패스 결제 유도(페이월①)
 → [확정 분기] 아이디어 flip 카드 — 앞면 고통 타임라인 → 뒷면 해결+근거, 공유해야 프롬프트 언락
 → [S3] 카드 발행 (/publish) ★여기서만 로그인 (9:16 카드 + 카톡/링크/스토리 공유)
 → [S5] 받은 사람이 응원 (/c/[slug], 무로그인, 4칩 긍정 전용) → "너라면?" 씨앗 프리필 → S1로 = 입소문 루프
 → [S5-VS] A/B 응원 대결 (/vs/[slug]) — 최근 카드 2장으로 만드는 별도 대결 링크, 역시 무로그인·긍정 전용
 → [S6] 대시보드 (/dashboard) — 무료: 도착 수 + 첫 반응 1개 / 5표 이상 시 수요 리포트 결제 유도(가짜 문)
```

> **1주차는 프런트만.** 상태는 브라우저 저장소(localStorage)에 두고, 공유 URL 자체가 데이터를 실어 나른다. **단, 응원(투표) 데이터는 이미 Supabase에 연결돼 있다**(§3 참고) — 로그인·결제만 아직 가짜다.

### 반드시 있어야 할 기능 (Must-have)

1. **취향이 반영된 빈 슬롯 채우기 + 고정/리롤** — 필수 4칸 + 옵션 마음 칸을 부채꼴 덱에서 채운다. 취향 시트가 덱을 80/20으로 가중하고, 씨앗과 안 맞는 카드는 막고(앵커), 잘 맞으면 ✨. [구현됨]
2. **근거 기반 카드 콘텐츠 + flip 확정 카드 + 공유 게이트** — trustmrr 실매출 앵커 90/10 생성, 고통 타임라인, 공유해야 열리는 프롬프트. [구현됨]
3. **카드 발행 + 로그인 없는 응원(단건 + A/B 대결) + 실제 응원 데이터** — 9:16 공유 카드, Supabase 백엔드, 응원 직후 씨앗 프리필로 재뽑기 루프. [구현됨]
4. **가짜 문 2개 + 측정 코드** — 결제 지점(오늘 패스·수요 리포트)의 클릭·이메일을 재고, 여정 전체 이벤트를 기록한다. [구현됨]
5. **디자인 리셋 롤아웃(P1~P5)** — moonlight 게임판 통합, 랜딩 폐기, 타로/오로라 카드아트 정리. [목표, 구현 대기]

### 4주차 시연 시나리오 (Demo Flow)

2. 🎲 전체 다시 뽑기 → 취향 시트에 "러닝" 입력 → 마음에 든 축은 🔒로 고정 → 나머지만 리롤 → 확정
3. flip 카드 뒷면에서 해결+근거 확인 → 공유해서 프롬프트 언락 → 카드 발행(로그인) → 카톡 링크 복사

## 5. 리스크 & 검증할 가정

### 검증해야 할 4가지 가정 (VUVF)

| 축 | 무엇을 확인하나 | 우리의 가정 |
|----------|---|---|
| **Value** (가치가 있나) | 사람들이 이걸 원하나 | 사용자는 "내 것에서 나온 아이디어 + 수요 신호"에 가치를 느낀다. |
| **Usability** (쓸 수 있나) | 설명 없이 쓸 수 있나 | 설명 없이도 '덱에서 채우기 → 탭 교체 → 확정 → 발행 → 응원' 흐름을 이해한다. |
| **Viability** (사업이 되나) | 우리 자원으로 가능한가 | 주 20시간(실질 15~16h) × 4주로 가짜 루프 전체 + 실측 2회가 가능하다. |
| **Feasibility** (기술이 되나) | 만들 수 있나 | 씨앗 기반 조합 + trustmrr 앵커 규칙만으로 "만들어볼 만한" 품질이 나온다. |

### 가장 위험한 가정과 검증 순서

1. **공유가 실제로 일어나는가** — Supabase 연동이 이미 끝났으니, 3주차를 기다릴 필요 없이 **지금 바로 실측 가능**하다. 안 일어나면 나머지 전부 무의미.
2. **씨앗 기반 조합이 진짜 "내 얘기"처럼 느껴지는가** — 카드 콘텐츠 v6/v7 롤아웃으로 상당 부분 해소 시도됨.
3. 결제 클릭으로 본 돈 낼 의사 (특히 수요 리포트 — 단, 리포트 본문 자체가 아직 비어 있어 클릭해도 실망 리스크 있음)
4. 받은 사람 → 보낸 사람으로 이어지는 입소문 루프가 실제로 닫히는가

> 전체 리스크 목록(R1~R26)은 archive[^improve]에 있다.

### 1~2주차 검증 실험 (Wizard-of-Oz)

처음부터 끝까지 이어지는 **가짜 루프**를 실제 유저에게 던진다. 결제 2지점은 가짜 문 클릭만 측정하고, 수요 리포트는 사람이 손으로 써서 전달한다.

**Day 1부터 페르소나별 테스터 5명을 동시에 모집한다** (사람 구하기가 진짜 병목이다). 실측이 지목한 **딱 한 단계**에만 3주차 실제 구현을 판다.

## 용어 한눈에

이 문서에 나오는 약어·기호를 처음 보는 사람용으로 모았다.

| 용어 | 뜻 |
|------|------|
| 🌱 씨앗 | 사용자가 좋아하거나 잘 아는 것. 모든 아이디어의 출발점 (예: 러닝) |
| 😖 불편 / 📦 형태 / 🎬 장면 / 💭 마음 | 아이디어를 조합하는 카드 축들 (불편=해결할 문제, 형태=결과물 모양, 장면=쓰이는 상황, 마음=건드릴 감정) |
| MVP | Minimum Viable Product. 검증에 꼭 필요한 최소 기능만 담은 첫 버전 |
| Job Story | "어떤 상황일 때 → 무엇을 하고 싶고 → 그래서 뭘 얻나"로 쓴 사용자 욕구 문장 |
| North Star Metric | 제품이 잘 되고 있는지를 대표하는 단 하나의 핵심 지표 |
| 바이럴 K | 한 사용자가 평균 몇 명의 새 사용자를 데려오는지. 1보다 크면 입소문으로 저절로 퍼짐 |
| 가짜 문 (fake door) | 실제 기능 없이 버튼만 두고, 클릭·이메일로 "돈 낼 의사·수요"를 재는 기법 |
| 공유 게이트 | 결제가 아니라 **공유해야** 다음 콘텐츠(만들기 프롬프트)가 열리는 장치. 결제 대신 바이럴을 강제 |
| 앵커 / 허용 목록 | 두 가지 뜻으로 쓰인다 — ① 씨앗과 안 어울리는 카드가 안 나오게 막는 제약(뽑기 단계) ② 카드 문구를 생성할 때 참고하는 실제 매출 있는 유사 제품(카드 콘텐츠 생성 단계, trustmrr 코퍼스) |
| 골든 조합 | 특별히 잘 맞아떨어지는 씨앗+축 조합. 완성 시 ✨ 특별 타이틀 |
| VUVF | Value·Usability·Viability·Feasibility. 새 제품이 검증해야 할 4가지 가정 축 |
| Wizard-of-Oz | 뒤에서 사람이 수동 처리하지만 사용자에겐 진짜 시스템처럼 보이게 해 검증하는 방법 |
| S0~S6 / L | 화면 단계 코드 (L=랜딩, S1=슬롯, S3=발행, S5=응원, S6=대시보드 등) |
| R1~R26 | 리스크 목록 번호 (전체는 archive 개선안 문서 §7) |
| f11 | toss simplicity 사이트의 특정 인트로 프레임(다크 + 블루 글로우 카드) — 디자인 리셋의 색 레퍼런스 |
| localStorage | 서버 없이 사용자의 브라우저 안에 데이터를 저장해두는 공간 |
| GA4 | 구글 애널리틱스 4. 사용자 행동을 측정하는 도구 |
