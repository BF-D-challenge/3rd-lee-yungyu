# 오늘 해볼까 — 아이디어 품질 개선 계획 (2026-07-07)

> 문제(사용자): "5개 조합 시 매우 이상한 아이디어만 나온다. 인기있는 그럴듯한 아이디어가 나오게 하려면?"
> 근거: 진단 2트랙(데이터 격차·조합 실패 모드 20만회 시뮬) + 리서치 2트랙 + 설계 3안 심사 + 적대 검증 워크플로우(11 에이전트). 원 리서치: [idea-tools-2026](../../research/idea-tools-2026/SUMMARY.md).

---

## 1. 진단 — "이상한 아이디어"는 4개 층 결함의 중첩

| 층 | 결함 | 실패율(20만회 시뮬) |
|---|---|---|
| ① 뽑기 | 80/20 전체풀 탈출구로 allowlist 밖 조합 + **pain↔format 호환 데이터 부재**(선택마비×스트릭 류) + 장면·마음 친화도 0(항상 랜덤 노이즈 접두) | 씨앗-불편 불일치 24.4% · **메커니즘 충돌 30%(최대 단일 원인)** |
| ② 큐레이션 | **golden 20개 중 뽑기 풀 도달 씨앗이 1개뿐**(running). 12개는 어떤 풀에도 없는 유령 씨앗 | 골든 적중률 **0.094%**(1,064번에 1번 = 사실상 비활성) |
| ③ 카드 | pains 16개가 추상 메타-불편("벤치마크 부재")이라 어떤 조합도 컨설팅 문체 | — (데모 v6는 실매출 제품서 증류한 구체 1인칭 182장) |
| ④ 렌더 | 따옴표가 조사 교정 차단, `format.short` 한 단어로 해결책 공허, **know 트랙에서 고른 장면·마음이 문장에서 증발** | 조사 오류 40% · 정보량0 25% · 축 증발 55% |

**핵심 통찰**: ①품질 격차의 절반이 조합이 아니라 **문장 렌더 층**에서 발생(같은 재료도 죽은 문장이 됨). ②적대 검증이 "데모 90% 정합=그대로 이식"을 반증 — 데모 10도메인엔 food가 없고 **카페×운세가 like 트랙의 76% 주경로**, 취미(골프·클라이밍·등산) 원천 0행. → 이식은 재태깅·audience 분리·보강을 **전제 조건**으로 게이팅한다.

## 2. 채택 접근 — 데모 이식+확장 (조건부 수정판)

검증 자산 3종(구체 1인칭 카드 182장 · 도메인 곱셈 가중 `weightedPick` · pain 따옴표 인용 문장)을 이식하되, 반증된 지점은 "이식"이 아니라 "재작업"으로 정직 재분류:
- 도메인 10→약 16종 재설계 후 182장 전량 재태깅.
- **pain 카드에 `audience`(consumer|maker) 1비트** → like=consumer, know=maker 하드 필터(시청자×크리에이터 오부착 구조 차단).
- **pain×format 호환 행렬**은 데모에 없던 신규 발명 — 인간 정답 ~150페어로 크리틱 일치율 κ≥0.7 선검증 후 전수 채점.
- 얇은 도메인(취미 7~9씨앗)은 app-store kr 수확 + 씨앗당 4~6장 손저작.

## 3. ★ P0 퀵윈 (0.5~1일, 순수 데이터 + 코드 ~10줄 — 체감 개선 상당분 즉시)

**코드 위험 거의 0. 사용자 불만을 바로 완화하는 것들:**
1. `.gitignore`에 `research/**/*.jsonl` + `data/build/` 추가(스크랩 원본 재게시=ToS 위반 차단, 모든 작업의 선행).
2. **유령 씨앗 12개를 tracks/presetSeeds로 승격 + 골든 보유 씨앗을 온보딩 앞정렬** → 죽어있던 골든 12개 즉시 부활.
3. **`sentenceTemplates` 데이터 교체**: pain 따옴표 1인칭 인용 + 조사가 따옴표 밖에 붙는 설계(조사 오류 40%→0) · `format.action` 사용(정보량0 해소) · know 트랙에 `{situation}/{psych}` 실삽입(축 증발 55%→0) · seed 2회 반복 제거.
4. `assemble.ts` 4줄: `painShort→pain.label`, `format.short→format.action` + `goldenFor→goldenNear`(씨앗+1축 일치 부분 스냅으로 골든 커버리지 증폭).
5. **기존 16×12=192 pain×format 행렬을 손채점해 combos.json에 정적 표로 삽입** → 최대 실패모드(메커니즘 충돌 30%) 즉시 차단.
6. `demand-board.tsx` 금칙어 2건 스윕("판정:"·"검증된 고유 투표자" — 피벗 P0과 병합).

**범위 제한**: know 트랙은 데모 work/biz pain 직이식 가능(손시뮬 합격). **like 트랙 덱 교체는 P1 완료 전 금지**(현행이면 카페×운세급이 주경로로 출고).

## 4. P1 (4~5일, 오프라인 파이프라인)

스크립트 4종(`extract`/`merge`/`build-affinity`/`assemble`+시뮬 게이트) → 도메인 16종 재설계·182장 재태깅·audience 비트 → 얇은 도메인 손저작 → ideas.jsonl 1,863행(rev≥$100 621행 우선) LLM 배치 증류(문체 버킷 쿼터) → pain×format 행렬 κ검증 후 채점 → **골든 20→~80 재생성**(도달 가능 트리플에, 상위 20 사람 리라이트) → 20만회 씨앗별 게이트 통과 → `combos.json` v2 + `pools.ts` weightedPick 이식.

**런타임 변경은 3파일 ~60줄뿐**(`lib/combos.ts` 타입 · `lib/pools.ts` weightedPick · `assemble.ts` 4줄) — 나머지는 전부 데이터. UI·slot-store·josa·draw·수신자 경로 무변경.

## 5. P2 (증분)

app-store kr 소비자앱 pain 수확(취미 보강) · 디스콰이엇+한국 실사례 30~50개 큐레이션 · 투표/좋아요 축적 시 저반응 트리플 가중 하향(정적 재배포, 백엔드 불필요).

## 6. 품질 게이트 (빌드 실패 조건으로 기계화)

- **G0 원천**: 스크랩 원본 git 영구 배제, 레포엔 재작성 최종본만(ToS).
- **G1 카드**: 실매출 증류·v6 검증·손저작 3출처만. 구체명사 ≥1 린트, 추상어 16개 블록리스트 회귀, general 보편 pain 20~25% 하한(🔥 모수 방어).
- **G2 페어**: pain×format 충돌 페어 가중 0(확률 아닌 구조). 행렬 κ≥0.7 선검증 + 씨앗별 유효 페어 ≥8.
- **G3 뽑기**: 씨앗별×축별 불일치 ≤5% + 유효 pain 풀 ≥6 + axis-coverage(전 도메인 양축 카드 ≥4). **전역 평균이 숨기던 구멍을 씨앗별 판정으로 봉쇄.**
- **G4 문장**: 20만회 렌더 린트(조사 오류·seed 반복·따옴표·금칙어 정규식·매출수치·크기 예산·수신자 라우트 combos 임포트 금지).
- **G5 인간**: 릴리스당 300문장을 응원칩 4지선다(🔥/🙌/👀/💪)로 채점 — 💪+👀 비율 상한으로 슬롭 판정. 출시 전 구/신 30문장 블라인드로 기준선 재측정.

## 7. 전후 예시 (시뮬 실측 → 개선)

- 메커니즘 충돌: `선택 마비를 스트릭으로 끝내는 구매/MD 도구` → `'뭘 들여올지 매주 결정을 못 하겠어요'라는 구매/MD 실무자를 위해 — 후보 3개를 3초 투표 카드로 지인에게 물어보는 도구`
- 도메인 공백(76% 주경로): `카페 좋아하는 사람 — '내 운세가 궁금해요'를 AI 말동무로` → `'가 본 카페를 자꾸 까먹어요'라는 카페 도장깨기족을 위해 — 방문 카페 한 줄 기록 지도 (주말 오전에 시작하기 좋아요)`
- audience 오부착: `유튜브 시청자 — '긴 영상 자르기 힘들어요'(크리에이터 pain)` → `'볼 건 많은데 뭘 볼지 못 정해요'라는 유튜브 정주행러 — 오늘 볼 영상 1개 뽑기 카드`
- 조사+know 증발: `점심시간 20분에…영화의 "반복 수동"를 확장으로` → `'성과를 매번 손으로 정리하는 게 지겨워요'라는 마케팅 실무자를 위해 — 주간 성과 자동 요약 리포트로 끝내는 도구 (회의 시작 전에 돌려보기 좋아요)`

## 8. 공수·리스크

- **총 6~7.5인일** (P0 0.5~1일 = 체감 개선 상당분 즉시, P1 4~5일, P2 증분).
- 리스크: v6 "90% 정합"은 도메인 태그 일치율이지 문장 그럴듯함이 아님(블라인드 재측정이 유일 기준선) · 취미 씨앗은 원천 0행이라 손저작 부채(1인 빌더 취향에 묶임) · pain×format 행렬 κ 미달 시 +1일 · 밸런스 상수는 뽑기 로그 부재로 감으로 정함(출시 후 튜닝).

## 9. 피벗과의 접점

칭찬받기 피벗의 응원칩(🔥나도 필요해/🙌알려줘/👀지켜볼게/💪너라면)이 **G5 인간 게이트의 채점 루브릭**과 동형 — "🔥를 부를 만한 그럴듯한 아이디어인가"가 품질 기준이 된다. 즉 품질 개선과 피벗이 같은 척도를 공유한다. demand-board 금칙어 스윕은 두 계획의 P0에서 병합.
