## 2. 통합 실행 순서 (남은 작업만, 우선순위 재조정)

### P0 — 반나절 내, 리스크 거의 0

1. **금칙어 스윕**: `demand-board.tsx`의 `"판정: ●●●"` · `"검증된 고유 투표자"` 삭제/교체.
2. **유령 씨앗 12개 승격**: `tracks`/`presetSeeds`에 12개 시드 추가(신규 68장 통합 때 썼던 패턴 재사용) → 기존 골든 12장 즉시 부활.
3. **sentenceTemplates 수정**: `{format}` → `format.action`으로 교체, know 트랙 템플릿에 `{situation}`(선택 시 `{psych}`) 삽입.
4. **per-slug 서버 OG 메타**: `/c/[slug]`·`/vs/[slug]`에 `generateMetadata(slug)` 추가(제목·설명 동적 생성) + 최소 `opengraph-image.tsx` 1개(정적 템플릿이라도) — 카톡 프리뷰 활성화.

### P1 — 결제 상품 실물화 (1~2일)

5. **수요 영수증 리포트 4층 실장**: ①규칙 기반 판정(🔥 비율·표본 임계값, AI 점수 아님) ②반응유형 실제 집계(현재 placeholder → `row.votes` 기반 계산) ③🔥 누른 사람 리드 목록 + 아웃리치 초안 문구 생성 ④재방문 드립 카피. — 이게 없으면 유료 CTA는 계속 빈 약속.
6. **블러 가짜문 → "완제품 맛보기+수량잠금" 전환**: R5 경계 개정(무료=도착수+찬반집계+첫코멘트1개, 유료=판정+유형분해+벤치마크+리드+아웃리치)을 실제 UI로.

### P2 — 실측 게이트 이후 (피벗 PRD P1 기준: 빌더 1명 실제 공유 테스트 후)

7. 데일리 골든 아이디어 로테이션.
8. 리빌(실명 옵트인) 여부 최종 결정 후 구현.

## 4. 리스크 · 한계

- 수요 영수증 리포트(P1)는 이번 세션에서 다룬 "카드 콘텐츠 품질"(29항목 평가)과는 **다른 축**이다 — 카드는 이미 검증됐지만, 그 카드에 대한 투표를 "결제할 만한 리포트"로 승격하는 로직 자체가 아직 없다.
- OG 메타(P0-4)는 코드량은 적지만 카카오톡 실제 프리뷰 렌더링은 로컬에서 검증하기 어려움 — 배포 후 실제 카톡 공유로 확인 필요.
- 유령 씨앗 12개를 승격하면 온보딩 씨앗 목록이 늘어난다 — UI 카테고리 배치(어느 대분류 밑에 넣을지) 결정이 별도로 필요.
- P1(수요 리포트 4층)은 실제 결제 연동(토스페이먼츠 등) 없이 "판정 로직 + UI"만 먼저 붙일지, 결제까지 묶어서 갈지 범위 결정이 필요 — 현재 전부 `fakeDoor` 계측이라 실제 결제 게이트웨이는 이 계획 범위 밖.
