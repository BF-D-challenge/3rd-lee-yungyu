## 0. 핵심 명제 (승리안 = 하이브리드)

**"오늘 해볼까"는 무료 실행 브리프(오늘 바로 만들 한 페이지 + 프롬프트 복사)를 리텐션 코어로 두고, 소셜 레이어를 '긍정 전용 응원 보드'로 바꾼 빌더 유틸리티다.** 수신자가 보는 건 전부 응원이지만, 응원의 강도가 곧 정직한 수요 그라디언트다.

> 심사: minimal 47.5 / **hybrid 49.5** / full 42.5. full은 감정·바이럴·WTP 천장이 최고지만 "슬롯을 격하"해 수요신호·정합·구현을 잃어 탈락. hybrid는 **"긍정은 톤이지 신호 축이 아니다"**로 감정↔수요검증 긴장을 정확히 해소.

## 1. 왜 Gas 모델인가 (리서치 팩트)

- Gas = 익명 '칭찬만' 4지선다 투표. 광고 0으로 App Store 1위, ~3개월 1,000만, 5개월 만에 Discord 인수. 같은 개발자 tbh는 9주 만에 Facebook에 $30M.
- **엔진 = 궁금증 격차**("누가 나를 칭찬했나"를 일부러 안 보여주고 유료로 조금씩 연다) + **긍정 전용**(부정 벡터 원천 차단 = 바이럴 안전장치) + **객관식 강제**(자유입력·DM 0 — 우리 설계원칙 1과 동형).
- 수익 = **God Mode $6.99**(투표자 이름 첫 글자/초성 힌트를 조금씩). 무료로 K, 궁금증에서 결제 — 우리 원칙 3과 정확히 일치.

## 2. 성인 빌더 전이 — 무엇이 넘어오고 무엇이 깨지나 (적대검증)

**넘어옴**: ① 긍정 감정 후크(나이 무관) ② 궁금증 결제(이미 원칙 3) ③ 수신자 무로그인=K 보호(Bier "매 탭은 기적") ④ 3초·객관식.

**깨짐(정직하게 인정)**: 성인은 4PM 재집결하는 고교식 밀집·동기 그래프가 없다. 초대는 나이 13→18에 매년 20% 감소, 성인망은 카톡·슬랙으로 파편화. **→ K는 구조적으로 낮다(base K≈0.1~0.2). "바이럴 소셜 앱"이 아니라 "가벼운 소셜 장식이 달린 빌더 유틸리티"로 정직하게 재프레이밍**하고, 지속가치는 무료 실행 브리프에 둔다. 성장은 오가닉 기대 삭제 → **빌더가 자기 원자셀(기수·커뮤니티·BF.D 동료)을 손으로 시딩**.

## 3. 투표 모델 — 긍정 톤 + 수요 그라디언트 (S5, 무로그인·객관식)

부정칩(🤔글쎄) 제거, 대신 **응원의 강도로 정직한 수요 신호**를 뽑는다:

| 칩 | 의미 | 신호 |
|---|---|---|
| 🔥 나도 이거 필요해 | 강수요·자기참조 | 리드 후보 |
| 🙌 완성하면 알려줘 | revealed preference(탭만, 연락처 0) | 중수요 |
| 👀 지켜볼게 | 약한 관심 | 약신호 |
| 💪 너라면 만들어 | 사람 응원 = 제품 수요 0 (긍정으로 포장된 정중한 거절) | 신호 0 |

- **판정 = count 아님. (🔥+🙌) 비율 + 표본 N 임계값 규칙 기반**(AI 점수 금지 — 신뢰 즉사). HN "I'd invest" 문법이라 긍정과 판정이 동시 성립.
- **소표본 방어(레드팀 fix)**: 빌더 실제 N=3~15(콜드스타트 0~3)에선 "🔥 부재로 부정 읽기"가 노이즈 → **명시 약신호 칩(👀/🤔)을 반드시 남기고**, 표본 N을 정직 표기, **"응원만 N명·🔥 0명 = 강한 보류"를 UI에서 명시 강조**(Risk#1 always-'만들어라' 편향 직접 차단).
- 주관식 없음 → 응원형 프리셋 칩("와 이거 나오면 대박" 등) + 맨 끝 "직접 쓸래요" 탈출구.

## 4. 수익화 — 구독 폐기, 일회성만 (Skrr 도박화 회피)

- **God Mode 구독·초성 스크램블·계단식 배급·가변보상 드립 티어 전면 삭제.** 이유: 엔지니어링된 불완전성 = 도박화이고, 우리 원칙("블러 가짜문 폐기"·"한 번에 다 주기")과 정면 충돌 + Skrr 결제율 19%→종료 반면교사.
- **무료** = 응원 도착 수 + 칩별 집계 + 첫 코멘트 1개(형태·원자료 공개).
- **유료(일회성·자동갱신 없음)**: (A) 힌트 1회권 ₩990(능동 단건, 주간 리셋 아님) (B) **응원 리포트 패스 1,900/4,900/9,900** — 리드 목록 + 아웃리치 초안을 "한 번에 정직하게 완전 공개".
- **제3자 동의 게이트**: 기본 완전 익명(관계군 "대학 동기 2·회사 1"까지만). 실명은 **응원 순간 옵트인 토글**을 누른 사람만 리드에 노출(개인정보·평판 방어선).
- **리빌 게이트**: 손발신 소표본은 빌더가 이미 신원을 알아 궁금증 갭이 0 → 리빌은 **"2차+ 지인 응원"에만 개방**, 확산이 실측 안 되면 리빌 상품 자체를 접는다. 리텐션은 궁금증 원샷이 아니라 **무료 브리프 + 데일리 골든 + 실제 리드**로 분산.

## 5. ★ 선결 차단 (적대검증 CONFIRMED — 코드 실측)

**현재 `lib/storage.ts`의 `addVote`가 "투표자 본인 localStorage"에 쓴다 → 빌더 대시보드는 영원히 0을 본다.** 응원·궁금증·리드·결제 동기가 전부 원천 0. **칩 리라벨은 이 블랙홀 컴포넌트의 화장일 뿐.** 그래서:

- **P0에서 Supabase votes 테이블을 당긴다**(3주차 → 1주차). 개인 계정(`fjhnwwdfuhmsffkvlbbo`) SQL Editor에 직접 붙여넣기(MCP 금지), 무로그인 insert(수신자 마찰 0) + 빌더 select. → **[사용자 승인 필요 결정]**: "1주차 백엔드 없음" 원칙을 이 한 테이블에 한해 깨는 것.
- **per-slug 서버 OG 메타**(`generateMetadata`): 지금 `/c/[slug]`·`/vs/[slug]`가 use client라 카톡 프리뷰가 빈 페이지 → 클릭 전에 죽는다. **카톡 오픈률 단일 최고 레버.**

## 6. 단계 계획

- **P0**(순수 코드/리팩터): ①votes Supabase 이관(선결) ②per-slug OG ③S5 칩 리라벨(🔥/🙌/👀/💪+소프트 리다이렉트) + 자유입력→프리셋 칩 ④금칙어 스윕 + "응원받으며 오늘 만들기" 카피 ⑤무료 실행 브리프+프롬프트 복사 전면화 ⑥리빌 스펙 = 일회성 완전공개(스크램블 금지) + 실명 옵트인.
- **P1**(최위험 가정 1회 실측 게이트): 빌더 1명이 진짜 카톡망에 카드 1장 손공유 → OG 렌더·오픈률·탭률·"예상 못한 응원자 수" 측정. **응원 5개 미만·서프라이즈 0이면 성인 1인 바이럴+리빌 수익 논제 반증 → 리빌 착수 중단.**
- **P2**(확산·서프라이즈 확인 후에만): 리빌 패스 결제(2차+ 게이트), 🎁 원탭 재공유 확산 씨앗, 데일리 골든 드립.
- **P3+**(4주 밖 보류): 푸시/이메일 알림, 옆 원자셀 복제.
