## 먼저 읽을 것

이 폴더의 다섯 문서는 **동시에 다섯 제품을 만드는 계획이 아니다.** 각각 하나를 선택했을 때 주당 20시간, 실제 구현 15~16시간으로 진행할 수 있는 독립적인 4주 프로젝트 후보다.

문서가 완성됐다는 것과 제품이 검증됐다는 것은 다르다.

- **PRD 완료**: 문제, 타겟, 성공 지표, MVP, 위험 가정, 1주차 실험, 4주 일정이 결정됨
- **제품 검증**: 실제 사용 행동과 결제가 각 문서의 통과 기준을 넘음
- **구현 시작**: 1주차 코드 없는 실험을 통과한 후보만 허용

수집 데이터가 보여주는 것은 시장의 방향과 기존 패턴이다. 실제 결제 의사, 재사용, 자연 공유는 이번 실험으로 직접 확인한다.

## 공통 B2C 원칙

### 1. 소비자가 직접 얻는 결과에 과금한다

- 공개 유튜브 영상이나 공개 장소 정보 열람에 과금하지 않는다.
- 단순 AI 답변 한 번보다 개인화, 자동화, 지속성, 추가 플레이에 과금한다.
- 광고주, 식당, 유튜버용 대시보드는 초기 MVP의 결제 근거로 삼지 않는다.

### 2. 무료 한 바퀴에서 핵심 가치를 전부 경험시킨다

- 가입이나 결제 전에 핵심 결과를 한 번 얻을 수 있어야 한다.
- 결제는 결과를 더 자주, 더 깊게, 더 오래 사용하려는 순간에만 제안한다.
- 실제 결제 전 4주 MVP에서는 가짜 문과 수기 제공으로 지불 의사를 확인한다.

### 3. K-factor를 배포와 혼동하지 않는다

- 공유 버튼, 초대 링크, 결과 카드가 있다는 이유만으로 high-K라고 쓰지 않는다.
- 수신자가 활성화되고 다시 새로운 수신자를 데려오는 연쇄가 계측돼야 K를 주장할 수 있다.
- 민감한 식단·고민 데이터는 기본 비공개이며, 공유는 사용자가 고른 안전한 요약만 허용한다.

### 4. 위험한 영역은 제품 범위 밖에 둔다

- 의료·섭식장애·법률·재무·자해·폭력 대응을 일반 소비자 AI가 대신하지 않는다.
- 미성년자, 민감한 사진, 위치, 사적 고민은 최소 수집·명시적 삭제 원칙을 따른다.
- 유해 출력 하나가 나와도 “AI니까 그럴 수 있음”으로 넘기지 않고 중단 조건으로 다룬다.

## 공통 현실 조건

| 항목 | 기준 |
|---|---|
| 주당 가용 시간 | 20시간 |
| 실제 구현 예산 | 주당 15~16시간 |
| AI·코딩 수준 | Lv.3 |
| 도구 스택 | Codex·Claude Code·Cursor + Next.js·Vercel·Supabase |
| 제품 형태 | 설치 없이 시작 가능한 모바일 퍼스트 웹 |
| 1주차 | 코드 없는 가장 위험한 가정 검증 |
| 2~4주차 | 실험을 통과한 후보 하나만 구현 |

## 포트폴리오 선택 규칙

각 후보의 1주차 실험 결과를 아래 네 축으로 기록한다.

| 축 | 질문 | 가중치 |
|---|---|---:|
| 가치 | 사용자가 실제 과제를 끝냈는가 | 40% |
| 반복 | 며칠 뒤 또는 다음 장면에 다시 왔는가 | 25% |
| 결제 | 말이 아니라 클릭·예약금·선결제가 있었는가 | 25% |
| 구현 | 4주 범위에서 정확도·비용·안전이 감당되는가 | 10% |

선택 조건:

1. 각 PRD의 개별 Kill 기준을 모두 피한다.
2. 결제 행동이 0이면 단순 호감도와 무관하게 우승 후보에서 제외한다.
3. 안전 위험이 발생한 후보는 점수와 무관하게 중단한다.
4. 가장 높은 점수의 후보 하나만 2주차 구현으로 넘어간다.

## 아직 사실이 아닌 것

- 다섯 아이디어 중 어느 것도 현재 수익성이 입증되지 않았다.
- 공개된 가격은 모두 가짜 문 테스트용 가설이며 최종 가격이 아니다.
- 기존 앱의 다운로드·차트·리뷰는 시장 신호이지 이 제품의 수요 증거가 아니다.
- 1주차 실험 참가자 수는 방향 판단용 소규모 표본이며 시장 규모 추정에 쓰지 않는다.
- PRD에 성장 루프가 있어도 실제 연쇄 공유를 측정하기 전에는 K>1이라고 부르지 않는다.
