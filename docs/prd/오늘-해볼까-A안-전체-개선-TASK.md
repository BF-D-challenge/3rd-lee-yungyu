# 오늘 해볼까 — A안 전체 개선 TASK

업데이트: 2026-07-15

상태: 구현 완료·검증 통과

기준: 아이디어 만들기와 받은 응원을 하나의 순환 여정으로 연결한다.

## 1. 목표

사용자가 네 장의 카드로 오늘의 아이디어를 만들고, 친구에게 응원을 요청하고, 도착한 응원을 카드로 확인한 뒤 다시 아이디어 만들기로 돌아오게 한다.

```text
아이디어 만들기
→ 친구에게 응원 요청
→ 친구가 아이디어 확인
→ 응원 작성 및 전송
→ 보낸 사람의 받은 응원에 도착
→ 응원 카드 확인
→ 새 아이디어 만들기
```

## 2. 성공 기준

- [x] 모든 화면이 위 순환 여정 안에서 앞뒤 화면과 연결된다.
- [x] 첫 방문자가 설명 없이 네 장의 카드를 모두 뽑을 수 있다.
- [x] 드래그앤드롭과 빈칸 탭 카드 비행이 모두 유지된다.
- [x] 화면에는 현재 선택할 카드 한 장이 중심으로 보이고, 다음 카드는 옆에서 일부만 보인다.
- [x] 카드 교체는 목록 편집이 아니라 새 카드를 뽑는 방식으로 동작한다.
- [x] 교체할 때 선택하지 않은 세 장은 바뀌지 않고 진행률은 `4 / 4`를 유지한다.
- [x] 네 장 완성 후 색으로 구분된 한 줄 아이디어가 잘리지 않고 표시된다.
- [x] 공유 화면에서 약속한 행동과 친구가 실제로 할 수 있는 행동이 같다.
- [x] 친구가 보낸 응원이 원래 사용자에게 실제 칭찬 카드로 도착한다.
- [x] 동작하지 않는 결제·미리 보기·알림 버튼이 없다.
- [x] 모바일과 데스크톱에서 가로 잘림, 카드 잘림, 텍스트 겹침이 없다.

## 3. 반드시 유지할 규칙

1. 카드는 빈칸으로 드래그해서 놓을 수 있어야 한다.
2. 빈칸을 누르면 카드가 덱에서 빈칸으로 날아가야 한다.
3. 카드와 빈자리의 비율은 같아야 한다.
4. 카드와 빈자리의 `border-radius`는 모두 8px로 맞춘다.
5. 덱과 이동 중인 카드는 화면에서 가장 높은 z축에 있어야 한다.
6. 카드 덱은 부모 컨테이너에 잘리지 않아야 한다.
7. 한 화면에서 네 장을 같은 크기로 경쟁시키지 않는다.
8. 중요한 문장에는 말줄임표를 사용하지 않는다.
9. 카드 교체용 후보 목록·직접 입력 편집 시트는 만들지 않는다.
10. 요청과 관계없는 코드 정리나 구조 변경은 하지 않는다.

## 4. 제외 범위

- 결제와 유료 이름 공개
- `990원에 지금 확인` 가짜 문
- 실제 데이터가 없는 `다음 칭찬 미리 보기`
- 로그인 강제
- 구현되지 않은 푸시 알림 약속
- 실행 중 AI가 새 카드 문구를 생성하는 기능
- 카드 후보 목록과 주관식 카드 편집기
- 이번 여정과 관계없는 과거 듀얼·투표 화면 리팩터링

## T01. 전역 메뉴와 명칭 통일

- `아이디어 뽑기`와 `오늘의 칭찬`이 서로 다른 제품처럼 보인다.

- `오늘의 칭찬`을 `받은 응원`으로 변경한다.
- 새 응원이 있을 때만 받은 응원 메뉴에 개수 배지를 표시한다.

- [x] 선택된 메뉴와 선택되지 않은 메뉴의 대비를 명확히 한다.
- [x] 응원 0개일 때 숫자 `0`을 표시하지 않는다.
- [x] 새 응원 개수는 텍스트가 아닌 배지로 구분한다.
- [x] 키보드와 스크린리더가 선택 상태를 알 수 있게 한다.

### 완료 조건

- [x] 메뉴만 읽어도 `만들기 → 받은 결과 확인` 관계를 이해할 수 있다.
- [x] 메뉴의 터치 영역이 최소 48×48px다.

## T04. 아이디어 시작 화면 정리

### AS-IS

- `오늘은 뭘 만들어볼까요?`만으로 네 장을 왜 뽑는지 알기 어렵다.
- 카드, 빈자리, 진행률, 덱이 동시에 시선을 가져간다.

### TO-BE

- 제목 아래에 네 장을 뽑는 이유를 한 줄로 설명한다.
- 첫 번째 카드와 첫 번째 빈자리를 가장 강하게 보여준다.
- 하단 고스트 박스로 조작 방법을 크게 안내한다.

### 목표 문구

```text
오늘은 뭘 만들어볼까요?
네 장의 카드를 조합해 오늘 시험할 아이디어를 만들어보세요.
```

```text
카드를 끌어 이곳에 놓거나
빈칸을 눌러 카드를 뽑아보세요.
```

### 구현 체크리스트

- [x] 시작 설명을 제목 바로 아래에 배치한다.
- [x] 진행률 `0 / 4`를 제목과 같은 행의 오른쪽에 배치한다.
- [x] 현재 빈자리의 라벨을 카드 가운데 위에 배치한다.
- [x] 고스트 박스 전체를 빈칸 탭 영역으로 사용한다.
- [x] 다른 빈자리는 현재 빈자리보다 약하게 표현한다.

### 완료 조건

- [x] 첫 방문자가 드래그 또는 빈칸 탭 중 하나로 시작할 수 있다.
- [x] 시작 화면의 대표 행동이 하나의 영역으로 보인다.

## T05. 한 장 집중 카드 뽑기

### AS-IS

- 네 장의 빈자리가 같은 무게로 보이면 지금 선택할 대상을 찾는 데 시간이 걸린다.
- 덱의 카드와 빈자리가 시각적으로 경쟁한다.

### TO-BE

- 현재 카드 한 장을 화면 중앙에 원본 비율로 크게 보여준다.
- 이전·다음 카드는 양옆에서 일부만 보이게 한다.
- 뽑는 동안 옆 카드는 다음 순서를 알려주는 단서로만 사용하고, 완성 후에는 이전·다음 카드를 탐색할 수 있게 한다.

### 구현 체크리스트

- [x] 활성 카드만 완전히 보이는 캐러셀 구조를 사용한다.
- [x] 옆 카드는 40~48px 정도만 보이게 한다.
- [x] 뽑는 동안 활성 카드만 드래그와 키보드 조작을 허용한다.
- [x] 뽑는 동안 옆 카드는 `aria-hidden`과 비활성 탭 순서를 적용한다.
- [x] 네 장 완성 후 옆 카드는 접근 가능한 이전·다음 탐색 버튼으로 전환한다.
- [x] 카드 원본 비율을 변경하지 않는다.
- [x] 덱 컨테이너의 `overflow` 때문에 카드가 잘리지 않게 한다.

### 완료 조건

- [x] 사용자가 지금 뽑을 카드 하나를 즉시 구분한다.
- [x] 390px 너비에서도 활성 카드와 옆 카드 단서가 함께 보인다.

## T06. 카드 도착과 다음 단계 전환

### TO-BE


- [x] 드래그 카드의 z축을 전체 카드 영역보다 높게 둔다.
- [x] 카드 도착 후에만 선택 값을 확정한다.
- [x] 도착한 빈자리에 짧은 강조 모션을 적용한다.
- [x] 진행률과 활성 축을 같은 상태 변경에서 갱신한다.
- [x] 다음 활성 카드가 오른쪽에서 들어오는 모션을 적용한다.
- [x] `prefers-reduced-motion`에서는 이동 대신 즉시 상태를 변경한다.

### 완료 조건

- [x] 카드가 잘리거나 빈자리 위에 떠 있는 상태로 남지 않는다.
- [x] 드래그와 빈칸 탭이 같은 완료 상태를 만든다.

## T07. 네 장 완성과 한 줄 아이디어

### AS-IS

- 완성된 한 줄 아이디어가 작거나 말줄임표로 잘릴 수 있다.
- 카드 네 장과 결과 문장의 중요도가 비슷하게 보인다.

## T21. 네 장 완성 후 결과 이해·친구 반응 재설계

### AS-IS

- 네 장을 모두 골라도 `원본 × 결제자 × 순간 × 한 끗` 조합명이 먼저 보여 제품을 단번에 이해하기 어렵다.
- 결과 화면의 UVP, 타겟, 차별점, 플로우가 좋은 반응을 받은 소개 문장 구조보다 추상적이다.
- 공유받은 친구는 막연한 응원만 보낼 수 있어 아이디어 개선에 필요한 정성 피드백이 부족하다.

### TO-BE

- 결과 첫 화면만 읽어도 `문제 순간 → 한 문장 UVP → 타겟 → 기존 앱과 차이 → 전체 플로우`가 순서대로 보인다.
- UVP는 경쟁 앱 이름을 억지로 붙이지 않고 `현실적인 입력 1개 → 처리 1회 → 결과 1개`를 한 문장으로 보여준다.
- 사용자는 `친구에게 알리고 시작하기`로 공유하고, 친구는 이해도·사용 의향·차별성에 대한 반응을 보낸다.
- 친구가 반응을 보낸 뒤 `나도 아이디어 만들기`로 유입되는 순환 구조는 유지한다.

### 구현 체크리스트

- [x] 결과 제목을 `오늘 만들 제품`으로 바꾼다.
- [x] 문제 순간과 한 문장 UVP를 분리해 짧게 표시한다.
- [x] 타겟, 기존에 잘되는 앱, 차별점, 검증 근거를 한 화면에 배치한다.
- [x] 원본의 검증된 플로우와 한 끗 결과를 번호 순서로 보여준다.
- [x] 네 장 원본 데이터는 접힌 상세 영역에서 확인할 수 있게 한다.
- [x] 대표 행동을 `친구에게 알리고 시작하기`로 바꾼다.
- [x] 공유 데이터에 전체 플로우를 추가하고 기존 공유 링크와 호환한다.
- [x] 수신자 반응을 `바로 이해`, `써보고 싶음`, `차별점 보완`으로 구체화한다.
- [x] 직접 작성 의견과 익명·이름 공개 선택을 유지한다.
- [x] 수신자의 `나도 아이디어 만들기` 재유입 행동을 유지한다.

### 완료 조건
