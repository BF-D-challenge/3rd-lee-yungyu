## W1. 직결과 플로우 — "확정" 단계 제거 (사용자 지시)

**문제**: 스핀 결과(4칸+황금문장 박스) → "✓ 확정" → 카드 오버레이, 2단 결과가 어색하고 한 단계 김. 독립 리뷰도 "확정 다음에 무슨 일이 일어나는지 모름"(HIGH).

**설계**:
1. **스핀 완료 → 아이디어 카드(flip) 자동 등장.** `spinAll` 애니메이션 종료 콜백에서 ConfirmBranch를 바로 연다. "✓ 확정" 버튼·`아이디어 완성!` 배너·**황금문장 박스(gold-sentence) 제거** — 문장은 카드 제목 폴백(`line` 프롭, 이미 배선됨)으로만 산다.
2. **카드에서 바로 다시 뽑기.** 카드 오버레이 하단에 `🎲 다시 뽑기 (n/5)` 추가 — 누르면 카드가 닫히며 재스핀→새 카드. **🔒 고정한 축은 유지**(기존 keepSeed/고정 로직 재사용). 캡 소진 시 기존 페이월 시트가 카드 위에서 뜬다.
3. **축 미세조정 탈출구 유지**: 카드의 `✕`(닫기) = 판으로 복귀. 판에서는 기존처럼 축별 🔒/✕/재탭 조작 후 재스핀 가능 — "고정+리믹스" 루프(v7 툴킷 규칙)는 판에 그대로 산다.
4. 첫 CTA 단일화(UX P1-1): 빈 판의 시작 버튼을 **"오늘 아이디어 뽑기"** 하나로. "🎲 전체 다시 뽑기"는 첫 스핀 전엔 숨김(독립 리뷰: "아직 안 뽑았는데 왜 '다시'?"). '씨앗' 라벨에 행동 언어 병기, 진행 표기는 스핀 카운트(n/5) 하나만 노출.

**파일**: slot-machine.tsx, slot-store.ts(스핀 종료 콜백), confirm-branch.tsx, gold-sentence.tsx(제거), copy.ts, slot-cell.tsx(캡션).
**DoD**: 스핀→카드 등장까지 추가 탭 0회. 다시 뽑기 시 고정 축 보존 실측. 첫 화면 CTA 1개.

## W2. 발행 여정 봉합 + 공유 실패 정직화 (UX 테스트 P0 × Gas ⑤마찰 제로)

1. **발행 성공 상태 명시**: 발행(finishAuth) 완료 화면에 "발행 완료 — 링크를 보내면 투표가 시작돼요" 상태 문구+체크 표시(수신자 T3의 "전달됐어요 ✓" 문법을 발행자 쪽에 이식).
2. **OAuth 복귀 연속성**: 이미 sessionStorage 복원이 있으나(UX 테스트에서 확인) **복귀 직후 "방금 카드 발행 완료" 상태가 화면에 보이게** — 로그인 버튼이 다시 보이는 일 없도록 로딩 상태 추가.
3. **대시보드 "발행 대기 중" 카드**: 대시보드 진입 시 sessionStorage의 확정본(`oneul:confirmed`)이 발행 목록에 없으면 **"발행 중이던 카드" 행으로 표시 + [이어서 발행] CTA** — "아직 카드가 없어요" 단독 노출 금지(3/3 합의 해소).
4. **공유 게이트 침묵 실패 수정**: [idea-flip-card.tsx:70](../../src/components/organisms/slot/idea-flip-card.tsx#L70) `onShare()` 실패 시 ① share-row의 `copyText`(execCommand 폴백) 재시도 ② 그래도 실패면 "링크 복사가 막혔어요 — 주소를 길게 눌러 복사해주세요" 토스트+URL 노출. 성공 시에도 "링크를 복사했어요 ✓" 확인 토스트(현재 무피드백).
5. **공유 문구 질문화**(Gas ①): `"…뽑았어. 어때?"` → `"…뽑았어. 너라면 쓸 것 같아?"` — 수신자 자기관련 질문으로.

**파일**: publish-flow.tsx, demand-board.tsx, idea-flip-card.tsx, share-row.tsx, share.ts, copy.ts.
**DoD**: 클립보드 차단 환경(헤드리스 재현 스크립트 보유)에서 공유 버튼 무반응 0. 확정 직후 대시보드 진입 시 카드 행 표시.

## W3. 타로 앞면 비주얼 — 아트 12장 + 프레임 규약 + 이미지 공유 (사용자 재확정 반영)

**v8 원칙 2 개정**: "모든 카드면 이미지 0" → **"뒷면·판 위 축 카드는 잉크 유지, 아이디어/공유 카드 앞면에만 앵커 아트 12장 허용"**. 뒷면(backSvg 로제트)은 **무변경**(사용자 확정).

1. **아트 12장 생성** (134장 아님 — 세트 일관성 리서치 결론):
   - 축 아르카나 4장(씨앗·불편·형태·장면) + 취향 카테고리 8장.
   - 스타일 앵커 = 승인된 "씨앗(THE SEED)" 샘플(적국화×학·진홍#7A1F1F·금#C9A962·학백, 하단 캡션 밴드 공백). 5필드 프롬프트로 codex image_gen 생성, 앵커 이미지 첨부로 일관성 고정.
   - 산출: `public/cards/*.webp`(카드폭 2x ≈ 600px, 장당 ≤80KB, 총 ≤1MB 예산). 원본 PNG는 `docs/design/tarot-deck/`에 보관.
2. **프레임 규약(코드 조판)**: 아이디어 카드에 이중 금선 보더 + 상단 아르카나 번호(Ⅰ씨앗~Ⅳ장면 또는 카테고리 문장) + **하단 캡션 밴드에 appName·카드명** 렌더. 플립 물리(300:485, 720ms)는 무변경.
3. **카드 이미지 공유**(Gas ③ "메인 콘텐츠가 곧 공유물"): 확정 카드(아트+캡션 조판)를 **PNG로 저장/공유하는 실기능** — share-row의 "📸 스토리" 버튼을 실동작으로(html-to-image 캔버스 캡처 → Web Share API files / 다운로드 폴백). 카톡 프사·인스타 스토리용 세로 포맷.
4. **UI 무대는 현행 다크 유지** — 진홍·금은 카드 안에만. 전역 토큰 변경 없음.

**파일**: card-face.tsx, idea-flip-card.tsx, publish-card.tsx, share-row.tsx, public/cards/(신규), 생성 스크립트(scratchpad).
**DoD**: 4화면(판·카드·발행·수신자)에서 덱 일관성 육안 확인, 카드 이미지 저장 실측(모바일 공유시트 포함), 카드 렌더 LCP 회귀 없음(webp lazy).
