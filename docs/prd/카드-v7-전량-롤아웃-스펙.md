슬롯의 **모든 유효 조합(seed × pain × format) = 1,649개**에 대해 "v7 품질" 골든 카드를 생성해

카드 품질 원칙은 확정됨: **생성모델=Claude Sonnet 5, 린 프롬프트 + 결정론 검증기(규칙 쌓기는 진동함). 실명·숫자·시나리오를 oneliner 맨 앞에.**

## "1,649"의 정확한 출처 (하드코딩 말고 코드로 재계산)
`combos.json.allow`는 seed → {pains:[id], formats:[id]}. 총 조합수 = Σ_seed (pains.length × formats.length).
```js
const d=require("./src/data/combos.json");
let total=0; for(const s of Object.keys(d.allow)){const a=d.allow[s]; total+=(a.pains.length)*(a.formats.length);}
// total === 1649, allow seed 수 === 58
```
이미 생성된 v7 = `d.golden.filter(g=>g.source==="v7")` (40개). 대상 = allow의 모든 (seed,pain,format) 중 v7로 아직 없는 것.

## 조합별 입력 조립 (생성 프롬프트에 넣을 재료)
combos.json에서 join:
- **seed 한글 라벨**: `tracks[track].categories[].seeds[]`(36개) + `presetSeeds[]`(10개)로 46개 커버.
  라벨 없는 12개는 이 표 사용: `escape-room=방탈출, dividend-etf=배당ETF, plant-care=반려식물, knitting=뜨개질, freelance-tax=프리랜서 세금, cafe-ops=카페운영, ux-ui=UX/UI, real-estate-agent=부동산중개, tutoring=과외, nursing-handoff=간호 인수인계, insurance-review=보험 보장분석, import-duty=수입 관세`
- **pain**: `pains[]`에서 id로 → `label`(긴 설명)·`short`(짧은 태그)
- **format**: `formats[]`에서 id로 → `label`·`desc`·`buildDays`·`action` (todayAction/buildPrompt는 이 format 능력에 맞춰)
- **track**: `allow`의 seed가 like/know 어디 소속인지는 `tracks`에서 역참조

## evidence(근거) 출처 = 실매출 앵커 데이터셋
`docs/research/trustmrr-acquire/ideas.jsonl` — 실제 앱 1,863개. 행 필드: `name, category, raw_description, revenue_30d_value(USD, 월매출), problem, current_alternative, mvp_angle, target_user, tags`.
- 이 조합의 seed×pain×format과 **의미가 맞는 앵커 1개**를 고른다(카테고리·problem 유사도). 없으면 인접 카테고리 폴백.
- **evidence 문장**: `"[유형 설명] 비슷한 해외 앱이 매달 약 {X}만원씩 벌어요"`. X = 앵커 `revenue_30d_value`(USD) × 약 1,350원 → 만원 단위 반올림.
- `evidenceType`: 매출 자체를 근거로 쓰면 `"revenue"`, 사용/수요 패턴이면 `"usage"`.
- `anchorName`: 앵커 실명(또는 한국인이 아는 동급 브랜드로 치환 — 예 러닝→Nike Run Club). `anchorDetail`: 1줄(순위·수요 맥락).
- ⚠️ 90/10 규칙: 검증 메커니즘·과금·사용 패턴은 앵커 그대로(90), 타깃·상황·소재만 이 조합의 한국 니치로 비틀기(10). **새 메커니즘 발명 금지.**

## 카피 레시피 (승자 = H1 생생함 + v4~v6 규칙 + 쉬운말) — 생성 프롬프트에 필수 주입
1. **생생한 구체성을 oneliner 맨 앞에**: 실명·숫자·시나리오("엄마 카톡 잡채 레시피", "나이키 런 클럽 47일")를 그대로. 일반화·target 뒤로 빼기 금지.
2. **뉴로마케팅 통증→이점 대조**를 oneliner에 구조적으로: [감정 실린 통증] → [감정 실린 해소]. (예 "…민망함, …바로 찾아 뿌듯")
3. **한국 브랜드 현지화**: 낯선 외래어 브랜드는 한국 대중 브랜드로(러닝→나이키 런 클럽, 지도→네이버·카카오맵, 배달→배민, 중고→당근, 레시피→만개의레시피). 해외 앵커 실명은 evidence 줄에만.
4. **도구 가시화**: oneliner에 뭘 만드는 도구인지(타이머·트래커·투표·계산기 등)가 보여야.
5. **페르소나 정합**: 상황이 정체성을 규정(포트폴리오 쓰는 사람=취준생/신입, 재직 주니어 아님). frontStory.persona와 target 일치.
6. **반응/비교형은 구체적 판단기준**: 막연한 "어때요?" 금지("프로 자세와 프레임별 비교" 식).
7. **쉬운말·약어 금지**: PB/PR/KPI/MAU/스트릭 등 축약어·전문용어 금지, 무조건 우리말로 풀어씀("스트릭"→"며칠 연속 기록"). 단 브랜드·앱 실명은 허용. $ 표기 금지(원으로).
8. **없는 도구 지어내기 금지**(러너가 '메모장' 쓴다 등). situation/psych는 seed×pain 실사용과 정합할 때만 녹이고 어색하면 생략.
9. **title = `[appName] — [기능+키워드]`**(스토어 SEO). appName은 짧은 한국어 브랜드명.
10. **frontStory**: persona 1줄 + timeline 3스텝(각 t/act/emo/pain), 마지막 2스텝은 pain:true(고통 고조).

## 결정론 검증기 (코드로 O/X, 위반 시 그 규칙만 콕 집어 최대 2회 재생성)
- 길이 하드컷: `title ≤ 26`(이름+기능이라 넉넉) · `oneliner ≤ 70` · `target ≤ 50` · `mvp 각 ≤ 22`
- `mvp` 정확히 4개 · 각 항목 서로 다름(verbatim 복제 금지)
- `evidence`에 실명 앵커 + 수치("약 N만원") 포함 · `$`·달러 표기 0건
- `todayAction`은 "오늘 반나절"로 시작 · `buildPrompt` 비어있지 않음 · `appName` 존재
- 약어 블랙리스트 정규식 미검출: `\b(PB|PR|KPI|MAU|DAU|ROI|CTR|CRUD|API)\b`, "스트릭"
- `frontStory.timeline` 길이 3 · `pain:true` ≥ 2 · `persona`가 `target`과 모순 없음(간이 체크)
- `(seed,pain,format)`이 `allow[seed]` 안에 실제로 있는지(무효 조합 생성 방지)

## 파이프라인 (Workflow, 규모 큼 → 배치 + 체크포인트)
1. **사전 스크립트(Node, FS 가능)**: allow 전개 → 1,649 워크리스트 생성, 이미 있는 v7 제외 → 남은 대상 + 각 조합의 조립 재료(seed라벨·pain·format·후보 앵커 3~5개)를 `scratchpad/rollout/worklist.json`에 저장. ⚠️ **Workflow 에이전트는 FS 접근 불가** → 청크(예 20~30행)마다 재료를 **args/프롬프트에 임베드**해 전달.
2. **Workflow**: `pipeline(chunks, generate, validate→regenerate)`. 행별 = Sonnet 생성 → 결정론 검증 → 실패 규칙만 재생성(≤2). 결과는 구조화 스키마로 반환.
3. **체크포인트**: 청크 결과를 `scratchpad/rollout/done.jsonl`에 append(재개 가능). 1,609행은 크니 배치로 나눠 여러 번 돌려도 됨.
4. **조립 스크립트(Node)**: done.jsonl → golden에 upsert(키 `seed|pain|format`, dedupe, `source:"v7"` 유지, 구 골든 덮기). `JSON.stringify(indent=2)`로 재직렬화.
5. **검증**: `npx tsc --noEmit` + `npm run build` 통과. `allow`의 모든 조합이 이제 `source:"v7"` 골든을 갖는지 assert(누락 0). 랜덤 N개를 실제 슬롯 플로우에서 렌더 확인(control-in-app-browser).
