### 3-2. Gas 벤치마크 비교 (리서치 코퍼스 [gas-app](../gas-app/SUMMARY.md) · [gap-5](../gas-app/gaps/gap-5-K팩터-심리학-근본원인.md) 기준)

| ⑤ 마찰 제로 인프라 | friendfinder에 리소스 90%, 자동완성 SMS, 학교 딥링크 | OG 미리보기 ✓ 수신자 무로그인 ✓ / 공유 실패 무피드백 ✗ | **부분** |

Gas UX 디테일 대비 (FLOWS.md 기준):

- **온보딩 사회적 증거**: Gas는 가입 전에 학교별 회원 수 뱃지("824 members")로 "이미 모였다"를 보여줌. 우리는 취향 시트만 있고 사회적 증거 0 — 초기엔 불가피하나, 수신자→발행 전환 화면("나도 뽑아보기" 랜딩)에 "지수님 카드에 N명이 응원했어요" 같은 증거를 실을 자리가 있음.
- **쿨다운+우회 루프**: Gas의 "60분 잠금 → 초대로 스킵"과 우리의 "스핀 5회 캡 → 공유로 +3회"는 동형. 차이는 우리가 카운트다운·강제 없이 구현했다는 것(다크패턴 체크리스트 통과) — 원칙 준수 확인.
- **리텐션 트리거**: Gas의 심장은 '불꽃' 푸시였다. 우리는 웹이라 대응물이 없음 — 투표 도착 시 알림(이메일/카카오 알림톡) 채널이 없으면 ①의 호기심 루프가 1회성으로 끝남(gap-3의 "응답 다 받은 뒤 돌아올 이유" 문제).
- **Gas의 실패에서 잘 피한 것**: 신원 공개 유료화 없음(NGL FTC 선례 회피), 매직링크 미사용(시간당 2통 자기파괴 회피), 투표 조작 상품 없음(Gas 코인 상점 반면교사). — 기존 점검에서 확인된 사항 재확인.

### 3-3. 걸린 설계 원칙

- 발견 #1·#2: **피드백 부재 + 시스템 상태 비가시성** — 되돌리기 어려운 행동(발행) 직후 결과가 안 보임.
- 발견 #4: 첫 화면의 **행동 언어 부재**(씨앗=시스템 언어) + 진행 지표 이중화.
- 공유 게이트 무피드백: **에러 상태의 침묵** — 실패를 사용자에게 알리지 않음.

## 4. 결론 + 개선 액션

**한 줄 결론: 수신자 루프(K의 심장)는 이미 강하다. 지금 부러져 있는 것은 발행자 루프의 마지막 20cm — "발행했다"는 확신을 주는 피드백이다.**

### 액션 아이템

- 🔴 **P0-1. 발행 여정 봉합** (3/3 + 독립): 발행 확정 직후 성공 상태를 명시("발행 완료 — 링크를 친구에게 보내세요" + 공유 CTA). OAuth 복귀 시 "방금 카드 이어가기" 상태 복원. 대시보드 빈 상태에서 발행 중이던 카드(sessionStorage 확정본)를 "발행 대기 중" 카드로 표시.
- 🔴 **P0-2. 공유 게이트 무피드백 수정** (실측+코드): `onShare` 실패 시 execCommand 폴백([share-row.tsx의 copyText](../../../src/components/organisms/journey/share-row.tsx) 재사용) + 실패 토스트. 성공 시에도 "링크를 복사했어요" 확인 노출.
- 🟠 **P1-1. 홈 3초 이해**: 첫 방문 CTA를 "오늘 아이디어 뽑기" 하나로, '씨앗'에 행동 언어 병기, 진행 표기(0/4·0/5) 화면당 하나로.
- 🟠 **P1-2. 확정 버튼에 다음 행동 예고**: "✓ 확정" → "확정하고 친구에게 물어보기".
