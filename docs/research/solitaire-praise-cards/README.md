# 솔리테어 문법을 익명 칭찬 카드로 번역하기

이 문서는 특정 솔리테어 게임을 닮게 만드는 아트 가이드가 아니다. 여러 솔리테어 제품에서 반복되는 **일반적인 시각 문법**만 추출해, 실제로 받은 익명 칭찬을 하루 한 장씩 공개하는 모바일 화면에 적용한다.

핵심은 다음 한 문장이다.

> 어두운 테이블 위에 오늘의 카드 한 장을 가장 선명하게 놓고, 뒤집어서 공개하며, 다음 카드는 내용을 숨긴 채 가장자리만 보여준다.

## 조사 범위와 근거

먼저 로컬 `docs/research`를 검색했다. 기존 수집물에는 Chrome Web Store의 Microsoft Solitaire Collection과 Scorpion Solitaire 목록만 있어, 카드 앞·뒷면과 모바일 상호작용을 판단하기에는 부족했다. 그래서 공식 제품 페이지와 공식 스토어 스크린샷을 보충 조사했다.

로컬 [Gas 종합 리서치](../gas-app/SUMMARY.md)도 함께 참고했다. Gas의 시간 잠금과 초대 스킵은 강한 재방문·초대 장치였지만, 익명 칭찬 카드에서는 이를 그대로 복제하지 않는다. **다음 카드 잠금은 공개 시각을 설명하는 상태**여야 하며, 강제 초대나 보낸 사람 정보 노출을 미끼로 삼지 않는다.

## 솔리테어 요소와 칭찬 카드의 대응

| 솔리테어 문법 | 익명 칭찬 카드에서의 의미 | 가져오지 않을 것 |
|---|---|---|
| 어둡고 넓은 테이블 | 카드가 주인공이 되는 안전하고 조용한 무대 | 특정 게임의 펠트 사진·나무 테두리 |
| face-down stock | 아직 공개되지 않은 실제 칭찬 큐 | 가짜 카드 수, 랜덤 보상 상자 |
| 오늘 한 장 뽑기 | 오늘 배정된 칭찬 한 장만 활성화 | 무한 리롤, 여러 장을 빠르게 소모하는 슬롯 감각 |
| 카드 뒤집기 | 익명 칭찬 문구를 공개하는 한 번의 핵심 행동 | 승패, 점수, 폭죽, 코인 보상 |
| 뒤에서 보이는 다음 카드 | 내일 이어질 실제 칭찬이 있다는 예고 | 잠긴 문구·보낸 사람 데이터의 미리 렌더링 |
| 앞면과 뒷면의 강한 대비 | `비공개 → 공개` 상태를 즉시 인지 | 특정 수트, 숫자, 킹·퀸 일러스트 |
| 카드 모서리 인덱스 | 오늘 카드임을 알리는 작은 상태 표식 | `A`, `K`, `♥`, `♠`의 그대로 사용 |

## 1. 초록·짙은 테이블 배경

솔리테어의 초록색은 장식이 아니라 흰 카드의 실루엣을 분리하는 넓은 음영 면이다. 칭찬 카드에서도 전체 화면을 어두운 청록 계열로 두고, 카드 주변만 아주 약하게 밝힌다.

권장 시작 토큰은 아래와 같다. 레퍼런스 제품에서 추출한 색이 아니라 이 제품을 위한 새 팔레트다.

```css
--praise-table-950: #061815;
--praise-table-900: #0a2722;
--praise-table-800: #123a32;
--praise-paper: #f7f1e7;
--praise-ink: #17201e;
--praise-back: #26334f;
--praise-back-line: rgb(220 229 255 / 16%);
```

- 화면 가장자리는 `--praise-table-950`, 카드 뒤는 `--praise-table-900` 정도로 두어 깊이를 만든다.
- 펠트 사진, 노이즈 이미지, 나무 프레임을 복사하지 않는다. 필요하면 2~3% 투명도의 자체 생성 노이즈나 부드러운 방사형 명암만 사용한다.
- 초록색만으로 솔리테어를 흉내 내지 않는다. 칩·금화·잭팟·과한 금색을 제거해 카지노 문법과 거리를 둔다.
- 모바일에서는 테이블이 `100dvh`를 채우고, 하단에는 `env(safe-area-inset-bottom)`을 포함한다.

## 2. 카드 스택과 깊이

한 화면에서 실제 데이터 카드의 역할은 두 개뿐이다.

1. `today`: 사용자가 지금 뒤집을 수 있는 최상단 카드
2. `next`: 공개 시각 전까지 내용을 알 수 없는 뒷면 카드

오늘 카드는 항상 `z-index`가 가장 높고 정면을 향한다. 다음 카드는 뒤에서 세로로 `16~24px`만 보이게 내려 놓는다. 이 정도면 “한 장 더 있다”는 사실은 전달하면서 문구가 읽히는 것을 막을 수 있다.

- 장식용 가짜 덱을 여러 장 쌓아 실제 큐가 많은 것처럼 보이지 않는다.
- 다음 카드가 실제로 없으면 뒷면 예고도 렌더하지 않는다.
- 다음 카드에는 메시지, 작성자, 카테고리를 DOM·접근성 트리·클라이언트 속성에 넣지 않는다. 필요한 값은 `availableAt`과 큐 위치뿐이다.
- 그림자는 카드마다 세게 중첩하지 않는다. 최상단 카드에만 명확한 그림자를 주고 뒤 카드는 얕게 처리한다.

권장 모바일 크기:

```css
.praise-card {
  width: min(78vw, 300px);
  aspect-ratio: 5 / 7;
  border-radius: 16px;
  box-shadow:
    0 1px 0 rgb(255 255 255 / 18%) inset,
    0 14px 36px rgb(0 0 0 / 28%);
}
```

390×844 화면에서 약 300×420px가 되어 제목, 상태 문구, 하단 행동을 함께 배치할 수 있다. 태블릿·데스크톱에서도 카드를 과도하게 키우지 말고 주변 여백만 늘린다.

## 3. 앞면과 뒷면 대비

앞면은 “읽는 면”, 뒷면은 “기다리는 면”이다. 색뿐 아니라 정보 밀도와 구조도 달라야 한다.

### 앞면

- 따뜻한 아이보리 바탕과 짙은 본문색을 사용한다.
- 중앙에는 칭찬 문구를 가장 크게 두고, 하단에는 `익명`과 도착일만 작게 둔다.
- 모서리 인덱스는 `TODAY`, 날짜 숫자, 또는 자체 별표 엠블럼으로 바꾼다.
- 실제 칭찬 분류가 없다면 하트·스페이드처럼 보이는 가짜 카테고리를 만들지 않는다.

### 뒷면

- 짙은 남색·청록처럼 앞면과 명도가 확실히 다른 색을 쓴다.
- 자체 브랜드 도형, 점, 선으로 만든 낮은 대비의 반복 패턴을 사용한다.
- 문양 중앙에는 하나의 추상 엠블럼만 둔다. 기존 카드 덱의 격자, 로제트, 수트 조합을 따라 그리지 않는다.
- “탭해서 오늘의 칭찬 열기”처럼 행동을 직접 설명한다.

잠긴 다음 카드는 오늘 카드의 뒷면과 같은 계열을 쓰되, 자물쇠 아이콘과 `내일 00:00 공개`를 함께 제공한다. 잠김을 불투명도나 색 하나로만 표현하지 않는다.

## 4. 한 장 뒤집기

뒤집기는 이 화면의 유일한 주인공 동작이다. 드래그, 스와이프, 랜덤 셔플을 동시에 요구하지 않는다.

권장 모션:

- 탭 후 `460~600ms` 동안 `rotateY(180deg)`로 한 번만 뒤집는다.
- 중간 지점에서 앞·뒷면이 교체되도록 `backface-visibility: hidden`을 쓴다.
- 뒤집는 동안 중복 입력을 막고, 다음 카드는 움직이지 않는다.
- 과한 튕김이나 여러 바퀴 회전 대신 작은 `scale(.98 → 1)` 정도만 보조로 사용한다.
- `prefers-reduced-motion: reduce`에서는 회전을 없애고 즉시 전환하거나 `120ms` 이하의 페이드로 바꾼다.
- 뒤집힌 뒤 포커스를 강제로 옮기지 말고, 버튼의 `aria-label`을 “오늘의 칭찬 다시 뒷면으로 보기”처럼 상태에 맞게 갱신한다.

## 5. 다음 카드 일부 노출과 잠금

다음 카드 예고는 호기심을 만들되 내용을 흘리지 않아야 한다.

- 현재 카드 아래쪽 또는 옆쪽으로 뒷면의 `12~18%`만 보인다.
- 보이는 영역에는 반복 패턴과 잠금 표식만 둔다. 문구의 첫 줄, 보낸 사람 이니셜, 카테고리 색은 노출하지 않는다.
- 잠긴 카드를 탭할 수 있게 한다면 “내일 공개” 안내 시트만 연다. 초대, 결제, 광고 시청으로 즉시 해제되는 것처럼 약속하지 않는다.
- 공개 시각은 모호한 카운트다운보다 현지 날짜 기준의 `내일 00:00 공개`처럼 설명한다.
- 큐가 비었으면 “다음 카드가 없어요” 상태로 바꾸고, 존재하지 않는 뒷면을 남기지 않는다.

상태별 최소 표현은 다음과 같다.

| 상태 | 오늘 카드 | 다음 카드 | 주 행동 |
|---|---|---|---|
| `empty` | 없음 | 없음 | 칭찬 요청 링크 공유 |
| `today-unopened` | 뒷면, 활성 | 뒷면 일부, 잠김 | 오늘 카드 열기 |
| `today-opened` | 앞면, 칭찬 표시 | 뒷면 일부, 잠김 | 다시 보기 또는 공유 |
| `waiting` | 오늘 배정 없음 | 다음 공개일만 표시 | 공개일 안내 |
| `caught-up` | 마지막 공개 카드 또는 기록 | 없음 | 새 칭찬 요청 |

## 6. 카드 모서리·그림자·문양 번역 규칙

| 카드 게임 장식 | 칭찬 카드 번역 |
|---|---|
| 좌상단 숫자와 수트 | `TODAY`, 날짜, 자체 별표 중 하나 |
| 우하단의 뒤집힌 인덱스 | 작은 `익명` 표식 또는 생략 |
| 중앙의 인물·수트 일러스트 | 칭찬 문장과 충분한 여백 |
| 파랑·빨강 카드 뒷면 격자 | 브랜드 원형·점·선으로 만든 새 기하 패턴 |
| 테이블 위 강한 낙하 그림자 | 카드가 들렸다는 정도의 부드러운 2단 그림자 |
| 승리 폭죽과 카드 비행 | 공개 순간의 짧은 빛 변화 또는 1회 뒤집기 |

모서리는 `14~18px` 범위에서 제품의 다른 카드와 맞추고, 외곽선은 밝은 1px 키라인으로 종이 가장자리만 분리한다. 그림자는 검정 한 덩어리보다 가까운 접촉 그림자와 넓은 주변 그림자를 나눈다.

## 7. 모바일 퍼스트 배치

390×844을 먼저 맞춘다.

```text
┌──────────────────────────┐
│ 오늘의 익명 칭찬   1장   │
│                          │
│        ┌────────┐        │
│        │ TODAY  │        │
│        │        │        │  ← 오늘 카드: 탭 가능
│        │        │        │
│        └────────┘        │
│         └───────┘ 🔒     │  ← 다음 카드: 일부만 노출
│      내일 00:00 공개      │
│                          │
│   [ 오늘의 칭찬 열기 ]    │
└──────────────────────────┘
```

- 좌우 안전 여백은 최소 `16px`, 모든 탭 영역은 최소 `44×44px`로 둔다.
- 카드가 작아지는 짧은 화면에서는 본문을 줄이기보다 장식 헤더와 주변 여백부터 줄인다.
- 호버 상태에 핵심 정보를 두지 않는다.
- 뒤집기 버튼은 카드 전체를 사용하되, 다음 카드와 행동 영역이 겹치지 않게 한다.
- 긴 칭찬 문구는 최대 줄 수로 잘라 의미를 훼손하지 말고, 앞면 내부를 스크롤하거나 상세 보기로 연결한다.

## 복제 금지선

가져와도 되는 것은 **배치 원리, 상태 대비, 한 장씩 공개하는 리듬, 겹침으로 다음 장을 예고하는 방식**이다.

다음은 가져오지 않는다.

- Microsoft, MobilityWare 또는 다른 게임의 로고, 정확한 색 조합, 타이포그래피
- 기존 덱의 카드 뒷면 패턴, 수트 모양, 숫자·인물 카드, 테이블 텍스처
- 스크린샷의 프로모션 프레임, 불꽃놀이, 왕관, 트로피, 코인 장식
- 정확한 카드 배치 좌표나 동일한 모션 타이밍의 복제
- 슬롯·가챠처럼 보이는 무한 뽑기, 확률 보상, 긴급 카운트다운
- 잠긴 칭찬 문구나 영구 익명을 선택한 보낸 사람 정보를 결제·초대로 해제한다는 표현

최종 판정 기준은 간단하다. 스크린샷을 나란히 놓았을 때 특정 게임이 바로 떠오르면 너무 많이 가져온 것이다. 반대로 **어두운 무대, 종이 카드, 뒷면에서 앞면으로의 한 번의 전환, 다음 장의 작은 예고**만 남으면 적절한 추상화다.
