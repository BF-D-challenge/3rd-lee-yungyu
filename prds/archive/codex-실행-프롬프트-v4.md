너는 이 레포에서 "오늘 해볼까" v4 다크 리뉴얼을 구현한다. 계획서가 이미 완성돼 있으니 새로 판단하지 말고 계획서를 그대로 실행하라.

## 반드시 먼저 읽을 파일 (순서대로)

1. `prds/오늘-해볼까-개선계획-v4-다크리뉴얼.md` — **유일한 실행 명세.** 여정(§2)·디자인 토큰(§3.1)·랜딩 10섹션(§3.2)·인트로 4단계(§3.3)·화면 ASCII 목업 11개(§4)·태스크 순서(§5 T1~T9)·시작점 코드(§6)·카피 규칙(§7)이 전부 여기 있다.
2. `research/moonlight-ref/moonlight-interaction.html` — **S1 슬롯 화면의 구현 원본.** 부채꼴 덱(슬리버 14px·rAF 드리프트 1.1°/s·보이는 호만 렌더+recycle), 호버 풀아웃, 드래그 틸트, 탭→아크 비행→슬롯 안착, 플립 750ms+퍼플 블룸, 🔒고정+리롤, 낙하 버림, 한 번에 뽑기, 조사 자동교정. 이 인터랙션을 React로 이식하되 시각만 다크+골드로 바꾼다. 원본에 있는 `_hov` 플래그 정리(탭 제거 시 pointerleave 미발생 → 휠 영구 감속 버그 수정)를 누락하지 마라.
3. `docs/references/simplicity-intro/README.md` — 인트로 실측 근거(참고용).

베이스 코드는 기존 Next.js 앱(`app/`, `components/oneul-app.tsx`)이고, dev 서버는 `npm run dev`(현재 3001 사용 중이면 다른 포트).

- T3 온보딩 S0 a~c — **탭 3번**(시간·레벨 화면 없음, S4 직전 지연 수집)

## 어기면 안 되는 규칙

1. **금칙어**: "아이디어 생성기·AI 추천·랜덤" — 전 화면 카피에서 금지. 서사는 "내 것(🌱) → 오늘 바로 → 지인에게 물어보기".
2. **S5(수신자 투표) 화면에 로그인·앱 설치 요구 문구 한 글자도 금지.**
3. **백엔드 없음**: Supabase·PG·이메일·매직링크 일절 연동 금지. 결제 3지점 전부 가짜 문("준비 중이에요" 시트+이메일 수집), 로그인 버튼도 가짜(클릭 기록 후 통과). 상태는 localStorage/sessionStorage.
4. **다크 단일 테마**: `--bg #0a0a0b`, 골드 `#c9a962`, 시그니처 그림자 `0 3px 0 0 #c9a962`(검정 하드섀도 금지 — 다크에서 안 보임). 플립 블룸만 퍼플 `#b288f6`.
5. **데스크탑(≥1024px)은 와이드**(§4 상단 규칙): 모바일 폭 가운데 박기 금지, 콘텐츠 max 1100~1200px. 예외는 S5(중앙 560px 유지).
6. 러닝 씨앗에서 물류 불편이 나오면 실패다 — 불편·형태 릴은 반드시 씨앗 allowlist 안에서만 추출(§6.3).
7. 애니메이션은 opacity/transform만. `prefers-reduced-motion` 폴백 필수.

## 완료 기준 (스모크)

각 태스크 후 `npm run typecheck` 통과 + 모바일 375px·데스크탑 1280px 확인. 최종:
**인트로 → 랜딩 → S0(3탭) → S1(스핀 2회, 부채꼴에서 탭/드래그) → 확정 → 분기 → S3 → 공유 링크 복사 → 새 탭에서 /c/[slug] 열어 투표 → "나도 뽑아보기" → 씨앗 프리필로 S1 재진입** — 이 여정이 한 번도 끊기지 않고 흘러야 한다. `localStorage.events`에 slot_spin·idea_confirmed·card_vote·plan_click 등이 찍혀 있어야 한다.

작업 중 계획서와 충돌하는 판단이 필요하면 임의로 정하지 말고 멈춰서 물어봐라.
