# 오늘 해볼까 — 아이디어 엔진 설계서

씨앗(seed) → 불편(pain) → 형태(format) 3릴을 씨앗 반경 안에서만 조합해 "4주 안에 혼자 만들 수 있는, ChatGPT가 못 하는" 아이디어를 생성하는 엔진의 taxonomy·규칙·검증 결과를 정리한다.

---

## 1. 씨앗 taxonomy

씨앗은 두 트랙으로 나뉜다. **좋아하는 것(likeTrack, 취향형)** 은 감성 톤·커뮤니티 페르소나, **잘 아는 것(knowTrack, 도메인형)** 은 실용 톤·직군 페르소나로 강제된다.

### 1-1. likeTrack (좋아하는 것 · 취향형)

| 대분류 | 소분류 |
| --- | --- |

## 2. 불편 축 · 형태 축

### 2-1. 불편 축 (pain, 16종)

| # | 불편 |
| --- | --- |
| 1 | 기록이 여기저기 흩어져 한눈에 안 모인다 |
| 2 | 초보라 뭘 사거나 골라야 할지 모른다 |
| 3 | 혼자라 동기부여·지속이 안 된다 |
| 4 | 정보가 파편화돼 매번 검색·취합해야 한다 |
| 5 | 비교·선택이 귀찮고 오래 걸린다 |
| 6 | 입문 장벽이 높다(초보-고수 정보 격차) |
| 7 | 반복 작업을 매번 손으로 한다 |
| 8 | 진행 상황·성과가 눈에 안 보인다(측정 부재) |
| 9 | 같이 할 사람·모임을 찾기 어렵다 |
| 10 | 결정을 못 내리고 미룬다(선택 마비) |
| 11 | 남들은 어떻게 하는지 궁금한데 알 길이 없다(벤치마크 부재) |
| 12 | 뭘 놓쳤는지 모른다(체크리스트·리마인드 부재) |
| 13 | 전문가에게 묻기엔 사소하고 부담된다 |
| 14 | 내 취향·조건에 맞는 추천이 없다(개인화 부재) |
| 15 | 시작은 했는데 꾸준히 관리가 안 된다(방치) |
| 16 | 후기·평판이 흩어져 믿을 정보를 못 찾는다 |

> 불편 릴은 전역 랜덤이 아니다. 씨앗 태그마다 인접 pain 화이트리스트를 매핑해(예: 러닝 → {지속 안 됨, 성과 측정 부재, 같이 할 사람, 초보 장비}) 씨앗 밖 pain은 후보에서 배제한다.

### 2-2. 형태 축 (format, 12종 · 4주 실현성 포함)

| 형태 | 설명 | 4주 실현성 |
| --- | --- | --- |
| 공유 링크툴 | 입력→결과 페이지 생성 후 URL 공유 | 정적 렌더+localStorage면 **3~4일**, 로그인 불필요 |
| 투표/반응 카드 | share_slug 공개 읽기 + 익명 insert 단일 공개 테이블 | 클라 중복방지만이면 **3~4일** (수요확인 트랙의 기본 형태) |
| 챗봇(GPT 래퍼) | 프롬프트+LLM API 1개 + 말풍선 UI | **2~3일**이나 차별화는 프롬프트·씨앗데이터에서 나옴 |
| 집계 대시보드 | 데이터 소스를 읽어 카운트·차트로 요약 | 소스만 확보되면 **3~5일** |
| 크롬 확장 | 특정 사이트 DOM에서 수집·주입(좁은 기능 1개) | manifest+content script로 **5~7일** |
| 랜딩+웨이팅리스트 | 정적 소개페이지 + 이메일 수집 | **반나절~1일**, 가짜문/수요검증 전용 |
| 간단 CRUD 웹앱 | 목록·추가·수정·삭제 | Supabase 테이블 1~2개면 **5~7일** |
| 자동 요약/알림 봇 | 소스 수집→요약→메일/슬랙/카톡 발송 | 크론+API, 범위 좁으면 **4~5일** |
| 규칙기반 계산기/진단 툴 | 입력→조건분기→결과 | 로직만이면 로그인 없이 **2~3일** |
| 큐레이션 리스트/디렉토리 | 수집 항목을 필터·검색·정렬 | 정적 데이터셋이면 **3~5일** |
| 템플릿/생성기 | 입력값으로 문서·이미지·프롬프트 산출 | 템플릿 엔진이면 **3~4일** |
| 타이머/스트릭 트래커 | 기록·연속일·통계 | localStorage 기반이면 로그인 없이 **2~3일** |

## 3. 조합 규칙 (금지 / 추천 / 궁합)

### 3-1. 대원칙 — 세 릴 모두 씨앗 반경 안에서

씨앗은 1릴에 고정될 뿐 아니라 나머지 두 릴의 후보 풀을 좁히는 **필터**다. 전역 랜덤은 없다. 상황·심리 축은 사용자에게 노출하지 않고 결과 문장 톤(감성/실용)만 결정하는 백엔드 변수로 남긴다.

**파이프라인:** 씨앗 고정 → 씨앗 태그로 pain 풀 필터 → (트랙×레벨×시간)으로 format 풀 게이팅 → 금지쌍 제거 → 남은 풀에서 추첨.

### 3-2. 금지 조합 (트랙별 형태 게이팅)

| 트랙 | 금지 형태 | 사유 / 허용 형태 |
| --- | --- | --- |
| 취향형 (러닝/카페투어/다꾸/덕질 등) | 간단 CRUD 웹앱 · 집계 대시보드 · 크롬확장 · 자동 알림 봇 | 감성·개인 취미를 무거운 B2B성 형태에 얹으면 '남의 아이디어' 이물감 + 4주 과적재. **허용:** 공유 링크툴 · 투표/반응 카드 · 타이머/스트릭 트래커 · 템플릿/생성기 · 큐레이션 리스트 · 챗봇 |
| 도메인형 (물류/회계/의료/커머스 등) | 타이머 트래커 · 다꾸형 템플릿 | **궁합:** 집계 대시보드 · 간단 CRUD · 자동 요약/알림 봇 · 계산기/진단 툴 · 챗봇 · 큐레이션 디렉토리. 단 첫 형태는 CRUD보다 계산기/진단·공유 링크툴로 유도(4주 실현성) |
| 모든 트랙 (레벨/시간 게이팅) | Lv.1에서 크롬 확장·간단 CRUD 가중치 하향 | Lv.1엔 링크툴·카드·계산기·타이머 우선, Lv.3에서 CRUD·대시보드·봇·확장 개방. 주당 5h 이하면 정적/localStorage 형태만 노출 |

### 3-3. 추천 궁합 (가중치 상향)

| 트랙 | 불편 | 형태 | 산출물 |
| --- | --- | --- | --- |
| 취향형 | 혼자라 지속 안 됨 / 성과 측정 부재 | 타이머·스트릭 트래커 | 러닝앱·습관앱류 (4주 실현성 최상) |
| 취향형 | 초보라 뭘 사야 할지 모름 / 비교 귀찮음 | 계산기·진단 툴 or 큐레이션 리스트 | 입문자 추천기 |
| 도메인형 | 반복 작업 수동 / 뭘 놓쳤는지 모름 | 자동 요약·알림 봇 or 체크리스트 CRUD | 실무 자동화 |
| 도메인형 | 벤치마크 부재 / 후기 흩어짐 | 집계 대시보드 or 디렉토리 | 업계 인사이트 툴 |
| 모든 씨앗 | 수요가 있나(사후형 pain) | 투표/반응 카드 | 바로 수요 카드 (기본 진입점) |

### 3-4. 뭉갬 방지 프리셋 칩

씨앗은 단일 대분류 강제 대신 복수 태그 조합(예: 음식+여행+콘텐츠='카페투어')을 허용. 소분류 상단에 실제 인기 씨앗을 프리셋 칩으로 원문 노출한다: **카페투어 · 빵집투어 · 러닝크루 · 다꾸 · 홈카페 · 캠핑 · 덕질/최애 · 사이드프로젝트 · 스마트스토어 · 물류검수**. 프리셋을 고르면 내부적으로 태그 세트+전용 pain·format 풀이 바인딩돼 '음식>카페'로 뭉개지지 않는다.

### 3-5. 콘텐츠 절제

콘텐츠는 50개가 아니라 **~20개**(핵심 씨앗 8개 × pain 3~4 × format 2~3)로 1주차 시작, 확정률 데이터가 나온 뒤 확장한다.

## 4. 아이디어 생성 템플릿

**입력:** (씨앗 seed, 불편 pain, 형태 format) + 프로필(weekly_hours, skill_level)

| 산출 항목 | 규칙 |
| --- | --- |
| **이름(title)** | 씨앗 명사 + 기능 동사/접미. 취향형은 감성 톤("러닝메이트", "빵집노트"), 도메인형은 실용 톤("검수 체크봇", "재고 알리미"). 카테고리 라벨이 아니라 고유명처럼. 확정 화면 무료 노출은 카테고리+분위기까지, 정확한 이름·한줄은 실행플랜(결제②)에 일부 잠금 |
| **한줄(oneliner)** | 공식: "[타겟]이 [pain]를 [format 동작]으로 해결하는 [씨앗] 도구" |
| **타겟(target)** | 씨앗에서 직접 도출된 구체 인물. 취향형=씨앗 커뮤니티 구성원, 도메인형=씨앗 직군 동료. '초보 1인 빌더' 같은 범용 타겟 금지 |
| **MVP 기능 3~4개** | ①format 핵심 동작 1개 · ②pain을 직접 해소하는 뷰 · ③씨앗 맞춤 요소(프리셋·추천 시드) · ④(선택) 공유/수요 훅 1개. 로그인·알림·결제 게이팅·RLS 상세읽기 같은 인프라 기능은 제외 |
| **시작 프롬프트** | 결제② 산출물. 순수 텍스트 플랜은 ChatGPT로 대체되므로, 사전검증된 금지조합 회피 근거 + 유사 발행 아이디어의 투표 전환 신호를 얹어 '집합 데이터 기반'으로 차별화 |

### 6-2. 남은 개선점

1. **트래커·스트릭 형태의 레드오션 리스크** — 러닝메이트·물주기·사이드 100일은 good=true지만 평균 2.3~2.7에 그침(Strava·Planta·습관앱과 정면충돌). 개선 방향: 순수 스트릭 단독을 지양하고 **씨앗×불편×'공유/응원 투표 카드'로 옮겨 제품 코어(수요 카드)와 인프라를 겹치기**. 알림은 웹푸시로 절제.

2. **디렉토리 형태의 콜드스타트** — 방탈출·앱 온보딩은 "데이터가 곧 제품"이라 1인 4주 수동 시딩이 병목. 개선: 극단적 니치화(지역 1곳·패턴 1종) + **초기 데이터를 UGC(유저 후기·투표)로 채우기**.

3. **정확도·책임 리스크 형태** — 세금·관세·보험·뜨개 도안은 룰이 틀리면 ChatGPT보다 못함. 개선: **범위를 좁힌 1케이스부터**(단순경비율 프리랜서 / 인기 품목 N개 / 핵심 담보 몇 개 / 목도리 1종) 정확도 확보 후 확장, 면책 문구 명시.

4. **B2B 니치의 비바이럴 배포** — 입고검수·카페 레시피·매물 수익률은 지불의사는 높으나 팀 도입·공유가 약함. 개선: 결과를 **공유 카드로 내보내 코어 수요카드 루프와 연결**(예: 매물 수익률 → 손님에게 브랜딩 리포트 발송).

5. **콘텐츠 확장 타이밍** — 현재 ~20개 핵심 씨앗으로 시작한 게 맞았음(노동 절제). 확정률 데이터가 쌓인 뒤에만 pain×format 조합을 늘려 과적재를 피할 것.
