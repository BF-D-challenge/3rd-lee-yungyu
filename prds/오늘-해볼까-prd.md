## 1. 문제 정의

### 겉으로 드러난 문제

초보 1인 개발자는 AI 코딩 툴 덕분에 **앱을 만들 수는 있게 됐다.** 그런데 정작 **무엇을 만들지를 몰라서 시작조차 못 한다.**

### 진짜 문제 (5 Whys로 파고들어 찾음)

AI 코딩 시대에 개발자의 첫 번째 벽은 '개발 실력'이 아니다. 두 가지다.

1. **지금 당장 만들 수 있을 만큼 구체적인 아이디어**를 얻는 것
2. 만들기 전에 **"이거 수요가 있을까?"라는 확신**을 얻는 것

1번만 풀면 우리 타겟 4명 중 1명(무엇을 만들지 막막한 사람)만 만족한다. 이미 아이디어가 있는 사람들(개발자·특정 분야 전문가)은 2번에서 멈춰 있다.

### 지금 사람들은 어떻게 버티고 있나 (현재 방식)

- ChatGPT/Claude에 "만들 아이디어 추천해줘"라고 물음 → 답이 너무 뻔함 → 계속 다시 묻는 무한 루프
- 노션에 아이디어 30개를 쌓아두고 하나도 못 고름 ("이거 수요 있나?"에서 멈춤)
- v0·Lovable·Claude Code를 열지만 **맨 첫 프롬프트를 못 정함**
- 커뮤니티 조언은 "툴 쓰지 말고 유저 5명이랑 얘기해봐" — 맞는 말이지만, 그걸 실제로 하게 만드는 장치가 없음

### 왜 지금인가 (Why now)

AI 코딩 툴로 **제작 비용이 급락**하면서, 병목이 "만들기"에서 "무엇을 만들지 + 수요 확신"으로 옮겨갔다.

단, 리서치로 확인한 함정이 있다.[^demand] "AI 아이디어 생성기"라는 형태 자체에 사람들은 냉소적이고, **순수 생성에 돈 낼 의사는 0에 가깝다**(대표 실패 사례: IdeasAI). 유일하게 돈이 되는 건 **1회성 소액 리포트**(예: DimeADozen $9)뿐이었다.

→ 그래서 **포지셔닝 금칙어**를 정했다. **"아이디어 생성기 · AI 추천 · 랜덤"이라고 절대 말하지 않는다.** 우리가 파는 이야기는 항상 이것이다:
**"내 것(🌱씨앗)에서 출발해 → 오늘 바로 만들고 → 지인에게 물어보는 도구."**
냉소파가 말하는 처방("유저와 대화하라")을 곧 이 제품의 **투표 카드** 기능으로 만든 것이다.

## 2. 타겟 & 그들이 원하는 것

### 타겟 사용자 (페르소나 4명 — 상세 프로필은 개선안 문서 참고[^improve])

'페르소나'는 대표 사용자를 한 명의 가상 인물로 그려본 것이다. **P1이 핵심 타겟**이고, 나머지는 확장 대상이다.

| 페르소나 | 상황 | 무엇에 끌리는가 (후크) | 어디서 결제하나 |
|---|---|---|---|
| **P1 지민** (29·주니어 마케터) — **핵심 타겟** | "사이드 프로젝트 할 거야"만 6개월째. 좋아하는 건 있음(러닝) | 씨앗을 넣는 순간 결과가 전부 내 얘기가 됨 | 실행 플랜 990원 |
| P2 현우 (33·백엔드 5년차) | 노션에 아이디어 30개, 못 고름 | 투표 카드 = 수요 조사 도구 | 수요 리포트 |
| P3 소연 (26·프리랜서 디자이너) | 빵집 투어 계정 운영, 랜덤을 싫어함 | 씨앗 고정 + SNS에 올릴 만큼 예쁜 카드 | 오늘 패스 1,900원 |
| P4 태호 (41·물류 15년차) | 업계 비효율을 매일 봄, 시간 없음 | 업계 단톡방에 수요 투표 돌리기 | 수요 리포트 (9,900원 상방 실험) |

### 사용자가 원하는 일 (Job Story)

'Job Story'는 **"어떤 상황일 때 → 무엇을 하고 싶고 → 그래서 뭘 얻고 싶은가"**를 한 문장으로 쓴 것이다.

- **상황:** AI 코딩 툴로 뭔가 만들고 싶은데, 무엇을 만들지 막막할 때
- **하고 싶은 것:** 내가 좋아하는 것/잘 아는 것(🌱씨앗)에서 출발한 조합을 슬롯머신처럼 돌려 **오늘 만들 아이디어 1개를 확정**하고, 그걸 카드로 만들어 **지인에게 수요 투표를 받는 것**
- **얻고 싶은 것:** "내 것"이라는 애착과 "누가 이걸 원한다"는 확신을 갖고, **오늘 바로 만들기 시작**하는 것

### 왜 이 제품이어야 하나 (경쟁 대비)

- **누구를 위한 건가:** 무엇을 만들지 모르거나 확신이 없는 초보 1인 개발자
- **기존 대안들이 못 주는 것:** ChatGPT 추천(내 취향과 무관·일반적), 노션 목록(검증이 없음), 커뮤니티 피드백(글이 묻혀 반응 없음) — **어느 것도 "내 것에서 출발 + 수요 확인"을 한 흐름으로 주지 못한다.**
- **우리가 주는 것:** 내 취향이 반영된 카드 덱에서 빈칸을 채워 아이디어를 좁히고 → 확정한 카드를 지인 투표로 검증. 결과는 **오늘 만들 아이디어 1개 + 지인 N명의 수요 신호 + 시작 프롬프트.**

## 3. 성공을 어떻게 잴 것인가 (임팩트 가설)

### 가장 중요한 단 하나의 지표 (North Star Metric)

제품이 잘 되고 있는지를 대표하는 핵심 지표다. 두 개를 나눠서 본다.

- **성장 지표 = 카드 공유율** = (카드를 공유한 횟수) ÷ (아이디어를 확정한 횟수)
  → 아이디어를 정한 사람이 실제로 지인에게 물어보러 가는가?
- **수익 지표 = 세션당 '가짜 문' 클릭 가치**
  → 4주차까지 결제는 전부 '가짜 문'(아래 [용어](#용어-한눈에) 참고)이므로, 클릭·이메일 남긴 정도로 "돈 낼 의사"를 측정한다.

### 앞단에서 미리 보는 지표 (선행 지표)

핵심 지표가 오르기 전에 먼저 움직이는, 원인 쪽 지표들이다.

1. 씨앗 입력 완료율 · 씨앗 종류별 아이디어 확정률 (= 씨앗이 얼마나 좋은가)
2. 발행한 카드 중 투표 5표 이상 받은 비율 (= 수요 검증이 실제로 돌아가는가)
3. **바이럴 K** = 공유 수 × (투표한 사람이 다시 아이디어를 뽑아보는 전환율)
4. 로그인 마찰 = (로그인 성공) ÷ (로그인 요청) — 로그인 요구가 사람을 얼마나 튕겨내는가

> 참고: 아직 GA4(구글 애널리틱스)를 안 붙인 단계에서는, 이벤트 기록 코드(`lib/track.ts`)가 콘솔 로그와 브라우저 저장소(localStorage)에 대신 쌓아둔다.

### 숫자가 아닌 성공의 모습

> "내가 좋아하는 걸로 오늘 만들 게 정해졌고, 친구들이 진짜 쓸지도 알게 됐어요."

## 4. 솔루션 (MVP)

'MVP'는 검증에 꼭 필요한 최소 기능만 담은 첫 버전이다.

### 한 문장 정리

**오늘 해볼까**는, 무엇을 만들지 모르는 초보 1인 개발자가 **좋아하는 것(🌱씨앗)에서 출발해 오늘 만들 아이디어 1개를 뽑고, 지인 투표로 수요까지 확인하는 도구다.**

### 화면에 등장하는 카드 축 (먼저 알아두기)

아이디어는 여러 '축'의 카드를 조합해서 만든다. 축은 아래 5가지다.

| 축 | 뜻 | 필수/옵션 |
|---|---|---|
| 🌱 씨앗 | 내가 좋아하거나 잘 아는 것 (예: 러닝) | 필수 |
| 😖 불편 | 해결하려는 불편·문제 | 필수 |
| 📦 형태 | 만들 결과물의 모양 (앱·봇·뉴스레터 등) | 필수 |
| 🎬 장면 | 그게 쓰이는 상황·장면 | 필수 |
| 💭 마음 | 건드리려는 감정·심리 | 옵션 (전체 뽑기 때 ~35%만 등장) |

### 절대 바꾸지 않는 확정 원칙 5 (2026-07-06 확정 — 바꾸려면 만든 사람 승인 필요)

1. **모든 입력은 객관식(칩 선택)이 기본**이다. 직접 타이핑하는 주관식은 맨 끝의 탈출구(`✏️ 직접 쓸래요`)로만 둔다. (한국 사용자는 주관식보다 객관식을 선호한다는 판단.)

2. **아이디어 만들기는 '빈 슬롯 채우기' 방식**이다 (정적 데모 v7 기준).
   - 필수 4칸(🌱씨앗·😖불편·📦형태·🎬장면) + 옵션 1칸(💭마음)으로 시작한다.
   - **빈 상태에서 시작** → 부채꼴로 펼쳐진 카드 덱에서 한 장씩 뽑아 칸을 채운다.
   - 이미 채워진 카드를 탭하면 → **그 축만 다른 카드로 교체**된다.
   - ✕를 누르면 카드 제거, 🎲를 누르면 **전체 다시 뽑기**(이것만 뽑기 횟수를 소모한다).
   - '씨앗을 처음부터 고정'하는 대신 **취향 질문 시트**를 중간에 한 번 띄운다. (첫 아이디어는 방해 없이 완성하게 두고, 두 번째 전체 뽑기부터 취향이 없으면 가끔(40%) 뜬다. 제출하거나 건너뛰면 다시 안 묻는다.)
   - 취향을 넣으면 이후 덱·뽑기가 **취향 80% / 다양성 20%**로 가중된다.
   - **불편·형태 카드는 씨앗과 어울리는 것만 나온다**(허용 목록 앵커). 예: 러닝 씨앗인데 물류 불편이 잔뜩 나오면 실패다. 특별히 잘 맞는 조합(골든 조합)이 완성되면 ✨특별 타이틀을 붙인다.

3. **돈 버는 지점은 뒤에 둔다.** 투표·공유는 **무료**(그래야 입소문이 퍼진다). 결제는 상세·추가 기능에서만 받는다.
   - ① 뽑기 횟수 소진 → 실행 패스 1,900원
   - ② 아이디어 확정 직후 → 실행 플랜 990원
   - ③ 카드 발행 시 선예약 + 대시보드 → 수요 리포트 1,900~9,900원
   - **4주차까지 이 결제는 전부 '가짜 문'**이다. 실제 결제 없이 클릭·이메일만 측정한다. 구독은 없다.

4. **로그인은 딱 한 곳, 카드를 발행하는 순간(S3)에만** 요구한다 (구글 원탭). **투표하는 사람에게는 로그인·설치를 한 글자도 요구하지 않는다.** (요구하면 입소문이 끊긴다.)

5. **디자인 톤**은 이렇게 고정한다.
   - 모바일 우선의 미니멀한 가운데 한 칸(최대 560px, 한 화면에 행동 버튼 1개).
   - 다크 배경(#0a0a0b) + 반투명 유리 질감(glassmorphism) + 오로라빛 그라데이션(주 색상, `--aurora-*` 토큰). 골드는 씨앗🌱·선택 상태에만 보조로.
   - 결과 카드는 타로 앱 Moonlight식 뒤집기 인터랙션. 인트로는 toss처럼 단순하게.
   - 애니메이션은 opacity·transform만 사용하고, 사용자가 '모션 줄이기'를 켜면(`prefers-reduced-motion`) 존중한다.

### 사용자가 밟는 여정

화면 코드(S1·S3 등)는 각 단계 이름이다.

```
[L]  미니멀 랜딩 (히어로 화면 + 단순한 인트로)
 → [S1] 빈 슬롯 채우기 🎰  (n/5칸) — 열자마자 바로 시작, 자동 뽑기 없음
        · 부채꼴 덱에서 한 장씩 채움 · 카드 탭 = 교체 · ✕ = 제거 · 🎲 = 전체 다시 뽑기(횟수 소모)
        · 중간에 취향 시트가 1회 떠서 덱을 내 취향 쪽으로 가중
        · 뽑기 횟수를 다 쓰면 → 결제 유도(페이월 ①)
        ※ [S0] 온보딩 화면은 이 여정 밖의 독립 페이지로 따로 둠
 → [확정 분기]  아이디어가 마음에 들 때: [플랜 990원 ②] 또는 [먼저 물어보기]
 → [S3] 카드 발행 ★여기서만 로그인  (9:16 세로 카드 + 카톡/링크/스토리 공유 + 리포트 선예약 ③)
 → [S5] 받은 사람이 투표  (로그인 없음, 🔥써볼래 / 💬공감 / 🤔글쎄 → "너라면?" 씨앗 미리 채워줌 → S1로 = 입소문 루프)
 → [S6] 대시보드  (무료: N명 도착 + 첫 1표만 / 유료: 수요 리포트 ③, 5표부터 열림)
```

> **1주차는 백엔드(서버·DB)를 아예 안 만든다.** 상태는 브라우저 저장소(localStorage)에 두고, 공유 URL 자체가 데이터를 실어 나른다. 데이터베이스(Supabase[^schema])는 **"공유가 진짜로 일어난다"가 실측된 뒤 3주차에만** 착수한다.

### 반드시 있어야 할 기능 3개 (Must-have)

1. **취향이 반영된 빈 슬롯 채우기** — 필수 4칸 + 옵션 마음 칸을 부채꼴 덱에서 채운다(탭=교체·✕=제거·🎲=전체 뽑기). 취향 시트가 덱을 80/20으로 가중하고, 씨앗과 안 맞는 카드는 막고(앵커), 잘 맞으면 ✨, 문장은 조사(을/를 등)까지 자동 교정한다. (조합 품질은 이미 실증됨 — 통과율 66.7%[^engine])
2. **카드 발행 + 로그인 없는 3초 투표** — 9:16 세로 공유 카드, 받은 사람은 버튼 3개로 투표, 투표 직후 "너라면?" 씨앗을 미리 채워줘 입소문 루프를 닫는다.
3. **가짜 문 3개 + 측정 코드** — 결제 3지점의 클릭·이메일을 재고, 여정 전체의 이벤트를 기록한다(`slot_spin` · `idea_confirmed` · `card_share` · `card_vote` · `pay_*_click`).

### 이번 4주 안에는 안 할 것 (Non-goals)

공개 피드 · 랭킹 · 커뮤니티 · 구독 · 카카오 로그인 · 이메일 인프라(매직링크·투표 알림 발송) · 뽑기 횟수 강제 제한 · 리텐션 장치(기기 간 유지·재방문 알림) · 진짜 결제(PG) 연동

### 4주차 시연 시나리오 (Demo Flow)

1. 인트로 → 랜딩 → 빈 슬롯으로 바로 진입 → 부채꼴 덱에서 카드 4장을 채워 첫 아이디어 완성
2. 🎲 전체 다시 뽑기 → 취향 시트에 "러닝" 입력 → 덱이 내 얘기로 바뀜 → 마음에 안 드는 축만 탭으로 교체 → 확정
3. 확정 분기에서 [먼저 물어보기] → 카드 발행(로그인) → 카톡 링크 복사
4. 받은 사람 화면(`/c/[slug]`)에서 로그인 없이 🔥투표 → "나도 뽑아보기" → 씨앗이 미리 채워진 슬롯으로 재진입 (입소문 루프 시연)
5. 대시보드에서 "N명 도착" 확인 + 수요 리포트 가짜 문 클릭

## 5. 리스크 & 검증할 가정

### 검증해야 할 4가지 가정 (VUVF)

새 제품을 만들 때 확인해야 하는 4가지 축이다. 각 앞글자를 따 VUVF라 부른다.

| 축 | 무엇을 확인하나 | 우리의 가정 |
|----------|---|---|
| **Value** (가치가 있나) | 사람들이 이걸 원하나 | 사용자는 "내 것에서 나온 아이디어 + 수요 신호"에 가치를 느낀다. (순수 랜덤 생성엔 돈 낼 의사가 0으로 실증됐고, **씨앗과 투표가 바로 그 반증 설계다.**) |
| **Usability** (쓸 수 있나) | 설명 없이 쓸 수 있나 | 설명 없이도 '덱에서 채우기 → 탭 교체 → 🎲전체 뽑기 → 확정 → 발행 → 투표' 흐름을 이해한다. 받은 사람은 3초 안에 투표한다. |
| **Viability** (사업이 되나) | 우리 자원으로 가능한가 | 주 20시간(실질 15~16h) × 4주로 가짜 루프 전체 + 실측 2회가 가능하다. 그래서 어려운 인프라(로그인·결제·이메일)는 전부 뒤로/밖으로 뺐다. |
| **Feasibility** (기술이 되나) | 만들 수 있나 | 씨앗 기반 조합 규칙만으로 "만들어볼 만한" 품질이 나온다. (1차 실증: 통과율 66.7%, 잘 맞는 골든 조합 20개 확보) |

### 가장 위험한 가정과 검증 순서

틀리면 제품 전체가 무너지는 순서대로 나열했다.

1. **공유가 실제로 일어나는가** — 안 일어나면 어떤 인프라도 무의미하다. (3주차 데이터베이스 착수의 판단 기준.)
2. **씨앗 기반 조합이 진짜 "내 얘기"처럼 느껴지는가**
3. 결제 클릭으로 본 돈 낼 의사 (특히 수요 리포트)
4. 받은 사람 → 보낸 사람으로 이어지는 입소문 루프가 실제로 닫히는가

> 전체 리스크 목록(R1~R26: 결제 배치 순서·입소문 구조·조합 엔진·분류 체계·인프라 범위)은 개선안 문서에 있다.[^improve]

### 1~2주차 검증 실험 (Wizard-of-Oz — 진짜 인프라 없이)

'Wizard-of-Oz'는 **뒤에서 사람이 수동으로 처리하면서, 사용자에겐 진짜 시스템처럼 보이게** 해 검증하는 방법이다.

처음부터 끝까지 이어지는 **가짜 루프**를 실제 유저에게 던진다:
- 카드 공유 → 정적 이미지·URL을 직접 카톡으로 보냄
- 결제 3지점 → 가짜 문 클릭만 측정
- 수요 리포트 → 사람이 손으로 써서 전달

**Day 1부터 페르소나별 테스터 5명을 동시에 모집한다** (사람 구하기가 진짜 병목이다). 실측이 지목한 **딱 한 단계**에만 3주차 실제 구현을 판다.
