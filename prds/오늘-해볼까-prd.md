## 1. 문제 정의

### 표면 문제

초보 1인 빌더는 AI 코딩 툴로 앱을 만들 수 있게 되었지만, 정작 **무엇을 만들지 몰라 시작하지 못한다.**

### 본질적 문제 (5 Whys로 도출, v3에서 두 번째 병목 추가)

AI 코딩 시대의 첫 병목은 개발 능력이 아니라 —
1. **지금 바로 실행할 만큼 구체적인 아이디어**를 얻는 것
2. 만들기 전에 **"이거 수요가 있을까"라는 확신**을 얻는 것

①만 풀면 4개 페르소나 중 1명(막막형)만 만족한다. 아이디어가 이미 있는 사람들(개발자·도메인 전문가)은 ②에서 멈춰 있다.

### 현재 사람들이 푸는 방식 (As-is)

- ChatGPT/Claude에 "만들 아이디어 추천해줘" → 너무 일반적 → 다시 묻는 루프 반복
- 노션에 아이디어 30개를 쌓아두고 못 고름 ("이거 수요 있나?"에서 정지)
- v0/Lovable/Claude Code를 열지만 첫 프롬프트를 못 정함
- 커뮤니티의 처방은 "툴 쓰지 말고 유저 5명과 대화하라" — 맞는 말이지만 실행 장치가 없음

### Why now?

AI 코딩 툴로 제작 비용이 급락하면서 병목이 "만들기"에서 "무엇을·확신"으로 이동했다. 단, 리서치 실증([수요 리서치 요약](../.claude/knowledge/research/oneul-haebolkka-demand/SUMMARY.md)): "AI 아이디어 생성기" 형태에는 냉소가 강하고 순수 생성의 지불의사는 0에 수렴(IdeasAI 실패). 성립하는 과금은 1회성 소액 리포트(DimeADozen $9)뿐.

→ **포지셔닝 금칙어: "아이디어 생성기 · AI 추천 · 랜덤"이라고 말하지 않는다.** 서사는 항상 **"내 것(🌱씨앗)에서 시작해 → 오늘 바로 만들고 → 지인에게 물어보는 도구"**. 냉소 진영의 처방("유저와 대화하라")이 곧 이 제품의 투표 카드다.

### Job Story

When **AI 코딩 툴로 뭔가 만들고 싶은데 무엇을 만들지 막막할 때**,
I want to **내가 좋아하는 것/잘 아는 것(🌱씨앗)에서 출발한 조합을 슬롯처럼 돌려 오늘 만들 아이디어 1개를 확정하고, 카드로 만들어 지인에게 수요 투표를 받고**,
so I can **"내 것"이라는 애착과 "누가 원한다"는 확신을 갖고 오늘 바로 만들기 시작할 수 있다.**

### Value Proposition

- **Who**: 무엇을 만들지 모르는(또는 확신이 없는) 초보 1인 빌더
- **Why**: AI 코딩 툴은 열 수 있지만 첫 아이디어와 수요 확신을 못 얻음
- **What before**: ChatGPT 추천 루프 반복, 노션 아이디어 방치, "만들었는데 아무도 안 쓰면?" 공포
- **How**: 취향(내 것)이 가중된 덱에서 카드를 채우는 빈 슬롯으로 아이디어를 좁히고, 확정한 카드를 지인 투표로 검증
- **What after**: 오늘 만들 아이디어 1개 + 지인 N명의 수요 신호 + 시작 프롬프트
- **Alternatives**: ChatGPT 아이디어 추천(일반적·씨앗 무관), 노션 목록(검증 없음), 커뮤니티 피드백(글 죽음). 어느 것도 "내 것에서 출발 + 수요 확인"을 한 흐름으로 주지 못함

## 3. 임팩트 가설

### North Star Metric

- **성장 NSM = 카드 공유율** (`card_share` / `idea_confirmed`) — 확정한 사람이 실제로 지인에게 물어보는가
- **수익 NSM = 세션당 가짜 문 클릭 가치** — 4주차까지 결제는 전부 가짜 문이므로 클릭·이메일로 지불의사 측정

### Input Metrics

1. 씨앗 입력 완료율 · 씨앗 태그별 확정률 (씨앗 품질)
2. 발행 카드 중 투표 5+ 도달률 (수요 검증 가동률)
3. 바이럴 K = 공유 수 × (투표→스핀 전환율)
4. 로그인 마찰 = `auth_done` / `auth_prompt`

GA4 미연결 단계에서는 `lib/track.ts`가 콘솔+localStorage 적재로 대체한다.

### 정성적 임팩트

> "내가 좋아하는 걸로 오늘 만들 게 정해졌고, 친구들이 진짜 쓸지도 알게 됐어요."

## 4. 솔루션 (MVP)

### 핵심 가치 한 문장

**오늘 해볼까**는 무엇을 만들지 모르는 초보 1인 빌더가, **좋아하는 것(🌱씨앗)에서 출발해 오늘 만들 아이디어 1개를 뽑고, 지인 투표로 수요까지 확인하는 도구다.**

### 확정 원칙 5 (2026-07-06 확정 — 변경 시 사용자 승인)

1. **입력은 전부 객관식 칩**, 주관식은 맨 끝 탈출구(`✏️ 직접 쓸래요`).
2. **슬롯은 빈 슬롯 모델**(2026-07-06 확정, 정적 데모 v7 충실): 필수 4칸(🌱씨앗·😖불편·📦형태·🎬장면) + 옵션 💭마음(전체 뽑기 시 ~35%만 등장). 빈 시작 → 부채꼴 덱에서 한 장씩 채움 / 채워진 카드 탭=그 축만 교체 / ✕=제거 / 🎲전체 다시 뽑기(유일한 캡 소모). **씨앗 고정 릴 대신 취향 질문 시트**(첫 아이디어는 무방해 완성 — 두 번째 전체 뽑기부터 미설정 시 가끔(40%) 등장, 제출/스킵 후 재질문 없음) — 이후 덱·뽑기가 취향 80%/다양성 20% 가중. **불편·형태는 씨앗(취향) allowlist 앵커 준수**(러닝에서 물류 불편이 주류면 실패), 골든 조합 일치 시 ✨타이틀.
3. **투표·공유는 무료**(바이럴 K), **결제는 상세·추가에서**: ① 스핀 캡→실행 패스 1,900 ② 확정 직후→플랜 990 ③ 발행 선예약+대시보드→수요 리포트 1,900~9,900. **4주차까지 전부 가짜 문**(클릭·이메일만 측정), 구독 없음.
4. **로그인은 카드 발행(S3) 단 한 곳**(구글 원탭). **투표 수신자에게는 로그인·설치 요구 0자.**
5. 디자인 = 미니멀 센터 컬럼(max 560px·모바일 우선·화면당 CTA 1개) × 다크(#0a0a0b) × glassmorphism 2.0 × Moonlight 카드 인터랙션 × toss simplicity급 인트로. 주요 색상은 Aurora Mesh(`--aurora-*` 토큰), 골드는 씨앗🌱·선택 상태 보조 액센트. 애니메이션은 opacity/transform만, `prefers-reduced-motion` 존중.

### 사용자 여정

```
[L] 미니멀 랜딩 (f11 히어로 + simplicity 인트로)
 → [S1] 빈 슬롯 🎰 n/5 — 즉시 진입, 4칸 빈 시작(자동 스핀 없음):
       부채꼴 덱에서 한 장씩 채움 · 카드 탭=교체 · ✕=제거 · 🎲전체 다시 뽑기(캡 소모)
       중간 취향 시트(1회)가 씨앗 고정을 대체, 덱을 취향 가중 / 캡 소진 → 페이월①
    ※ [S0] 온보딩 라우트는 여정 밖(독립 페이지로 존치)
 → [확정 분기] 뜨거울 때: [플랜 990 ②] or [먼저 물어보기]
 → [S3] 발행 ★로그인 (9:16 카드 + 카톡/링크/스토리 + 리포트 선예약 ③)
 → [S5] 수신자 투표 (무로그인, 🔥써볼래/💬공감/🤔글쎄 → "너라면?" 씨앗 프리필 → S1 = K 루프)
 → [S6] 대시보드 (무료: N명 도착+첫 1표만 / 유료: 리포트 ③, 5표 게이트)
```

1주차는 백엔드 없음: 상태는 localStorage, 공유 URL이 데이터 캐리어. Supabase([schema.sql](../supabase/schema.sql) — 개인 계정 SQL Editor로만 적용)는 "공유가 실제로 일어난다"가 실측된 뒤 3주차에만 착수.

### Must-have 기능 3개

1. **취향-conditioned 빈 슬롯** — 필수 4칸+옵션 마음을 부채꼴 덱에서 채우고(탭=교체·✕=제거·🎲전체 뽑기), 취향 시트가 덱을 80/20 가중, allowlist 앵커·골든 조합 ✨, 조사 자동교정 문장 템플릿 (아이디어 엔진 통과율 66.7% 실증: [assets/오늘-해볼까-아이디어엔진.md](assets/오늘-해볼까-아이디어엔진.md))
2. **카드 발행 + 무로그인 3초 투표** — 9:16 공유 카드, 수신자 투표 3버튼, 투표 직후 "너라면?" 씨앗 프리필로 K 루프 폐쇄
3. **가짜 문 3개 + 계측** — 결제 3지점 클릭·이메일 측정, 전 여정 이벤트 적재(`slot_spin`·`idea_confirmed`·`card_share`·`card_vote`·`pay_*_click`)

### Non-goals (4주 안에 안 할 것)

공개 피드 · 랭킹 · 커뮤니티 · 구독 · 카카오 로그인 · 이메일 인프라(매직링크·투표 알림 SMTP) · 스핀 캡 강제 · 리텐션 기계(기기간 유지·재방문 알림) · 진짜 PG 연동

### Demo Flow (4주차 시연)

1. 인트로 → 랜딩 → 빈 슬롯 즉시 진입, 부채꼴 덱에서 카드 4장을 채워 첫 아이디어 완성
2. 🎲 전체 다시 뽑기 → 취향 시트에 "러닝" 입력 → 덱이 내 얘기로 바뀜 → 마음에 안 드는 축만 탭 교체 → 확정
3. 확정 분기에서 [먼저 물어보기] → 카드 발행(로그인) → 카톡 링크 복사
4. 수신자 화면(/c/[slug])에서 무로그인 🔥투표 → "나도 뽑아보기" → 씨앗 프리필 슬롯 재진입 (K 루프 시연)
5. 대시보드에서 "N명 도착" + 수요 리포트 가짜 문 클릭

## 5. 리스크 & 가정

### VUVF 가정 매핑

| 카테고리 | 가정 |
|----------|------|
| Value | 사용자는 "내 것에서 나온 아이디어 + 수요 신호"에 가치를 느낀다 (순수 랜덤 생성은 지불의사 0으로 실증됨 — 씨앗과 투표가 그 반증 설계) |
| Usability | 설명 없이 덱에서 카드 채우기→탭 교체→🎲전체 뽑기→확정→발행→투표 흐름을 이해한다. 투표 수신자는 3초 안에 참여한다 |
| Viability | 주 20시간(유효 15~16h) × 4주로 가짜 루프 전체 + 실측 2회가 가능하다 — 그래서 어려운 인프라(Auth·PG·이메일)는 전부 뒤로/밖으로 뺐다 |
| Feasibility | 씨앗-conditioned 조합 규칙만으로 "만들어볼 만하다" 품질이 나온다 (1차 실증: 통과율 66.7%, 골든 조합 20개 확보) |

### 가장 위험한 가정 (검증 순서)

1. **공유가 실제로 일어나는가** — 안 일어나면 어떤 인프라도 무의미. 3주차 Supabase 착수의 게이트
2. **씨앗-conditioned 조합의 체감 품질** — "이게 내 얘기"라는 느낌이 실제로 드는가
3. 결제 클릭 지불의사 (특히 수요 리포트)
4. 수신자→발신자 K 루프가 실제로 닫히는가

전체 리스크 대장 R1~R26(결제 배치 역배치·바이럴 구조·조합엔진·택소노미·인프라 범위)은 [archive/오늘-해볼까-개선안.md](archive/오늘-해볼까-개선안.md) §7.

### 1~2주차 검증 실험 (Wizard-of-Oz — 진짜 인프라 없이)

end-to-end **가짜 루프**를 실유저에게 투하한다: 카드 공유는 정적 이미지/URL을 직접 카톡 발송, 결제 3지점은 가짜 문 클릭 측정, 수요 리포트는 수기 작성 전달. **Day 1부터 페르소나별 테스터 5명 병렬 모집**(리크루팅이 진짜 long-pole). 실측이 지목한 병목 한 단계에만 3주차 실구현을 판다.
