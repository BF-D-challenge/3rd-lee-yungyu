# 오늘 해볼까 — 최종 가치 개선 계획 (2026-07-07, 계획만·구현 없음)

> 문제의식(사용자): "최종 가치가 너무 별로다." 현 딜리버러블 = 조합 문장 한 줄 + 지인 투표 + 블러 가짜문.
> 근거: [idea-tools-2026 리서치](../.claude/knowledge/research/idea-tools-2026/SUMMARY.md) — SaaS 15개 스캔 · 대표 4개 딜리버러블 해부 · Reddit/HN VOC · OSS 17개. 본 문서의 모든 설계 결정에 실증 번호(→팩트 N)를 붙였다.

## 1. 진단 — 무엇이 비었나

유행 제품이 결제 순간 건네는 것은 예외 없이 **4층 구조**인데, 우리는 1층도 절반뿐이다:

| 층 | 시장 표준 (예) | 오늘 해볼까 현재 |
|---|---|---|
| ① 판정 | "build / don't build" verdict (DimeADozen $129) | 없음 — 투표는 감상 수집에 머묾 |
| ② 근거(영수증) | 인용 800개·퍼머링크+업보트·검색량+YoY | 없음 — 블러 가짜문은 "있는 척"조차 구조화 안 됨 |
| ③ 다음 행동 | 실험 3개(표본·예산·기간)·빌드 프롬프트·고객 리드+아웃리치 문구 | 없음 — v1 PRD의 "프롬프트 복사"가 여정 개편 때 실종 |
| ④ 재방문 드립 | 데일리 아이디어 1개(완제품)+이메일 | 없음 — 내일 또 올 이유 부재 |

**치명적 역설**: 우리의 유일무이한 자산(지인 실투표 = 아무도 없는 **1차 자체생산 수요 증거**, →팩트 10)이 ①②층으로 승격되지 않고 "N명 도착"에서 끝난다. 경쟁 전부는 스크랩한 남의 데이터로 증거를 빌리다 플랫폼 리스크로 죽는데(GummySearch, →팩트 2) 우리는 그 리스크가 없는 데이터를 갖고도 안 쓰고 있다.

## 2. 설계 원칙 (리서치 실증 — 하지 말 것 포함)

1. **AI 점수/등급 절대 금지** — "전부 75점" 밈으로 신뢰 즉사(→팩트 4). 판정의 근거는 오직 사람 투표.
2. **구독 금지, 일회성·패스만** — $5~29가 검증된 충동구매 지대(→팩트 2·3). 기존 "구독 없음" 원칙 유지·강화.
3. **블러 가짜문 문법 폐기** — 무료 단계에서 유료 딜리버러블의 '형태'가 완전히 보여야 한다(→팩트 5). 가짜 문은 "완제품 맛보기 + 수량/깊이 잠금" 문법으로 전환.
4. **출처 없는 수치 금지** — 가짜 검색량 의심은 치명상(IdeaBrowser, →팩트 6). 근거 레이어는 검증 가능한 것만.
5. **리포트의 끝은 오늘 만들기** — 유저는 결과를 Claude Code에 복붙하러 간다(→팩트 7). 빌드 프롬프트는 장식이 아니라 최종 가치.
6. **타 플랫폼 API 의존 최소화** — GummySearch 사인(→팩트 2). 1차 데이터(투표) 중심, 외부 데이터는 보조.
7. 기존 포지셔닝 금칙어("아이디어 생성기·AI 추천·랜덤") 유지 — 생성기는 슬롭·셔블로 조롱받지만 "내 것+실데이터 확인"과 "가벼운 소셜 투표"는 열광(VOC §5-1).

## 3. 새 최종 가치물 설계

### 3.1 확정 딜리버러블: 조합 문장 → **"실행 브리프"** (무료 완제품)

확정 순간 카드 한 장이 아니라 **한 페이지 브리프**를 준다. 전 섹션이 기존 데이터(combos.json)만으로 조립 가능:

```
[실행 브리프 — 무료, 전체 공개]
1. 오늘 만들 한 개        (기존 조합 문장 + ✨골든 타이틀)
2. 누구의 어떤 순간        target × situation × psych (이미 있는 숨은 축을 서술로 승격)
3. 만들 것 (MVP)          골든 mvp 4개 or format.desc/action + "buildDays: 3~4일"
4. 첫 실험 = 지인 투표     "12명에게 오늘 물어보기 — 비용 0원, 소요 3초/인"
                          (DimeADozen '실험 카드' 문법을 우리 투표로 치환 — 검증 도구 자체가 실험)
5. 📋 빌드 프롬프트 복사    Claude Code/v0용 — 이름·타깃·문제·MVP 4개·첫 화면 포함
                          (v1 PRD 기능 복원, VOC 최고 인기 아웃풋 →팩트 7)
```

- 왜 무료 완제품인가: IdeaBrowser는 무료 1개/일도 11섹션 완제품 — 무료가 유료 리포트의 '형태'를 학습시킨다(→팩트 5). 우리의 결제 상품은 브리프가 아니라 **수요 리포트**(3.2)이므로 브리프는 아낌없이 준다.
- 발행 카드(S3)·수신자 카드(S5)도 이 브리프의 요약을 공유 — 투표자가 보는 것도 풍부해져 투표 질이 오른다.

### 3.2 투표의 승격: "N명 도착" → **"수요 영수증 리포트"** (유료 주력)

투표 데이터를 4층 구조로 승격한 것이 우리의 결제 상품. AI가 아니라 **투표 집계 규칙**으로 만든다:

```
[수요 영수증 리포트 — 결제 ③, 1,900~]
① 판정      "12명 중 7명 🔥써볼래 → 만들어 볼 만함" / 보류 / 재설계
            (규칙 기반: 🔥비율·표본수 임계값. AI 점수 아님 →원칙 1. 표본 한계 문구 유지=기존 R7)
② 영수증    반응자 코멘트 원문 인용(지인판 퍼머링크 문법, →팩트 1)
            + 반응 유형 분해(🔥/💬/🤔 누가·언제) + 씨앗 태그 벤치마크(같은 씨앗 카드들의
            평균 반응 — 우리 1차 데이터 축적, 외부 API 불요 →원칙 6)
③ 다음 행동  🔥 누른 사람 목록 = **첫 고객 리드** + "이 3명에게 오늘 보낼 메시지" 초안
            (WorthBuild $5 문법: 검증과 첫 고객 획득을 한 딜리버러블에 →팩트 7)
④ 드립      "새 투표 3개 도착 — 판정이 바뀌었어요" 재방문 트리거
```

- **무료/유료 경계 개정 (기존 R5 변경 — 사용자 승인 필요)**: 현행 R5는 "도착 수만 공개, 찬반 비공개". 개정안은 GummySearch 문법(원자료 무료·해석 유료, →팩트 8): **무료 = 도착 수 + 찬반 집계 + 첫 코멘트 1개**(원자료·형태 공개) / **유료 = 판정 + 유형 분해 + 벤치마크 + 리드 목록 + 아웃리치 문구**(해석·행동). 근거: 블러 가짜문 폐기(→원칙 3)와 디스콰이엇 교훈(반응만으론 결제 없음, 문서로 굳혀야 →팩트 9).

### 3.3 재방문 드립: **"오늘의 골든 아이디어"** 1개/일 (무료)

- 골든 조합 20개를 실행 브리프 완제품 형태로 하루 1개 랜딩·재방문 화면에 로테이션(IdeaBrowser 데일리 문법의 초경량판, →팩트 5). "남의 아이디어 구경 무료 → 내 것 검증 유료" 전환선.
- 이메일 드립은 4주 범위 밖(기존 이메일 인프라 비접촉 원칙 유지) — URL 재방문만.

### 3.4 가격 재배치 (가짜 문 3개 재정의 — 실측 후 확정)

| 기존 | 개정안 | 근거 |
|---|---|---|
| ② 실행 플랜 990 | **폐지 → 실행 브리프로 무료화.** "내 아이디어 심층 브리프"(경쟁 유사제품+실적+검증 가능 출처)를 2~3주차 Wizard-of-Oz(수기)로 실험 후 승격 여부 결정 | 990 산출물은 ChatGPT 무료 대체(기존 R4) + 무료 형태 공개 원칙(→팩트 5). 심층 리포트는 유일하게 궁금해하는 유료 기능(→팩트 6) |
| ③ 수요 리포트 1,900~9,900 | **주력 유지·강화** — §3.2 영수증 리포트로 실물 스펙 확정. 5표 게이트·가치사다리(1,900/4,900/9,900) 유지 | 순간 결제 수용(→팩트 3) + 1차 증거 독점(→팩트 10) |
| ① 실행 패스 1,900 (스핀 캡) | 유지하되 기대 낮춤 — 캡은 결제 트리거로 약함("아이디어는 싸다" →VOC §3-④). 실측 클릭이 0에 수렴하면 제거 | v3 원칙: 실측이 결정 |

## 4. 단계별 계획 (구현은 승인 후 착수)

- **P0 (1주차 잔여 — 외부 데이터 0으로 가능)**: ①실행 브리프(확정 화면 개편+빌드 프롬프트 복사 복원) ②투표 기반 판정·리드 목록·아웃리치 초안(S6 리포트 실물화 — 가짜문이 아니라 무료/유료 경계만 가짜 결제) ③S5 카드에 브리프 요약 노출.
- **P1 (2주차)**: ④데일리 골든 아이디어 로테이션 ⑤씨앗 태그 벤치마크(투표 축적 시작) ⑥심층 브리프 수기 실험(Wizard-of-Oz).
- **P2 (3주차+, 실측 게이트)**: ⑦한국형 근거 레이어 최소 1지표 — 네이버 데이터랩 검색량(키리스·공개 API, subscope 접근법 참고) — 반드시 출처 링크 동반(→원칙 4).
- **비접촉**: AI 점수·구독·커뮤니티 피드·LLM 순수생성 리포트·이메일 인프라.

## 5. 지표 연결

기존 NSM 유지 + 신규: `prompt_copied`(부활 — 브리프 최종 가치 실측) · 판정 열람→리포트 결제 클릭 전환 · 데일리 아이디어 재방문율 · 리드 아웃리치 복사율.

## 6. 사용자 승인 필요 결정 3개

1. **R5 개정**: 찬반 집계를 무료로 공개(해석·리드만 유료)로 바꿀지 — §3.2.
2. **플랜 990 폐지**(실행 브리프 무료화)와 심층 브리프로의 대체 실험 — §3.4.
3. **빌드 프롬프트 복사 복원** — 금칙어("AI") 회피 카피 필요: "오늘 만들 준비물 복사" 등.

## 7. 리스크

- 판정이 항상 "만들어라"로 기울면 신뢰 붕괴 — 보류/재설계 케이스 UI 강조(기존 R7)와 임계값 설계가 핵심.
- 벤치마크는 데이터 축적 전까지 표본 부족 — 초기엔 "표본 N개" 정직 표기, 골든 조합 자체 데이터로 시딩.
- 리드 목록은 투표자 개인정보 아님(발신자의 지인, 이름 이니셜/관계만) — 수신자 무마찰 원칙 유지.
